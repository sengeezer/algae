## Why It Surfaces

Every substring is a prefix of some suffix. If you insert all suffixes into a trie, each brand-new node you create corresponds to one distinct non-empty substring that has never appeared before.

## Worked Example

For `ababa`, inserting suffixes creates nodes for `a`, `b`, `ab`, `ba`, `aba`, `bab`, `abab`, `baba`, and `ababa`. The total number of non-root nodes created is `9`, which is the number of distinct non-empty substrings.

## Reference Implementations

<ExamplePair>

```js
class DistinctSubstringTrieNode {
  constructor() {
    this.children = new Map();
  }
}

function countOfDistinctSubstrings(text) {
  if (typeof text !== 'string') {
    throw new TypeError('Text must be a string');
  }

  const root = new DistinctSubstringTrieNode();
  let distinctCount = 0;

  for (let start = 0; start < text.length; start += 1) {
    let node = root;

    for (let end = start; end < text.length; end += 1) {
      const char = text[end];

      if (!node.children.has(char)) {
        node.children.set(char, new DistinctSubstringTrieNode());
        distinctCount += 1;
      }

      node = node.children.get(char);
    }
  }

  return distinctCount;
}
```

## Common Pitfalls

- Counting every substring occurrence instead of only distinct ones.
- Forgetting that the root node does not correspond to a non-empty substring.
- Building all substring strings explicitly before deduplicating them.
