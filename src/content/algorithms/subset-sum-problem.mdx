## Why It Surfaces

Subset sum is one of the standard DP recognition tests. The key observation is that every state answers a yes-or-no question: after considering some prefix of the array, which target sums are still achievable?

## Worked Example

For `values = [3, 34, 4, 12, 5, 2]` and `target = 9`, the answer is true because `4 + 5 = 9`. The DP reaches `9` by first marking `4` reachable and later extending it with `5`.

## Reference Implementations

<ExamplePair>

```js
function subsetSumProblem(values, target) {
  const reachable = new Array(target + 1).fill(false);
  reachable[0] = true;

  for (const value of values) {
    for (let current = target; current >= value; current -= 1) {
      if (reachable[current - value]) {
        reachable[current] = true;
      }
    }
  }

  return reachable[target];
}
```

```ts
export function subsetSumProblem(values: number[], target: number): boolean {
  const reachable = new Array<boolean>(target + 1).fill(false);

  for (const value of values) {
    for (let current = target; current >= value; current -= 1) {
      if (reachable[current - value]) {
        reachable[current] = true;
      }
    }
  }

  return reachable[target];
}
```

## Follow-up Questions

- Return one actual subset that reaches the target.
- Count how many subsets hit the target instead of returning a boolean.
- Explain why the one-dimensional update must run right to left.

## Common Pitfalls

- Updating the reachable array left to right and accidentally reusing an item multiple times.
- Confusing boolean feasibility with the counting version of subset sum.
- Forgetting that `0` is always reachable via the empty subset.
