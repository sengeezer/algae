## Why It Surfaces

Boyer-Moore matters when the interviewer wants more than KMP or a rolling hash. It preprocesses the pattern so that a mismatch near the end of the window can skip ahead by more than one character.

## JavaScript

```js
function buildLastOccurrence(pattern) {
  const last = new Map();

  for (let index = 0; index < pattern.length; index += 1) {
    last.set(pattern[index], index);
  }

  return last;
}

function boyerMoore(text, pattern) {
  if (pattern.length === 0) {
    return 0;
  }

  const last = buildLastOccurrence(pattern);
  let start = 0;

  while (start <= text.length - pattern.length) {
    let patternIndex = pattern.length - 1;

    while (patternIndex >= 0 && text[start + patternIndex] === pattern[patternIndex]) {
      patternIndex -= 1;
    }

    if (patternIndex < 0) {
      return start;
    }

    const mismatch = text[start + patternIndex];
    const skipIndex = last.get(mismatch) ?? -1;
    start += Math.max(1, patternIndex - skipIndex);
  }

  return -1;
}
```

## Interview Notes

- State that comparisons run from the end of the pattern toward the front.
- Explain the bad-character rule before reaching for more advanced good-suffix optimizations.
- Contrast it with KMP when the interviewer asks about preprocessing tradeoffs.
