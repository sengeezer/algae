## Why It Surfaces

This is the canonical polygon-area prompt once the boundary vertices are already ordered. The main interview step is recognizing that the shoelace sum gives the signed area in one pass without needing to split the polygon into individual triangles manually.

## Worked Example

For the ordered square `(0, 0)`, `(4, 0)`, `(4, 4)`, `(0, 4)`, the shoelace formula returns area `16`.

## Reference Implementations

<ExamplePair>

```js
function areaOfAPolygonWithGivenNOrderedVertices(vertices) {
  if (!Array.isArray(vertices) || vertices.length < 3) {
    throw new RangeError("Polygon must contain at least three ordered vertices");
  }

  let doubledArea = 0;

  for (let index = 0; index < vertices.length; index += 1) {
    const current = vertices[index];
    const next = vertices[(index + 1) % vertices.length];
    doubledArea += current.x * next.y - next.x * current.y;
  }

  return Math.abs(doubledArea) / 2;
}
```

## Follow-up Questions

- Why does reversing the vertex order flip the sign but not the final area?
- How does the shoelace formula relate to summing triangle cross products against the origin?
- What changes if the polygon vertices are not provided in boundary order?

## Common Pitfalls

- Forgetting the wraparound term from the last vertex back to the first.
- Returning the signed result directly instead of its absolute value.
- Treating this as the same problem as the simpler fixed-shape `Area` helper.
