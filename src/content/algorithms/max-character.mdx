## Why It Surfaces

This is one of the cleanest examples of the frequency-map pattern. It shows up often because the same shape also powers anagram checks, sliding-window counts, and character-mode style questions.

## Worked Example

For `"Hello World!"`, ignoring nonletters leaves `H`, `e`, `l`, `l`, `o`, `W`, `o`, `r`, `l`, `d`, so `"l"` is the max character with three occurrences.

## Reference Implementations

<ExamplePair>

```js
function maxCharacter(value, ignorePattern) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  if (value.length === 0) {
    throw new Error('Value must be nonempty');
  }

  const safeIgnorePattern = ignorePattern
    ? new RegExp(ignorePattern.source, ignorePattern.flags.replace(/[gy]/g, ''))
    : undefined;
  const counts = new Map();
  let bestChar = '';
  let bestCount = 0;

  for (const char of value) {
    if (safeIgnorePattern?.test(char)) {
      continue;
    }

    const nextCount = (counts.get(char) ?? 0) + 1;
    counts.set(char, nextCount);

    if (nextCount > bestCount) {
      bestChar = char;
      bestCount = nextCount;
    }
  }

  return bestChar;
}
```

## Follow-up Questions

- Return both the character and its frequency.
- Normalize case so `A` and `a` count together.
- Break ties deterministically by earliest appearance or lexical order.

## Common Pitfalls

- Leaving tie-breaking rules implicit.
- Reusing a global regex with `.test()` and getting stateful false negatives.
- Forgetting to reject or define behavior for the empty string.
