## Why It Surfaces

This problem looks like raw graph traversal, but the graph never changes. That makes the real state `(index in the string, current vertex)`. Once you model the task that way, you can solve feasibility from the back and then reconstruct the lexicographically least walk greedily.

## Worked Example

For `ABB`, the lexicographically least valid walk is `[0, 1, 6]`. Vertex `0` contributes `A`, vertex `1` contributes `B`, and the spoke from `1` to `6` gives the second `B`. If no suffix-feasible neighbor exists at any step, then the target string cannot be realized at all.

```js
function aPetersonGraphProblem(pattern) {
  if (pattern.length === 0) {
    return [];
  }

  const text = pattern.toUpperCase();
  if (/[^A-E]/.test(text)) {
    return null;
  }

  const labels = ['A', 'B', 'C', 'D', 'E', 'A', 'B', 'C', 'D', 'E'];
  const graph = [
    [1, 4, 5],
    [0, 2, 6],
    [1, 3, 7],
    [2, 4, 8],
    [0, 3, 9],
    [0, 7, 8],
    [1, 8, 9],
    [2, 5, 9],
    [3, 5, 6],
    [4, 6, 7],
  ];
  const possible = Array.from({ length: text.length }, () => new Array(10).fill(false));

  for (let vertex = 0; vertex < 10; vertex += 1) {
    possible[text.length - 1][vertex] = labels[vertex] === text[text.length - 1];
  }

  for (let index = text.length - 2; index >= 0; index -= 1) {
    for (let vertex = 0; vertex < 10; vertex += 1) {
      if (labels[vertex] !== text[index]) {
        continue;
      }

      for (const neighbor of graph[vertex]) {
        if (possible[index + 1][neighbor]) {
          possible[index][vertex] = true;
          break;
        }
      }
    }
  }

  let current = -1;
  for (let vertex = 0; vertex < 10; vertex += 1) {
    if (possible[0][vertex]) {
      current = vertex;
      break;
    }
  }

  if (current === -1) {
    return null;
  }

  const walk = [current];

  for (let index = 1; index < text.length; index += 1) {
    let next = -1;

    for (const neighbor of graph[current]) {
      if (possible[index][neighbor]) {
        next = neighbor;
        break;
      }
    }

    if (next === -1) {
      return null;
    }

    walk.push(next);
    current = next;
  }

  return walk;
}
```

## Follow-up Questions

- Reconstruct the lexicographically greatest valid walk instead.
- Return only a boolean if the interviewer does not need the actual walk.
- Generalize the same suffix-feasibility idea to a larger labeled graph.

## Common Pitfalls

- Greedily picking the first locally valid next vertex without checking whether the remaining suffix is still realizable.
- Forgetting that each letter `A` through `E` appears on both an outer and an inner Petersen-graph vertex.
- Treating the answer as an ordinary shortest-path problem when the real constraint is matching a label sequence exactly.
