## Why It Surfaces

This is one of the cleanest examples of a merge-cost greedy proof. If an item is going to be charged again in later merges, you want its current contribution to be as small as possible, which is why the two shortest ropes must be merged first.

## Worked Example

For `ropes = [4, 3, 2, 6]`, merge `2` and `3` first for cost `5`, leaving `[4, 5, 6]`. Then merge `4` and `5` for cost `9`, leaving `[6, 9]`. Finally merge `6` and `9` for cost `15`, so the total cost is `5 + 9 + 15 = 29`.

```js
function connectNRopesWithMinimumCost(lengths) {
  if (lengths.length <= 1) {
    return 0;
  }

  const heap = [...lengths];

  function swap(left, right) {
    [heap[left], heap[right]] = [heap[right], heap[left]];
  }

  function siftUp(index) {
    let child = index;

    while (child > 0) {
      const parent = Math.floor((child - 1) / 2);

      if (heap[parent] <= heap[child]) {
        break;
      }

      swap(parent, child);
      child = parent;
    }
  }

  function siftDown(index) {
    let parent = index;

    while (true) {
      let smallest = parent;
      const left = parent * 2 + 1;
      const right = parent * 2 + 2;

      if (left < heap.length && heap[left] < heap[smallest]) {
        smallest = left;
      }

      if (right < heap.length && heap[right] < heap[smallest]) {
        smallest = right;
      }

      if (smallest === parent) {
        break;
      }

      swap(parent, smallest);
      parent = smallest;
    }
  }

  function buildHeap() {
    for (let index = Math.floor(heap.length / 2) - 1; index >= 0; index -= 1) {
      siftDown(index);
    }
  }

  function push(value) {
    heap.push(value);
    siftUp(heap.length - 1);
  }

  function pop() {
    const min = heap[0];
    const last = heap.pop();

    if (heap.length > 0 && last !== undefined) {
      heap[0] = last;
      siftDown(0);
    }

    return min;
  }

  buildHeap();
  let totalCost = 0;

  while (heap.length > 1) {
    const first = pop();
    const second = pop();
    const merged = first + second;
    totalCost += merged;
    push(merged);
  }

  return totalCost;
}
```

## Follow-up Questions

- Explain why this is the same greedy structure as Huffman-style merging.
- Return the full merge tree instead of only the total cost.
- Discuss when repeated sorting is acceptable versus when a heap is the right tool.

## Common Pitfalls

- Treating the answer as the final rope length instead of the sum of all merge costs.
- Repeatedly sorting the whole array when a min-heap is the intended optimization.
- Forgetting that every intermediate merged rope goes back into the candidate set.
