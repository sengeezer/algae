## Why It Surfaces

The trick is to stop thinking about the remaining elements as two separate pieces. If `removed = xor(L..R)` and `total = xor(all values)`, then the answer is just `total ^ removed` because every element inside `[L, R]` cancels exactly once.

## Worked Example

For `[4, 7, 8, 5]` and query `[2, 3]` using one-based indexing, the removed XOR is `7 ^ 8 = 15`. The total XOR is `4 ^ 7 ^ 8 ^ 5 = 14`, so the remaining XOR is `14 ^ 15 = 1`, which matches `4 ^ 5`.

## Reference Implementations

<ExamplePair>

```js
function nitikaAndHerQueries(values, queries) {
  if (!Array.isArray(values) || values.length === 0) {
    throw new RangeError('Values must be a non-empty array');
  }

  const prefix = new Array(values.length + 1).fill(0);

  for (let index = 0; index < values.length; index += 1) {
    const value = values[index];
    if (!Number.isSafeInteger(value)) {
      throw new RangeError('Array values must be safe integers');
    }

    prefix[index + 1] = prefix[index] ^ value;
  }

  const totalXor = prefix[values.length];

  return queries.map(([left, right]) => {
    if (!Number.isInteger(left) || !Number.isInteger(right) || left < 1 || right > values.length || left > right) {
      throw new RangeError('Each query must be a valid one-based inclusive range');
    }

    if (left === 1 && right === values.length) {
      throw new RangeError('The resulting array must not be empty');
    }

    const removedXor = prefix[right] ^ prefix[left - 1];
    return totalXor ^ removedXor;
  });
}
```

## Follow-up Questions

- How would the solution change if queries asked for the sum outside `[L, R]` instead of XOR?
- What if the array allowed point updates between queries?
- Why does prefix XOR need the same one-extra-slot pattern as prefix sums?

## Common Pitfalls

- Mixing one-based query indices with zero-based array indices.
- Computing the XOR of the remaining prefix and suffix separately when `total ^ removed` is simpler.
- Forgetting that removing the whole array is invalid in the original problem statement.
