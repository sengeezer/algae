## Why It Surfaces

Odd-even sort is a clean way to separate adjacent swaps into two predictable phases. One phase fixes pairs starting at odd indexes, the next fixes pairs starting at even indexes, and the algorithm repeats until both phases make no changes.

## Worked Example

On `[5, 2, 4, 3]`, the odd phase compares `(2, 4)` first, then the even phase compares `(5, 2)` and `(4, 3)`. Repeating the two-phase cycle eventually bubbles every value into place.

```js
function oddEvenSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const items = [...values];
  let sorted = false;

  while (!sorted) {
    sorted = true;

    for (let index = 1; index < items.length - 1; index += 2) {
      if (items[index] > items[index + 1]) {
        [items[index], items[index + 1]] = [items[index + 1], items[index]];
        sorted = false;
      }
    }

    for (let index = 0; index < items.length - 1; index += 2) {
      if (items[index] > items[index + 1]) {
        [items[index], items[index + 1]] = [items[index + 1], items[index]];
        sorted = false;
      }
    }
  }

  return items;
}
```

## Follow-up Questions

- Compare odd-even sort with bubble sort on the same input.
- Explain why the phase structure makes sense for parallel local communication.
- Modify it to sort descending without changing the phase schedule.

## Common Pitfalls

- Running only the odd phase or only the even phase.
- Assuming the parity split changes the worst-case quadratic behavior.
- Forgetting to reset the `sorted` flag at the start of each full cycle.
