## Why It Surfaces

The greedy-looking part of this problem is really a shortest-path transformation. Every forward edge costs `0`, every reversed edge costs `1`, and 0-1 BFS always expands the currently cheapest frontier by pushing zero-cost moves to the front of a deque.

## Worked Example

For `n = 7`, `src = 0`, `dst = 6`, and

```text
[[0, 1], [2, 1], [2, 3], [6, 3], [6, 4], [4, 5], [5, 1]]
```

the best route is `0 -> 1 -> 2 -> 3 -> 6`, which requires reversing the edges `2 -> 1` and `6 -> 3`. The minimum number of reversals is therefore `2`.

```js
function minimumEdgesToReverseToMakePathFromASourceToADestination(
  nodeCount,
  edges,
  source,
  destination,
) {
  const maxNode = Math.max(
    source,
    destination,
    ...edges.flatMap(([from, to]) => [from, to]),
  );
  const size = Math.max(nodeCount, maxNode + 1);
  const graph = Array.from({ length: size }, () => []);

  for (const [from, to] of edges) {
    graph[from].push([to, 0]);
    graph[to].push([from, 1]);
  }

  const distances = Array(size).fill(Number.POSITIVE_INFINITY);
  const deque = {};
  let head = 0;
  let tail = 0;

  function pushFront(value) {
    head -= 1;
    deque[head] = value;
  }

  function pushBack(value) {
    deque[tail] = value;
    tail += 1;
  }

  function popFront() {
    const value = deque[head];
    delete deque[head];
    head += 1;
    return value;
  }

  distances[source] = 0;
  pushBack(source);

  while (head !== tail) {
    const node = popFront();

    for (const [nextNode, weight] of graph[node]) {
      const nextDistance = distances[node] + weight;

      if (nextDistance >= distances[nextNode]) {
        continue;
      }

      distances[nextNode] = nextDistance;

      if (weight === 0) {
        pushFront(nextNode);
      } else {
        pushBack(nextNode);
      }
    }
  }

  return distances[destination] === Number.POSITIVE_INFINITY
    ? -1
    : distances[destination];
}
```

## Follow-up Questions

- Rework the solution with Dijkstra and explain why 0-1 BFS is better here.
- Return one optimal path in addition to the minimum reversal count.
- Adapt the approach if some reversals cost more than `1`.

## Common Pitfalls

- Treating the graph as purely directed and forgetting to add the synthetic reverse edges.
- Using `shift()` and `unshift()` on arrays while claiming O(1) deque operations in JavaScript.
- Returning the first path found instead of the minimum-cost one.
