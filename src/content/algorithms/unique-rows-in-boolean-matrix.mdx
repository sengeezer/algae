## Why It Surfaces

Each row is just a binary string of fixed width, so a trie with only two child choices per node works naturally. The first time a full row path ends at a non-terminal node, that row is new; every later visit to the same terminal path is a duplicate.

## Worked Example

For `[[1, 1, 0, 1], [1, 0, 0, 1], [1, 1, 0, 1]]`, the first row creates a new terminal path and is emitted, the second row creates another new terminal path and is emitted, and the third row lands on the first terminal path so it is skipped.

## Reference Implementations

<ExamplePair>

```js
class BooleanRowTrieNode {
  constructor() {
    this.children = [null, null];
    this.isRow = false;
  }
}

function uniqueRowsInBooleanMatrix(matrix) {
  if (!Array.isArray(matrix)) {
    throw new TypeError('Matrix must be an array of rows');
  }

  const root = new BooleanRowTrieNode();
  const uniqueRows = [];

  for (const row of matrix) {
    if (!Array.isArray(row)) {
      throw new TypeError('Each row must be an array');
    }

    let node = root;
    for (const bit of row) {
      if (bit !== 0 && bit !== 1) {
        throw new RangeError('Rows must contain only 0 and 1');
      }

      if (node.children[bit] === null) {
        node.children[bit] = new BooleanRowTrieNode();
      }

      node = node.children[bit];
    }

    if (!node.isRow) {
      node.isRow = true;
      uniqueRows.push([...row]);
    }
  }

  return uniqueRows;
}
```

## Common Pitfalls

- Forgetting to preserve first appearance order when deduplicating.
- Reusing the same row array reference instead of copying it into the result.
- Treating a shared prefix as proof that the full row already exists.
