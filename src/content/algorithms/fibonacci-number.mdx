## Why It Surfaces

Fibonacci is the standard entry point for dynamic programming because the recurrence is tiny but the naive recursive tree grows exponentially. It is the cleanest way to show why overlapping subproblems matter.

## Worked Example

With `F(0) = 0` and `F(1) = 1`, every later value is the sum of the previous two. Instead of recomputing `F(3)` again and again while solving `F(5)`, carry the last two totals forward in one loop.

## Reference Implementations

<ExamplePair>

```js
function fibonacciNumber(index) {
  if (index === 0) {
    return 0;
  }

  let previous = 0;
  let current = 1;

  for (let step = 1; step < index; step += 1) {
    const next = previous + current;
    previous = current;
    current = next;
  }

  return current;
}
```

## Common Pitfalls

- Reverting to exponential recursion after identifying the DP pattern.
- Mishandling the small base cases around zero and one.
- Storing the full table when only two prior values are needed.
