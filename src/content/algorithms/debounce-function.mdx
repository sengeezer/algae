## Why It Surfaces

Debounce is one of the most common small frontend and systems-design utilities because it turns noisy event bursts into deliberate trailing work. The key idea is that each new call cancels the previous pending callback and schedules a fresh one.

## Worked Example

If a debounced search function has a `300ms` delay and the user types five keys quickly, the timer resets on every keystroke. Only one search runs, `300ms` after typing stops.

## Reference Implementations

<ExamplePair>

```js
function debounce(fn, delay) {
  if (typeof fn !== 'function' || typeof delay !== 'number') {
    throw new TypeError('Expected a function and a numeric delay');
  }

  let timerId;

  return function debounced(...args) {
    clearTimeout(timerId);
    timerId = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}
```

```ts
export function debounce<TArgs extends unknown[]>(
  fn: (...args: TArgs) => void,
  delay: number,
): (...args: TArgs) => void {
  let timerId: ReturnType<typeof setTimeout> | undefined;

  return function debounced(this: unknown, ...args: TArgs): void {
    if (timerId !== undefined) {
      clearTimeout(timerId);
    }

    timerId = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}
```

## Common Pitfalls

- Forgetting to preserve `this` and the latest arguments.
- Triggering every call immediately and only delaying side effects superficially.
- Confusing debounce with throttle.
