## Why It Surfaces

Perfect-number questions are mostly about divisor handling. The useful step is recognizing that proper divisors arrive in pairs, so a square-root scan replaces a full walk from `1` to `n - 1`.

## Worked Example

`28` has proper divisors `1`, `2`, `4`, `7`, and `14`. Their sum is `28`, so `28` is perfect. `12` is not, because its proper divisors sum to `16`.

## Reference Implementations

<ExamplePair>

```js
function perfectNumber(value) {
  if (!Number.isInteger(value) || value <= 1) {
    return false;
  }

  let divisorSum = 1;

  for (let divisor = 2; divisor * divisor <= value; divisor += 1) {
    if (value % divisor !== 0) {
      continue;
    }

    divisorSum += divisor;
    const pairedDivisor = value / divisor;

    if (pairedDivisor !== divisor) {
      divisorSum += pairedDivisor;
    }
  }

  return divisorSum === value;
}
```

## Common Pitfalls

- Including the number itself in the proper-divisor sum.
- Double-counting a square root divisor on perfect squares.
- Scanning all numbers up to `n - 1` when divisor pairing already gives a better bound.
