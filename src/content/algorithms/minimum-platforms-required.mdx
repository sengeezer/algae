## Why It Surfaces

This is one of the standard overlap-count problems. The trick is to stop thinking in terms of per-train decisions and instead process the entire timeline as a merged stream of arrival and departure events.

## Worked Example

For `arr = [900, 940, 950, 1100, 1500, 1800]` and `dep = [910, 1200, 1120, 1130, 1900, 2000]`, sorted events show three trains present at `1100`, before the departures at `1120` and `1130` reduce the count again. That maximum overlap means `3` platforms are required.

## Reference Implementations

<ExamplePair>

```js
function minimumPlatformsRequired(arrivals, departures) {
  if (arrivals.length === 0) {
    return 0;
  }

  const sortedArrivals = [...arrivals].sort((left, right) => left - right);
  const sortedDepartures = [...departures].sort((left, right) => left - right);

  let platformsInUse = 1;
  let maxPlatforms = 1;
  let arrivalIndex = 1;
  let departureIndex = 0;

  while (arrivalIndex < sortedArrivals.length && departureIndex < sortedDepartures.length) {
    if (sortedArrivals[arrivalIndex] <= sortedDepartures[departureIndex]) {
      platformsInUse += 1;
      maxPlatforms = Math.max(maxPlatforms, platformsInUse);
      arrivalIndex += 1;
      continue;
    }

    platformsInUse -= 1;
    departureIndex += 1;
  }

  return maxPlatforms;
}
```

## Common Pitfalls

- Treating a departure at the same time as an arrival as freeing a platform early, which this prompt explicitly disallows.
- Merging intervals instead of counting the maximum number of overlaps.
- Sorting one array but forgetting that departures must also be sorted before the sweep.
