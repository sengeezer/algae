## Why It Surfaces

This is the classic coin-row counterexample to greedy reasoning. Picking the larger visible coin is not enough because the right move is the one that leaves the opponent with the weakest future position, not the one with the largest immediate reward.

## Worked Example

For `coins = [8, 15, 3, 7]`, the best guaranteed answer is `22`. Taking `7` first is stronger than taking `8` because it leads to the winning path `7 + 15` under optimal play.

## Reference Implementations

<ExamplePair>

```js
function optimalStrategyForAGame(coins) {
  const size = coins.length;
  if (size === 0) {
    return 0;
  }

  const dp = new Array(size).fill(0);
  let total = 0;

  for (let left = size - 1; left >= 0; left -= 1) {
    total += coins[left];

    for (let right = left; right < size; right += 1) {
      if (left === right) {
        dp[right] = coins[right];
      } else {
        dp[right] = Math.max(coins[left] - dp[right], coins[right] - dp[right - 1]);
      }
    }
  }

  return (total + dp[size - 1]) / 2;
}
```

## Follow-up Questions

- Reconstruct one optimal sequence of left-right choices, not only the final amount.
- Compare the score-difference DP with the more direct 2D recurrence that uses `min` on the opponent's response.
- Explain why the final score equals `(total + difference) / 2`.

## Common Pitfalls

- Assuming the locally larger end coin is always the right first move.
- Forgetting that the opponent is also optimal, not random or greedy.
- Returning the score difference instead of the first player's actual amount.
