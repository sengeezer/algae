## Why It Surfaces

This is a small greedy matching problem with a simple invariant: never spend a larger cookie on a smaller greed requirement if a smaller cookie would already work. Sorting both lists makes that invariant easy to enforce with one left-to-right scan.

## Worked Example

If `greed = [1, 10, 3]` and `cookies = [1, 2, 3]`, sort them to `[1, 3, 10]` and `[1, 2, 3]`. The first child gets cookie `1`, the second child with greed `3` gets cookie `3`, and the child with greed `10` cannot be satisfied. The answer is `2`.

## Reference Implementations

<ExamplePair>

```js
function assignMaximumCookies(greed, cookies) {
  const sortedGreed = [...greed].sort((left, right) => left - right);
  const sortedCookies = [...cookies].sort((left, right) => left - right);
  let child = 0;
  let cookie = 0;
  let satisfied = 0;

  while (child < sortedGreed.length && cookie < sortedCookies.length) {
    if (sortedCookies[cookie] >= sortedGreed[child]) {
      satisfied += 1;
      child += 1;
      cookie += 1;
      continue;
    }

    cookie += 1;
  }

  return satisfied;
}
```

## Follow-up Questions

- Return the actual child-to-cookie assignments instead of only the count.
- Explain why trying to satisfy the greediest child first can fail.
- Adapt the problem if each cookie could be split among multiple children.

## Common Pitfalls

- Assigning a large cookie too early when a smaller one would have been enough.
- Forgetting to sort both arrays before using the two-pointer scan.
- Mutating the caller's arrays unintentionally when sorting.
