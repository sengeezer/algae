## Why It Surfaces

Chinese Postman appears when Eulerian thinking is almost right but the graph is weighted and not quite Eulerian yet. The optimization insight is that you do not choose an entirely new route from scratch; you add just enough duplicated travel to make a closed Eulerian walk possible at minimum extra cost.

## Worked Example

If a weighted graph has four odd-degree vertices, an Eulerian tour is impossible until they are paired up. After computing shortest paths between those odd vertices, the problem becomes: which pairing adds the least extra distance to the total sum of all original edge weights?

function chinesePostmanProblem(graph) {
  const nodeCount = graph.length;
  if (nodeCount === 0) {
    return 0;
  }

  const distances = Array.from({ length: nodeCount }, (_, from) =>
    Array.from({ length: nodeCount }, (_, to) => (from === to ? 0 : graph[from][to])),
  );
  const degree = new Array(nodeCount).fill(0);
  let totalWeight = 0;

  for (let from = 0; from < nodeCount; from += 1) {
    for (let to = from + 1; to < nodeCount; to += 1) {
      if (Number.isFinite(graph[from][to])) {
        totalWeight += graph[from][to];
        degree[from] += 1;
        degree[to] += 1;
      }
    }
  }

  for (let via = 0; via < nodeCount; via += 1) {
    for (let from = 0; from < nodeCount; from += 1) {
      for (let to = 0; to < nodeCount; to += 1) {
        const candidate = distances[from][via] + distances[via][to];
        if (candidate < distances[from][to]) {
          distances[from][to] = candidate;
        }
      }
    }
  }

  const oddVertices = degree.flatMap((value, node) => (value % 2 === 1 ? [node] : []));
  if (oddVertices.length === 0) {
    return totalWeight;
  }

  const memo = new Map();

  function pairOddVertices(mask) {
    if (mask === 0) {
      return 0;
    }

    if (memo.has(mask)) {
      return memo.get(mask);
    }

    let first = 0;
    while (((mask >> first) & 1) === 0) {
      first += 1;
    }

    let best = Number.POSITIVE_INFINITY;
    for (let second = first + 1; second < oddVertices.length; second += 1) {
      if ((mask & (1 << second)) === 0) {
        continue;
      }

      const candidate =
        distances[oddVertices[first]][oddVertices[second]]
        + pairOddVertices(mask ^ (1 << first) ^ (1 << second));
      best = Math.min(best, candidate);
    }

    memo.set(mask, best);
    return best;
  }

  return totalWeight + pairOddVertices((1 << oddVertices.length) - 1);
}

## Follow-up Questions

- Reconstruct the actual duplicated edges, not just the minimum route cost.
- Compare Chinese Postman with Eulerian classification and with Hamiltonian tour problems.
- Explain why only odd-degree vertices matter when fixing an undirected graph for a closed tour.

## Common Pitfalls

- Confusing "cover every edge" with "visit every vertex once."
- Forgetting to add the original sum of edge weights before the odd-vertex pairing cost.
- Using shortest paths between odd vertices without first making sure the graph is connected enough for the route to exist.
