## Why It Surfaces

Permutations are the cleanest example of full search with state restoration. Unlike subset questions, every slot in the output must be filled, so the recursion is really about which unused value should occupy the next position.

## Worked Example

For `[1, 2, 3]`, the first choice can be `1`, `2`, or `3`. Once `2` is chosen first, the second position only considers `1` and `3`, and so on until the path reaches length `3`.

## Reference Implementations

<ExamplePair>

```js
function permutations(nums) {
  const results = [];
  const path = [];
  const used = new Array(nums.length).fill(false);

  function backtrack() {
    if (path.length === nums.length) {
      results.push([...path]);
      return;
    }

    for (let index = 0; index < nums.length; index += 1) {
      if (used[index]) {
        continue;
      }

      used[index] = true;
      path.push(nums[index]);
      backtrack();
      path.pop();
      used[index] = false;
    }
  }

  backtrack();
  return results;
}
```

## Follow-up Questions

- Avoid duplicate permutations when the input contains repeated values.
- Generate the permutations lazily instead of materializing them all.
- Compare permutations with combinations and subsets.

## Common Pitfalls

- Forgetting to mark elements unused again on backtrack.
- Solving combinations instead of order-sensitive permutations.
- Copying the path too late and mutating the saved answer.
