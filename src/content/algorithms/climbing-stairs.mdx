## Why It Surfaces

Climbing Stairs is the friendly entry point into dynamic programming because the recurrence is easy to see but still exposes the full DP workflow: define the state, write the transition, and then compress the memory.

## Worked Example

To reach step `5`, the final move must have come from step `4` or step `3`, so the total number of ways is `ways[4] + ways[3]`.

## Reference Implementations

<ExamplePair>

```js
function climbStairs(steps) {
  if (steps <= 2) {
    return steps;
  }

  let previousTwo = 1;
  let previousOne = 2;

  for (let current = 3; current <= steps; current += 1) {
    const next = previousOne + previousTwo;
    previousTwo = previousOne;
    previousOne = next;
  }

  return previousOne;
}
```

## Follow-up Questions

- Allow jumps of one, two, or three steps.
- Block some steps and explain how the recurrence changes.
- Compare the rolling-state version with a full `dp` array.

## Common Pitfalls

- Writing exponential recursion and stopping before memoizing or tabulating it.
- Forgetting the small base cases for `1` and `2` steps.
- Missing the fact that the recurrence is Fibonacci-shaped, not factorial or combinatorial by brute force.
