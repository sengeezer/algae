## Why It Surfaces

This is one of the sequence problems where the right state is not just the index. Once a subsequence picks its second element, the common difference is fixed, so the DP has to remember both where the subsequence ends and what difference it is trying to maintain.

## Worked Example

For `values = [1, 2, 3]`, the answer is `8` in the GeeksforGeeks version because it counts the empty sequence, all three singletons, the three length-two subsequences, and the full arithmetic subsequence `[1, 2, 3]`.

## Reference Implementations

<ExamplePair>

```js
function countOfAPArithmeticProgressionSubsequencesInAnArray(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('values must be an array of integers');
  }

  for (const value of values) {
    if (!Number.isInteger(value)) {
      throw new TypeError('values must be an array of integers');
    }
  }

  const length = values.length;
  if (length === 0) {
    return 1;
  }

  const dp = Array.from({ length }, () => new Map());
  let total = length + 1;

  for (let end = 0; end < length; end += 1) {
    for (let start = 0; start < end; start += 1) {
      const difference = values[end] - values[start];
      const previous = dp[start].get(difference) ?? 0;
      const current = (dp[end].get(difference) ?? 0) + previous + 1;

      dp[end].set(difference, current);
      total += previous + 1;
    }
  }

  return total;
}
```

## Follow-up Questions

- Adjust the answer for interview variants that exclude the empty subsequence or single-element subsequences.
- Return only arithmetic subsequences of length at least `3`.
- Compare the per-index hash-map DP with the range-based optimization that depends on `max - min`.

## Common Pitfalls

- Solving contiguous arithmetic subarrays instead of subsequences.
- Forgetting that the GeeksforGeeks formulation counts the empty sequence and singletons.
- Collapsing the state to index only and losing the common difference information.
