## Why It Surfaces

This problem shows up when subsets are encoded as bitmasks and you need the next mask with the same number of chosen elements. The key is not to increment blindly, but to relocate the rightmost movable `1` and then repack the trailing ones as far right as possible.

## Worked Example

`12` is `1100` in binary and has two set bits. The next higher number with two set bits is `17`, which is `10001`.

## Reference Implementations

<ExamplePair>

```js
function nextHigherNumberWithSameNumberOfSetBits(value) {
  if (!Number.isInteger(value) || value < 0 || value > 0xffffffff) {
    throw new RangeError('Value must be an unsigned 32-bit integer');
  }

  const number = BigInt(value >>> 0);
  if (number === 0n) {
    throw new RangeError('Value must be positive');
  }

  const rightOne = number & -number;
  const nextHigherOneBit = number + rightOne;
  const rightOnesPattern = ((number ^ nextHigherOneBit) / rightOne) >> 2n;
  const next = nextHigherOneBit | rightOnesPattern;

  if (next > 0xffffffffn) {
    throw new RangeError('No higher unsigned 32-bit integer has the same number of set bits');
  }

  return Number(next);
}
```

## Common Pitfalls

- Incrementing upward linearly until the popcount matches again.
- Forgetting that some inputs near the top of the 32-bit range have no valid higher answer.
- Breaking the trailing one-bit pattern instead of packing it back to the far right.
