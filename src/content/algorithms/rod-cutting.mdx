## Why It Surfaces

Rod Cutting is the classic DP framing for repeated choice under composition. Unlike 0/1 knapsack, one cut size can be reused again and again, so the solution behaves like an unbounded optimization problem.

## Worked Example

If a length-4 rod has prices `[1, 5, 8, 9]`, then cutting it into two length-2 pieces yields `10`, which beats selling the full rod for `9`. The DP finds that by trying every possible first cut for each smaller length.

## Reference Implementations

<ExamplePair>

```js
function rodCutting(prices, length) {
  const dp = new Array(length + 1).fill(0);

  for (let rod = 1; rod <= length; rod += 1) {
    let best = Number.NEGATIVE_INFINITY;

    for (let cut = 1; cut <= rod; cut += 1) {
      best = Math.max(best, prices[cut - 1] + dp[rod - cut]);
    }

    dp[rod] = best;
  }

  return dp[length];
}
```

## Follow-up Questions

- Return the sequence of cuts, not only the best revenue.
- Compare this with 0/1 knapsack and coin change.
- Add a fixed cutting cost and update the recurrence.

## Common Pitfalls

- Treating each cut size as usable only once.
- Forgetting that prices are indexed by cut length.
- Using a greedy highest-price piece instead of DP over all first cuts.
