## Why It Surfaces

This is the cleanest demonstration that subsets can be treated as integers. Every bit position answers one question: is the corresponding element in the current subset or not?

## Worked Example

For `[1, 2, 3]`, mask `000` gives `[]`, mask `101` gives `[1, 3]`, and mask `111` gives `[1, 2, 3]`. Sweeping the masks from `0` to `7` visits the full power set.

## Reference Implementations

<ExamplePair>

```js
function generateSubsets(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Input must be an array');
  }

  if (values.length > 31) {
    throw new RangeError('Bitmask subset generation supports at most 31 items');
  }

  const subsets = [];
  const totalMasks = 2 ** values.length;

  for (let mask = 0; mask < totalMasks; mask += 1) {
    const subset = [];

    for (let index = 0; index < values.length; index += 1) {
      if (mask & (1 << index)) {
        subset.push(values[index]);
      }
    }

    subsets.push(subset);
  }

  return subsets;
}
```

    const subset: T[] = [];

      if (mask & (1 << index)) {
        subset.push(values[index]);
      }
    }

    subsets.push(subset);
  }

  return subsets;
}

## Follow-up Questions

- Generate only subsets of a fixed size `k`.
- Compare this approach with recursive include-or-exclude subset generation.
- Use subset masks inside a dynamic-programming state instead of materializing every answer.

## Common Pitfalls

- Forgetting to include the empty subset.
- Using bit shifts past the practical width of JavaScript bitwise operators.
- Confusing subset generation with permutation generation.
