## Why It Surfaces

Universal sink problems are interesting because they look like they require checking every row and column, but they do not. One adjacency-matrix comparison at a time is enough to eliminate impossible candidates until only one vertex can still qualify.

## Worked Example

If `matrix[i][j] === 1`, then vertex `i` cannot be a universal sink because it has an outgoing edge. If `matrix[i][j] === 0`, then vertex `j` cannot be a universal sink because some other vertex does not point to it. After one scan, the survivor is verified by checking its full row and column.

## Reference Implementations

<ExamplePair>

```js
function universalSinkInDirectedGraph(matrix) {
  const size = matrix.length;
  let row = 0;
  let column = 0;

  while (row < size && column < size) {
    if (matrix[row][column] === 1) {
      row += 1;
    } else {
      column += 1;
    }
  }

  const candidate = row;
  if (candidate >= size) {
    return -1;
  }

  for (let index = 0; index < size; index += 1) {
    if (index === candidate) {
      continue;
    }

    if (matrix[candidate][index] !== 0 || matrix[index][candidate] !== 1) {
      return -1;
    }
  }

  return candidate;
}
```

## Follow-up Questions

- Explain why the elimination phase is linear even though the input is a matrix.
- Compare this with the celebrity problem, which uses the same core idea.
- Modify the routine to return all sinks if the definition changes to out-degree zero only.

## Common Pitfalls

- Returning the surviving candidate without verifying its full row and column.
- Assuming a sink may point to itself in a matrix that is defined with zeros on the diagonal.
- Reverting to an O(n^2) full scan before using the elimination observation.
