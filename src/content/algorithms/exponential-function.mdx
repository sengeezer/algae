## Why It Surfaces

This is a compact series-approximation problem. It checks whether you can accumulate terms efficiently and reason about how truncating an infinite series affects accuracy.

## Worked Example

For `x = 1`, the first few Taylor terms are `1`, `1`, `1 / 2`, `1 / 6`, and `1 / 24`, which quickly approach the value of `e`.

## Reference Implementations

<ExamplePair>

```js
function exponentialFunction(power, terms) {
  if (![power, terms].every((value) => typeof value === 'number' && Number.isFinite(value))) {
    throw new TypeError('Power and term count must be finite numbers');
  }

  if (!Number.isInteger(terms) || terms < 0) {
    throw new RangeError('Term count must be a non-negative integer');
  }

  if (terms === 0) {
    return 1;
  }

  let total = 0;
  let factorial = 1;

  for (let index = 0; index < terms; index += 1) {
    if (index > 0) {
      factorial *= index;
    }

    total += power ** index / factorial;
  }

  return total;
}
```

## Follow-up Questions

- Reuse each term to derive the next one without recomputing `power ** index` from scratch.
- Measure how many terms are needed for a chosen error tolerance.
- Compare this series approximation with directly calling `Math.exp`.

## Common Pitfalls

- Treating the term count as a real number instead of an integer.
- Recomputing factorials from scratch inside the loop.
- Forgetting that the result is an approximation whose quality improves with more terms.
