## Why It Surfaces

This is where repeated primality checks stop scaling well. If the goal is to sum every prime below a limit, sieve preprocessing is the natural tool.

## Worked Example

Below `10`, the primes are `2`, `3`, `5`, and `7`, so the answer is `17`. A sieve marks composites in bulk and then sums the indices that remain prime.

## Reference Implementations

```js
function problem010(limit) {
  if (!Number.isInteger(limit) || limit < 2) {
    throw new RangeError('Limit must be an integer at least 2');
  }

  const isPrime = new Array(limit).fill(true);
  isPrime[0] = false;
  isPrime[1] = false;

  for (let value = 2; value * value < limit; value += 1) {
    if (!isPrime[value]) {
      continue;
    }

    for (let multiple = value * value; multiple < limit; multiple += value) {
      isPrime[multiple] = false;
    }
  }

  let sum = 0;
  for (let value = 2; value < limit; value += 1) {
    if (isPrime[value]) {
      sum += value;
    }
  }

  return sum;
}
```

## Common Pitfalls

- Using trial division for every number in the range.
- Forgetting that the prompt says “below” the limit, not “up to and including”.
- Starting sieve marking at `2 * p` instead of `p * p`.
