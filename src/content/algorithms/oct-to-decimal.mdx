## Why It Surfaces

Octal-to-decimal is the same positional parsing idea used for binary and hexadecimal, which makes it a good reinforcement problem. The only real change is the base value.

## Worked Example

`56` in octal becomes `46` in decimal because $5 \times 8 + 6 = 46$. For longer inputs, each new digit shifts the accumulated value by one place in base `8`.

## Reference Implementations

<ExamplePair>

```js
function octToDecimal(input) {
  const octal = String(input);
  if (!/^[0-7]+$/.test(octal)) {
    throw new TypeError('Octal values may only contain digits 0 through 7');
  }

  let decimal = 0;
  for (const digit of octal) {
    decimal = decimal * 8 + Number(digit);
  }

  return decimal;
}
```

```ts
export function octToDecimal(input: string | number): number {
  const octal = String(input);
  if (!/^[0-7]+$/.test(octal)) {
    throw new TypeError('Octal values may only contain digits 0 through 7');
  }

  let decimal = 0;
  for (const digit of octal) {
    decimal = decimal * 8 + Number(digit);
  }

  return decimal;
}
```

## Follow-up Questions

- Explain why Horner's method works for any base.
- Support leading zeros without changing the numeric result.
- Generalize the parser to bases between `2` and `36`.

## Common Pitfalls

- Accepting digits `8` or `9` in an octal parser.
- Treating the input as base-10 just because it uses decimal glyphs.
- Recomputing powers of eight instead of folding incrementally.
