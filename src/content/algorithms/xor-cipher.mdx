## Why It Surfaces

XOR cipher appears when an interviewer wants a very small symmetric transformation with bitwise flavor. The central insight is that `x ^ k ^ k = x`, so the same operation both encrypts and decrypts.

## Worked Example

If you XOR the character code for `A` with key `42`, you get a transformed code. XORing that transformed code with `42` again returns the original `A`.

## Reference Implementations

<ExamplePair>

```js
function xorCipher(value, key) {
  if (typeof value !== 'string' || !Number.isInteger(key)) {
    throw new TypeError('Value must be a string and key must be an integer');
  }

  return value
    .split('')
    .map((char) => String.fromCharCode(char.charCodeAt(0) ^ key))
    .join('');
}
```

```ts
export function xorCipher(value: string, key: number): string {
  if (!Number.isInteger(key)) {
    throw new TypeError('Key must be an integer');
  }

  return value
    .split('')
    .map((char) => String.fromCharCode(char.charCodeAt(0) ^ key))
    .join('');
}
```

## Follow-up Questions

- Return the transformed bytes as hex or base64 instead of raw characters.
- Explain why reusing the same XOR key is insecure.
- Extend the function to XOR against a repeated multi-byte key.

## Common Pitfalls

- Assuming the output will always stay printable text.
- Forgetting that the same operation decrypts the ciphertext.
- Mixing Unicode expectations with UTF-16 code-unit level operations.
