## Why It Surfaces

This is one of the standard backtracking templates. The crucial detail is that choosing a value does not force you to advance to the next index, because the same candidate may be reused any number of times.

## Worked Example

For candidates `[1, 2, 3]` and target `5`, the valid combinations are `[1, 1, 1, 1, 1]`, `[1, 1, 1, 2]`, `[1, 1, 3]`, `[1, 2, 2]`, and `[2, 3]`. The recursive tree keeps the index on the same value after including it so repeated picks remain legal.

## Reference Implementations

<ExamplePair>

```js
function targetSumCombinations(values, target) {
  const sorted = [...new Set(values)].sort((left, right) => left - right);
  const result = [];
  const combination = [];

  function backtrack(index, remaining) {
    if (remaining === 0) {
      result.push([...combination]);
      return;
    }

    if (remaining < 0 || index === sorted.length) {
      return;
    }

    combination.push(sorted[index]);
    backtrack(index, remaining - sorted[index]);
    combination.pop();

    backtrack(index + 1, remaining);
  }

  backtrack(0, target);
  return result;
}
```

```ts
export function targetSumCombinations(values: number[], target: number): number[][] {
  const sorted = [...new Set(values)].sort((left, right) => left - right);
  const result: number[][] = [];
  const combination: number[] = [];

  const backtrack = (index: number, remaining: number): void => {
    if (remaining === 0) {
      result.push([...combination]);
      return;
    }

    if (remaining < 0 || index === sorted.length) {
      return;
    }

    combination.push(sorted[index]);
    backtrack(index, remaining - sorted[index]);
    combination.pop();

    backtrack(index + 1, remaining);
  };

  backtrack(0, target);
  return result;
}
```

## Follow-up Questions

- How does the recursion change when each candidate may be used at most once?
- Why is keeping the same index after inclusion the defining step here?
- How would you sort the output combinations if the prompt required stable lexicographic order?

## Common Pitfalls

- Advancing to `index + 1` immediately after including a value, which incorrectly forbids reuse.
- Forgetting to prune when the remaining target goes negative.
- Emitting duplicate combinations when the input contains repeated candidates.
