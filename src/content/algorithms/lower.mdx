## Why It Surfaces

This is the symmetric companion to uppercase conversion. The interview value is usually in explaining what ASCII arithmetic buys you and where that hand-written approach stops being correct for full Unicode text.

## Worked Example

`"He_LLO"` becomes `"he_llo"`: uppercase letters shift into lowercase, while underscores and existing lowercase letters remain unchanged.

## Reference Implementations

<ExamplePair>

```js
function lower(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  return value.replace(/[A-Z]/g, (char) =>
    String.fromCharCode(char.charCodeAt(0) + 32),
  );
}
```

```ts
export function lower(value: string): string {
  return value.replace(/[A-Z]/g, (char) =>
    String.fromCharCode(char.charCodeAt(0) + 32),
  );
}
```

</ExamplePair>

## Follow-up Questions

- Compare this manual ASCII approach with `value.toLowerCase()`.
- Implement locale-aware lowercase conversion requirements.
- Handle both uppercase and lowercase conversion behind one reusable helper.

## Common Pitfalls

- Assuming ASCII conversion is enough for every language.
- Using the wrong code-point offset and corrupting characters.
- Forgetting that non-letter characters should stay unchanged.
