## Why It Surfaces

Longest path is hard on general graphs, so interviewers often add the DAG restriction to make the problem solvable. Once you have a topological order, the problem becomes a clean relaxation pass where every edge is processed only after its predecessor distance is finalized.

## Worked Example

If the graph is already topologically ordered as `0, 1, 2, 3`, then any longest path to `3` must come through one of the earlier nodes. That lets you update `distance[3]` by taking the best among `distance[1] + w(1,3)` and `distance[2] + w(2,3)` without revisiting settled cycles, because no cycles exist.

```js
function longestPathInADirectedAcyclicGraph(graph, source) {
  const size = graph.length;
  const visited = new Array(size).fill(false);
  const order = [];

  function dfs(node) {
    visited[node] = true;
    for (const [neighbor] of graph[node]) {
      if (!visited[neighbor]) {
        dfs(neighbor);
      }
    }
    order.push(node);
  }

  for (let node = 0; node < size; node += 1) {
    if (!visited[node]) {
      dfs(node);
    }
  }

  const distance = new Array(size).fill(Number.NEGATIVE_INFINITY);
  distance[source] = 0;

  while (order.length > 0) {
    const node = order.pop();
    if (distance[node] === Number.NEGATIVE_INFINITY) {
      continue;
    }

    for (const [neighbor, weight] of graph[node]) {
      distance[neighbor] = Math.max(distance[neighbor], distance[node] + weight);
    }
  }

  return distance;
}
```

```ts
export function longestPathInADirectedAcyclicGraph(
  graph: Array<Array<[number, number]>>,
  source: number,
): number[] {
  const size = graph.length;
  const visited = new Array<boolean>(size).fill(false);
  const order: number[] = [];

  function dfs(node: number): void {
    visited[node] = true;
    for (const [neighbor] of graph[node]) {
      if (!visited[neighbor]) {
        dfs(neighbor);
      }
    }
    order.push(node);
  }

  for (let node = 0; node < size; node += 1) {
    if (!visited[node]) {
      dfs(node);
    }
  }

  const distance = new Array<number>(size).fill(Number.NEGATIVE_INFINITY);
  distance[source] = 0;

  while (order.length > 0) {
    const node = order.pop()!;
    if (distance[node] === Number.NEGATIVE_INFINITY) {
      continue;
    }

    for (const [neighbor, weight] of graph[node]) {
      distance[neighbor] = Math.max(distance[neighbor], distance[node] + weight);
    }
  }

  return distance;
}
```

## Follow-up Questions

- Reconstruct the actual longest path, not just the distances.
- Explain why the same idea fails on graphs with cycles.

## Common Pitfalls

- Applying the algorithm to a graph that is not acyclic.
- Using minimum relaxation instead of maximum relaxation.
- Forgetting to initialize unreachable distances to negative infinity.
