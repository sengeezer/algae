## Why It Surfaces

This problem is the enumeration counterpart to subset-sum DP. Instead of asking whether a target can be reached, the interviewer wants every valid subset, which shifts the focus back to recursive search and pruning.

## Worked Example

For `nums = [2, 3, 5, 6, 8]` and `target = 10`, valid answers include `[2, 3, 5]` and `[2, 8]`. Once a branch reaches `11` or `12`, it can stop immediately because all inputs are non-negative.

<Callout title="Pruning assumption" tone="warning">
  The simple `sum > target` prune only works when the remaining values cannot lower the total. If negative numbers are allowed, the search logic must change.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function sumOfSubset(nums, target) {
  const results = [];
  const path = [];

  function dfs(index, sum) {
    if (sum === target) {
      results.push([...path]);
      return;
    }

    if (sum > target) {
      return;
    }

    for (let next = index; next < nums.length; next += 1) {
      path.push(nums[next]);
      dfs(next + 1, sum + nums[next]);
      path.pop();
    }
  }

  dfs(0, 0);
  return results;
}
```

## Follow-up Questions

- Return only whether at least one subset exists.
- Count valid subsets instead of listing them.
- Handle duplicate values without returning duplicate subsets.

## Common Pitfalls

- Forgetting that enumeration needs the path itself, not just the running sum.
- Using `sum > target` pruning when negative values are present.
- Reusing the same element more than once in a supposedly 0/1 subset search.
