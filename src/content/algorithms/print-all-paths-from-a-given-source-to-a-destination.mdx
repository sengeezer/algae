## Why It Surfaces

This is the enumeration version of path search. Once the graph is a DAG, the recursion can safely grow a current route, emit it at the destination, and then pop the last node to continue exploring siblings.

## Worked Example

In a graph with edges `0 -> 1`, `1 -> 2`, `1 -> 3`, and `2 -> 3`, the paths from `0` to `3` are `[0, 1, 3]` and `[0, 1, 2, 3]`. DFS naturally discovers one, backtracks, and then discovers the other.

## Reference Implementations

<ExamplePair>

```js
function printAllPathsFromAGivenSourceToADestination(vertexCount, edges, source, destination) {
  const graph = Array.from({ length: vertexCount }, () => []);
  for (const [from, to] of edges) {
    graph[from].push(to);
  }

  const path = [];
  const result = [];

  function dfs(node) {
    path.push(node);

    if (node === destination) {
      result.push([...path]);
    } else {
      for (const next of graph[node]) {
        dfs(next);
      }
    }

    path.pop();
  }

  dfs(source);
  return result;
}
```

## Common Pitfalls

- Forgetting to remove the current node from the path during backtracking.
- Reusing the same `path` array reference in the result without copying it.
- Applying this directly to cyclic graphs without a visited-on-path check.
