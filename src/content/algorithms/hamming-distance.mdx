## Why It Surfaces

Hamming distance is the simplest string-difference metric, but it is only correct under a strict equal-length assumption. That makes it a useful contrast with edit distance, where insertions and deletions are allowed.

## Worked Example

Comparing `"karolin"` with `"kathrin"` yields mismatches at positions `3`, `4`, and `5`, so the Hamming distance is `3`.

## Reference Implementations

<ExamplePair>

```js
function hammingDistance(left, right) {
  if (left.length !== right.length) {
    throw new Error('Strings must be of the same length');
  }

  let distance = 0;
  for (let index = 0; index < left.length; index += 1) {
    if (left[index] !== right[index]) {
      distance += 1;
    }
  }

  return distance;
}
```

## Follow-up Questions

- Compute the distance between integers with XOR and bit counting.
- Compare Hamming distance with edit distance on strings of different lengths.
- Weight different positions differently instead of counting every mismatch equally.

## Common Pitfalls

- Applying the metric to strings with different lengths.
- Confusing mismatched positions with the number of distinct characters.
- Reaching for dynamic programming when a single linear scan is enough.
