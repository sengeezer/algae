## Why It Surfaces

This is a good example of turning factorization into a shrinking search problem. Once a small factor is removed, the remaining value becomes smaller and can eventually be the largest prime factor itself.

## Worked Example

For `13195`, the prime factors are `5`, `7`, `13`, and `29`, so the answer is `29`. The shrinking version repeatedly divides out small factors until only the largest prime factor remains.

## Reference Implementations

<ExamplePair>

```js
function problem003(value = 600851475143) {
  if (!Number.isInteger(value) || value < 2) {
    throw new RangeError('Value must be an integer at least 2');
  }

  let remaining = value;
  let divisor = 2;

  while (divisor * divisor <= remaining) {
    if (remaining % divisor === 0) {
      remaining /= divisor;
    } else {
      divisor += 1;
    }
  }

  return remaining;
}
```

## Follow-up Questions

- Return the full list of prime factors instead of only the largest one.
- Skip even divisors after handling `2`.
- Explain why the leftover remainder is prime when the loop ends.

## Common Pitfalls

- Continuing to test divisors against the original input after the remainder has shrunk.
- Treating `1` as a prime factor.
- Using an upper bound that does not update as factors are removed.
