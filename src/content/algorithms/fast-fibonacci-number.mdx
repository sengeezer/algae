## Why It Surfaces

The ordinary Fibonacci recurrence is often used as a warm-up, then the interviewer tightens the constraints. Fast doubling is the next step when `O(n)` iteration is no longer good enough.

## Worked Example

If you know `F(k)` and `F(k + 1)`, then you can compute `F(2k)` and `F(2k + 1)` directly. Recurse on `n / 2`, then rebuild the answer from those two formulas instead of stepping through every value up to `n`.

## Reference Implementations

<ExamplePair>

```js
function fibonacciPair(index) {
  if (index === 0) {
    return [0, 1];
  }

  const [a, b] = fibonacciPair(Math.floor(index / 2));
  const c = a * (2 * b - a);
  const d = a * a + b * b;

  if (index % 2 === 0) {
    return [c, d];
  }

  return [d, c + d];
}

function fastFibonacci(index) {
  return fibonacciPair(index)[0];
}
```

export function fastFibonacci(index: number): number {

## Follow-up Questions

- Return the value modulo a large prime.
- Compare fast doubling with matrix exponentiation.
- Handle the first few base cases explicitly and explain why they still fit the formulas.

## Common Pitfalls

- Reverting to linear recursion and losing the whole optimization.
- Mixing up the fast-doubling formulas for even and odd indices.
- Ignoring integer overflow when the language has fixed-size numeric types.
