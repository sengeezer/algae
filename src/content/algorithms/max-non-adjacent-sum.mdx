## Why It Surfaces

Max Non Adjacent Sum is the stripped-down version of many cooldown or spacing problems. The whole recurrence comes from one question: if you take this value, what previous state is still legal?

## Worked Example

For `[1, 5, 3, 7, 2]`, taking `5` blocks `3`, then taking `7` gives a total of `12`. The DP tracks the best answer when the current position is included and when it is skipped.

## Reference Implementations

<ExamplePair>

```js
function maxNonAdjacentSum(nums) {
  let take = 0;
  let skip = 0;

  for (const value of nums) {
    const nextTake = skip + value;
    const nextSkip = Math.max(skip, take);
    take = nextTake;
    skip = nextSkip;
  }

  return Math.max(take, skip);
}
```

```ts
export function maxNonAdjacentSum(nums: number[]): number {
  let take = 0;
  let skip = 0;

  for (const value of nums) {
    const nextTake = skip + value;
    const nextSkip = Math.max(skip, take);
    take = nextTake;
    skip = nextSkip;
  }

  return Math.max(take, skip);
}
```

## Follow-up Questions

- Return the chosen indices as well as the best total.
- Explain how this maps directly to House Robber.
- Clarify whether taking nothing is allowed when all values are negative.

## Common Pitfalls

- Treating this as a contiguous-subarray problem like Kadane.
- Using the updated `take` value too early when computing `skip`.
- Forgetting that each choice only conflicts with immediate neighbors.
