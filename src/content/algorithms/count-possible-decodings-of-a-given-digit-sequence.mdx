## Why It Surfaces

Decode Ways is a useful interview problem because the recurrence is small, but the edge cases are not. A prefix can extend by one digit or two digits, yet zeroes invalidate many apparently natural moves, so the DP has to combine counting with local parsing rules.

## Worked Example

For `digits = "226"`, the valid decodings are `"2|2|6"`, `"22|6"`, and `"2|26"`, so the answer is `3`. The two-digit chunks work only because `22` and `26` both stay within the `10` to `26` range.

## Reference Implementations

<ExamplePair>

```js
function countPossibleDecodingsOfAGivenDigitSequence(digits) {
  if (!/^\d*$/.test(digits)) {
    throw new TypeError('digits must contain only numeric characters');
  }

  if (digits.length === 0 || digits[0] === '0') {
    return 0;
  }

  let previousTwo = 1;
  let previousOne = 1;

  for (let index = 1; index < digits.length; index += 1) {
    let current = 0;

    if (digits[index] !== '0') {
      current += previousOne;
    }

    const pair = Number(digits.slice(index - 1, index + 1));
    if (digits[index - 1] !== '0' && pair >= 10 && pair <= 26) {
      current += previousTwo;
    }

    previousTwo = previousOne;
    previousOne = current;
  }

  return previousOne;
}
```

## Follow-up Questions

- Return one actual decoding string instead of only the count.
- Allow `*` wildcards that can stand for multiple digits.
- Compare the rolling-state version with a full prefix DP array.

## Common Pitfalls

- Treating `0` as a valid one-digit decoding.
- Allowing two-digit values above `26` like `27` or `30`.
- Forgetting that an empty-prefix base case of `1` is still needed inside the DP.
