## Why It Surfaces

This looks like a graph problem, but the hidden simplification is strong: every city has at most one outgoing ticket and the data is acyclic. That means you do not need a general Eulerian-path construction. You only need to find the unique start and walk the chain.

## Worked Example

If the unordered tickets are `Bombay -> Delhi`, `Delhi -> Goa`, `Goa -> Chennai`, and `Chennai -> Bangalore`, then `Bombay` is the only source that never appears as a destination. Start there and repeatedly follow the hash map to rebuild the full itinerary.

## Reference Implementations

<ExamplePair>

```js
function findItineraryFromAGivenListOfTickets(tickets) {
  if (tickets.length === 0) {
    return [];
  }

  const nextBySource = new Map();
  const destinations = new Set();

  for (const [from, to] of tickets) {
    nextBySource.set(from, to);
    destinations.add(to);
  }

  let start = null;
  for (const [from] of tickets) {
    if (!destinations.has(from)) {
      start = from;
      break;
    }
  }

  if (start === null) {
    return [];
  }

  const itinerary = [start];
  const seen = new Set([start]);

  while (nextBySource.has(start)) {
    const next = nextBySource.get(start);
    if (seen.has(next)) {
      return [];
    }

    itinerary.push(next);
    seen.add(next);
    start = next;
  }

  return itinerary.length === tickets.length + 1 ? itinerary : [];
}
```

```ts
export function findItineraryFromAGivenListOfTickets(
  tickets: Array<[string, string]>,
): string[] {
  if (tickets.length === 0) {
    return [];
  }

  const nextBySource = new Map<string, string>();
  const destinations = new Set<string>();

  for (const [from, to] of tickets) {
    nextBySource.set(from, to);
    destinations.add(to);
  }

  let start: string | null = null;
  for (const [from] of tickets) {
    if (!destinations.has(from)) {
      start = from;
      break;
    }
  }

  if (start === null) {
    return [];
  }

  const itinerary: string[] = [start];
  const seen = new Set<string>([start]);

  while (nextBySource.has(start)) {
    const next = nextBySource.get(start)!;
    if (seen.has(next)) {
      return [];
    }

    itinerary.push(next);
    seen.add(next);
    start = next;
  }

  return itinerary.length === tickets.length + 1 ? itinerary : [];
}
```

## Follow-up Questions

## Common Pitfalls

- Starting from an arbitrary source instead of the source missing from the destination set.
- Using a general graph traversal when the tickets already form a unique chain.
- Forgetting that duplicate outgoing tickets from the same city violate the simplifying assumptions.
