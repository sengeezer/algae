## Why It Surfaces

This is the smallest reusable bit-inspection question. Interviewers use it to check whether you understand how masks and shifts isolate a single position without converting the whole number into a binary string first.

## Worked Example

`7` is `111` in binary. At zero-based position `2`, the bit is `1`, so the answer is `true`. By contrast, `5` is `101`, so position `1` is `0` and the answer is `false`.

## Reference Implementations

<ExamplePair>

```js
function kThBitIsSetOrNot(value, bitPosition) {
  if (!Number.isInteger(value) || value < 0 || !Number.isInteger(bitPosition) || bitPosition < 0) {
    throw new TypeError('Value must be a non-negative integer and bitPosition must be a non-negative integer');
  }

  return ((value >> bitPosition) & 1) === 1;
}
```

```ts
export function kThBitIsSetOrNot(value: number, bitPosition: number): boolean {
  if (!Number.isInteger(value) || value < 0 || !Number.isInteger(bitPosition) || bitPosition < 0) {
    throw new TypeError('Value must be a non-negative integer and bitPosition must be a non-negative integer');
  }

  return ((value >> bitPosition) & 1) === 1;
}
```

## Follow-up Questions

- Rewrite the same test with a left-shifted mask instead of a right shift.
- Adjust the function if the interviewer uses one-based bit indexing.
- Use the same logic to set, clear, or toggle the chosen bit.

## Common Pitfalls

- Mixing up zero-based and one-based bit positions.
- Forgetting that negative values and bitwise coercion interact awkwardly in JavaScript.
- Converting the whole integer to a binary string for a one-bit check.
