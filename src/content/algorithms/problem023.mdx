## Why It Surfaces

This problem becomes manageable once the abundant numbers are known. After that, the task is just reachability: which totals can be formed as the sum of two members of that set?

## Worked Example

`12` is the smallest abundant number because its proper divisors sum to `16`. Once enough abundant numbers are collected, mark every achievable pair sum and add only the positive integers that remain unmarked.

function sumProperDivisors(value) {
  if (value === 1) {
    return 0;
  }

  let sum = 1;
  const root = Math.floor(Math.sqrt(value));

  for (let divisor = 2; divisor <= root; divisor += 1) {
    if (value % divisor !== 0) {
      continue;
    }

    const paired = value / divisor;
    sum += divisor;
    if (paired !== divisor) {
      sum += paired;
    }
  }

  return sum;
}

function problem023(limit = 28123) {
  if (!Number.isInteger(limit) || limit < 1) {
    throw new RangeError('Limit must be a positive integer');
  }

  const abundant = [];
  for (let value = 1; value <= limit; value += 1) {
    if (sumProperDivisors(value) > value) {
      abundant.push(value);
    }
  }

  const writable = new Array(limit + 1).fill(false);

  for (let left = 0; left < abundant.length; left += 1) {
    for (let right = left; right < abundant.length; right += 1) {
      const sum = abundant[left] + abundant[right];
      if (sum > limit) {
        break;
      }
      writable[sum] = true;
    }
  }

  let total = 0;
  for (let value = 1; value <= limit; value += 1) {
    if (!writable[value]) {
      total += value;
    }
  }

  return total;
}

## Follow-up Questions

- Replace repeated divisor scans with a sieve that precomputes aliquot sums for all values up to the limit.
- Return the non-representable values themselves, not only their sum.
- Explain why the inner pair loop can break as soon as the sum exceeds the limit.

## Common Pitfalls

- Rechecking whether a value is abundant from scratch during every pair test.
- Enumerating both `(a, b)` and `(b, a)` when one ordering is enough.
- Forgetting that the proven upper bound is part of the problem setup.
