## Why It Surfaces

This is the simplest selection problem, but interviewers still use it to check whether you reach for the right linear pattern immediately instead of over-solving with sorting or extra structures.

## Worked Example

For `[4, 1, 9, 3]`, start with `4` as the running maximum, keep it when `1` arrives, replace it with `9`, and ignore `3` because it is smaller than the best value already seen.

## Reference Implementations

<ExamplePair>

```js
function findMaximumInArray(values) {
  if (!Array.isArray(values) || values.length === 0) {
    throw new TypeError('Expected a non-empty array');
  }

  let best = values[0];

  for (let index = 1; index < values.length; index += 1) {
    if (values[index] > best) {
      best = values[index];
    }
  }

  return best;
}
```

## Follow-up Questions

- Track both the maximum value and its index.
- Extend the scan to return the top two distinct values.
- Explain when `Math.max(...values)` becomes a worse choice than a loop.

## Common Pitfalls

- Sorting the array when only one maximum is needed.
- Forgetting to guard against an empty array.
- Using the spread operator blindly on very large arrays.
