## Why It Surfaces

Decimal-to-hex conversion is the natural follow-up once repeated division in base 2 makes sense. The only new idea is the digit map for remainders 10 through 15.

## Worked Example

For `77`, repeated division by `16` gives remainder `13` then remainder `4`. Mapping `13` to `D` and reversing the collected digits yields `4D`.

## Reference Implementations

<ExamplePair>

```js
const HEX_DIGITS = '0123456789ABCDEF';

function decimalToHex(value) {
  if (!Number.isInteger(value) || value < 0) {
    throw new RangeError('The input must be a non-negative integer');
  }

  if (value === 0) {
    return '0';
  }

  const digits = [];
  let current = value;

  while (current > 0) {
    digits.push(HEX_DIGITS[current % 16]);
    current = Math.floor(current / 16);
  }

  return digits.reverse().join('');
}
```

## Follow-up Questions

- Explain why the least significant digit appears first during division.
- Compare a manual digit map with a generalized arbitrary-base alphabet.
- Emit lowercase hex or fixed-width padded output for downstream protocols.

## Common Pitfalls

- Forgetting the special case for zero.
- Mishandling remainders above `9`.
- Using floating-point input where repeated division assumes integers.
