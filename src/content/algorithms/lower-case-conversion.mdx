## Why It Surfaces

Lowercasing looks trivial when a standard library exists, but it is still a useful character-code exercise. It turns abstract ASCII tables into a concrete rule: lowercase letters are `32` positions above uppercase letters.

## Worked Example

`AlGae 123` becomes `algae 123`. Only characters in the uppercase ASCII range change; everything else is copied through unchanged.

## Reference Implementations

<ExamplePair>

```js
function lowerCaseConversion(input) {
  if (typeof input !== 'string') {
    throw new TypeError('Expected a string input');
  }

  return input
    .split('')
    .map((char) => {
      const code = char.charCodeAt(0);
      if (code >= 65 && code <= 90) {
        return String.fromCharCode(code + 32);
      }
      return char;
    })
    .join('');
}
```

```ts

}
```

## Common Pitfalls

- Assuming the ASCII offset works for all languages and scripts.
- Mutating non-letter characters that should pass through unchanged.
- Forgetting to validate non-string input in a utility API.
