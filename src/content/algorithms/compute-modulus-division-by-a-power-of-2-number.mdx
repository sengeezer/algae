## Why It Surfaces

This is the classic example of how binary structure can replace arithmetic. Once the divisor is a power of two, the remainder depends only on the low bits, so a mask gives the answer immediately.

## Worked Example

For `n = 6` and `d = 4`, compute `6 & (4 - 1)`, which is `110 & 011 = 010`, so the remainder is `2`.

## Reference Implementations

<ExamplePair>

```js
function computeModulusDivisionByAPowerOf2Number(value, divisor) {
  if (!Number.isInteger(value) || value < 0 || !Number.isInteger(divisor) || divisor <= 0) {
    throw new TypeError('Value must be a non-negative integer and divisor must be a positive integer');
  }

  if ((divisor & (divisor - 1)) !== 0) {
    throw new RangeError('Divisor must be a power of two');
  }

  return value & (divisor - 1);
}
```

```ts
export function computeModulusDivisionByAPowerOf2Number(value: number, divisor: number): number {
  if (!Number.isInteger(value) || value < 0 || !Number.isInteger(divisor) || divisor <= 0) {
    throw new TypeError('Value must be a non-negative integer and divisor must be a positive integer');
  }

  if ((divisor & (divisor - 1)) !== 0) {
    throw new RangeError('Divisor must be a power of two');
  }

  return value & (divisor - 1);
}
```

## Common Pitfalls

- Applying the mask trick when the divisor is not a power of two.
- Forgetting that `divisor = 0` is invalid.
- Treating the trick as a general replacement for modulo instead of a constrained shortcut.
