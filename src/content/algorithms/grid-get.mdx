## Why It Surfaces

This is a compact coordinate-mapping helper. It matters because many grid problems flatten a 2D board into a 1D array and still expect you to recover row and column positions cleanly.

## Worked Example

With `10` columns, index `3` maps to `(4, 1)` in one-based coordinates, while index `49` maps to `(10, 5)`.

## Reference Implementations

<ExamplePair>

```js
function validateGridInput(columns, index) {
  if (!Number.isInteger(columns) || columns <= 0) {
    throw new RangeError('Columns must be a positive integer');
  }

  if (!Number.isInteger(index) || index < 0) {
    throw new RangeError('Index must be a non-negative integer');
  }
}

function gridGetX(columns, index) {
  validateGridInput(columns, index);
  return (index % columns) + 1;
}

function gridGetY(columns, index) {
  validateGridInput(columns, index);
  return Math.floor(index / columns) + 1;
}
```

export function gridGetX(columns: number, index: number): number {

  return (index % columns) + 1;
}

  return Math.floor(index / columns) + 1;

## Follow-up Questions

- Return zero-based coordinates instead of one-based ones.
- Convert `(x, y)` back into a flat index.
- Use the recovered coordinates to measure distance between two cells.

## Common Pitfalls

- Mixing zero-based and one-based coordinate systems.
- Repeatedly subtracting `columns` when division and modulo already solve the problem.
- Forgetting to validate that the column count is positive.
