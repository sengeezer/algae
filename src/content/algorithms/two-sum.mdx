## Why It Surfaces

Two Sum is a standard signal problem for hash-based thinking. The important shift is recognizing that once you know the target and one number, the missing value is determined immediately.

## Worked Example

For `nums = [2, 7, 11, 15]` and `target = 9`, start with `2` and remember its index. At `7`, the complement `2` has already been seen, so the answer is `[0, 1]`.

## Reference Implementations

<ExamplePair>

```js
function twoSum(nums, target) {
  if (!Array.isArray(nums)) {
    throw new TypeError('Nums must be an array');
  }

  const seen = new Map();

  for (let index = 0; index < nums.length; index += 1) {
    const value = nums[index];
    const complement = target - value;

    if (seen.has(complement)) {
      return [seen.get(complement), index];
    }

    seen.set(value, index);
  }

  return [];
}
```

```ts
export function twoSum(nums: number[], target: number): number[] {
  const seen = new Map<number, number>();

  for (let index = 0; index < nums.length; index += 1) {
    const value = nums[index];
    const complement = target - value;

    if (seen.has(complement)) {
      return [seen.get(complement) as number, index];
    }

    seen.set(value, index);
  }

  return [];
}
```

## Common Pitfalls

- Storing the current value before checking its complement and accidentally reusing the same index.
- Forgetting that duplicate values can still form the answer.
- Returning all pairs when the prompt asks for only one valid pair.
