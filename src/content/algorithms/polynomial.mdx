## Why It Surfaces

This is the compact representation interviewers use before they ask you to add, multiply, or evaluate algebraic expressions programmatically.

## Worked Example

The coefficient array `[1, -3, 2]` means `1 - 3x + 2x^2`, because the array index is the exponent. Evaluating that polynomial at `x = 4` gives `1 - 12 + 32 = 21`.

```js
class Polynomial {
  constructor(coefficients) {
    if (!Array.isArray(coefficients) || coefficients.length === 0) {
      throw new TypeError('Expected a non-empty coefficient array');
    }

    if (!coefficients.every((value) => typeof value === 'number' && Number.isFinite(value))) {
      throw new TypeError('Coefficients must be finite numbers');
    }

    this.coefficients = [...coefficients];
  }

  display() {
    const terms = this.coefficients
      .map((coefficient, exponent) => {
        if (coefficient === 0) {
          return null;
        }

        if (exponent === 0) {
          return `(${coefficient})`;
        }

        if (exponent === 1) {
          return `(${coefficient}x)`;
        }

        return `(${coefficient}x^${exponent})`;
      })
      .filter(Boolean)
      .reverse();

    return terms.length > 0 ? terms.join(' + ') : '0';
  }

  evaluate(value) {
    if (!Number.isFinite(value)) {
      throw new TypeError('Evaluation point must be a finite number');
    }

    return this.coefficients.reduce(
      (total, coefficient, exponent) => total + coefficient * value ** exponent,
      0,
    );
  }
}
```

## Follow-up Questions

- Replace direct powered evaluation with Horner's method.
- Add polynomial addition or multiplication using the same coefficient representation.
- Explain how trailing zero coefficients affect the displayed form.

## Common Pitfalls

- Forgetting that the array index is the exponent.
- Leaving zero-coefficient terms in the rendered string.
- Evaluating at non-finite inputs without defining the behavior.
