## Why It Surfaces

Roman-to-decimal conversion is the inverse of Roman encoding, but parsing is slightly trickier because smaller symbols sometimes subtract instead of add. A reverse scan makes that rule easy to apply.

## Worked Example

For `MCMXCIV`, reading from right to left means `V` adds `5`, `I` subtracts `1`, `C` adds `100`, `X` subtracts `10`, and so on until the total becomes `1994`.

## Reference Implementations

<ExamplePair>

```js
const romanValues = {
  I: 1,
  V: 5,
  X: 10,
  L: 50,
  C: 100,
  D: 500,
  M: 1000,
};

function romanToDecimal(roman) {
  if (!/^[IVXLCDM]+$/i.test(roman)) {
    throw new TypeError('Roman numerals may only contain I, V, X, L, C, D, and M');
  }

  let total = 0;
  let largestSeen = 0;

  for (let index = roman.length - 1; index >= 0; index -= 1) {
    const value = romanValues[roman[index].toUpperCase()];

    if (value < largestSeen) {
      total -= value;
    } else {
      total += value;
      largestSeen = value;
    }
  }

  return total;
}
```

## Follow-up Questions

- Explain why scanning from right to left simplifies subtractive notation.
- Add stricter validation for malformed numerals like `IC`.
- Pair the parser with Roman encoding in a round-trip test.

## Common Pitfalls

- Treating every symbol as additive and missing subtractive forms.
- Ignoring lowercase input normalization.
- Accepting invalid Roman strings without defining what the parser should do.
