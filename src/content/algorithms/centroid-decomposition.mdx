## Why It Surfaces

Centroid decomposition is useful when a static tree needs repeated global queries but no single root provides a balanced view. By recentering recursively around centroids, the tree turns into a shallower centroid tree that is easier to climb for aggregate updates and queries.

## Worked Example

If a 9-node tree has a centroid whose removal leaves components of sizes 4, 3, and 1, that centroid becomes the new decomposition root. Each component is then decomposed independently, and the recursion depth stays small because no child component exceeds half the current size.

```js
function centroidDecomposition(tree) {
  const size = tree.length;
  const removed = new Array(size).fill(false);
  const subtree = new Array(size).fill(0);
  const centroidParent = new Array(size).fill(-1);

  function computeSize(node, parent) {
    subtree[node] = 1;

    for (const neighbor of tree[node]) {
      if (neighbor !== parent && !removed[neighbor]) {
        computeSize(neighbor, node);
        subtree[node] += subtree[neighbor];
      }
    }
  }

  function findCentroid(node, parent, totalSize) {
    for (const neighbor of tree[node]) {
      if (neighbor !== parent && !removed[neighbor] && subtree[neighbor] > totalSize / 2) {
        return findCentroid(neighbor, node, totalSize);
      }
    }

    return node;
  }

  function decompose(entry, parent) {
    computeSize(entry, -1);
    const centroid = findCentroid(entry, -1, subtree[entry]);
    removed[centroid] = true;
    centroidParent[centroid] = parent;

    for (const neighbor of tree[centroid]) {
      if (!removed[neighbor]) {
        decompose(neighbor, centroid);
      }
    }

    return centroid;
  }

  const root = decompose(0, -1);
  return { root, centroidParent };
}
```

## Follow-up Questions

- Use the centroid tree for nearest red-node queries or distance updates.
- Compare centroid decomposition with heavy-light decomposition.
- Explain why centroid-tree depth is O(log n).

## Common Pitfalls

- Forgetting to mark centroids as removed before recursing.
- Reusing stale subtree sizes after the tree is partitioned.
- Confusing centroid decomposition with binary-tree balancing.
