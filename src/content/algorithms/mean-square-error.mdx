## Why It Surfaces

Mean square error is one of the most common loss functions in prediction tasks. The coding task is simple, but it still checks whether you validate paired-array lengths and use squared differences rather than absolute ones.

## Worked Example

If predicted values are `[2, 4, 6]` and expected values are `[1, 4, 7]`, the squared errors are `1`, `0`, and `1`, so the mean square error is `2 / 3`.

<ExamplePair>

```js
function meanSquareError(predicted, expected) {
  if (!Array.isArray(predicted) || !Array.isArray(expected)) {
    throw new TypeError('Expected two arrays');
  }

  if (predicted.length === 0 || predicted.length !== expected.length) {
    throw new RangeError('Arrays must be non-empty and have the same length');
  }

  if (![predicted, expected].every((list) => list.every((value) => typeof value === 'number' && Number.isFinite(value)))) {
    throw new TypeError('Arrays must contain only finite numbers');
  }

  let error = 0;

  for (let index = 0; index < expected.length; index += 1) {
    error += (expected[index] - predicted[index]) ** 2;
  }

  return error / expected.length;
}
```

## Follow-up Questions

- Compare mean square error with mean absolute error.
- Return root mean square error instead.
- Explain why large errors matter more under squared loss.

## Common Pitfalls

- Forgetting to validate equal lengths before indexing both arrays.
- Using absolute errors instead of squared errors.
- Allowing empty arrays and dividing by zero.
