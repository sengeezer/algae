## Why It Surfaces

Absolute value is tiny, but it is one of the most reused normalization steps in algorithm work. Interviewers sometimes use it as a warm-up because it exposes whether you are explicit about inputs like `NaN` and signed zero.

## Worked Example

`abs(-10)` returns `10`, `abs(50)` returns `50`, and `abs(-0)` should normalize to `0`. The magnitude stays the same; only the sign handling changes.

## Reference Implementations

<ExamplePair>

```js
function abs(value) {
  if (typeof value !== 'number' || Number.isNaN(value)) {
    throw new TypeError('Expected a numeric input');
  }

  if (value === 0) {
    return 0;
  }

  return value < 0 ? -value : value;
}
```

```ts
export function abs(value: number): number {
  if (Number.isNaN(value)) {
    throw new TypeError('Expected a numeric input');
  }

  if (value === 0) {
    return 0;
  }

  return value < 0 ? -value : value;
}
```

## Common Pitfalls

- Quietly coercing non-numeric inputs into numbers.
- Returning `-0` instead of `0` for signed zero.
- Mixing magnitude normalization with unrelated rounding behavior.
