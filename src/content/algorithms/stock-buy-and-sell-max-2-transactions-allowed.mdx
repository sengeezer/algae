## Why It Surfaces

This is the smallest stock-trading variant that needs real state instead of a single running minimum. The insight is that each price only needs to improve one of four milestones: after the first buy, after the first sell, after the second buy, or after the second sell.

## Worked Example

For `prices = [10, 22, 5, 75, 65, 80]`, the answer is `87`. The best plan is buy at `10`, sell at `22`, then buy at `5` and sell at `80`.

## Reference Implementations

<ExamplePair>

```js
function stockBuyAndSellMax2TransactionsAllowed(prices) {
  let firstBuy = Number.NEGATIVE_INFINITY;
  let firstSell = 0;
  let secondBuy = Number.NEGATIVE_INFINITY;
  let secondSell = 0;

  for (const price of prices) {
    firstBuy = Math.max(firstBuy, -price);
    firstSell = Math.max(firstSell, firstBuy + price);
    secondBuy = Math.max(secondBuy, firstSell - price);
    secondSell = Math.max(secondSell, secondBuy + price);
  }

  return secondSell;
}
```

## Common Pitfalls

- Allowing overlapping transactions by buying again before selling the previous stock.
- Resetting the second buy state as if it were independent of the first completed sell.
- Solving the unlimited-transactions variant by mistake.
