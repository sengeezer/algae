## Why It Surfaces

Gray codes show up when the order of the bit patterns matters as much as the patterns themselves. The construction is memorable because it reuses the old answer by reflecting it rather than recomputing each code independently.

## Worked Example

For `n = 2`, start from `[0, 1]`. Reflect that list to get `[1, 0]`, prefix the reflected values with the new highest bit to get `[3, 2]`, and append them for `[0, 1, 3, 2]`.

## Reference Implementations

<ExamplePair>

```js
function grayCodes(bitCount) {
  if (bitCount <= 0) {
    return [0];
  }

  const codes = [0, 1];

  for (let bit = 1; bit < bitCount; bit += 1) {
    const highestBit = 1 << bit;

    for (let index = codes.length - 1; index >= 0; index -= 1) {
      codes.push(highestBit | codes[index]);
    }
  }

  return codes;
}
```

## Follow-up Questions

- Convert each decimal code into a fixed-width binary string.
- Explain why reflection preserves the one-bit-difference property.
- Generate Gray codes directly from the formula `i ^ (i >> 1)` and compare it with reflection.

## Common Pitfalls

- Appending the old sequence in forward order instead of reverse order.
- Forgetting that `n <= 0` still has a valid base sequence `[0]`.
- Returning ordinary binary counting order instead of Gray-code order.
