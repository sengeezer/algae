## Why It Surfaces

This is a clean entry point into geometric predicates. Instead of inventing a custom case split, you compute the triangle area once, then see whether the query point decomposes that triangle into three smaller pieces whose total area matches the original.

## Worked Example

For `A = [0, 0]`, `B = [10, 30]`, `C = [20, 0]`, and `P = [10, 15]`, the three sub-triangle areas add up exactly to the area of `ABC`, so `P` lies inside the triangle.

## Reference Implementations

<ExamplePair>

```js
function checkWhetherAGivenPointLiesInsideATriangleOrNot(a, b, c, point) {
  function twiceArea(first, second, third) {
    return Math.abs(
      first[0] * (second[1] - third[1])
      + second[0] * (third[1] - first[1])
      + third[0] * (first[1] - second[1])
    );
  }

  const whole = twiceArea(a, b, c);

  if (whole === 0) {
    return false;
  }

  const part1 = twiceArea(point, b, c);
  const part2 = twiceArea(a, point, c);
  const part3 = twiceArea(a, b, point);

  return Math.abs(whole - (part1 + part2 + part3)) < 1e-9;
}
```

```ts
type Point = [number, number];

export function checkWhetherAGivenPointLiesInsideATriangleOrNot(
  a: Point,
  b: Point,
  c: Point,
  point: Point,
): boolean {
  const twiceArea = (first: Point, second: Point, third: Point): number => {
    return Math.abs(
      first[0] * (second[1] - third[1])
      + second[0] * (third[1] - first[1])
      + third[0] * (first[1] - second[1])
    );
  };

  const whole = twiceArea(a, b, c);

  if (whole === 0) {
    return false;
  }

  const part1 = twiceArea(point, b, c);
  const part2 = twiceArea(a, point, c);
  const part3 = twiceArea(a, b, point);

  return Math.abs(whole - (part1 + part2 + part3)) < 1e-9;
}
```

## Common Pitfalls

- Forgetting to reject collinear triangle vertices.
- Using exact floating-point equality without a tolerance.
- Treating edge points as outside when the intended interpretation includes the boundary.
