## Why It Surfaces

Ternary search is not as common as binary search on sorted arrays, but it surfaces when the interviewer wants to compare search strategies or transition into unimodal optimization. The key idea is the same: use ordered structure to discard most of the remaining space quickly.

## Worked Example

For a sorted array, choose `midLeft` and `midRight`. If the target is smaller than `midLeft`, discard the right two-thirds. If it is larger than `midRight`, discard the left two-thirds. Otherwise it must lie in the middle region.

## Reference Implementations

<ExamplePair>

```js
function ternarySearch(nums, target) {
  let left = 0;
  let right = nums.length - 1;

  while (left <= right) {
    const third = Math.floor((right - left) / 3);
    const midLeft = left + third;
    const midRight = right - third;

    if (nums[midLeft] === target) {
      return midLeft;
    }

    if (nums[midRight] === target) {
      return midRight;
    }

    if (target < nums[midLeft]) {
      right = midLeft - 1;
    } else if (target > nums[midRight]) {
      left = midRight + 1;
    } else {
      left = midLeft + 1;
      right = midRight - 1;
    }
  }

  return -1;
}
```

## Follow-up Questions

- Explain why binary search is usually preferred on sorted arrays.
- Apply ternary search to a unimodal function instead of a sorted list.
- Return the insertion region when the target is absent.

## Common Pitfalls

- Using ternary search where binary search is simpler and more practical.
- Miscomputing the two midpoints and skipping candidate values.
- Forgetting that unimodal optimization and sorted-array search are related but not identical settings.
