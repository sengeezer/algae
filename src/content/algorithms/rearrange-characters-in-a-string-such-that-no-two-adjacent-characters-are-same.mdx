## Why It Surfaces

The greedy constraint is driven by the most frequent character. If it cannot be separated from itself, no solution exists. If it can, placing it into every other slot first leaves the least risky positions for the remaining characters.

## Worked Example

For `s = "aaabc"`, the character `a` appears `3` times, which is still within the feasible limit of `(5 + 1) / 2 = 3`. Filling even positions first gives `a _ a _ a`, and then placing `b` and `c` into the remaining slots yields `abaca`.

```js
function rearrangeCharactersInAStringSuchThatNoTwoAdjacentCharactersAreSame(
  text,
) {
  if (text.length <= 1) {
    return text;
  }

  const frequencies = new Map();

  for (const char of text) {
    frequencies.set(char, (frequencies.get(char) ?? 0) + 1);
  }

  const entries = [...frequencies.entries()].sort(
    (left, right) => right[1] - left[1] || left[0].localeCompare(right[0]),
  );

  if (entries[0][1] > Math.floor((text.length + 1) / 2)) {
    return '';
  }

  const result = new Array(text.length);
  let index = 0;

  for (const [char, count] of entries) {
    for (let placed = 0; placed < count; placed += 1) {
      if (index >= text.length) {
        index = 1;
      }

      result[index] = char;
      index += 2;
    }
  }

  return result.join('');
}
```

## Follow-up Questions

- Validate a returned arrangement instead of constructing one.
- Rework the solution with a max heap that always picks the best currently safe character.
- Explain why the `(n + 1) / 2` frequency bound is both necessary and sufficient.

## Common Pitfalls

- Ignoring the impossible case before constructing the result.
- Treating any sorted order as safe without spacing out the highest-frequency character first.
- Comparing against one exact output string even though many valid rearrangements can exist.
