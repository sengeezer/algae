## Why It Surfaces

The triangle layout makes this problem feel unfamiliar at first, but the recurrence is small and local. Every cell only has two legal descendants, so the clean solution is to fold the triangle upward from the base until the top cell becomes the answer.

## Worked Example

For `[[2], [3, 4], [6, 5, 7], [4, 1, 8, 3]]`, the best route is `2 -> 3 -> 5 -> 1`, which totals `11`. A bottom-up pass discovers that by replacing each cell with its own value plus the cheaper of its two children.

## Reference Implementations

<ExamplePair>

```js
function minimumSumPathInATriangle(triangle) {
  if (triangle.length === 0) {
    return 0;
  }

  const dp = triangle[triangle.length - 1].slice();

  for (let row = triangle.length - 2; row >= 0; row -= 1) {
    for (let col = 0; col < triangle[row].length; col += 1) {
      dp[col] = triangle[row][col] + Math.min(dp[col], dp[col + 1]);
    }
  }

  return dp[0];
}
```

## Follow-up Questions

- Recover the actual minimum-sum path, not only its total.
- Compare the top-down memoized recurrence with the bottom-up reduction.
- Adapt the same idea to a rectangular grid with right and down moves.

## Common Pitfalls

- Treating the triangle like a full rectangular matrix and reading invalid neighbors.
- Solving the maximum-sum variant by mistake.
- Mutating caller-owned input directly without making it clear.
