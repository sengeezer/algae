## Why It Surfaces

Eulerian questions test whether you can separate structural criteria from traversal construction. Before you ever build the path, you should know that an undirected graph has an Euler circuit with zero odd-degree vertices and an Euler path with exactly two.

## Worked Example

If every non-isolated vertex belongs to one connected component and exactly two vertices have odd degree, then an Euler path exists and must start at one odd-degree vertex and end at the other. If no vertices have odd degree, the path can close into a circuit.

```js
function classifyEulerianGraph(graph) {
  const start = graph.findIndex((neighbors) => neighbors.length > 0);
  if (start === -1) {
    return 'circuit';
  }

  const visited = new Array(graph.length).fill(false);

  function dfs(node) {
    visited[node] = true;

    for (const neighbor of graph[node]) {
      if (!visited[neighbor]) {
        dfs(neighbor);
      }
    }
  }

  dfs(start);

  for (let node = 0; node < graph.length; node += 1) {
    if (graph[node].length > 0 && !visited[node]) {
      return 'none';
    }
  }

  const oddCount = graph.reduce(
    (count, neighbors) => count + (neighbors.length % 2 === 1 ? 1 : 0),
    0,
  );

  if (oddCount === 0) {
    return 'circuit';
  }

  if (oddCount === 2) {
    return 'path';
  }

  return 'none';
}
```

## Follow-up Questions

- Construct the actual traversal using Hierholzer's algorithm.
- Adapt the conditions for directed graphs.
- Explain why isolated vertices do not invalidate Eulerian classification.

## Common Pitfalls

- Confusing Euler paths with Hamiltonian paths.
- Forgetting to ignore zero-degree vertices during connectivity checks.
- Applying the undirected degree rule to directed graphs without adjustment.
