## Why It Surfaces

Linear search is not glamorous, but it is the correct baseline for a lot of real inputs. If the collection is unsorted or you only need one pass, building a more complex index can cost more than just scanning once.

## Worked Example

Searching for `7` in `[4, 9, 1, 7, 3]` checks `4`, then `9`, then `1`, and finally finds `7` at index `3`. If the target never appears, the scan simply reaches the end and returns failure.

## Reference Implementations

<ExamplePair>

```js
function linearSearch(values, target) {
  for (let index = 0; index < values.length; index += 1) {
    if (values[index] === target) {
      return index;
    }
  }

  return -1;
}
```

```ts
export function linearSearch<T>(values: T[], target: T): number {
  for (let index = 0; index < values.length; index += 1) {
    if (values[index] === target) {
      return index;
    }
  }

  return -1;
}
```

## Follow-up Questions

- Explain when linear search is better than binary search in practice.
- Return all matching indexes instead of only the first.
- Adapt the scan for linked lists or streaming input.

## Common Pitfalls

- Sorting first even though the prompt only asks for one lookup.
- Forgetting that binary search needs sorted input.
- Returning a one-based position when the surrounding API expects zero-based indexes.
