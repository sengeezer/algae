## Why It Surfaces

This is one of the cleanest BFS interview problems because the grid is just an implicit graph with uniform edge cost. The main signal is that all moves cost the same, so the first time BFS reaches the destination, you already have the shortest distance.

## Worked Example

If the source is `(0, 1)` and the destination is `(2, 2)`, BFS expands all cells at distance `1`, then all cells at distance `2`, and so on. When the search first reaches `(2, 2)`, that layer count is the minimum number of steps needed to get there.

```js
function shortestPathInABinaryMaze(grid, source, destination) {
  const rowCount = grid.length;
  const columnCount = rowCount === 0 ? 0 : grid[0].length;
  if (rowCount === 0 || columnCount === 0) {
    return -1;
  }

  const [startRow, startColumn] = source;
  const [targetRow, targetColumn] = destination;
  if (
    startRow < 0
    || startRow >= rowCount
    || startColumn < 0
    || startColumn >= columnCount
    || targetRow < 0
    || targetRow >= rowCount
    || targetColumn < 0
    || targetColumn >= columnCount
    || grid[startRow][startColumn] === 0
    || grid[targetRow][targetColumn] === 0
  ) {
    return -1;
  }

  const directions = [
    [1, 0],
    [-1, 0],
    [0, 1],
    [0, -1],
  ];
  const visited = Array.from({ length: rowCount }, () => new Array(columnCount).fill(false));
  const queue = [[startRow, startColumn, 0]];
  visited[startRow][startColumn] = true;

  for (let head = 0; head < queue.length; head += 1) {
    const [row, column, distance] = queue[head];
    if (row === targetRow && column === targetColumn) {
      return distance;
    }

    for (const [rowDelta, columnDelta] of directions) {
      const nextRow = row + rowDelta;
      const nextColumn = column + columnDelta;

      if (
        nextRow < 0
        || nextRow >= rowCount
        || nextColumn < 0
        || nextColumn >= columnCount
        || visited[nextRow][nextColumn]
        || grid[nextRow][nextColumn] === 0
      ) {
        continue;
      }

      visited[nextRow][nextColumn] = true;
      queue.push([nextRow, nextColumn, distance + 1]);
    }
  }

  return -1;
}
```

## Follow-up Questions

- How would you reconstruct the actual path, not just its length?
- What changes if diagonal moves are allowed?
- When would Dijkstra or A* be more appropriate than plain BFS?

## Common Pitfalls

- Reaching for Dijkstra even though every move has the same cost.
- Forgetting to reject blocked source or destination cells up front.
- Marking cells visited too late and enqueuing duplicates.
