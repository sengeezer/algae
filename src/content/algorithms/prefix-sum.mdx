## Why It Surfaces

Prefix sums are the cleanest answer when the question repeats range queries or asks you to compare the current running state against earlier ones. They also rescue cases where negative numbers make a standard sliding window invalid.

## JavaScript

```js
function buildPrefixSums(nums) {
  const prefix = new Array(nums.length + 1).fill(0);

  for (let index = 0; index < nums.length; index += 1) {
    prefix[index + 1] = prefix[index] + nums[index];
  }

  return prefix;
}

function rangeSum(prefix, left, right) {
  return prefix[right + 1] - prefix[left];
}
```

## TypeScript

```ts
export function countSubarraysWithSum(values: number[], target: number): number {
  const seen = new Map<number, number>([[0, 1]]);
  let prefix = 0;
  let count = 0;

  for (const value of values) {
    prefix += value;
    count += seen.get(prefix - target) ?? 0;
    seen.set(prefix, (seen.get(prefix) ?? 0) + 1);
  }

  return count;
}
```

## Interview Notes

- Add a leading zero slot so `prefix[right + 1] - prefix[left]` works cleanly.
- Mention 2D prefix sums when the prompt says matrix region.
- Call out that prefix sums still work when negative values break monotonic windows.
