## Why It Surfaces

Reverse Polish notation is the clean version of expression evaluation because precedence disappears. The main test is whether you keep operand order correct when an operator consumes the two most recent values.

## Worked Example

For `3 4 + 2 *`, push `3`, push `4`, apply `+` to get `7`, push `2`, then apply `*` to get `14`. For division, order matters: `8 2 /` means `8 / 2`, not `2 / 8`.

```js
function reversePolishNotation(expression) {
  if (typeof expression !== 'string' || expression.trim() === '') {
    throw new TypeError('Expected a non-empty postfix expression');
  }

  const operators = {
    '+': (left, right) => left + right,
    '-': (left, right) => left - right,
    '*': (left, right) => left * right,
    '/': (left, right) => left / right,
  };

  const stack = [];
  for (const token of expression.trim().split(/\s+/)) {
    const operation = operators[token];

    if (operation) {
      if (stack.length < 2) {
        throw new RangeError('Expression does not have enough operands');
      }

      const right = stack.pop();
      const left = stack.pop();
      stack.push(operation(left, right));
      continue;
    }

    const value = Number(token);
    if (!Number.isFinite(value)) {
      throw new TypeError(`Invalid token: ${token}`);
    }

    stack.push(value);
  }

  if (stack.length !== 1) {
    throw new RangeError('Expression must reduce to exactly one result');
  }

  return stack[0];
}
```

```ts
export function reversePolishNotation(expression: string): number {
  if (expression.trim() === '') {
    throw new TypeError('Expected a non-empty postfix expression');
  }

  const operators: Record<string, (left: number, right: number) => number> = {
    '+': (left, right) => left + right,
    '-': (left, right) => left - right,
    '*': (left, right) => left * right,
    '/': (left, right) => left / right,
  };

  const stack: number[] = [];
  for (const token of expression.trim().split(/\s+/)) {
    const operation = operators[token];

    if (operation) {
      if (stack.length < 2) {
        throw new RangeError('Expression does not have enough operands');
      }

      const right = stack.pop()!;
      const left = stack.pop()!;
      stack.push(operation(left, right));
      continue;
    }

    const value = Number(token);
    if (!Number.isFinite(value)) {
      throw new TypeError(`Invalid token: ${token}`);
    }

    stack.push(value);
  }

  if (stack.length !== 1) {
    throw new RangeError('Expression must reduce to exactly one result');
  }

  return stack[0];
}
```

## Follow-up Questions

- Add exponentiation or unary operators to the evaluator.
- Convert infix expressions to postfix before evaluation.
- Explain why division and subtraction must preserve operand order.

## Common Pitfalls

- Reversing operand order for subtraction and division.
- Ignoring malformed expressions with too many or too few operands.
- Splitting only on single spaces instead of general whitespace.
