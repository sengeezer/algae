## Why It Surfaces

This is the shortest path from number-theory vocabulary to executable logic. Once you know that two numbers are coprime exactly when their gcd is `1`, the whole problem becomes a standard Euclidean loop.

## Worked Example

`14` and `25` are coprime because `gcd(14, 25) = 1`. By contrast, `21` and `35` are not coprime because their gcd is `7`.

## Reference Implementations

<ExamplePair>

```js
function gcd(a, b) {
  let left = Math.abs(a);
  let right = Math.abs(b);

  while (right !== 0) {
    [left, right] = [right, left % right];
  }

  return left;
}

function coPrimeCheck(left, right) {
  return gcd(left, right) === 1;
}
```

```ts
function gcd(a: number, b: number): number {
  let left = Math.abs(a);
  let right = Math.abs(b);

  while (right !== 0) {
    [left, right] = [right, left % right];
  }

  return left;
}

export function coPrimeCheck(left: number, right: number): boolean {
  return gcd(left, right) === 1;
}
```

## Follow-up Questions

- Explain why coprimality is the requirement for a modular inverse.

- Handle zero explicitly and explain the result.

## Common Pitfalls

- Reaching for factorization when gcd already answers the question.
- Forgetting that negative signs do not affect coprimality after taking absolute values.
- Assuming every pair that is not divisible by `2` must be coprime.
