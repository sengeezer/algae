## Why It Surfaces

This problem looks combinatorial, but the clean implementation is a memoized search over counts. The only extra piece of state beyond the remaining balls is the type of the last placed ball, because that is what blocks one of the three possible next moves.

## Worked Example

For `p = 1`, `q = 1`, and `r = 1`, every permutation of the three distinct types is valid, so the answer is `6`.

```js
function waysToArrangeBallsSuchThatAdjacentBallsAreOfDifferentTypes(p, q, r) {
  const memo = new Map();

  function solve(remainingP, remainingQ, remainingR, lastPlaced) {
    if (remainingP < 0 || remainingQ < 0 || remainingR < 0) {
      return 0;
    }

    if (remainingP === 0 && remainingQ === 0 && remainingR === 0) {
      return 1;
    }

    const key = `${remainingP}|${remainingQ}|${remainingR}|${lastPlaced}`;
    const cached = memo.get(key);
    if (cached !== undefined) {
      return cached;
    }

    let ways = 0;

    if (lastPlaced !== 0 && remainingP > 0) {
      ways += solve(remainingP - 1, remainingQ, remainingR, 0);
    }

    if (lastPlaced !== 1 && remainingQ > 0) {
      ways += solve(remainingP, remainingQ - 1, remainingR, 1);
    }

    if (lastPlaced !== 2 && remainingR > 0) {
      ways += solve(remainingP, remainingQ, remainingR - 1, 2);
    }

    memo.set(key, ways);
    return ways;
  }

  return solve(p, q, r, 3);
}
```

## Follow-up Questions

- Generalize the solution from three colors to an arbitrary multiset of colors.
- Return one valid arrangement when the count is non-zero.
- Add modulo arithmetic when the counts become large.

## Common Pitfalls

- Tracking only the remaining counts and forgetting that the previous color changes what is legal next.
- Returning zero instead of one when all counts reach zero.
- Double-counting states by recomputing the same `(p, q, r, last)` combination.
