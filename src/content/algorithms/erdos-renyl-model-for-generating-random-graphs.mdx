## Why It Surfaces

Most graph interview problems start with a fixed input. This one flips the job around: you need to generate the input itself. The `G(n, p)` model is the cleanest randomized baseline because every possible edge is treated symmetrically and independently.

## Worked Example

If `n = 4` and `p = 0.5`, there are six possible undirected edges. The generator flips one probability check for each pair such as `(0, 1)`, `(0, 2)`, and `(2, 3)`. Every pair survives independently with chance `0.5`, so the final graph density varies from sample to sample.

```js
function erdosRenylModelForGeneratingRandomGraphs(
  nodeCount,
  edgeProbability,
  random = Math.random,
) {
  if (nodeCount < 0) {
    throw new RangeError('nodeCount must be non-negative');
  }

  if (edgeProbability < 0 || edgeProbability > 1) {
    throw new RangeError('edgeProbability must be between 0 and 1');
  }

  const graph = Array.from({ length: nodeCount }, () => []);

  for (let from = 0; from < nodeCount; from += 1) {
    for (let to = from + 1; to < nodeCount; to += 1) {
      if (random() >= edgeProbability) {
        continue;
      }

      graph[from].push(to);
      graph[to].push(from);
    }
  }

  return graph;
}
```

## Follow-up Questions

- Add the `G(n, M)` variant that samples exactly `M` edges instead of using a probability.
- Generate a directed random graph by checking every ordered pair instead of every unordered pair.
- Estimate expected edge count, average degree, or connectivity thresholds as `p` changes.

## Common Pitfalls

- Iterating over both `(u, v)` and `(v, u)` when you intend to build an undirected graph.
- Using one shared random decision for a whole vertex instead of one independent decision per edge.
- Forgetting to validate that `p` lies in `[0, 1]`.
