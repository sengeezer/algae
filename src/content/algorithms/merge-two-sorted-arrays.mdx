## Why It Surfaces

This is the smallest useful two-pointer merge problem. It shows up on its own, but it is also the exact combine step behind merge sort, interval sweeps, and many streaming joins.

## Worked Example

For `[1, 3, 5]` and `[2, 4, 6]`, compare the current fronts, emit `1`, then `2`, then `3`, and keep going until one side runs out. Append the remaining tail from the other side at the end.

## Reference Implementations

<ExamplePair>

```js
function mergeTwoSortedArrays(left, right) {
  if (!Array.isArray(left) || !Array.isArray(right)) {
    throw new TypeError('Both inputs must be arrays');
  }

  const merged = [];
  let leftIndex = 0;
  let rightIndex = 0;

  while (leftIndex < left.length && rightIndex < right.length) {
    if (left[leftIndex] <= right[rightIndex]) {
      merged.push(left[leftIndex]);
      leftIndex += 1;
    } else {
      merged.push(right[rightIndex]);
      rightIndex += 1;
    }
  }

  return merged.concat(left.slice(leftIndex), right.slice(rightIndex));
}
```

## Follow-up Questions

- Merge the arrays in place when one buffer has extra trailing capacity.
- Count inversions while performing the merge.
- Generalize the merge to objects with a custom comparator.

## Common Pitfalls

- Concatenating both arrays and sorting again instead of exploiting the existing order.
- Forgetting to append the leftover tail once one pointer reaches the end.
- Advancing both pointers on every comparison.
