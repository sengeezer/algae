## Why It Surfaces

Length conversion is the clearest example of normalization through a shared base unit. Instead of writing one formula for every pair of units, you convert into meters once and scale back out.

## Worked Example

To convert `3` miles to feet, first turn miles into meters, then divide by the meters-per-foot factor. The algorithm stays the same no matter which pair of units you choose.

## Reference Implementations

<ExamplePair>

```js
const metersPerUnit = {
  mm: 0.001,
  cm: 0.01,
  m: 1,
  km: 1000,
  inch: 0.0254,
  ft: 0.3048,
  yd: 0.9144,
  mi: 1609.34,
};

function lengthConversion(length, fromUnit, toUnit) {
  if (typeof length !== 'number' || Number.isNaN(length)) {
    throw new TypeError('Length must be a valid number');
  }

  if (!(fromUnit in metersPerUnit) || !(toUnit in metersPerUnit)) {
    throw new RangeError('Invalid length units');
  }

  const lengthInMeters = length * metersPerUnit[fromUnit];
  return lengthInMeters / metersPerUnit[toUnit];
}
```

## Common Pitfalls

- Writing one-off formulas for every unit pair instead of normalizing once.
- Failing to validate unsupported unit strings.
- Mixing conversion factors that are not all defined relative to the same base unit.
