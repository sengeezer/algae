## Why It Surfaces

This problem turns ordinary cycle detection into exact counting. The challenge is not just finding one cycle, but searching paths of a precise length, ensuring vertices stay distinct, and correcting for the fact that undirected cycles are discovered multiple times.

## Worked Example

If the graph has vertices `0, 1, 2, 3` forming a square, then the cycle `0 -> 1 -> 2 -> 3 -> 0` can also be discovered as `0 -> 3 -> 2 -> 1 -> 0`. A bounded DFS finds both traversals, so you divide by two after the search to count that undirected cycle only once per start-set strategy.

## Reference Implementations

<ExamplePair>

```js
function cyclesOfLengthNInAnUndirectedAndConnectedGraph(graph, cycleLength) {
  const size = graph.length;
  if (cycleLength < 2 || cycleLength > size) {
    return 0;
  }

  const visited = new Array(size).fill(false);
  let count = 0;

  function dfs(start, node, remainingEdges) {
    visited[node] = true;

    if (remainingEdges === 0) {
      visited[node] = false;
      if (graph[node][start]) {
        count += 1;
      }
      return;
    }

    for (let neighbor = 0; neighbor < size; neighbor += 1) {
      if (!graph[node][neighbor] || visited[neighbor]) {
        continue;
      }

      dfs(start, neighbor, remainingEdges - 1);
    }

    visited[node] = false;
  }

  for (let start = 0; start <= size - cycleLength; start += 1) {
    dfs(start, start, cycleLength - 1);
    visited[start] = true;
  }

  return Math.floor(count / 2);
}
```

  graph: boolean[][],
  cycleLength: number,
): number {
  const size = graph.length;
  if (cycleLength < 2 || cycleLength > size) {
    return 0;
  }

  const visited = new Array<boolean>(size).fill(false);
  let count = 0;

  function dfs(start: number, node: number, remainingEdges: number): void {
    visited[node] = true;

    if (remainingEdges === 0) {
      visited[node] = false;
      if (graph[node][start]) {
        count += 1;
      }
      return;
    }

    for (let neighbor = 0; neighbor < size; neighbor += 1) {
      if (!graph[node][neighbor] || visited[neighbor]) {
        continue;
      }

      dfs(start, neighbor, remainingEdges - 1);
    }

    visited[node] = false;
  }

  for (let start = 0; start <= size - cycleLength; start += 1) {
    dfs(start, start, cycleLength - 1);
    visited[start] = true;
  }

  return Math.floor(count / 2);
}

## Follow-up Questions

- How would you adapt the count for directed graphs?

## Common Pitfalls

- Counting the same undirected cycle once in each direction.
- Allowing a vertex to reappear before the cycle closes.
- Solving only generic cycle detection instead of exact-length counting.
