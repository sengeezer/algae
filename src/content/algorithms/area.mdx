## Why It Surfaces

Area formulas are direct, but they still test whether you match the right formula to the given measurements and reject impossible inputs instead of blindly computing nonsense.

## Worked Example

For a triangle with sides `5`, `6`, and `7`, the semiperimeter is `9`, and Heron's formula gives `sqrt(9 * 4 * 3 * 2)`, which is about `14.697`.

function validateNonNegative(value, label) {
  if (typeof value !== 'number' || Number.isNaN(value) || value < 0) {
    throw new RangeError(`${label} must be a non-negative number`);
  }
}

function areaRectangle(length, width) {
  validateNonNegative(length, 'Length');
  validateNonNegative(width, 'Width');
  return length * width;
}

function areaCircle(radius) {
  validateNonNegative(radius, 'Radius');
  return Math.PI * radius ** 2;
}

function areaTriangle(base, height) {
  validateNonNegative(base, 'Base');
  validateNonNegative(height, 'Height');
  return (base * height) / 2;
}

function areaTriangleWithAllThreeSides(a, b, c) {
  validateNonNegative(a, 'Side a');
  validateNonNegative(b, 'Side b');
  validateNonNegative(c, 'Side c');

  if (a + b <= c || a + c <= b || b + c <= a) {
    throw new RangeError('Triangle sides must satisfy the triangle inequality');
  }

  const semi = (a + b + c) / 2;
  return Math.sqrt(semi * (semi - a) * (semi - b) * (semi - c));
}

## Follow-up Questions

- Add parallelogram, trapezium, and rhombus helpers to the same module.
- Compare base-height triangle area with Heron's formula.
- Explain why negative dimensions should be rejected instead of absolutized.

## Common Pitfalls

- Applying a formula to impossible triangle sides.
- Mixing surface area and 2D area in the same prompt.
- Forgetting that radius, base, and height should be non-negative.
