## Why It Surfaces

This prompt looks like string sorting at first, but the real constraint is that the array order is fixed. The only freedom is whether each string stays as-is or is reversed, which turns the problem into a left-to-right DP with two states per position.

## Worked Example

For `strings = ["aa", "ba", "ac"]` and `costs = [1, 3, 1]`, the answer is `1`. Keeping `"aa"` and `"ba"` is already valid, but the last string must become `"ca"` to stay lexicographically after `"ba"`, so reversing only the final string is optimal.

function minimumCostToSortStringsUsingReversalsOfDifferentCosts(strings, costs) {
  if (!Array.isArray(strings) || !Array.isArray(costs) || strings.length !== costs.length) {
    throw new RangeError('strings and costs must be arrays of equal length');
  }

  const count = strings.length;
  if (count === 0) {
    return 0;
  }

  for (let index = 0; index < count; index += 1) {
    if (typeof strings[index] !== 'string' || !Number.isInteger(costs[index]) || costs[index] < 0) {
      throw new TypeError('strings must contain strings and costs must contain non-negative integers');
    }
  }

  const reversed = strings.map((value) => [...value].reverse().join(''));
  const inf = Number.POSITIVE_INFINITY;
  let previousKeep = 0;
  let previousReverse = costs[0];

  for (let index = 1; index < count; index += 1) {
    let currentKeep = inf;
    let currentReverse = inf;

    if (strings[index] >= strings[index - 1] && previousKeep !== inf) {
      currentKeep = Math.min(currentKeep, previousKeep);
    }

    if (strings[index] >= reversed[index - 1] && previousReverse !== inf) {
      currentKeep = Math.min(currentKeep, previousReverse);
    }

    if (reversed[index] >= strings[index - 1] && previousKeep !== inf) {
      currentReverse = Math.min(currentReverse, previousKeep + costs[index]);
    }

    if (reversed[index] >= reversed[index - 1] && previousReverse !== inf) {
      currentReverse = Math.min(currentReverse, previousReverse + costs[index]);
    }

    previousKeep = currentKeep;
    previousReverse = currentReverse;
  }

  const answer = Math.min(previousKeep, previousReverse);
  return Number.isFinite(answer) ? answer : -1;
}

## Follow-up Questions

- Reconstruct which indices should be reversed instead of returning only the minimum cost.
- Compare the O(1)-state version with the full `dp[i][2]` table.
- Explain why greedy local reversal decisions can fail even though each position has only two choices.

## Common Pitfalls

- Sorting the array directly even though the prompt forbids reordering.
- Comparing only against the original previous string and forgetting the reversed previous state.
- Returning `Infinity` instead of `-1` when no valid lexicographic sequence exists.
