## Why It Surfaces

Computing the optimum cost is only half of the interview question when the caller also wants to see the actual multiplication order. The fix is to store the midpoint that won for each interval so the final bracket structure can be rebuilt afterward.

## Worked Example

For `dimensions = [40, 20, 30, 10, 30]`, the optimal parenthesization is `((A(BC))D)`. That order achieves the minimum multiplication cost among all valid bracketings.

```js
function printingBracketsInMatrixChainMultiplicationProblem(dimensions) {
  const matrixCount = dimensions.length - 1;
  if (matrixCount <= 0) {
    return '';
  }

  const cost = Array.from({ length: matrixCount }, () => new Array(matrixCount).fill(0));
  const split = Array.from({ length: matrixCount }, () => new Array(matrixCount).fill(0));

  for (let chainLength = 2; chainLength <= matrixCount; chainLength += 1) {
    for (let left = 0; left + chainLength - 1 < matrixCount; left += 1) {
      const right = left + chainLength - 1;
      cost[left][right] = Number.POSITIVE_INFINITY;

      for (let mid = left; mid < right; mid += 1) {
        const current =
          cost[left][mid]
          + cost[mid + 1][right]
          + dimensions[left] * dimensions[mid + 1] * dimensions[right + 1];

        if (current < cost[left][right]) {
          cost[left][right] = current;
          split[left][right] = mid;
        }
      }
    }
  }

  function label(index) {
    return index < 26 ? String.fromCharCode(65 + index) : `M${index + 1}`;
  }

  function build(left, right) {
    if (left === right) {
      return label(left);
    }

    const mid = split[left][right];
    return `(${build(left, mid)}${build(mid + 1, right)})`;
  }

  return build(0, matrixCount - 1);
}
```

## Follow-up Questions

- Return both the bracket order and the minimum cost together.
- Explain why a one-row optimized cost DP is not enough to reconstruct the order directly.
- Switch matrix labels from letters to numbered tokens when the chain grows beyond `26` matrices.

## Common Pitfalls

- Recomputing the whole cost DP during reconstruction instead of storing split points.
- Mixing matrix indices with dimension indices and shifting the multiplication formula off by one.
- Forgetting that a single matrix should print without extra parentheses.
