## Why It Surfaces

Kruskal's algorithm is the greedy MST solution that thinks in terms of edges first. It is a common follow-up whenever Union-Find appears, because the disjoint-set structure makes cycle detection cheap while the graph grows.

## JavaScript

```js
function minimumSpanningTree(vertexCount, edges) {
  const sortedEdges = [...edges].sort((left, right) => left[2] - right[2]);
  const dsu = new UnionFind(vertexCount);
  let totalWeight = 0;

  for (const [from, to, weight] of sortedEdges) {
    if (dsu.union(from, to)) {
      totalWeight += weight;
    }
  }

  return totalWeight;
}

class UnionFind {
  constructor(size) {
    this.parent = Array.from({ length: size }, (_, index) => index);
    this.rank = new Array(size).fill(0);
  }

  find(node) {
    if (this.parent[node] !== node) {
      this.parent[node] = this.find(this.parent[node]);
    }

    return this.parent[node];
  }

  union(left, right) {
    const rootLeft = this.find(left);
    const rootRight = this.find(right);

    if (rootLeft === rootRight) {
      return false;
    }

    if (this.rank[rootLeft] < this.rank[rootRight]) {
      this.parent[rootLeft] = rootRight;
    } else if (this.rank[rootLeft] > this.rank[rootRight]) {
      this.parent[rootRight] = rootLeft;
    } else {
      this.parent[rootRight] = rootLeft;
      this.rank[rootLeft] += 1;
    }

    return true;
  }
}
```

## Interview Notes

- Say explicitly that sorting edges is why the runtime is O(E log E).
- Mention Union-Find by name; that is usually part of what the interviewer wants to hear.
- If the graph is disconnected, call out whether the problem expects an MST or a minimum spanning forest.
