## Why It Surfaces

This is the pointer-mutation baseline for linked lists. It looks simple, but it exposes whether you can update a structure in place without losing access to the remainder of the list.

## Worked Example

For `1 -> 2 -> 3 -> null`, once `1.next` is rewired to `null`, the only safe way forward is if you already saved the original `2` pointer before the rewrite.

## Reference Implementations

<ExamplePair>

```js
function reverseList(head) {
  let previous = null;
  let current = head;

  while (current !== null) {
    const next = current.next;
    current.next = previous;
    previous = current;
    current = next;
  }

  return previous;
}
```

```ts
type Node = {
  value: number;
  next: Node | null;
};

export function reverseList(head: Node | null): Node | null {
  let previous: Node | null = null;
  let current = head;

  while (current !== null) {
    const next: Node | null = current.next;
    current.next = previous;
    previous = current;
    current = next;
  }

  return previous;
}
```

## Common Pitfalls

- Overwriting `next` before saving it.
- Returning the old head instead of the new one.
- Allocating a second list when the prompt asks for in-place reversal.
