## Why It Surfaces

This is a simple indexed search layered on top of primality testing. It is useful because it forces a choice between direct trial division and a more precomputed sieve-style approach.

## Worked Example

The first six primes are `2`, `3`, `5`, `7`, `11`, and `13`, so the `6`th prime is `13`. The search continues until exactly `n` primes have been counted.

```js
function isPrime(value) {
  if (value < 2) {
    return false;
  }

  if (value === 2) {
    return true;
  }

  if (value % 2 === 0) {
    return false;
  }

  for (let divisor = 3; divisor * divisor <= value; divisor += 2) {
    if (value % divisor === 0) {
      return false;
    }
  }

  return true;
}

function problem007(index) {
  if (!Number.isInteger(index) || index < 1) {
    throw new RangeError('Index must be a positive integer');
  }

  let count = 0;
  let candidate = 1;

  while (count < index) {
    candidate += 1;
    if (isPrime(candidate)) {
      count += 1;
    }
  }

  return candidate;
}
```

## Follow-up Questions

- Replace repeated trial division with a sieve and an upper-bound estimate.
- Explain why even candidates after `2` can be skipped.
- Compare this indexed search with summing primes below a limit.

## Common Pitfalls

- Counting `1` as prime.
- Forgetting to validate that the requested index is positive.
- Checking every divisor instead of only odd divisors up to the square root.
