## Why It Surfaces

This is the counting follow-up to subset sum. Once feasibility is solved, the next natural question is how many valid subsets reach the target, which changes the DP transition from boolean logic to addition.

## Worked Example

For `nums = [1, 2, 3, 3]` and `target = 6`, the valid subsets include `[1, 2, 3]` in two different ways because there are two distinct `3`s, plus `[3, 3]`.

<Callout title="Backward iteration" tone="warning">
  Update the target sums from high to low. Forward iteration would reuse the same number multiple times in one row and change the problem into an unbounded variant.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function countSubsets(nums, target) {
  const dp = new Array(target + 1).fill(0);
  dp[0] = 1;

  for (const value of nums) {
    for (let sum = target; sum >= value; sum -= 1) {
      dp[sum] += dp[sum - value];
    }
  }

  return dp[target];
}
```

```ts
export function countSubsets(nums: number[], target: number): number {
  const dp = new Array<number>(target + 1).fill(0);
  dp[0] = 1;

  for (const value of nums) {
    for (let sum = target; sum >= value; sum -= 1) {
      dp[sum] += dp[sum - value];
    }
  }

  return dp[target];
}
```

## Follow-up Questions

- Count subsets modulo a large prime.

- Reconstruct one valid subset in addition to the count.

## Common Pitfalls

- Iterating target sums forward and reusing the same item twice.
- Solving only the boolean subset-sum version.
- Forgetting that duplicate values still represent different choices when they occupy different indices.
