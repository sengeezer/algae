## Why It Surfaces

This problem is the classic follow-up to ordinary BFS and Dijkstra. Uniform weights make plain BFS enough, arbitrary nonnegative weights suggest Dijkstra, and the special 0-or-1 case lands in the middle where a deque gives you the right ordering without heap overhead.

## Worked Example

If relaxing an edge of weight `0` improves a neighbor, that neighbor should be processed as soon as possible because its distance is no larger than the current node's distance. If the edge weight is `1`, the neighbor belongs one layer later, so it goes to the back of the deque.

## Reference Implementations

<ExamplePair>

```js
function shortestPathInABinaryWeightGraph(nodeCount, edges, source) {
  const graph = Array.from({ length: nodeCount }, () => []);
  for (const [from, to, weight] of edges) {
    graph[from].push([to, weight]);
    graph[to].push([from, weight]);
  }

  const distance = new Array(nodeCount).fill(Number.POSITIVE_INFINITY);
  const deque = new Map([[0, source]]);
  let front = 0;
  let back = 1;
  distance[source] = 0;

  while (front < back) {
    const node = deque.get(front);
    deque.delete(front);
    front += 1;

    if (node === undefined) {
      continue;
    }

    for (const [neighbor, weight] of graph[node]) {
      if (weight !== 0 && weight !== 1) {
        throw new RangeError('Edge weights must be either 0 or 1');
      }

      const candidate = distance[node] + weight;
      if (candidate >= distance[neighbor]) {
        continue;
      }

      distance[neighbor] = candidate;
      if (weight === 0) {
        front -= 1;
        deque.set(front, neighbor);
      } else {
        deque.set(back, neighbor);
        back += 1;
      }
    }
  }

  return distance;
}
```

## Follow-up Questions

- Why is this faster than Dijkstra when weights are only 0 and 1?
- How does the method change if weights range from 0 to `W` instead?
- Can you reconstruct the actual shortest path tree, not just the distances?

## Common Pitfalls

- Treating the graph like a plain unweighted BFS problem.
- Using a heap even though the deque ordering is sufficient.
- Forgetting to push 0-weight relaxations to the front.
