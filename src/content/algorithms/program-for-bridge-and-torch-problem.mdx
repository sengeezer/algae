## Why It Surfaces

This puzzle is really about choosing between two repeatable transfer patterns. Once the crossing times are sorted, each step asks whether it is cheaper to have the two fastest shuttle the torch or to let the fastest person escort each of the two slowest separately.

## Worked Example

For `times = [1, 2, 5, 8]`, the best total is `15`. The optimal sequence is `(1, 2) -> 2`, `1` returns, `(5, 8) -> 8`, `2` returns, then `(1, 2) -> 2` again.

```js
function programForBridgeAndTorchProblem(times) {
  const sorted = [...times].sort((left, right) => left - right);
  const size = sorted.length;

  if (size === 0) {
    return 0;
  }

  if (size === 1) {
    return sorted[0];
  }

  if (size === 2) {
    return sorted[1];
  }

  if (size === 3) {
    return sorted[0] + sorted[1] + sorted[2];
  }

  let prevTwo = sorted[1];
  let prevOne = sorted[0] + sorted[1] + sorted[2];
  let current = prevOne;

  for (let count = 4; count <= size; count += 1) {
    const optionOne = sorted[1] + sorted[0] + sorted[count - 1] + sorted[1];
    const optionTwo = sorted[count - 1] + sorted[0] + sorted[count - 2] + sorted[0];
    current = Math.min(prevTwo + optionOne, prevTwo + optionTwo);
    prevTwo = prevOne;
    prevOne = current;
  }

  return current;
}
```

- Print the actual crossing schedule, not only the minimum total time.

## Common Pitfalls

- Pairing people greedily without considering the torch-return cost.
- Forgetting the special base cases for one, two, and three people.
- Updating the rolling state as if the recurrence depended on every earlier group size.
