## Why It Surfaces

Dutch National Flag Sort is the standard answer for “sort 0s, 1s, and 2s in one pass.” The key idea is to maintain three regions: already-placed lows on the left, already-placed highs on the right, and unknown elements in the middle.

## Worked Example

For `[2, 0, 2, 1, 1, 0]`, swapping the leading `2` toward the high end grows the right region. Encountering `0` grows the left region, while `1` values stay in the middle and simply advance the scan.

## Reference Implementations

<ExamplePair>

```js
function dutchNationalFlagSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array');
  }

  const items = [...values];
  let low = 0;
  let mid = 0;
  let high = items.length - 1;

  while (mid <= high) {
    if (items[mid] === 0) {
      [items[low], items[mid]] = [items[mid], items[low]];
      low += 1;
      mid += 1;
    } else if (items[mid] === 1) {
      mid += 1;
    } else if (items[mid] === 2) {
      [items[mid], items[high]] = [items[high], items[mid]];
      high -= 1;
    } else {
      throw new RangeError('Values must contain only 0, 1, and 2');
    }
  }

  return items;
}
```

## Follow-up Questions

- Explain why `mid` does not advance immediately after swapping with `high`.
- Generalize the idea to partition around a pivot in quicksort.
- Compare this with counting the number of 0s, 1s, and 2s first.

## Common Pitfalls

- Incrementing `mid` after a swap with `high` and skipping an unclassified value.
- Forgetting that the algorithm is specialized to exactly three categories.
- Describing it as a generic replacement for arbitrary comparison sorting.
