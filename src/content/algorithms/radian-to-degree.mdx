## Why It Surfaces

This is the inverse of degree-to-radian conversion, so it often appears as a quick check that the candidate actually understands the unit relationship instead of memorizing one formula by itself.

## Worked Example

`pi / 2` radians becomes `90` degrees. The conversion is again linear because both units describe the same underlying rotation.

## Reference Implementations

<ExamplePair>

```js
function radianToDegree(radians) {
  if (typeof radians !== 'number' || Number.isNaN(radians)) {
    throw new TypeError('Expected a numeric radian value');
  }

  return radians * (180 / Math.PI);
}
```

```ts
export function radianToDegree(radians: number): number {
  if (Number.isNaN(radians)) {
    throw new TypeError('Expected a numeric radian value');
  }

  return radians * (180 / Math.PI);
}
```

## Follow-up Questions

- Show that converting degrees to radians and back is only approximately identity under floating-point arithmetic.
- Normalize any resulting angle into the `0` to `360` range.
- Explain why using the wrong conversion constant can still look plausible in small tests.

## Common Pitfalls

- Reusing the forward conversion constant by mistake.
- Forgetting to label units at API boundaries.
- Expecting exact integer output for every radian input.
