## Why It Surfaces

Autocomplete is one of the most recognizable trie applications. The whole point is that you only pay once to walk the typed prefix, and after that you search a much smaller subtree instead of rechecking every stored word.

## Worked Example

If the trie stores `hell`, `hello`, `help`, `helping`, and `helps`, then the prefix `hel` lands at the shared prefix node and a DFS below it returns all five suggestions.

```js
class AutoCompleteTrieNode {
  constructor() {
    this.children = new Map();
    this.isWord = false;
  }
}

function autoCompleteFeatureUsingTrie(words, prefix) {
  if (!Array.isArray(words) || typeof prefix !== 'string') {
    throw new TypeError('Expected an array of words and a string prefix');
  }

  const root = new AutoCompleteTrieNode();

  for (const word of words) {
    if (typeof word !== 'string' || word.length === 0) {
      throw new RangeError('Words must be non-empty strings');
    }

    let node = root;
    for (const char of word) {
      if (!node.children.has(char)) {
        node.children.set(char, new AutoCompleteTrieNode());
      }
      node = node.children.get(char);
    }
    node.isWord = true;
  }

  let node = root;
  for (const char of prefix) {
    node = node.children.get(char);
    if (!node) {
      return [];
    }
  }

  const suggestions = [];

  const collect = (currentNode, currentWord) => {
    if (currentNode.isWord) {
      suggestions.push(currentWord);
    }

    const entries = [...currentNode.children.entries()].sort((left, right) =>
      left[0].localeCompare(right[0]),
    );

    for (const [char, child] of entries) {
      collect(child, currentWord + char);
    }
  };

  collect(node, prefix);
  return suggestions;
}
```

class AutoCompleteTrieNode {
  children = new Map<string, AutoCompleteTrieNode>();
  isWord = false;
}

export function autoCompleteFeatureUsingTrie(words: string[], prefix: string): string[] {
  const root = new AutoCompleteTrieNode();

  for (const word of words) {
    if (word.length === 0) {
      throw new RangeError('Words must be non-empty strings');
    }

    let node = root;
    for (const char of word) {
      if (!node.children.has(char)) {
        node.children.set(char, new AutoCompleteTrieNode());
      }
      node = node.children.get(char) as AutoCompleteTrieNode;
    }
    node.isWord = true;
  }

  let node: AutoCompleteTrieNode | undefined = root;
  for (const char of prefix) {
    node = node.children.get(char);
    if (!node) {
      return [];
    }
  }

  const suggestions: string[] = [];

  const collect = (currentNode: AutoCompleteTrieNode, currentWord: string): void => {
    if (currentNode.isWord) {
      suggestions.push(currentWord);
    }

    const entries = [...currentNode.children.entries()].sort((left, right) =>
      left[0].localeCompare(right[0]),
    );

    for (const [char, child] of entries) {
      collect(child, currentWord + char);
    }
  };

  collect(node, prefix);
  return suggestions;
}

## Follow-up Questions

- How would you rank results by popularity instead of pure lexicographic order?
- How would you cap output at the top `k` suggestions?
- What would change if queries arrive continuously and the trie is reused across many requests?

## Common Pitfalls

- Scanning every stored word even after the trie already identifies the prefix subtree.
- Returning the prefix alone even when it is not actually stored as a complete word.
- Forgetting that no matching prefix should usually return an empty list rather than throw.
