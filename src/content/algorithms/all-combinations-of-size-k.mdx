## Why It Surfaces

This is the cleanest combination generator in the catalog. It looks similar to permutations at first, but the real constraint is that each partial answer must stay in increasing order so the same group is not emitted multiple times.

## Worked Example

For `n = 4` and `k = 2`, start with `1` and recurse into `[1, 2]`, `[1, 3]`, and `[1, 4]`. Then backtrack and start with `2`, producing `[2, 3]` and `[2, 4]`, and finally `[3, 4]`.

## Reference Implementations

<ExamplePair>

```js
function allCombinationsOfSizeK(n, k) {
  const combinations = [];
  const path = [];

  function backtrack(start) {
    if (path.length === k) {
      combinations.push([...path]);
      return;
    }

    for (let value = start; value <= n; value += 1) {
      path.push(value);
      backtrack(value + 1);
      path.pop();
    }
  }

  backtrack(1);
  return combinations;
}
```

```ts

    if (path.length === k) {
      combinations.push([...path]);
      return;
    }

    for (let value = start; value <= n; value += 1) {
      path.push(value);
      backtrack(value + 1);
      path.pop();
    }
  }

  backtrack(1);
  return combinations;
}
```

## Follow-up Questions

- Prune branches when the remaining values cannot fill the rest of the path.
- Generate combinations from an input array instead of the numeric range `1..n`.
- Compare the branching pattern with permutations and subsets.

## Common Pitfalls

- Treating different orders of the same values as distinct answers.
- Forgetting to pop from the current path on backtrack.
- Reusing the same value more than once in a single combination.
