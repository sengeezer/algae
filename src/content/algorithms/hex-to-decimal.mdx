## Why It Surfaces

Hex-to-decimal conversion extends the same fold used in binary parsing, but adds a digit map for letters. It is a good reminder that positional notation is the same algorithm no matter how the digits are named.

## Worked Example

Reading `4D` left to right gives `((0 * 16 + 4) * 16 + 13) = 77`. The only extra step compared with decimal parsing is translating `D` to `13`.

## Reference Implementations

<ExamplePair>

```js
function hexToDecimal(hexString) {
  if (!/^[0-9A-F]+$/i.test(hexString)) {
    throw new TypeError('Hex strings may only contain digits 0-9 and letters A-F');
  }

  let value = 0;

  for (const digit of hexString.toUpperCase()) {
    const digitValue = Number.parseInt(digit, 16);
    value = value * 16 + digitValue;
  }

  return value;
}
```

## Common Pitfalls

- Forgetting to normalize lowercase and uppercase letters consistently.
- Accepting invalid letters outside the hex alphabet.
- Recomputing powers of sixteen instead of folding incrementally.
