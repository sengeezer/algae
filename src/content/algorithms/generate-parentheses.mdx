## Why It Surfaces

Generate Parentheses is a clean pruning problem. A brute-force solution would build every length-`2n` string and filter the bad ones later, but the interview version expects you to reject invalid prefixes immediately.

## Worked Example

For `n = 3`, a prefix like `())` is already invalid because it closes more groups than it has opened. The search should cut that branch right away instead of exploring it further.

<Callout title="Prefix invariant" tone="warning">
  The number of closing parentheses used can never exceed the number of opening parentheses used at any prefix.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function generateParentheses(pairs) {
  const results = [];

  function backtrack(path, openUsed, closeUsed) {
    if (path.length === pairs * 2) {
      results.push(path);
      return;
    }

    if (openUsed < pairs) {
      backtrack(path + "(", openUsed + 1, closeUsed);
    }

    if (closeUsed < openUsed) {
      backtrack(path + ")", openUsed, closeUsed + 1);
    }
  }

  backtrack("", 0, 0);
  return results;
}
```

```ts
    }

    if (closeUsed < openUsed) {
      backtrack(path + ")", openUsed, closeUsed + 1);
    }
  }

  backtrack("", 0, 0);
  return results;
}
```

## Follow-up Questions

- Count the valid strings without generating all of them.
- Generate multiple bracket types instead of only parentheses.
- Explain the Catalan-number growth of the result size.

## Common Pitfalls

- Closing more groups than have been opened.
- Generating all `2^(2n)` strings and filtering afterward.
- Forgetting that the recursion ends when the string length reaches `2n`, not when opens run out.
