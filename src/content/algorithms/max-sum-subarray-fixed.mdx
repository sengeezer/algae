## Why It Surfaces

This is usually the first sliding-window problem because the invariant is simple: the window size never changes. Once the first sum is built, the rest of the algorithm is just a rolling update.

## Worked Example

For `[2, 1, 5, 1, 3, 2]` with `k = 3`, the window sums are `8`, `7`, `9`, and `6`. The answer is `9`, coming from the subarray `[5, 1, 3]`.

## Reference Implementations

<ExamplePair>

```js
function maxSumSubarrayFixed(values, size) {
  if (!Number.isInteger(size) || size < 1 || size > values.length) {
    throw new RangeError('Window size must be between 1 and the array length');
  }

  let windowSum = 0;

  for (let index = 0; index < size; index += 1) {
    windowSum += values[index];
  }

  let best = windowSum;

  for (let right = size; right < values.length; right += 1) {
    windowSum += values[right] - values[right - size];
    best = Math.max(best, windowSum);
  }

  return best;
}
```

## Follow-up Questions

- Return the start and end indices of the best window.
- Generalize the rolling update to averages or weighted scores.
- Explain how the fixed window differs from Kadane's algorithm.

## Common Pitfalls

- Recomputing each window sum from scratch.
- Forgetting to seed the first full window before sliding.
- Accepting invalid `k` values that are zero or larger than the array.
