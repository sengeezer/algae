## Why It Surfaces

Z Function is a classic string-preprocessing algorithm that tests whether you can preserve a window invariant and reuse previous matches instead of recomputing them character by character.

## Worked Example

For `"aabxaabx"`, the Z value at index `4` is `4` because the suffix starting there, `"aabx"`, matches the whole prefix of the same length.

## Reference Implementations

<ExamplePair>

```js
function zFunction(text) {
  const zArray = new Array(text.length).fill(0);
  let left = 0;
  let right = 0;

  for (let index = 1; index < text.length; index += 1) {
    if (index <= right) {
      zArray[index] = Math.min(right - index + 1, zArray[index - left]);
    }

    while (
      index + zArray[index] < text.length &&
      text[zArray[index]] === text[index + zArray[index]]
    ) {
      zArray[index] += 1;
    }

    if (index + zArray[index] - 1 > right) {
      left = index;
      right = index + zArray[index] - 1;
    }
  }

  return zArray;
}
```

## Follow-up Questions

- Use the Z array on `pattern + '#' + text` to implement substring search.
- Compare the invariant here with the prefix-table invariant in KMP.
- Return the longest border or repeated prefix implied by the Z array.

## Common Pitfalls

- Starting the main loop at index `0` instead of `1`.
- Reusing a prior Z value without capping it to the current window boundary.
- Forgetting to update the `[left, right]` window after extending a match.
