## Why It Surfaces

Bubble sort shows up mostly as a teaching or warm-up question. The value is not the algorithm itself, but recognizing how an early-exit flag changes the best case from quadratic to linear on nearly sorted input.

## Worked Example

For `[5, 1, 4, 2]`, the first pass swaps `5` rightward until it reaches the end. Each additional pass fixes one more suffix position until no swaps happen and the algorithm can stop.

## Reference Implementations

<ExamplePair>

```js
function bubbleSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const items = [...values];

  for (let end = items.length - 1; end > 0; end -= 1) {
    let swapped = false;

    for (let index = 0; index < end; index += 1) {
      if (items[index] > items[index + 1]) {
        [items[index], items[index + 1]] = [items[index + 1], items[index]];
        swapped = true;
      }
    }

    if (!swapped) {
      break;
    }
  }

  return items;
}
```

## Follow-up Questions

- Explain why the early-exit flag gives a linear best case.
- Compare bubble sort with insertion sort on nearly sorted arrays.
- Sort in descending order with the same pattern.

## Common Pitfalls

- Forgetting that the unsorted boundary shrinks after each pass.
- Mutating the original input when the caller expects a copied result.
- Overselling bubble sort as competitive with better quadratic sorts.
