## Why It Surfaces

This is the duplicate-aware version of subset generation. The interview trick is recognizing that duplicate values only cause trouble when you branch on them from the same recursion depth, which is why sorting and skipping equal neighbors works.

## Worked Example

For `[1, 2, 2]`, the valid unique subsets are `[]`, `[1]`, `[2]`, `[1, 2]`, `[2, 2]`, and `[1, 2, 2]`. The second `2` is allowed only when the first `2` is already part of the current branch.

## Reference Implementations

<ExamplePair>

```js
function findAllUniqueSubsets(values) {
  const sorted = [...values].sort((left, right) => left - right);
  const result = [];
  const subset = [];

  function backtrack(start) {
    result.push([...subset]);

    for (let index = start; index < sorted.length; index += 1) {
      if (index > start && sorted[index] === sorted[index - 1]) {
        continue;
      }

      subset.push(sorted[index]);
      backtrack(index + 1);
      subset.pop();
    }
  }

  backtrack(0);
  return result;
}
```

## Common Pitfalls

- Skipping every duplicate globally instead of only duplicates at the same recursion depth.
- Forgetting to sort first, which makes duplicate skipping unreliable.
- Mutating and storing the same subset array without copying it.
