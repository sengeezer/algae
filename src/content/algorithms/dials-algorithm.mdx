## Why It Surfaces

Dial's algorithm is the kind of follow-up that appears after you correctly propose Dijkstra and then notice extra structure in the input. If weights are small integers, a general-purpose priority queue is often more machinery than you need because distance buckets can process nodes in order directly.

## Worked Example

When every edge weight is between `0` and `W`, the shortest simple path to any node uses at most `V - 1` edges, so reachable distances stay inside a manageable bucket range. That lets the algorithm walk the buckets from smallest distance upward instead of paying heap costs on every push and pop.

```js
function dialsAlgorithm(graph, source, maxWeight) {
  if (graph.length === 0) {
    return [];
  }

  const inf = Number.POSITIVE_INFINITY;
  const maxDistance = maxWeight * Math.max(1, graph.length - 1);
  const buckets = Array.from({ length: maxDistance + 1 }, () => []);
  const distances = new Array(graph.length).fill(inf);
  distances[source] = 0;
  buckets[0].push(source);

  for (let distance = 0; distance <= maxDistance; distance += 1) {
    while (buckets[distance].length > 0) {
      const node = buckets[distance].shift();

      if (distance !== distances[node]) {
        continue;
      }

      for (const [neighbor, weight] of graph[node]) {
        if (!Number.isInteger(weight) || weight < 0 || weight > maxWeight) {
          throw new RangeError('Edge weights must be integers in the declared range');
        }

        const candidate = distances[node] + weight;
        if (candidate < distances[neighbor]) {
          distances[neighbor] = candidate;
          buckets[candidate].push(neighbor);
        }
      }
    }
  }

  return distances;
}
```

## Follow-up Questions

- State exactly which weight constraints make Dial's algorithm worthwhile.
- Compare bucket queues with a binary heap and with 0-1 BFS.
- Show how to derive the bucket count bound from the maximum path length in a simple graph.

## Common Pitfalls

- Applying the algorithm to negative or non-integer edge weights.
- Underestimating the distance range and allocating too few buckets.
- Forgetting that stale bucket entries can still appear and must be skipped.
