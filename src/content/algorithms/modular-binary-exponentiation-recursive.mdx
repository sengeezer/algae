## Why It Surfaces

This is the modular version of fast exponentiation. Interviewers use it to check whether you know the logarithmic recurrence and why reduction after each multiplication matters.

## Worked Example

To compute `7^13 mod 11`, recurse on smaller exponents, square the half result for even exponents, and multiply by one more `7` for odd exponents. The work grows with `log2(13)`, not `13`.

## Reference Implementations

<ExamplePair>

```js
function normalize(value, modulus) {
  return ((value % modulus) + modulus) % modulus;
}

function modularBinaryExponentiationRecursive(base, exponent, modulus) {
  if (!Number.isInteger(base) || !Number.isInteger(exponent) || !Number.isInteger(modulus)) {
    throw new TypeError('Base, exponent, and modulus must be integers');
  }

  if (exponent < 0) {
    throw new RangeError('Exponent must be non-negative');
  }

  if (modulus <= 0) {
    throw new RangeError('Modulus must be a positive integer');
  }

  const normalizedBase = normalize(base, modulus);

  if (exponent === 0) {
    return 1 % modulus;
  }

  if (exponent % 2 === 1) {
    return (
      modularBinaryExponentiationRecursive(normalizedBase, exponent - 1, modulus) * normalizedBase
    ) % modulus;
  }

  const half = modularBinaryExponentiationRecursive(normalizedBase, exponent / 2, modulus);
  return (half * half) % modulus;
}
```

## Follow-up Questions

- Replace the recursive version with an iterative loop.
- Explain why the base should be normalized before the recursion starts.
- Use the same idea to implement modular inverse when the modulus is prime.

## Common Pitfalls

- Computing `base^exponent` first and applying `% modulus` only at the end.
- Forgetting that exponent `0` returns `1 mod modulus`.
- Accepting a non-positive modulus and leaving the arithmetic undefined.
