## Why It Surfaces

This is another exact-arithmetic problem, but the multiplication factor changes on every step. The right structure is still a reversed digit array with carry propagation.

## Worked Example

`10! = 3628800`, so the digit sum is `3 + 6 + 2 + 8 + 8 + 0 + 0 = 27`. The computation never needs to form a floating-point approximation of the factorial.

## Reference Implementations

<ExamplePair>

```js
function problem020(limit = 100) {
  if (!Number.isInteger(limit) || limit < 0) {
    throw new RangeError('Limit must be a non-negative integer');
  }

  const digits = [1];

  for (let value = 2; value <= limit; value += 1) {
    let carry = 0;

    for (let index = 0; index < digits.length; index += 1) {
      const product = digits[index] * value + carry;
      digits[index] = product % 10;
      carry = Math.floor(product / 10);
    }

    while (carry > 0) {
      digits.push(carry % 10);
      carry = Math.floor(carry / 10);
    }
  }

  return digits.reduce((sum, digit) => sum + digit, 0);
}
```

## Follow-up Questions

- Return the exact decimal digits of `n!` rather than just their sum.
- Compare this digit-array approach with a `BigInt` implementation.

## Common Pitfalls

- Computing the factorial in a native `Number` and losing exact digits.
- Forgetting that `0! = 1`.
- Dropping the remaining carry after each multiplication pass.
