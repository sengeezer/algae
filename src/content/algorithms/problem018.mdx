## Why It Surfaces

This is the classic case where local choices are not enough, but the problem still decomposes cleanly. The best sum at any position depends only on the best of the two positions directly below it.

## Worked Example

For the triangle `3 / 7 4 / 2 4 6 / 8 5 9 3`, the best route is `3 + 7 + 4 + 9 = 23`. A bottom-up pass turns the last row into reusable subproblem answers for the rows above it.

function parseTriangle(input) {
  if (typeof input === 'string') {
    return input
      .trim()
      .split(/\r?\n/)
      .map((row) => row.trim().split(/\s+/).map(Number));
  }

  if (!Array.isArray(input) || input.length === 0) {
    throw new TypeError('Triangle must be a non-empty string or matrix');
  }

  return input.map((row) => [...row]);
}

function problem018(triangle) {
  const rows = parseTriangle(triangle);

  for (let row = rows.length - 2; row >= 0; row -= 1) {
    for (let column = 0; column < rows[row].length; column += 1) {
      rows[row][column] += Math.max(rows[row + 1][column], rows[row + 1][column + 1]);
    }
  }

  return rows[0][0];
}

## Common Pitfalls

- Trying greedy local choices from the top instead of solving subproblems from the bottom.
- Mutating the original caller-owned triangle unexpectedly without copying it first.
- Forgetting that each row is one element longer than the row above it.
