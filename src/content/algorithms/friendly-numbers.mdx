## Why It Surfaces

Friendly numbers are a neat number-theory exercise because they turn divisor sums into a comparison problem. The key implementation choice is whether you compare ratios directly or cross-multiply to stay exact.

## Worked Example

`6` and `28` are friendly because both are perfect numbers, so each has abundancy index `2`.

function sumDivisors(value) {
  if (value === 1) {
    return 1;
  }

  let total = 1 + value;

  for (let divisor = 2; divisor * divisor <= value; divisor += 1) {
    if (value % divisor !== 0) {
      continue;
    }

    total += divisor;
    const paired = value / divisor;
    if (paired !== divisor) {
      total += paired;
    }
  }

  return total;
}

function friendlyNumbers(first, second) {
  if (![first, second].every((value) => Number.isInteger(value) && value > 0)) {
    throw new RangeError('Values must be positive integers');
  }

  if (first === second) {
    throw new RangeError('Values must be distinct');
  }

  return sumDivisors(first) * second === sumDivisors(second) * first;
}

## Follow-up Questions

- Return the abundancy index itself in reduced fractional form.
- Compare friendly numbers with perfect and amicable numbers.
- Precompute divisor sums when evaluating many pairs.

## Common Pitfalls

- Comparing divisor-sum ratios with floating-point equality.
- Forgetting that the two values must be distinct.
- Using an `O(n)` divisor scan when a square-root-bounded scan is enough.
