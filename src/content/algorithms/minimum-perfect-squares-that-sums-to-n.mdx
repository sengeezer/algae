## Why It Surfaces

This is a good recognition problem because it looks mathematical at first but behaves exactly like an unbounded minimum-coin DP. The only difference is that the available "coins" are all perfect squares up to `n`.

## Worked Example

For `n = 12`, the best answer is `3` because `12 = 4 + 4 + 4`. Using `9` first would leave `3`, which needs three more `1`s and is therefore worse.

## Reference Implementations

<ExamplePair>

```js
function minimumPerfectSquaresThatSumsToN(target) {
  if (target <= 0) {
    return 0;
  }

  const dp = new Array(target + 1).fill(Number.POSITIVE_INFINITY);
  dp[0] = 0;

  for (let value = 1; value <= target; value += 1) {
    for (let root = 1; root * root <= value; root += 1) {
      const square = root * root;
      dp[value] = Math.min(dp[value], dp[value - square] + 1);
    }
  }

  return dp[target];
}
```

## Follow-up Questions

- Return one actual square decomposition, not only the count.
- Compare this DP with theorems that characterize answers for special values.
- Generalize the idea to cubes or other reusable figurate numbers.

## Common Pitfalls

- Treating each square number as usable only once.
- Greedily taking the largest square first and assuming it must be optimal.
- Forgetting that `1` is always a valid square fallback.
