## Why It Surfaces

This is a DFS question disguised as a weighted-graph puzzle. The core idea is to carry the remaining threshold downward through recursion and to backtrack the visited set because the same node may participate in different candidate paths.

## Worked Example

If the source is `0` and the target is `58`, a path like `0 -> 7 -> 1 -> 2 -> 8 -> 6 -> 5 -> 3 -> 4` can succeed even though some individual edges are not the heaviest available at each step. That is why the search must consider full paths, not greedy edge picks.

## Reference Implementations

<ExamplePair>

```js
function pathOfMoreThanKWeightFromSource(vertexCount, edgeList, source, minimumWeight) {
  const graph = Array.from({ length: vertexCount }, () => []);
  for (const [from, to, weight] of edgeList) {
    graph[from].push([to, weight]);
    graph[to].push([from, weight]);
  }

  const visited = new Set([source]);

  function dfs(node, remaining) {
    if (remaining <= 0) {
      return true;
    }

    for (const [next, weight] of graph[node]) {
      if (visited.has(next)) {
        continue;
      }

      visited.add(next);
      if (dfs(next, remaining - weight)) {
        return true;
      }
      visited.delete(next);
    }

    return false;
  }

  return dfs(source, minimumWeight);
}
```

export function pathOfMoreThanKWeightFromSource(
  vertexCount: number,
  edgeList: Array<[number, number, number]>,
  source: number,
  minimumWeight: number,
): boolean {
  const graph: Array<Array<[number, number]>> = Array.from({ length: vertexCount }, () => []);

  for (const [from, to, weight] of edgeList) {
    graph[from].push([to, weight]);
    graph[to].push([from, weight]);
  }

  const visited = new Set<number>([source]);

  const dfs = (node: number, remaining: number): boolean => {
    if (remaining <= 0) {
      return true;
    }

    for (const [next, weight] of graph[node]) {
      if (visited.has(next)) {
        continue;
      }

      visited.add(next);
      if (dfs(next, remaining - weight)) {
        return true;
      }
      visited.delete(next);
    }

    return false;
  };

  return dfs(source, minimumWeight);
}

## Common Pitfalls

- Reusing the same visited set without removing nodes during backtracking.
- Assuming the locally largest edge must belong to a successful path.
- Treating the graph as directed when the input edges are undirected.
