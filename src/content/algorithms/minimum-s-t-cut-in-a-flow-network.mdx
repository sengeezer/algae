## Why It Surfaces

Minimum s-t cut is the theorem-backed companion to maximum flow. After the flow is maximized, the interesting interview step is to use the final residual graph to read off the exact source side and sink side instead of recomputing the cut from scratch.

## Worked Example

Once no more augmenting path exists, perform a traversal from the source using only residual edges with positive capacity. Every original edge that crosses from a reachable vertex to an unreachable vertex belongs to the minimum s-t cut.

```js
function minimumSTCutInAFlowNetwork(capacity, source, sink) {
  const size = capacity.length;
  const residual = capacity.map((row) => [...row]);
  const parent = new Array(size).fill(-1);

  function bfs() {
    parent.fill(-1);
    parent[source] = source;
    const queue = [source];

    for (let head = 0; head < queue.length; head += 1) {
      const node = queue[head];

      for (let neighbor = 0; neighbor < size; neighbor += 1) {
        if (parent[neighbor] === -1 && residual[node][neighbor] > 0) {
          parent[neighbor] = node;
          queue.push(neighbor);
        }
      }
    }

    return parent[sink] !== -1;
  }

  while (bfs()) {
    let flow = Number.POSITIVE_INFINITY;
    for (let node = sink; node !== source; node = parent[node]) {
      flow = Math.min(flow, residual[parent[node]][node]);
    }

    for (let node = sink; node !== source; node = parent[node]) {
      residual[parent[node]][node] -= flow;
      residual[node][parent[node]] += flow;
    }
  }

  const reachable = new Array(size).fill(false);
  const queue = [source];
  reachable[source] = true;

  for (let head = 0; head < queue.length; head += 1) {
    const node = queue[head];

    for (let neighbor = 0; neighbor < size; neighbor += 1) {
      if (!reachable[neighbor] && residual[node][neighbor] > 0) {
        reachable[neighbor] = true;
        queue.push(neighbor);
      }
    }
  }

  const cutEdges = [];
  for (let from = 0; from < size; from += 1) {
    for (let to = 0; to < size; to += 1) {
      if (reachable[from] && !reachable[to] && capacity[from][to] > 0) {
        cutEdges.push([from, to]);
      }
    }
  }

  return cutEdges;
}
```

## Follow-up Questions

- Return the source-side and sink-side partitions along with the cut edges.
- Swap the inner max-flow routine from Edmonds-Karp to Dinic or Push Relabel and compare the tradeoffs.
- Explain why a cut edge can be saturated in the final residual graph.

## Common Pitfalls

- Forgetting that cut recovery happens after max flow, not before it.
- Traversing the original graph instead of the residual graph to find the reachable set.
- Returning every saturated edge instead of only those that cross from reachable to unreachable vertices.
