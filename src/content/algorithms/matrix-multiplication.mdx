## Why It Surfaces

Matrix multiplication is the core operation that powers matrix exponentiation, linear recurrences, and many state-transition optimizations. Interview questions often use it as a smaller building block before asking for something faster or more abstract.

## Worked Example

If the left matrix is `2 x 3` and the right matrix is `3 x 2`, each result cell comes from the dot product of one left row and one right column. That compatibility on the shared inner dimension is the first thing to verify.

## Reference Implementations

<ExamplePair>

```js
function matrixMultiplication(left, right) {
  const rows = left.length;
  const inner = left[0].length;
  const cols = right[0].length;
  const result = Array.from({ length: rows }, () => new Array(cols).fill(0));

  for (let row = 0; row < rows; row += 1) {
    for (let col = 0; col < cols; col += 1) {
      for (let index = 0; index < inner; index += 1) {
        result[row][col] += left[row][index] * right[index][col];
      }
    }
  }

  return result;
}
```

  const rows = left.length;
  const inner = left[0].length;
  const cols = right[0].length;
  const result = Array.from({ length: rows }, () => new Array<number>(cols).fill(0));

  for (let row = 0; row < rows; row += 1) {
    for (let col = 0; col < cols; col += 1) {
      for (let index = 0; index < inner; index += 1) {
        result[row][col] += left[row][index] * right[index][col];
      }
    }
  }

  return result;
}

## Follow-up Questions

- Validate matrix dimensions before multiplying.
- Explain how this operation supports matrix exponentiation.
- Apply a modulus after each accumulation for competitive-programming prompts.

## Common Pitfalls

- Confusing element-wise multiplication with matrix multiplication.
- Forgetting to check the inner dimensions for compatibility.
- Swapping row and column loops and indexing the wrong cell.
