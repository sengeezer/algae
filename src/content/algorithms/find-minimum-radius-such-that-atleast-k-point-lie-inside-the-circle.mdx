## Why It Surfaces

The problem reads like a search over radii, but once the circle center is fixed at the origin, the radius only needs to reach the kth nearest point. That turns the problem into sorting squared distances and selecting the kth value.

## Worked Example

For points `(1, 1)`, `(-1, -1)`, and `(1, -1)` with `k = 3`, the squared distances are all `2`, so the minimum required radius squared is `2`.

## Reference Implementations

<ExamplePair>

```js
function findMinimumRadiusSuchThatAtleastKPointLieInsideTheCircle(points, k) {
  if (!Array.isArray(points) || !Number.isInteger(k) || k < 1 || k > points.length) {
    throw new RangeError("k must be between 1 and the number of points");
  }

  const squaredDistances = points.map(([x, y]) => {
    if (!Number.isFinite(x) || !Number.isFinite(y)) {
      throw new RangeError("Point coordinates must be finite numbers");
    }

    return x * x + y * y;
  }).sort((left, right) => left - right);

  return squaredDistances[k - 1];
}
```

export function findMinimumRadiusSuchThatAtleastKPointLieInsideTheCircle(
  points: Array<[number, number]>,
  k: number,
): number {
  if (!Number.isInteger(k) || k < 1 || k > points.length) {
    throw new RangeError('k must be between 1 and the number of points');
  }

  const squaredDistances = points
    .map(([x, y]) => {
      if (!Number.isFinite(x) || !Number.isFinite(y)) {
        throw new RangeError('Point coordinates must be finite numbers');
      }

      return x * x + y * y;
    })
    .sort((left, right) => left - right);

  return squaredDistances[k - 1];
}

## Common Pitfalls

- Taking square roots even though they are not needed for ordering.
- Forgetting that the answer is the kth smallest distance, not the maximum distance.
- Returning the radius instead of the requested squared radius.
