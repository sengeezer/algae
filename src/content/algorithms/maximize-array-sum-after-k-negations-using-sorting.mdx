## Why It Surfaces

Negating a negative value is always helpful, so sorting exposes the best first choices immediately. Once every negative has been fixed, the only remaining question is whether the leftover number of operations is odd; if it is, the smallest absolute value must absorb that last unavoidable flip.

## Worked Example

For `arr = [-2, 0, 5, -1, 2]` and `k = 4`, sorting gives `[-2, -1, 0, 2, 5]`. Flipping `-2` and `-1` makes the array nonnegative, and the remaining two operations can both be spent on `0`, so the maximum sum is `10`.

## Reference Implementations

<ExamplePair>

```js
function maximizeArraySumAfterKNegationsUsingSorting(values, operations) {
  const sortedValues = [...values].sort((left, right) => left - right);

  for (
    let index = 0;
    index < sortedValues.length && operations > 0 && sortedValues[index] < 0;
    index += 1
  ) {
    sortedValues[index] *= -1;
    operations -= 1;
  }

  let total = 0;
  let minimumAbsoluteValue = Number.POSITIVE_INFINITY;

  for (const value of sortedValues) {
    total += value;
    minimumAbsoluteValue = Math.min(minimumAbsoluteValue, Math.abs(value));
  }

  if (operations % 2 === 1) {
    total -= 2 * minimumAbsoluteValue;
  }

  return total;
}
```

## Common Pitfalls

- Continuing to flip large positive values instead of the smallest absolute value after negatives are exhausted.
- Forgetting that zero can absorb any number of leftover negations harmlessly.
- Assuming the array must be resorted after each flip even though the parity-based final step avoids that extra work.
