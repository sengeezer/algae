## Why It Surfaces

This is a generated-sequence search with a sharp helper test. The key is to avoid storing every pentagonal number indefinitely by checking pentagonality through the inverse formula.

## Worked Example

The `n`th pentagonal number is `P(n) = n(3n - 1) / 2`. A candidate difference `D = P(k) - P(j)` is valid only when both `P(k) - P(j)` and `P(k) + P(j)` are pentagonal.

function pentagonal(index) {
  return (index * (3 * index - 1)) / 2;
}

function isPentagonal(value) {
  const candidate = (Math.sqrt(24 * value + 1) + 1) / 6;
  return Number.isInteger(candidate);
}

function problem044(startIndex = 1) {
  if (!Number.isInteger(startIndex) || startIndex < 1) {
    throw new RangeError('Start index must be a positive integer');
  }

  for (let k = startIndex + 1; ; k += 1) {
    const pk = pentagonal(k);

    for (let j = k - 1; j >= 1; j -= 1) {
      const pj = pentagonal(j);
      const difference = pk - pj;

      if (isPentagonal(difference) && isPentagonal(pk + pj)) {
        return difference;
      }
    }
  }
}

## Common Pitfalls

- Treating approximate floating-point roots as exact without an integer check.
- Searching sums and differences by scanning a list linearly instead of using a closed-form membership test.
- Forgetting that the search should minimize the difference, not the pentagonal values themselves.
