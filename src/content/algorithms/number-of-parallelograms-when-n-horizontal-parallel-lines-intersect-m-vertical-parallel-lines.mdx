## Why It Surfaces

The geometric picture can distract from the simple counting structure. A parallelogram is fully determined by picking any two horizontal parallels and any two vertical parallels, so the answer is just a product of two binomial coefficients.

## Worked Example

For `n = 3` and `m = 2`, the count is `C(3, 2) * C(2, 2) = 3 * 1 = 3`.

```js
function numberOfParallelogramsWhenNHorizontalParallelLinesIntersectMVerticalParallelLines(n, m) {
  if (!Number.isInteger(n) || !Number.isInteger(m) || n < 0 || m < 0) {
    throw new RangeError("Line counts must be non-negative integers");
  }

  if (n < 2 || m < 2) {
    return 0;
  }

  return (n * (n - 1) * m * (m - 1)) / 4;
}
```

## Common Pitfalls

- Confusing the number of lines with the number of cells between lines.
- Forgetting that at least two lines from each family are required.
- Enumerating every possible shape instead of recognizing the direct combination count.
