## Why It Surfaces

This question checks whether you can reuse a familiar bit trick instead of restarting from scratch. The key step is noticing that not every power of two is a power of four.

## Worked Example

`16` is `10000` in binary, so it passes the one-set-bit check, and `16 % 3 === 1`, so it is a power of four. `8` also has one set bit, but `8 % 3 === 2`, so it is only a power of two.

## Reference Implementations

<ExamplePair>

```js
function isPowerOfFour(value) {
  return value > 0 && (value & (value - 1)) === 0 && value % 3 === 1;
}
```

```ts
export function isPowerOfFour(value: number): boolean {
  return value > 0 && (value & (value - 1)) === 0 && value % 3 === 1;
}
```

</ExamplePair>

## Follow-up Questions

- Rework the solution with a bit mask such as `0x55555555` instead of the modulo test.
- Explain why every power of four leaves remainder `1` when divided by `3`.
- Compare this with repeated division by `4` for readability and cost.

## Common Pitfalls

- Forgetting that the power-of-two check alone accepts values like `2`, `8`, and `32`.
- Omitting the positivity guard and accidentally accepting `0`.
- Using bitwise operators without remembering JavaScript treats them as 32-bit integer operations.
