## Why It Surfaces

This prompt usually appears right after someone has seen longest palindromic subsequence or edit distance. The key distinction is that you are not allowed to delete anything, so a mismatch at the ends forces one insertion and then the problem continues from the better of the two adjacent intervals.

## Worked Example

For `"geeks"`, the answer is `3`. One valid result is `"skgeegks"`, which inserts three characters to mirror the original string.

## Reference Implementations

<ExamplePair>

```js
function minimumInsertionsToFormAPalindrome(text) {
  const size = text.length;
  if (size <= 1) {
    return 0;
  }

  const dp = new Array(size).fill(0);

  for (let left = size - 2; left >= 0; left -= 1) {
    let diagonal = 0;

    for (let right = left + 1; right < size; right += 1) {
      const saved = dp[right];

      if (text[left] === text[right]) {
        dp[right] = diagonal;
      } else {
        dp[right] = Math.min(dp[right], dp[right - 1]) + 1;
      }

      diagonal = saved;
    }
  }

  return dp[size - 1];
}
```

## Follow-up Questions

- Reconstruct one minimum-insertion palindrome, not only the count.
- Show why the answer is also `text.length - LPS(text)`.
- Compare this interval DP with edit distance to the reversed string.

## Common Pitfalls

- Solving the deletion version and assuming the same recurrence always applies.
- Forgetting that matching ends shrink the interval without adding cost.
- Overwriting the diagonal dependency too early in the space-optimized version.
