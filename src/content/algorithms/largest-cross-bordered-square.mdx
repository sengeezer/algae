## Why It Surfaces

This is the square-border variant where checking every border from scratch becomes too expensive. The right answer is to precompute how many consecutive `X` cells extend rightward and downward from every position, then use those counts to validate a candidate border with a couple of comparisons.

## Worked Example

For

```txt
X X
X X
```

the answer is `2`, because the whole matrix is already a square whose entire border is `X`.

```js
function largestCrossBorderedSquare(matrix) {
  const rowCount = matrix.length;
  if (rowCount === 0) {
    return 0;
  }

  const colCount = matrix[0].length;
  const right = Array.from({ length: rowCount }, () => new Array(colCount).fill(0));
  const down = Array.from({ length: rowCount }, () => new Array(colCount).fill(0));

  for (let row = rowCount - 1; row >= 0; row -= 1) {
    for (let col = colCount - 1; col >= 0; col -= 1) {
      if (matrix[row][col] !== 'X') {
        continue;
      }

      right[row][col] = 1 + (col + 1 < colCount ? right[row][col + 1] : 0);
      down[row][col] = 1 + (row + 1 < rowCount ? down[row + 1][col] : 0);
    }
  }

  let best = 0;

  for (let row = 0; row < rowCount; row += 1) {
    for (let col = 0; col < colCount; col += 1) {
      const limit = Math.min(right[row][col], down[row][col]);

      for (let side = limit; side > best; side -= 1) {
        const bottomRow = row + side - 1;
        const rightCol = col + side - 1;

        if (
          bottomRow < rowCount
          && rightCol < colCount
          && right[bottomRow][col] >= side
          && down[row][rightCol] >= side
        ) {
          best = side;
          break;
        }
      }
    }
  }

  return best;
}
```

- Return the top-left coordinate of one maximum bordered square.

## Common Pitfalls

- Checking only the top and left borders and forgetting the bottom and right borders.
- Assuming the matrix must be square when the same precomputation works on rectangular input.
- Recomputing border scans from scratch for every candidate square.
