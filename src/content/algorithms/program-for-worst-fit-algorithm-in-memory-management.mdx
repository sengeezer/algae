## Why It Surfaces

Worst fit is the mirror image of best fit: instead of minimizing slack immediately, it spends the largest hole first so smaller and medium holes remain untouched. That makes it a strong teaching example for why different greedy objectives can feel plausible yet still fail differently on the same sequence of requests.

## Worked Example

For `blockSizes = [100, 500, 200, 300, 600]` and `processSizes = [212, 417, 112, 426]`, worst fit places `212` into block `5`, `417` into block `2`, `112` into the remaining space of block `5`, and cannot place `426`. The resulting 1-based allocation array is `[5, 2, 5, -1]`.

## Reference Implementations

<ExamplePair>

```js
function programForWorstFitAlgorithmInMemoryManagement(blockSizes, processSizes) {
  const remainingBlocks = [...blockSizes];
  const allocation = Array(processSizes.length).fill(-1);

  for (let processIndex = 0; processIndex < processSizes.length; processIndex += 1) {
    let worstBlockIndex = -1;

    for (let blockIndex = 0; blockIndex < remainingBlocks.length; blockIndex += 1) {
      if (remainingBlocks[blockIndex] < processSizes[processIndex]) {
        continue;
      }

      if (
        worstBlockIndex === -1 ||
        remainingBlocks[blockIndex] > remainingBlocks[worstBlockIndex]
      ) {
        worstBlockIndex = blockIndex;
      }
    }

    if (worstBlockIndex === -1) {
      continue;
    }

    allocation[processIndex] = worstBlockIndex + 1;
    remainingBlocks[worstBlockIndex] -= processSizes[processIndex];
  }

  return allocation;
}
```

## Follow-up Questions

- Give an input where worst fit allocates fewer processes than first fit.
- Return the remaining free blocks after all allocations, not only the chosen block numbers.
- Replace the repeated scans with a max-heap keyed by remaining block capacity.

## Common Pitfalls

- Choosing the largest block overall even when it still cannot fit the current process.
- Forgetting that the same block may be selected again later after its remaining size shrinks.
- Assuming worst fit automatically leaves enough room for future large processes.
