## Why It Surfaces

Radix sort is the next step after counting sort once the keys have multiple digits. The interview challenge is usually not the buckets themselves, but recognizing that stable passes let earlier work survive later digit groupings.

## Worked Example

For `[170, 45, 75, 90, 802, 24, 2, 66]`, sort first by ones digit, then tens digit, then hundreds digit. Because each pass is stable, the final order is globally sorted once the most significant processed digit is complete.

## Reference Implementations

<ExamplePair>

```js
function radixSort(values, base = 10) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of nonnegative integers');
  }

  if (values.some((value) => !Number.isInteger(value) || value < 0)) {
    throw new TypeError('Radix sort expects nonnegative integers');
  }

  let output = [...values];
  let max = 0;

  for (const value of output) {
    if (value > max) {
      max = value;
    }
  }

  for (let place = 1; Math.floor(max / place) > 0; place *= base) {
    const buckets = Array.from({ length: base }, () => []);

    for (const value of output) {
      const digit = Math.floor(value / place) % base;
      buckets[digit].push(value);
    }

    output = buckets.flat();
  }

  return output;
}
```

## Follow-up Questions

- Replace per-digit buckets with a counting-sort pass.
- Extend the approach to signed integers or fixed-width strings.
- Explain why the digit passes must be stable.

## Common Pitfalls

- Applying radix sort directly to negative numbers without defining a policy.
- Forgetting that unstable digit passes break the final order.
- Overstating the benefit when the digit width or base is large.
