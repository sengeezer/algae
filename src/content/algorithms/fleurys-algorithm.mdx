## Why It Surfaces

Fleury's algorithm is the classical construction method that turns Eulerian theory into an actual undirected traversal. The key interview move is to avoid burning a bridge while another choice still exists, because a premature bridge can strand unused edges.

## Worked Example

If the current vertex has three incident edges and one of them is a bridge, Fleury prefers one of the other two first. Only when an edge is the last remaining option does the algorithm accept a bridge and move through it.

```js
function fleurysAlgorithm(graph) {
  const remaining = graph.map((neighbors) => [...neighbors]);
  const trail = [];

  function removeEdge(from, to) {
    remaining[from].splice(remaining[from].indexOf(to), 1);
    remaining[to].splice(remaining[to].indexOf(from), 1);
  }

  function addEdge(from, to) {
    remaining[from].push(to);
    remaining[to].push(from);
  }

  function countReachable(node, seen) {
    seen[node] = true;
    let count = 1;

    for (const neighbor of remaining[node]) {
      if (!seen[neighbor]) {
        count += countReachable(neighbor, seen);
      }
    }

    return count;
  }

  function isValidNextEdge(from, to) {
    if (remaining[from].length === 1) {
      return true;
    }

    const seenBefore = new Array(remaining.length).fill(false);
    const reachableBefore = countReachable(from, seenBefore);
    removeEdge(from, to);
    const seenAfter = new Array(remaining.length).fill(false);
    const reachableAfter = countReachable(from, seenAfter);
    addEdge(from, to);
    return reachableBefore === reachableAfter;
  }

  function walk(node) {
    while (remaining[node].length > 0) {
      let next = remaining[node][0];
      for (const candidate of [...remaining[node]]) {
        if (isValidNextEdge(node, candidate)) {
          next = candidate;
          break;
        }
      }

      removeEdge(node, next);
      trail.push([node, next]);
      walk(next);
    }
  }

  const odd = remaining.findIndex((neighbors) => neighbors.length % 2 === 1);
  const start = odd !== -1 ? odd : remaining.findIndex((neighbors) => neighbors.length > 0);
  if (start === -1) {
    return trail;
  }

  walk(start);
  return trail;
}
```

## Follow-up Questions

- Compare Fleury with Hierholzer and explain why Hierholzer is preferred in practice.
- Add an explicit Eulerian feasibility check before construction begins.
- Return the vertices in the tour instead of the traversed edges.

## Common Pitfalls

- Removing a bridge too early when another unused edge is available.
- Forgetting that Fleury is for undirected Euler tours in this form.
- Mutating the caller's adjacency lists instead of working on a copy.
