## Why It Surfaces

Kebab-case checks are common when strings need to be safe for routes, CSS classes, or command names. The interviewer often wants to see whether you encode the style precisely instead of checking only for the presence of hyphens.

## Worked Example

`"user-name"` passes because each segment is lowercase and separated by a single hyphen. `"user_Name"` fails because underscores are not part of kebab-case.

## Reference Implementations

<ExamplePair>

```js
function checkKebabCase(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  return /^[a-z]+(?:-[a-z]+)*$/.test(value);
}
```

```ts
export function checkKebabCase(value: string): boolean {
  return /^[a-z]+(?:-[a-z]+)*$/.test(value);
}
```

</ExamplePair>

## Follow-up Questions

## Common Pitfalls

- Accepting underscores or spaces.
- Allowing leading, trailing, or repeated hyphens.
- Forgetting to reject uppercase letters.
