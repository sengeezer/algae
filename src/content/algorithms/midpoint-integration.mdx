## Why It Surfaces

This is a compact numerical-method question that checks whether you can translate an approximation formula into a loop without mixing up step size, endpoints, and sample locations.

## Worked Example

To approximate `integral from 0 to 2 of x^2 dx` with `4` subintervals, use width `0.5` and sample at `0.25`, `0.75`, `1.25`, and `1.75`. The midpoint rule gives `0.5 * (0.0625 + 0.5625 + 1.5625 + 3.0625) = 2.625`, which is close to the exact value `8 / 3`.

## Reference Implementations

<ExamplePair>

```js
function midpointIntegration(segments, left, right, fn) {
  if (!Number.isInteger(segments) || segments <= 0) {
    throw new RangeError('Segment count must be a positive integer');
  }

  if (!Number.isFinite(left) || !Number.isFinite(right)) {
    throw new TypeError('Bounds must be finite numbers');
  }

  if (typeof fn !== 'function') {
    throw new TypeError('Expected a function to integrate');
  }

  if (left > right) {
    throw new RangeError('Left bound must not exceed right bound');
  }

  if (left === right) {
    return 0;
  }

  const width = (right - left) / segments;
  let total = 0;

  for (let index = 0; index < segments; index += 1) {
    const midpoint = left + (index + 0.5) * width;
    const value = fn(midpoint);

    if (!Number.isFinite(value)) {
      throw new RangeError('Function must return a finite number inside the interval');
    }

    total += value;
  }

  return total * width;
}
```

## Follow-up Questions

- Compare midpoint integration with left-endpoint and trapezoidal approximations.
- Increase the segment count until the approximation error drops below a target threshold.
- Explain why a domain error inside the interval should fail fast instead of silently returning `NaN`.

## Common Pitfalls

- Sampling at the interval endpoints instead of the midpoints.
- Forgetting that a zero-width interval integrates to `0`.
- Letting a `NaN` function value propagate through the whole approximation silently.
