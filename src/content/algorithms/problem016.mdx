## Why It Surfaces

This is another precision-management problem, but the input comes from repeated multiplication instead of external strings. The standard `Number` type cannot represent the full power exactly, so the digits have to be carried manually.

## Worked Example

`2^15 = 32768`, and the digit sum is `3 + 2 + 7 + 6 + 8 = 26`. A digit-array simulation multiplies by `2` repeatedly and stores the resulting decimal digits exactly.

## Reference Implementations

```js
function problem016(base = 2, exponent = 1000) {
  if (!Number.isInteger(base) || base < 0 || !Number.isInteger(exponent) || exponent < 0) {
    throw new RangeError('Base and exponent must be non-negative integers');
  }

  const digits = [1];

  for (let power = 0; power < exponent; power += 1) {
    let carry = 0;

    for (let index = 0; index < digits.length; index += 1) {
      const product = digits[index] * base + carry;
      digits[index] = product % 10;
      carry = Math.floor(product / 10);
    }

    while (carry > 0) {
      digits.push(carry % 10);
      carry = Math.floor(carry / 10);
    }
  }

  return digits.reduce((sum, digit) => sum + digit, 0);
}
```

## Follow-up Questions

- Return the full decimal representation instead of only the digit sum.
- Replace repeated multiplication with exponentiation by squaring plus exact big-number multiplication.
- Compare this manual approach with a `BigInt` implementation.

## Common Pitfalls

- Using floating-point exponentiation and losing exact digits.
- Forgetting the special case `exponent = 0`, where the value should start from `1`.
- Dropping the final carry after a multiplication pass.
