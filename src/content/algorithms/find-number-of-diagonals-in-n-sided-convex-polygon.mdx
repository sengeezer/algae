## Why It Surfaces

This is a clean combinatorics-on-geometry prompt. From any vertex in a convex `n`-gon, you can connect to every other vertex except itself and the two adjacent neighbors, which leaves `n - 3` diagonal choices.

## Worked Example

For `n = 5`, the polygon has `5 * (5 - 3) / 2 = 5` diagonals.

## Reference Implementations

<ExamplePair>

```js
function findNumberOfDiagonalsInNSidedConvexPolygon(sides) {
  if (!Number.isInteger(sides) || sides < 3) {
    throw new RangeError("A polygon must have at least three sides");
  }

  return (sides * (sides - 3)) / 2;
}
```

```ts
export function findNumberOfDiagonalsInNSidedConvexPolygon(sides: number): number {
  if (!Number.isInteger(sides) || sides < 3) {
    throw new RangeError('A polygon must have at least three sides');
  }

  return (sides * (sides - 3)) / 2;
}
```

## Common Pitfalls

- Forgetting to divide by `2` after counting from every vertex.
- Allowing polygons with fewer than three sides as if they were valid inputs.
- Confusing diagonals with all line segments between vertices, including edges.
