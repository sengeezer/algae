## Why It Surfaces

Date-to-day conversion is one of the classic calendar arithmetic exercises. It is useful because it turns a human-readable date into a weekday label without iterating through months or relying entirely on a runtime helper.

## Worked Example

`18/12/2020` maps to `Friday`. The formula shifts January and February into the previous year and then applies modular arithmetic to recover the weekday index.

## Reference Implementations

<ExamplePair>

```js
const WEEKDAYS = ['Saturday', 'Sunday', 'Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday'];

function parseDateString(input) {
  const match = /^(\d{1,2})\/(\d{1,2})\/(\d{4})$/.exec(input);
  if (!match) {
    throw new TypeError('Dates must use dd/mm/yyyy format');
  }

  const day = Number(match[1]);
  const month = Number(match[2]);
  const year = Number(match[3]);
  const candidate = new Date(year, month - 1, day);

  if (candidate.getFullYear() !== year || candidate.getMonth() !== month - 1 || candidate.getDate() !== day) {
    throw new RangeError('Invalid calendar date');
  }

  return { day, month, year };
}

function dateToDay(input) {
  const parsed = parseDateString(input);
  let year = parsed.year;
  let month = parsed.month;

  if (month < 3) {
    year -= 1;
    month += 12;
  }

  const yearDigits = year % 100;
  const century = Math.floor(year / 100);
  const weekday =
    (parsed.day +
      Math.floor((month + 1) * 2.6) +
      yearDigits +
      Math.floor(yearDigits / 4) +
      Math.floor(century / 4) +
      5 * century) %
    7;

  return WEEKDAYS[weekday];
}
```

## Common Pitfalls

- Forgetting the month and year adjustment for January and February.
- Producing a negative modulo result in languages with different remainder rules.
- Accepting malformed or impossible dates without validation.
