## Why It Surfaces

Reverse-words problems look trivial until whitespace rules matter. The interviewer usually wants to know whether you are reversing tokens, characters, or preserving exact spacing.

## Worked Example

For `"  blue   sky here  "`, splitting on whitespace gives `['blue', 'sky', 'here']`, and reversing the words yields `"here sky blue"`.

## Reference Implementations

<ExamplePair>

```js
function reverseWords(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  return value
    .trim()
    .split(/\s+/)
    .filter(Boolean)
    .reverse()
    .join(' ');
}
```

```ts
export function reverseWords(value: string): string {
  return value
    .trim()
    .split(/\s+/)
    .filter(Boolean)
    .reverse()
    .join(' ');
}
```

</ExamplePair>

## Follow-up Questions

- Reverse each word's characters in place while preserving word order.
- Preserve the exact spacing layout instead of normalizing it.
- Solve the problem in place on a mutable character array.

## Common Pitfalls

- Reversing characters instead of words.
- Forgetting to trim leading and trailing spaces when normalization is expected.
- Collapsing repeated spaces even when the prompt requires preserving them exactly.
