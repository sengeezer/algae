## Why It Surfaces

Fisher-Yates is the shuffle interview problem because it tests both algorithmic efficiency and probability reasoning. The key argument is that every remaining element must have an equal chance to land in the current position.

## Worked Example

For `[A, B, C, D]`, first choose a random index from `0..3` for the last slot, then a random index from `0..2` for the next slot, and so on. Each pass fixes one final position without disturbing the suffix that is already done.

## Reference Implementations

<ExamplePair>

```js
function fisherYatesShuffle(values, rng = Math.random) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array');
  }

  const items = [...values];

  for (let index = items.length - 1; index > 0; index -= 1) {
    const swapIndex = Math.floor(rng() * (index + 1));
    [items[index], items[swapIndex]] = [items[swapIndex], items[index]];
  }

  return items;
}
```

```ts
export function fisherYatesShuffle<T>(
  values: T[],
  rng: () => number = Math.random,
): T[] {
  const items = [...values];

  for (let index = items.length - 1; index > 0; index -= 1) {
    const swapIndex = Math.floor(rng() * (index + 1));
    [items[index], items[swapIndex]] = [items[swapIndex], items[index]];
  }

  return items;
}
```

## Follow-up Questions

- Prove that each permutation is equally likely.
- Shuffle in place instead of returning a copied array.
- Sample only `k` random elements without fully shuffling the input.

## Common Pitfalls

- Picking the swap index from the full array every time instead of the shrinking prefix.
- Using a random comparator sort, which is both biased and slower.
- Off-by-one errors when choosing the inclusive upper bound for the random index.
