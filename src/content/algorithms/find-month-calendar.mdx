## Why It Surfaces

Find Month Calendar is a good date-arithmetic exercise because every small rule matters. Leap years, month lengths, and the weekday offset of the first day all need to line up before the final grid can be printed correctly.

## Worked Example

To render `1/2021`, first compute the weekday for January 1st, then prepend blank cells before day `1`, then stream the remaining dates into rows of seven slots until the month is exhausted.

## Reference Implementations

<ExamplePair>

```js
function findMonthCalendar(month, year) {
  const monthLengths = [31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31];
  const weekdayLabels = ['M', 'T', 'W', 'Th', 'F', 'S', 'Su'];

  function isLeapYear(value) {
    return (value % 400 === 0) || (value % 4 === 0 && value % 100 !== 0);
  }

  function daysInMonth(targetMonth, targetYear) {
    if (targetMonth === 2 && isLeapYear(targetYear)) {
      return 29;
    }

    return monthLengths[targetMonth - 1];
  }

  let dayOffset = 0;
  for (let currentYear = 1900; currentYear < year; currentYear += 1) {
    dayOffset += isLeapYear(currentYear) ? 366 : 365;
  }

  for (let currentMonth = 1; currentMonth < month; currentMonth += 1) {
    dayOffset += daysInMonth(currentMonth, year);
  }

  const startIndex = dayOffset % 7;
  const cells = new Array(startIndex).fill('   ');

  for (let day = 1; day <= daysInMonth(month, year); day += 1) {
    cells.push(String(day).padEnd(3, ' '));
  }

  const rows = [weekdayLabels.join(' ')];
  for (let index = 0; index < cells.length; index += 7) {
    rows.push(cells.slice(index, index + 7).join(' ').trimEnd());
  }

  return rows;
}
```

## Follow-up Questions

- Support different week starts or localized weekday labels.
- Compute the weekday with Zeller's congruence instead of an epoch walk.
- Return a matrix structure instead of preformatted strings.

## Common Pitfalls

- Mishandling leap years in February.
- Using the wrong weekday basis for the chosen epoch.
- Forgetting to prepend empty cells before day `1`.
