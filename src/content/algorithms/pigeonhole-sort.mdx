## Why It Surfaces

Pigeonhole sort is the clean answer when every item is an integer and the range is small enough to index directly. Instead of comparing values against each other, you count how many times each value occurs and then rebuild the array.

## Worked Example

For `[8, 3, 2, 7, 4, 6, 8]`, the range is `2..8`, so create one counter for each value in that interval. After counting frequencies, rewrite the values from the smallest occupied hole to the largest.

## Reference Implementations

<ExamplePair>

```js
function pigeonholeSort(values) {
  if (!Array.isArray(values) || values.some((value) => !Number.isInteger(value))) {
    throw new TypeError('Values must be an array of integers');
  }

  if (values.length < 2) {
    return [...values];
  }

  const items = [...values];
  let min = items[0];
  let max = items[0];

  for (const value of items) {
    if (value < min) {
      min = value;
    }

    if (value > max) {
      max = value;
    }
  }

  const holes = new Array(max - min + 1).fill(0);

  for (const value of items) {
    holes[value - min] += 1;
  }

  let writeIndex = 0;

  for (let offset = 0; offset < holes.length; offset += 1) {
    while (holes[offset] > 0) {
      items[writeIndex] = offset + min;
      writeIndex += 1;
      holes[offset] -= 1;
    }
  }

  return items;
}
```

## Common Pitfalls

- Ignoring negative integers and forgetting the offset by `min`.
- Using pigeonhole sort when the value range is vastly larger than the input.
- Treating it as a generic comparison sort for arbitrary data types.
