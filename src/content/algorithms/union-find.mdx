## Why It Surfaces

Union-Find is the compact answer when components merge over time. It shows up in connectivity, redundant connection, account merge, and minimum spanning tree style questions.

## JavaScript

```js
class UnionFind {
  constructor(size) {
    this.parent = Array.from({ length: size }, (_, index) => index);
    this.rank = new Array(size).fill(0);
  }

  find(node) {
    if (this.parent[node] !== node) {
      this.parent[node] = this.find(this.parent[node]);
    }

    return this.parent[node];
  }

  union(left, right) {
    const rootLeft = this.find(left);
    const rootRight = this.find(right);

    if (rootLeft === rootRight) {
      return false;
    }

    if (this.rank[rootLeft] < this.rank[rootRight]) {
      this.parent[rootLeft] = rootRight;
    } else if (this.rank[rootLeft] > this.rank[rootRight]) {
      this.parent[rootRight] = rootLeft;
    } else {
      this.parent[rootRight] = rootLeft;
      this.rank[rootLeft] += 1;
    }

    return true;
  }
}
```

## Interview Notes

- Name path compression and union by rank explicitly.
- Use it when repeated connectivity checks would make graph traversal too expensive.
- Mention Kruskal's algorithm if the interviewer pivots toward MSTs.
