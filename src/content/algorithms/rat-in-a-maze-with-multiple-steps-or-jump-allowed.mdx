## Why It Surfaces

This is a good follow-up to the standard rat-in-a-maze problem because the state transition changes from one-step moves to a range of legal jumps. That forces you to search over candidate jump lengths while still preserving the path and backtracking cleanly.

## Worked Example

For

```txt
[
  [2, 1, 0, 0],
  [3, 0, 0, 1],
  [0, 1, 0, 1],
  [0, 0, 0, 1],
]
```

one shortest solution is to go from `(0, 0)` down to `(1, 0)`, jump right to `(1, 3)`, and then move down to the destination `(3, 3)`. The returned matrix marks exactly those cells with `1`.

```js
function ratInAMazeWithMultipleStepsOrJumpAllowed(mat) {
  const n = mat.length;

  if (n === 0 || mat.some((row) => row.length !== n) || mat[0][0] === 0 || mat[n - 1][n - 1] === 0) {
    return [[-1]];
  }

  const path = Array.from({ length: n }, () => Array(n).fill(0));
  let bestPath = null;
  let bestHops = Number.POSITIVE_INFINITY;

  function search(row, column, hops) {
    if (row < 0 || row >= n || column < 0 || column >= n || mat[row][column] === 0) {
      return;
    }

    if (hops >= bestHops) {
      return;
    }

    path[row][column] = 1;

    if (row === n - 1 && column === n - 1) {
      bestHops = hops;
      bestPath = path.map((currentRow) => [...currentRow]);
      path[row][column] = 0;
      return;
    }

    const jumpLimit = mat[row][column];

    for (let jump = 1; jump <= jumpLimit; jump += 1) {
      search(row, column + jump, hops + 1);
    }

    for (let jump = 1; jump <= jumpLimit; jump += 1) {
      search(row + jump, column, hops + 1);
    }

    path[row][column] = 0;
  }

  search(0, 0, 0);
  return bestPath ?? [[-1]];
}
```

## Common Pitfalls

- Treating the cell value as an exact jump instead of a maximum jump.
- Returning the first path found even when a shorter one exists.
- Forgetting that the required output is the marked path matrix, not the move list.
