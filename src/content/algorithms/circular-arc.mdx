## Why It Surfaces

Circular-arc formulas are small, but they still test whether you remember that the clean formulas use radians, not degrees. That unit conversion is the main trap.

## Worked Example

With radius `3` and angle `45` degrees, the angle in radians is `pi / 4`, so the arc length is `3 * pi / 4`, or about `2.356`.

## Reference Implementations

<ExamplePair>

```js
function degreeToRadian(degrees) {
  return degrees * (Math.PI / 180);
}

function circularArcLength(radius, degrees) {
  if (![radius, degrees].every((value) => typeof value === 'number' && Number.isFinite(value) && value >= 0)) {
    throw new RangeError('Radius and degrees must be non-negative finite numbers');
  }

  return radius * degreeToRadian(degrees);
}

function circularArcArea(radius, degrees) {
  if (![radius, degrees].every((value) => typeof value === 'number' && Number.isFinite(value) && value >= 0)) {
    throw new RangeError('Radius and degrees must be non-negative finite numbers');
  }

  return (radius * radius * degreeToRadian(degrees)) / 2;
}
```

## Follow-up Questions

- Accept radian input directly and skip the conversion step.
- Explain why the full-circle case with `360` degrees reduces to circumference and full area.
- Add chord length as another derived circle quantity.

## Common Pitfalls

- Applying the formulas directly to degree values.
- Forgetting that radius and angle should be non-negative in this helper.
- Mixing arc length and sector area even though they scale differently with radius.
