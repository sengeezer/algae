## Why It Surfaces

Pascal Triangle is a compact way to test whether you can generate structured output from previously computed state. It also opens the door to follow-ups about combinations, binomial coefficients, and row-space optimization.

## Worked Example

Starting from `[1]`, the next row is `[1, 1]`. The next row becomes `[1, 2, 1]` because the center value comes from `1 + 1`. Repeating that rule generates the full triangle.

## Reference Implementations

<ExamplePair>

```js
function pascalTriangle(numRows) {
  if (numRows === 0) {
    return [];
  }

  const triangle = [[1]];

  for (let row = 1; row < numRows; row += 1) {
    const previous = triangle[row - 1];
    const nextRow = [1];

    for (let index = 0; index < previous.length - 1; index += 1) {
      nextRow.push(previous[index] + previous[index + 1]);
    }

    nextRow.push(1);
    triangle.push(nextRow);
  }

  return triangle;
}
```

## Follow-up Questions

- Return only the `k`th row using `O(k)` extra space.
- Relate row values to combination counts like `n choose r`.
- Explain how this differs from a closed-form combinatorics solution.

## Common Pitfalls

- Forgetting that every row starts and ends with `1`.
- Mutating the previous row before the next row is fully computed.
- Overcomplicating a simple constructive DP problem.
