## Why It Surfaces

Recursive Fibonacci is the smallest example of overlapping subproblems. The formula is easy to state, but the call tree duplicates work immediately, which makes it a clean setup for explaining memoization and dynamic programming.

## Worked Example

To compute `F(5)`, the recursion expands into `F(4) + F(3)`, then `F(4)` expands again into `F(3) + F(2)`. That repeated `F(3)` is the signal that the direct recursion is correct but inefficient.

## Reference Implementations

<ExamplePair>

```js
function fibonacciNumberRecursive(index) {
  if (!Number.isInteger(index) || index < 0) {
    throw new RangeError('Index must be a non-negative integer');
  }

  if (index < 2) {
    return index;
  }

  return (
    fibonacciNumberRecursive(index - 1)
    + fibonacciNumberRecursive(index - 2)
  );
}
```

  );

## Follow-up Questions

- Add memoization and compare the new time complexity.
- Rewrite the solution iteratively with only two rolling variables.
- Estimate how fast the recursion tree grows as `n` increases.

## Common Pitfalls

- Using the naive recursion for large `n` and timing out.
- Forgetting that the base cases are `0` and `1`.
- Describing this as dynamic programming before any caching or tabulation is added.
