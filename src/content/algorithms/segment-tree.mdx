## Why It Surfaces

Segment trees appear when the array changes over time and the query needs information about arbitrary intervals. They recursively partition the array so each node owns a segment and can answer or combine only the parts that overlap a query.

## Worked Example

To query the sum over `[2, 6]`, a segment tree skips nodes whose ranges are fully outside, returns nodes fully inside, and recursively splits only the partial overlaps. That is where the logarithmic behavior comes from.

<Callout title="When to choose it" tone="note">
  If the problem only needs dynamic prefix sums, a Fenwick tree is lighter. If it needs range minimums, richer combines, or a path to lazy propagation, a segment tree is the safer abstraction.
</Callout>

```js
class SegmentTree {
  constructor(values) {
    this.size = values.length;
    this.tree = new Array(this.size * 4).fill(0);
    this.build(values, 1, 0, this.size - 1);
  }

  build(values, node, left, right) {
    if (left === right) {
      this.tree[node] = values[left];
      return;
    }

    const mid = Math.floor((left + right) / 2);
    this.build(values, node * 2, left, mid);
    this.build(values, node * 2 + 1, mid + 1, right);
    this.tree[node] = this.tree[node * 2] + this.tree[node * 2 + 1];
  }

  update(index, value, node = 1, left = 0, right = this.size - 1) {
    if (left === right) {
      this.tree[node] = value;
      return;
    }

    const mid = Math.floor((left + right) / 2);
    if (index <= mid) {
      this.update(index, value, node * 2, left, mid);
    } else {
      this.update(index, value, node * 2 + 1, mid + 1, right);
    }

    this.tree[node] = this.tree[node * 2] + this.tree[node * 2 + 1];
  }

  query(queryLeft, queryRight, node = 1, left = 0, right = this.size - 1) {
    if (queryRight < left || right < queryLeft) {
      return 0;
    }

    if (queryLeft <= left && right <= queryRight) {
      return this.tree[node];
    }

    const mid = Math.floor((left + right) / 2);
    return (
      this.query(queryLeft, queryRight, node * 2, left, mid)
      + this.query(queryLeft, queryRight, node * 2 + 1, mid + 1, right)
    );
  }
}
```

## Interview Notes

- Define what each node stores before talking about code.
- Call out the three overlap cases: none, full, and partial.
- Mention lazy propagation only if the interviewer introduces range updates.
