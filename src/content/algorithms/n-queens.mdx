## Why It Surfaces

N-Queens is less about chess and more about search discipline. The interviewer wants to hear how you track forbidden columns and diagonals so every recursive step rules out large parts of the search space.

## Worked Example

For `n = 4`, there are two valid boards. The key observation is that once a queen is placed, the next row only needs to consider columns not already claimed by a column, main diagonal, or anti-diagonal.

<Callout title="Pruning rule" tone="tip">
  Put the state in sets or bitmasks. Recomputing whether a whole board is valid on every recursive call is the avoidable brute-force version.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function solveNQueens(n) {
  const boards = [];
  const columns = new Set();
  const diagonals = new Set();
  const antiDiagonals = new Set();
  const placement = new Array(n).fill("");

  function backtrack(row) {
    if (row === n) {
      boards.push([...placement]);
      return;
    }

    for (let col = 0; col < n; col += 1) {
      const diagonal = row - col;
      const antiDiagonal = row + col;

      if (
        columns.has(col)
        || diagonals.has(diagonal)
        || antiDiagonals.has(antiDiagonal)
      ) {
        continue;
      }

      columns.add(col);
      diagonals.add(diagonal);
      antiDiagonals.add(antiDiagonal);
      placement[row] = ".".repeat(col) + "Q" + ".".repeat(n - col - 1);

      backtrack(row + 1);

      columns.delete(col);
      diagonals.delete(diagonal);
      antiDiagonals.delete(antiDiagonal);
    }
  }

  backtrack(0);
  return boards;
}
```

```ts
export function totalNQueens(n: number): number {
  const columns = new Set<number>();
  const diagonals = new Set<number>();
  const antiDiagonals = new Set<number>();

  function backtrack(row: number): number {
    if (row === n) {
      return 1;
    }

    let count = 0;

    for (let col = 0; col < n; col += 1) {
      const diagonal = row - col;
      const antiDiagonal = row + col;

      if (
        columns.has(col)
        || diagonals.has(diagonal)
        || antiDiagonals.has(antiDiagonal)
      ) {
        continue;
      }

      columns.add(col);
      diagonals.add(diagonal);
      antiDiagonals.add(antiDiagonal);
      count += backtrack(row + 1);
      columns.delete(col);
      diagonals.delete(diagonal);
      antiDiagonals.delete(antiDiagonal);
    }

    return count;
  }

  return backtrack(0);
}
```

## Follow-up Questions

- Count the valid boards instead of generating them.
- Replace sets with bitmasks for tighter state tracking.
- Explain why row-by-row placement is enough.

## Common Pitfalls

- Checking the whole board every time instead of tracking attacks incrementally.
- Forgetting that diagonals can be identified by `row - col` and `row + col`.
- Mutating shared board state without undoing it on backtrack.
