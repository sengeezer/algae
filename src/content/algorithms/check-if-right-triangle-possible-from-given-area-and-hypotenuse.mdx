## Why It Surfaces

This problem looks geometric, but the clean solution is algebraic. If the unknown legs are `a` and `b`, then `ab = 2A` from the area and `a^2 + b^2 = H^2` from the Pythagorean theorem. Eliminating one variable gives a quadratic in the squared leg length.

## Worked Example

With area `6` and hypotenuse `5`, the only right triangle is `[3, 4, 5]`, so the function should return those sides in ascending order.

```js
function checkIfRightTrianglePossibleFromGivenAreaAndHypotenuse(area, hypotenuse) {
  if (area <= 0 || hypotenuse <= 0) {
    return null;
  }

  const hypotenuseSquared = hypotenuse * hypotenuse;
  const discriminant = hypotenuseSquared * hypotenuseSquared - 16 * area * area;

  if (discriminant < 0) {
    return null;
  }

  const sqrtDiscriminant = Math.sqrt(discriminant);
  const legSquareA = (hypotenuseSquared + sqrtDiscriminant) / 2;
  const legSquareB = (hypotenuseSquared - sqrtDiscriminant) / 2;

  if (legSquareA < 0 || legSquareB < 0) {
    return null;
  }

  const legA = Math.sqrt(legSquareA);
  const legB = Math.sqrt(legSquareB);

  if (
    !Number.isFinite(legA)
    || !Number.isFinite(legB)
    || Math.abs(0.5 * legA * legB - area) > 1e-9
    || Math.abs(Math.hypot(legA, legB) - hypotenuse) > 1e-9
  ) {
    return null;
  }

  return [legA, legB, hypotenuse]
    .sort((left, right) => left - right)
    .map((value) => (Math.abs(value - Math.round(value)) < 1e-9 ? Math.round(value) : value));
}
```

## Follow-up Questions

## Common Pitfalls

- Forgetting that both area and hypotenuse constraints must be satisfied at once.
- Accepting negative squared-leg roots.
- Assuming every non-negative discriminant automatically yields a valid triangle without checking the reconstructed legs.
