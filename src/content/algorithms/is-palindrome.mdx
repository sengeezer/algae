## Why It Surfaces

This is the cleanest palindrome interview prompt because it separates exact symmetry from the normalized or rearranged variants. The important skill is recognizing that you only need to compare mirrored positions until the pointers cross.

## Worked Example

`12321` is a palindrome because the first and last digits match, then the next pair matches, and the middle digit does not need further comparison.

## Reference Implementations

<ExamplePair>

```js
function isPalindrome(value) {
  if (typeof value !== 'string' && typeof value !== 'number') {
    throw new TypeError('Value must be a string or number');
  }

  const text = value.toString();

  for (let left = 0, right = text.length - 1; left < right; left += 1, right -= 1) {
    if (text[left] !== text[right]) {
      return false;
    }
  }

  return true;
}
```

```ts
export function isPalindrome(value: string | number): boolean {
  const text = value.toString();

  for (let left = 0, right = text.length - 1; left < right; left += 1, right -= 1) {
    if (text[left] !== text[right]) {
      return false;
    }
  }

  return true;
}
```

## Follow-up Questions

- Solve the numeric-only version without converting the value into a string.
- Ignore punctuation and case, turning the prompt into the normalized palindrome variant.
- Check whether a linked list is a palindrome while minimizing extra memory.

## Common Pitfalls

- Solving the normalized version when the prompt wants exact character equality.
- Reversing the whole string without mentioning the avoidable extra memory.
- Forgetting that empty strings and one-character inputs are still palindromes.
