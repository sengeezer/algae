## Why It Surfaces

The direct `n choose 3` scan works, but the real idea is to stop treating every line as unique. What matters first is whether two lines are parallel. Once lines are grouped by direction, every valid triangle comes from choosing one line from each of three different slope classes.

## Worked Example

For lines `x + 2y = 5`, `2x + 4y = 7`, `3x + 5y = 8`, and `4x + 5y = 6`, the first two are parallel and the other two have distinct slopes, so there are exactly `2` triangles.

function trianglesFormedFromASetOfLinesInEuclideanPlane(lines) {
  function gcd(left, right) {
    let a = Math.abs(left);
    let b = Math.abs(right);

    while (b !== 0) {
      [a, b] = [b, a % b];
    }

    return a;
  }

  function directionKey(a, b) {
    if (a === 0 && b === 0) {
      throw new RangeError("Each line needs a non-zero normal vector");
    }

    const divisor = gcd(a, b) || 1;
    let normalizedA = a / divisor;
    let normalizedB = b / divisor;

    if (normalizedA < 0 || (normalizedA === 0 && normalizedB < 0)) {
      normalizedA = -normalizedA;
      normalizedB = -normalizedB;
    }

    return `${normalizedA}/${normalizedB}`;
  }

  const groupCounts = new Map();

  for (const [a, b] of lines) {
    const key = directionKey(a, b);
    groupCounts.set(key, (groupCounts.get(key) ?? 0) + 1);
  }

  let sum1 = 0;
  let sum2 = 0;
  let sum3 = 0;

  for (const count of groupCounts.values()) {
    sum3 += sum2 * count;
    sum2 += sum1 * count;
    sum1 += count;
  }

  return sum3;
}

## Follow-up Questions

- Why do three lines from three distinct slope classes form a triangle only under the no-concurrency assumption?
- How would you adapt the count if multiple non-parallel line pairs could intersect at the same point?
- Why is a normalized `(a, b)` key safer than floating-point slope comparison?

## Common Pitfalls

- Comparing slopes with raw floating-point division instead of normalized coefficients.
- Forgetting that the constant term `c` does not affect whether two lines are parallel.
- Counting triplets of lines directly even after the problem collapses into grouped combinatorics.
