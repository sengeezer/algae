## Why It Surfaces

This question is really a power-of-two test with one extra step. Once you know the number is one-hot, the remaining work is just counting how many shifts it takes to move that bit to position zero.

## Worked Example

`2` is `10` in binary, so it has exactly one set bit and that bit is at position `2` when counting from the least-significant side starting at `1`. For `5`, the binary form is `101`, which has two set bits, so the correct answer is `-1`.

## Reference Implementations

<ExamplePair>

```js
function positionOfTheOnlySetBit(value) {
  if (!Number.isInteger(value) || value < 0) {
    throw new TypeError('Value must be a non-negative integer');
  }

  if (value === 0 || (value & (value - 1)) !== 0) {
    return -1;
  }

  let position = 1;
  let remaining = value;

  while ((remaining & 1) === 0) {
    remaining >>= 1;
    position += 1;
  }

  return position;
}
```

## Follow-up Questions

- Replace the shift loop with `Math.log2` and explain the tradeoff.
- Connect this result to the broader power-of-two identity.
- Adjust the function if the interviewer wants zero-based indexing instead.

## Common Pitfalls

- Forgetting to reject `0` and numbers with more than one set bit.
- Returning a zero-based position when the prompt wants one-based counting.
- Using floating-point logs without first enforcing the one-set-bit condition.
