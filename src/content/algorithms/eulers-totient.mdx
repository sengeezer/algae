## Why It Surfaces

Euler's Totient appears when brute-force gcd checks are too expensive and the interviewer wants you to exploit prime structure. The main insight is that each distinct prime factor removes a fixed fraction of the remaining candidates.

## Worked Example

For `n = 12`, the distinct prime factors are `2` and `3`. Starting from `12`, apply the totient formula: `12 * (1 - 1/2) * (1 - 1/3) = 4`, so the numbers coprime to `12` in `[1, 12]` are `1, 5, 7, 11`.

## Reference Implementations

<ExamplePair>

```js
function eulersTotient(value) {
  let remaining = value;
  let result = value;

  for (let factor = 2; factor * factor <= remaining; factor += 1) {
    if (remaining % factor !== 0) {
      continue;
    }

    while (remaining % factor === 0) {
      remaining = Math.floor(remaining / factor);
    }

    result -= Math.floor(result / factor);
  }

  if (remaining > 1) {
    result -= Math.floor(result / remaining);
  }

  return result;
}
```

## Follow-up Questions

- Relate the function to Euler's theorem in modular arithmetic.
- Explain why repeated powers of the same prime do not create new formula terms.

## Common Pitfalls

- Applying the formula for every occurrence of a prime factor instead of once per distinct prime.
- Forgetting the final leftover prime factor after the loop.
- Using gcd checks across the whole range and missing the arithmetic shortcut.
