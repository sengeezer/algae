## Why It Surfaces

LIS is a standard interview checkpoint because it rewards both dynamic-programming intuition and the ability to replace a slower state transition with binary search. The important distinction is subsequence versus subarray: the numbers do not need to stay contiguous.

## JavaScript

```js
function lengthOfLIS(values) {
  const tails = [];

  for (const value of values) {
    let left = 0;
    let right = tails.length;

    while (left < right) {
      const middle = Math.floor((left + right) / 2);

      if (tails[middle] < value) {
        left = middle + 1;
      } else {
        right = middle;
      }
    }

    tails[left] = value;
  }

  return tails.length;
}
```

## Interview Notes

- Lead with the O(n^2) DP if needed, then upgrade to the tails array when the interviewer asks for better performance.
- Emphasize that `tails[length]` stores the smallest possible tail for subsequences of that length.
- If they want the actual subsequence, mention predecessor indices and reconstruction arrays.
