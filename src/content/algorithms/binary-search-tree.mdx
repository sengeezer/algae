## Why It Surfaces

BST questions test whether you can use local ordering invariants to avoid scanning an entire structure. Once you trust the left subtree to contain only smaller values and the right subtree to contain only larger ones, search, insert, and delete all become path problems instead of full traversals.

## Worked Example

Inserting `6` into a BST rooted at `8` with left child `3` compares `6 < 8`, then `6 > 3`, so the insertion continues only into the right child of `3`.

<Callout title="Shape matters" tone="note">
  A BST is only as fast as its height. Interviewers often follow up by asking when balancing is needed or how a skewed tree changes the runtime.
</Callout>

## Reference Implementations

<ExamplePair>

```js
class TreeNode {
  constructor(value) {
    this.value = value;
    this.left = null;
    this.right = null;
  }
}

class BinarySearchTree {
  constructor() {
    this.root = null;
  }

  insert(value) {
    const node = new TreeNode(value);

    if (this.root === null) {
      this.root = node;
      return;
    }

    let current = this.root;

    while (true) {
      if (value < current.value) {
        if (current.left === null) {
          current.left = node;
          return;
        }

        current = current.left;
      } else {
        if (current.right === null) {
          current.right = node;
          return;
        }

        current = current.right;
      }
    }
  }

  contains(value) {
    let current = this.root;

    while (current !== null) {
      if (current.value === value) {
        return true;
      }

      current = value < current.value ? current.left : current.right;
    }

    return false;
  }
}
```

```ts
type TreeNode = {
  value: number;
  left: TreeNode | null;
  right: TreeNode | null;
};

export function inorder(root: TreeNode | null, output: number[] = []): number[] {
  if (root === null) {
    return output;
  }

  inorder(root.left, output);
  output.push(root.value);
  inorder(root.right, output);
  return output;
}
```

## Follow-up Questions

- Delete a node with zero, one, or two children.
- Return the kth smallest value with an inorder traversal.
- Explain when you would upgrade to a self-balancing tree.

## Common Pitfalls

- Forgetting that duplicate-value policy must be explicit.
- Assuming BST operations are always `O(log n)` even on skewed trees.
- Mixing ordinary binary trees with BST-specific ordering logic.
