## Why It Surfaces

This is a useful variation on subsequence DP because the state is keyed by value instead of just position. When the next number arrives, only subsequences ending at its immediate numeric neighbors matter, which makes a hash map more natural than a quadratic index-to-index scan.

## Worked Example

For `[10, 9, 4, 5, 4, 8, 6]`, the answer is `3`. Valid subsequences of length `3` include `[10, 9, 8]`, `[4, 5, 4]`, and `[4, 5, 6]`.

## Reference Implementations

<ExamplePair>

```js
function longestSubsequenceSuchThatDifferenceBetweenAdjacentsIsOne(values) {
  const bestEndingAt = new Map();
  let best = 0;

  for (const value of values) {
    const candidate = 1 + Math.max(
      bestEndingAt.get(value - 1) ?? 0,
      bestEndingAt.get(value + 1) ?? 0,
    );
    const updated = Math.max(bestEndingAt.get(value) ?? 0, candidate);
    bestEndingAt.set(value, updated);
    best = Math.max(best, updated);
  }

  return best;
}
```

## Follow-up Questions

- Reconstruct one longest valid subsequence instead of only its length.
- Change the rule to allow adjacent differences up to `k`.
- Compare the hash-map solution with the slower index-based O(n^2) DP.

## Common Pitfalls

- Sorting the array and accidentally destroying the subsequence constraint.
- Checking only `value - 1` or only `value + 1` instead of both neighbors.
- Overwriting a better chain already stored for the same value with a shorter one.
