## Why It Surfaces

This is one of the most common introductory string questions because it makes the costs of immutable strings visible. It also leads naturally into in-place array reversal and palindrome checks.

## Worked Example

`"algae"` reversed character by character from the end becomes `"eagla"`.

## Reference Implementations

<ExamplePair>

```js
function reverseString(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  let reversed = '';

  for (let index = value.length - 1; index >= 0; index -= 1) {
    reversed += value[index];
  }

  return reversed;
}
```

```ts
export function reverseString(value: string): string {
  let reversed = '';

  for (let index = value.length - 1; index >= 0; index -= 1) {
    reversed += value[index];
  }

  return reversed;
}
```

</ExamplePair>

## Follow-up Questions

- Reverse the string in place when the input is a mutable character array.
- Reverse only letters while leaving punctuation in place.
- Compare the iterative build with `split('').reverse().join('')` for readability and cost.

## Common Pitfalls

- Reversing words instead of characters.
- Assuming strings are mutable in JavaScript or TypeScript.
- Ignoring Unicode grapheme-cluster issues when emojis or combining marks are involved.
