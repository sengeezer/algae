## Why It Surfaces

This problem shows whether you can read multiplication through the binary expansion of one operand. Instead of forming every partial product directly, you double one side, halve the other, and add only when the current low bit says that partial product is needed.

## Worked Example

For `2` and `5`, the multiplier `5` is binary `101`, so the algorithm adds `2` on the first step, skips the middle doubled value, then adds `8` on the last step. The final result is `10`.

## Reference Implementations

<ExamplePair>

```js
function russianPeasantMultiplyTwoNumbersUsingBitwiseOperators(left, right) {
  if (!Number.isInteger(left) || !Number.isInteger(right) || left < 0 || right < 0) {
    throw new TypeError('Inputs must be non-negative integers');
  }

  let multiplicand = left;
  let multiplier = right;
  let result = 0;

  while (multiplier > 0) {
    if ((multiplier & 1) === 1) {
      result += multiplicand;
    }

    multiplicand <<= 1;
    multiplier >>= 1;
  }

  return result;
}
```

## Common Pitfalls

- Forgetting to add the current multiplicand when the multiplier is odd.
- Shifting the wrong operand and breaking the halving-doubling invariant.
- Ignoring that JavaScript bitwise shifts operate on signed 32-bit integers.
