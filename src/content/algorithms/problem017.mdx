## Why It Surfaces

This is a rules engine disguised as arithmetic. The point is not to generate pretty strings, but to decompose each value into known word-length fragments and aggregate the result across a range.

## Worked Example

For `1` through `5`, the words are `one`, `two`, `three`, `four`, and `five`, which use `3 + 3 + 5 + 4 + 4 = 19` letters. The range answer is just the sum of each individual word-length count.

```js
const ones = [0, 3, 3, 5, 4, 4, 3, 5, 5, 4, 3, 6, 6, 8, 8, 7, 7, 9, 8, 8];
const tens = [0, 0, 6, 6, 5, 5, 5, 7, 6, 6];
const hundred = 7;
const thousand = 8;
const andLength = 3;

function wordLength(value) {
  if (!Number.isInteger(value) || value < 0 || value > 1000) {
    throw new RangeError('Value must be an integer from 0 to 1000');
  }

  if (value < 20) {
    return ones[value];
  }

  if (value < 100) {
    return tens[Math.floor(value / 10)] + ones[value % 10];
  }

  if (value < 1000) {
    const remainder = value % 100;
    return ones[Math.floor(value / 100)] + hundred + (remainder === 0 ? 0 : andLength + wordLength(remainder));
  }

  return ones[1] + thousand;
}

function problem017(limit = 1000) {
  if (!Number.isInteger(limit) || limit < 1 || limit > 1000) {
    throw new RangeError('Limit must be an integer from 1 to 1000');
  }

  let total = 0;
  for (let value = 1; value <= limit; value += 1) {
    total += wordLength(value);
  }

  return total;
}
```

## Follow-up Questions

- Generate the actual British-English words instead of only their lengths.
- Extend the helper beyond `1000` while preserving the same decomposition strategy.
- Explain why British usage inserts `and` in values like `342`.

## Common Pitfalls

- Forgetting that the prompt ignores spaces and hyphens.
- Omitting the British `and` in numbers like `one hundred and one`.
- Building full strings when the problem only asks for letter counts.
