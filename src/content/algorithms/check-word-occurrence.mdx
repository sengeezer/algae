## Why It Surfaces

Word-occurrence problems are the token-level counterpart to character frequency questions. The interview signal is usually whether you separate tokenization from counting cleanly.

## Worked Example

For `"to be or not to be"`, tokenizing on spaces yields `['to', 'be', 'or', 'not', 'to', 'be']`, so the counts are `{ to: 2, be: 2, or: 1, not: 1 }`.

## Reference Implementations

<ExamplePair>

```js
function checkWordOccurrence(value, caseSensitive = false) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  if (typeof caseSensitive !== 'boolean') {
    throw new TypeError('caseSensitive must be a boolean');
  }

  const normalized = caseSensitive ? value : value.toLowerCase();
  const counts = {};

  for (const word of normalized.trim().split(/\s+/).filter(Boolean)) {
    counts[word] = (counts[word] ?? 0) + 1;
  }

  return counts;
}
```

```ts
export function checkWordOccurrence(
  value: string,
  caseSensitive = false,
): Record<string, number> {
  const normalized = caseSensitive ? value : value.toLowerCase();
  const counts: Record<string, number> = {};

  for (const word of normalized.trim().split(/\s+/).filter(Boolean)) {
    counts[word] = (counts[word] ?? 0) + 1;
  }

  return counts;
}
```

## Follow-up Questions

- Strip punctuation before counting words.
- Return the most frequent word instead of the whole map.
- Stream the input token by token instead of splitting the full string up front.

## Common Pitfalls

- Forgetting to define case sensitivity clearly.
- Counting empty tokens when multiple spaces appear.
- Mixing word frequency with substring frequency.
