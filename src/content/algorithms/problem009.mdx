## Why It Surfaces

This is a constraint-combination search problem. The important step is noticing that once the total sum is fixed, the third value is determined by the first two, which removes one whole loop.

## Worked Example

For target sum `1000`, search values `a < b < c` with `c = 1000 - a - b`. The one triple that satisfies `a^2 + b^2 = c^2` gives the desired product `abc`.

## Reference Implementations

<ExamplePair>

```js
function problem009(target = 1000) {
  if (!Number.isInteger(target) || target < 3) {
    throw new RangeError('Target sum must be an integer at least 3');
  }

  for (let a = 1; a < target / 3; a += 1) {
    for (let b = a + 1; b < target / 2; b += 1) {
      const c = target - a - b;
      if (a * a + b * b === c * c) {
        return a * b * c;
      }
    }
  }

  return null;
}
```

## Follow-up Questions

- Return the triplet itself instead of only the product.
- Compare the constrained search with Euclid's formula for generating triplets.
- Explain why `c` can be derived directly once `a` and `b` are known.

## Common Pitfalls

- Using three full loops even though the target sum already determines `c`.
- Forgetting the ordering constraint `a < b < c`.
- Returning `0` or `NaN` when no triplet exists instead of a clear sentinel value.
