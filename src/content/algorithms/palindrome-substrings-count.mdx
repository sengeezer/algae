## Why It Surfaces

This prompt usually appears right after a palindrome-recognition question stops being enough. The important distinction is that the count is over contiguous ranges, so once a mismatch appears the current interval cannot be rescued by skipping characters.

## Worked Example

For `text = "abbaeae"`, the answer is `4`. The palindromic substrings of length greater than `1` are `"bb"`, `"abba"`, `"aea"`, and `"eae"`.

## Reference Implementations

<ExamplePair>

```js
function palindromeSubstringsCount(text) {
  const size = text.length;
  const dp = Array.from({ length: size }, () => new Array(size).fill(false));
  let count = 0;

  for (let left = size - 1; left >= 0; left -= 1) {
    dp[left][left] = true;

    for (let right = left + 1; right < size; right += 1) {
      if (text[left] === text[right] && (right === left + 1 || dp[left + 1][right - 1])) {
        dp[left][right] = true;
        count += 1;
      }
    }
  }

  return count;
}
```

export function palindromeSubstringsCount(text: string): number {
  const size = text.length;
  const dp = Array.from({ length: size }, () => new Array<boolean>(size).fill(false));
  let count = 0;

        dp[left][right] = true;
        count += 1;
      }
    }
  }

  return count;
}

## Follow-up Questions

- Return the distinct palindromic substrings instead of only the count.
- Reduce the memory footprint with center expansion and compare tradeoffs.
- Explain why the interval iteration order must move from shorter inner substrings to longer outer ones.

## Common Pitfalls

- Counting single characters when the prompt starts at length `2`.
- Confusing contiguous substrings with palindromic subsequences.
- Filling the DP table in an order that reads `dp[left + 1][right - 1]` before it exists.
