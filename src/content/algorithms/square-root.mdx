## Why It Surfaces

Square Root is one of the most common introductions to numerical methods in interviews. The main test is whether you can translate an update formula into a stable iterative routine.

## Worked Example

For `25`, start with estimate `1`. Newton's update moves the estimate through values like `13`, `7.46`, `5.41`, and then near `5`. Each step corrects the previous guess using both the current estimate and the target number.

## Reference Implementations

<ExamplePair>

```js
function squareRoot(value, iterations = 6) {
  if (!Number.isFinite(value)) {
    throw new TypeError('Value must be finite');
  }

  if (!Number.isInteger(iterations) || iterations < 1) {
    throw new RangeError('Iterations must be a positive integer');
  }

  if (value < 0) {
    throw new RangeError('Value must be non-negative');
  }

  if (value === 0) {
    return 0;
  }

  let estimate = 1;

  for (let iteration = 0; iteration < iterations; iteration += 1) {
    estimate -= (estimate * estimate - value) / (2 * estimate);
  }

  return estimate;
}
```

## Follow-up Questions

- Stop when the estimate changes by less than an epsilon instead of after a fixed iteration count.
- Return the integer floor square root rather than a floating-point approximation.
- Compare Newton's method with a binary-search-based root finder.

## Common Pitfalls

- Forgetting to reject negative inputs when working over real numbers.
- Starting from zero and dividing by zero in the Newton update.
- Expecting a finite number of iterations to produce exact answers for every non-square.
