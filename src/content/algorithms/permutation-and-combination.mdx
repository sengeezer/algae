## Why It Surfaces

Permutation and Combination is the fast way to test whether someone understands the effect of order. Many interview counting questions are really asking you to choose between `nPr` and `nCr` before any calculation starts.

## Worked Example

Choosing 2 officers from 5 candidates is a combination problem if the roles are identical, so the answer is `C(5, 2) = 10`. If the roles are president and vice president, order matters, so the answer becomes `P(5, 2) = 20`.

## Reference Implementations

<ExamplePair>

```js
function factorial(value) {
  if (value <= 1) {
    return 1;
  }

  return value * factorial(value - 1);
}

function permutationCount(n, r) {
  return factorial(n) / factorial(n - r);
}

function combinationCount(n, r) {
  return factorial(n) / (factorial(r) * factorial(n - r));
}
```

```ts
function factorial(value: number): number {
  if (value <= 1) {
    return 1;
  }

  return value * factorial(value - 1);
}

export function permutationCount(n: number, r: number): number {
  return factorial(n) / factorial(n - r);
}

export function combinationCount(n: number, r: number): number {
  return factorial(n) / (factorial(r) * factorial(n - r));
}
```

## Follow-up Questions

- Compare the formula approach with generating the actual arrangements.
- Explain why combinations divide permutations by `r!`.
- Replace factorial recursion with a multiplicative formula to reduce overflow risk.

## Common Pitfalls

- Using combinations when order matters, or permutations when it does not.
- Ignoring invalid inputs such as `r > n`.
- Treating this formula sheet as a substitute for understanding the underlying counting model.
