## Why It Surfaces

Binary Equivalent is a lightweight recursion exercise that makes the call stack visible. Each call shrinks the input with integer division, and the actual digit order emerges during the unwind when remainders are appended.

## Worked Example

For `13`, recurse on `6`, then `3`, then `1`. On the way back up, append the remainders `1`, `0`, and `1` to produce `1101`.

## Reference Implementations

<ExamplePair>

```js
function binaryEquivalent(value) {
  if (value === 0 || value === 1) {
    return String(value);
  }

  return binaryEquivalent(Math.floor(value / 2)) + String(value % 2);
}
```

```ts
export function binaryEquivalent(value: number): string {
  if (value === 0 || value === 1) {
    return String(value);
  }

  return binaryEquivalent(Math.floor(value / 2)) + String(value % 2);
}
```

## Follow-up Questions

- Rewrite the same conversion iteratively.
- Extend the approach to arbitrary bases, not just binary.
- Explain why the digits appear in the correct order during unwinding.

## Common Pitfalls

- Returning digits in reverse order by appending before the recursive call.
- Forgetting the small base cases for `0` and `1`.
- Treating string assembly as numeric addition.
