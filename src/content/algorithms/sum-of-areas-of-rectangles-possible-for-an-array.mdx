## Why It Surfaces

The brute-force framing tempts you to try every group of four lengths, but the largest total area always comes from preserving the biggest usable sides first. Once the lengths are sorted descending, every valid adjacent pair can be harvested greedily, and every two harvested sides can be combined immediately into the next best rectangle.

## Worked Example

For `a = [10, 10, 10, 10, 11, 10, 11, 10]`, sorting descending gives `[11, 11, 10, 10, 10, 10, 10, 10]`. Harvest side pairs `11`, `10`, `10`, and `10`, then combine them into rectangles `11 * 10 = 110` and `10 * 10 = 100`, for a total area of `210`.

## Reference Implementations

<ExamplePair>

```js
function sumOfAreasOfRectanglesPossibleForAnArray(values) {
  const sortedValues = [...values].sort((left, right) => right - left);
  let pendingSide = null;
  let totalArea = 0;

  for (let index = 0; index + 1 < sortedValues.length; index += 1) {
    if (sortedValues[index] - sortedValues[index + 1] > 1) {
      continue;
    }

    const currentSide = sortedValues[index + 1];

    if (pendingSide === null) {
      pendingSide = currentSide;
    } else {
      totalArea += pendingSide * currentSide;
      pendingSide = null;
    }

    index += 1;
  }

  return totalArea;
}
```

## Follow-up Questions

- Return the individual rectangles, not only their total area.
- Explain why using the smaller value of a near-equal pair is optimal when the larger one is reduced by one.
- Adapt the routine if each length may be reduced by up to `d` instead of only one.

## Common Pitfalls

- Pairing small sides too early and wasting larger compatible lengths.
- Forgetting that a difference of exactly one is still usable after one reduction.
- Reusing a stick after it has already been consumed in a side pair.
