## Why It Surfaces

Once the array is sorted, any group you raise to a common value should target the window's largest element. The sliding-window invariant measures how many total increments are needed to raise the rest of the window up to that right boundary.

## Worked Example

For `arr = [1, 2, 3, 4]` and `k = 3`, sort the array and grow a window. Raising `1` by `2` and `2` by `1` makes `[3, 3, 3, 4]`, so the maximum equal count is `3`.

## Reference Implementations

<ExamplePair>

```js
function maximumElementsThatCanBeMadeEqualWithKUpdates(values, k) {
  const sorted = [...values].sort((left, right) => left - right);
  let left = 0;
  let windowSum = 0;
  let best = 0;

  for (let right = 0; right < sorted.length; right += 1) {
    windowSum += sorted[right];

    while ((right - left + 1) * sorted[right] > windowSum + k) {
      windowSum -= sorted[left];
      left += 1;
    }

    best = Math.max(best, right - left + 1);
  }

  return best;
}
```

## Follow-up Questions

- Return the target value as well as the maximum count.
- Explain why a window should always target its rightmost sorted value.
- Adapt the solution if decrements are also allowed.

## Common Pitfalls

- Trying to equalize a window to something other than its current maximum.
- Forgetting to maintain the running sum when shrinking the window.
- Treating the problem as unrestricted replacement instead of increment-only updates.
