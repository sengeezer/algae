## Why It Surfaces

This is a formula recall problem disguised as arithmetic work. The real win is recognizing that both quantities have closed forms and can be computed in constant time.

## Worked Example

For `10`, the square of the sum is `(1 + ... + 10)^2 = 55^2 = 3025`, and the sum of the squares is `1^2 + ... + 10^2 = 385`, so the difference is `2640`.

## Reference Implementations

<ExamplePair>

```js
function problem006(limit = 100) {
  if (!Number.isInteger(limit) || limit < 1) {
    throw new RangeError('Limit must be a positive integer');
  }

  const sum = (limit * (limit + 1)) / 2;
  const sumOfSquares = (limit * (limit + 1) * (2 * limit + 1)) / 6;
  return sum ** 2 - sumOfSquares;
}
```

```ts
export function problem006(limit = 100): number {
  if (!Number.isInteger(limit) || limit < 1) {
    throw new RangeError('Limit must be a positive integer');
  }

  const sum = (limit * (limit + 1)) / 2;
  const sumOfSquares = (limit * (limit + 1) * (2 * limit + 1)) / 6;
  return sum ** 2 - sumOfSquares;
}
```

## Follow-up Questions

## Common Pitfalls

- Mixing up the square of the sum with the sum of the squares.
- Using loops when the prompt is meant to test formula recall.
- Forgetting to validate that the input is a positive integer.
