## Why It Surfaces

Derangements show up when a counting problem forbids fixed points instead of just asking for unrestricted permutations. The useful reduction is that once the first element moves away from its original slot, the rest of the problem falls into two smaller cases that depend only on the previous two answers.

## Worked Example

For `n = 4`, the answer is `9`. The recurrence gives `D(4) = 3 * (D(3) + D(2)) = 3 * (2 + 1) = 9`.

## Reference Implementations

<ExamplePair>

```js
function countDerangements(size) {
  if (size < 0) {
    throw new RangeError('size must be non-negative');
  }

  if (size === 0) {
    return 1;
  }

  if (size === 1) {
    return 0;
  }

  if (size === 2) {
    return 1;
  }

  let previousTwo = 0;
  let previousOne = 1;

  for (let current = 3; current <= size; current += 1) {
    const next = (current - 1) * (previousOne + previousTwo);
    previousTwo = previousOne;
    previousOne = next;
  }

  return previousOne;
}
```

## Follow-up Questions

- Return the answer modulo a large prime for bigger `n`.
- Derive the same count from inclusion-exclusion and compare it with the DP.
- Construct one derangement instead of only counting them.

## Common Pitfalls

- Forgetting the base cases `D(1) = 0` and `D(2) = 1`.
- Confusing derangements with permutations that only avoid adjacent equality.
- Recomputing the full DP table when only two prior values are needed.
