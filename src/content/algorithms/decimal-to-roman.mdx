## Why It Surfaces

Decimal-to-Roman conversion is a good counterexample to place-value systems. Instead of repeated division by a fixed base, the algorithm greedily subtracts from a descending table of symbol values.

## Worked Example

To encode `944`, take `900` as `CM`, then `40` as `XL`, then `4` as `IV`. The final numeral is `CMXLIV`.

## Reference Implementations

<ExamplePair>

```js
const romanSymbols = [
  ['M', 1000],
  ['CM', 900],
  ['D', 500],
  ['CD', 400],
  ['C', 100],
  ['XC', 90],
  ['L', 50],
  ['XL', 40],
  ['X', 10],
  ['IX', 9],
  ['V', 5],
  ['IV', 4],
  ['I', 1],
];

function decimalToRoman(value) {
  if (!Number.isInteger(value) || value <= 0 || value >= 4000) {
    throw new RangeError('Roman numeral conversion expects an integer from 1 to 3999');
  }

  let current = value;
  let result = '';

  for (const [symbol, amount] of romanSymbols) {
    while (current >= amount) {
      result += symbol;
      current -= amount;
    }
  }

  return result;
}
```

## Follow-up Questions

- Explain why the greedy choice is always safe for the standard Roman table.
- Support larger values by defining an extension to the symbol set.
- Compare Roman numerals with positional systems like decimal or hexadecimal.

## Common Pitfalls

- Forgetting subtractive pairs like `IV`, `IX`, `XL`, and `CM`.
- Accepting zero or negative values even though standard Roman numerals do not represent them.
- Treating the problem like a fixed-base conversion when the numeral system is not positional.
