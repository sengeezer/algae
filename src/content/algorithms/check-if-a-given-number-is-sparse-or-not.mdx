## Why It Surfaces

Sparse-number questions reward knowing a compact bit identity rather than reaching for string conversion or a manual scan. The key observation is that any adjacent `1` bits survive one right shift and then overlap in an AND.

## Worked Example

`72` is `1001000` in binary, so no adjacent `1` bits overlap and the function returns `true`. `12` is `1100`, so the leading `11` creates an overlap and the function returns `false`.

## Reference Implementations

<ExamplePair>

```js
function checkIfAGivenNumberIsSparseOrNot(value) {
  if (!Number.isInteger(value) || value < 0 || value > 0xffffffff) {
    throw new RangeError('Value must be an unsigned 32-bit integer');
  }

  const number = value >>> 0;
  return (number & (number >>> 1)) === 0;
}
```

```ts
export function checkIfAGivenNumberIsSparseOrNot(value: number): boolean {
  if (!Number.isInteger(value) || value < 0 || value > 0xffffffff) {
    throw new RangeError('Value must be an unsigned 32-bit integer');
  }

  const number = value >>> 0;
  return (number & (number >>> 1)) === 0;
}
```

## Follow-up Questions

- Return the positions of the first adjacent set-bit pair instead of only a boolean.
- Compare this bitwise identity with a loop that inspects bits one by one.
- Count how many sparse numbers exist in a bounded range.

## Common Pitfalls

- Forgetting that `0` is sparse because it contains no adjacent set bits.
- Converting to a string when a constant-time bit test is available.
- Letting JavaScript bitwise coercion silently truncate values outside the unsigned 32-bit range.
