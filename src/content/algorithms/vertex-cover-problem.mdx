## Why It Surfaces

Vertex cover is one of the cleanest ways to show that interview graph problems do not always want exact optimization. The usual teaching version is not "solve the NP-hard problem perfectly," but "state a fast approximation with a defensible guarantee and explain why it works."

## Worked Example

If edge `(u, v)` is still uncovered, any valid cover must contain `u` or `v`. The greedy approximation takes both endpoints, deletes every incident edge from consideration, and repeats; that may overshoot the optimum, but it never overshoots by more than a factor of two.

## Reference Implementations

<ExamplePair>

```js
function approximateVertexCover(nodeCount, edges) {
  const incidentEdges = Array.from({ length: nodeCount }, () => []);

  edges.forEach(([from, to], index) => {
    incidentEdges[from].push(index);
    incidentEdges[to].push(index);
  });

  const active = new Array(edges.length).fill(true);
  const chosen = new Array(nodeCount).fill(false);

  for (let index = 0; index < edges.length; index += 1) {
    if (!active[index]) {
      continue;
    }

    const [from, to] = edges[index];
    chosen[from] = true;
    chosen[to] = true;

    for (const edgeIndex of incidentEdges[from]) {
      active[edgeIndex] = false;
    }

    for (const edgeIndex of incidentEdges[to]) {
      active[edgeIndex] = false;
    }
  }

  return chosen.flatMap((take, node) => (take ? [node] : []));
}
```

## Follow-up Questions

- Explain why this greedy routine is a 2-approximation.
- Contrast general vertex cover with the bipartite case, where matching gives stronger structure.
- Discuss what changes if vertices have weights instead of uniform cost.

## Common Pitfalls

- Confusing vertex cover with selecting vertices that reach every other vertex.
- Claiming the greedy 2-approximation is always optimal.
- Forgetting that covering edges, not vertices, is the actual requirement.
