## Why It Surfaces

LCIS is the point where knowing LCS and LIS separately is no longer enough. A common value can extend the sequence only if it keeps the result strictly increasing, so the DP has to remember the best compatible smaller match seen so far.

## Worked Example

For `left = [3, 4, 9, 1]` and `right = [5, 3, 8, 9, 10, 2, 1]`, the answer length is `2` because `[3, 9]` is common and increasing.

## Reference Implementations

<ExamplePair>

```js
function longestCommonIncreasingSubsequenceLcsLis(left, right) {
  const dp = new Array(right.length).fill(0);

  for (let leftIndex = 0; leftIndex < left.length; leftIndex += 1) {
    let bestSoFar = 0;

    for (let rightIndex = 0; rightIndex < right.length; rightIndex += 1) {
      if (left[leftIndex] === right[rightIndex]) {
        dp[rightIndex] = Math.max(dp[rightIndex], bestSoFar + 1);
      } else if (left[leftIndex] > right[rightIndex]) {
        bestSoFar = Math.max(bestSoFar, dp[rightIndex]);
      }
    }
  }

  return dp.length === 0 ? 0 : Math.max(...dp);
}
```

## Common Pitfalls

- Treating LCIS as ordinary LCS and allowing non-increasing matches.
- Updating `bestSoFar` with values that are not strictly smaller than the current element.
- Forgetting that equal values in the wrong order still cannot form an increasing subsequence.
