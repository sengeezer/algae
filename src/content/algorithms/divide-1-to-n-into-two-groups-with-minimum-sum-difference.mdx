## Why It Surfaces

At first glance this resembles subset sum, but the input structure matters. Because the numbers are exactly `1` through `n`, you can aim directly at half of the total triangular sum and fill that target greedily from the largest remaining numbers downward.

## Worked Example

For `n = 5`, the total sum is `15`, so the first group should get as close as possible to `7`. Starting from `5`, take it, then skip `4` because `5 + 4` would overshoot `7`, then take `2`. The result can be `[[5, 2], [4, 3, 1]]`, whose sums differ by only `1`.

## Reference Implementations

<ExamplePair>

```js
function divideOneToNIntoTwoGroupsWithMinimumSumDifference(n) {
  if (n <= 2) {
    throw new RangeError('n must be greater than 2');
  }

  const firstGroup = [];
  const secondGroup = [];
  const target = Math.floor((n * (n + 1)) / 4);
  let firstSum = 0;

  for (let value = n; value >= 1; value -= 1) {
    if (firstSum + value <= target) {
      firstGroup.push(value);
      firstSum += value;
      continue;
    }

    secondGroup.push(value);
  }

  return [firstGroup, secondGroup];
}
```

## Follow-up Questions

- Return the minimum difference value along with the groups.
- Explain why the difference is always `0` or `1` for consecutive integers.
- Contrast this structured case with arbitrary-set partition problems that need DP.

## Common Pitfalls

- Reaching for generic subset-sum DP even though the consecutive-range structure gives a direct greedy construction.
- Using floating-point half sums instead of the integer target derived from the triangular total.
- Forgetting that the problem asks for groups, not just the difference value.
