## Why It Surfaces

Totient is a classic number-theory helper. Even the direct implementation checks whether you can combine a clean Euclidean GCD routine with a counting loop over candidates.

## Worked Example

For `n = 9`, the coprime values in `1..9` are `1`, `2`, `4`, `5`, `7`, and `8`, so `phi(9) = 6`.

```js
function gcd(left, right) {
  let a = Math.abs(left);
  let b = Math.abs(right);

  while (b !== 0) {
    const remainder = a % b;
    a = b;
    b = remainder;
  }

  return a;
}

function eulersTotientFunction(value) {
  if (!Number.isInteger(value) || value <= 0) {
    throw new RangeError('Value must be a positive integer');
  }

  let count = 0;

  for (let candidate = 1; candidate <= value; candidate += 1) {
    if (gcd(candidate, value) === 1) {
      count += 1;
    }
  }

  return count;
}
```

## Follow-up Questions

- Derive the faster multiplicative formula from the prime factors of `n`.
- Explain why `phi(p) = p - 1` for prime `p`.
- Reuse the totient values for all numbers up to a limit with a sieve-like approach.

## Common Pitfalls

- Forgetting that `1` is coprime with every positive integer.
- Confusing divisibility with coprimality.
- Using a brute-force factor test when the Euclidean algorithm is simpler and safer.
