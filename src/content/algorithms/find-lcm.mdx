## Why It Surfaces

Find LCM is usually a gcd question in disguise. Once you know the product-over-gcd identity, the hard part is mostly input validation and avoiding unnecessary overflow or brute-force search.

## Worked Example

For `12` and `18`, the gcd is `6`, so the lcm is `(12 / 6) * 18 = 36`. That is the smallest positive number divisible by both inputs.

## Reference Implementations

<ExamplePair>

```js
function gcd(a, b) {
  let left = a;
  let right = b;

  while (right !== 0) {
    [left, right] = [right, left % right];
  }

  return left;
}

function findLcm(left, right) {
  if (!Number.isInteger(left) || !Number.isInteger(right)) {
    throw new TypeError('Inputs must be whole numbers');
  }

  if (left < 1 || right < 1) {
    throw new RangeError('Inputs must be positive');
  }

  return (left / gcd(left, right)) * right;
}
```

function gcd(a: number, b: number): number {
  let left = a;
  let right = b;

  while (right !== 0) {
    [left, right] = [right, left % right];
  }

  return left;
}

export function findLcm(left: number, right: number): number {
  if (!Number.isInteger(left) || !Number.isInteger(right)) {
    throw new TypeError('Inputs must be whole numbers');
  }

  if (left < 1 || right < 1) {
    throw new RangeError('Inputs must be positive');
  }

  return (left / gcd(left, right)) * right;
}

## Follow-up Questions

- Extend the lcm to an array of values.
- Explain why dividing before multiplying is safer than `left * right / gcd`.
- Use gcd and lcm identities together in scheduling or fraction problems.

## Common Pitfalls

- Searching upward through multiples instead of using gcd.
- Multiplying before dividing and risking larger intermediate values.
- Forgetting that lcm is defined over positive integers in the standard interview formulation.
