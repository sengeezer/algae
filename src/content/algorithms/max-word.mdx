## Why It Surfaces

This is the word-level version of the max-character problem. The interesting part is less about counting and more about being explicit about normalization and tie-breaking rules.

## Worked Example

For `"Lala lili lala"`, lowercasing and tokenizing produces `['lala', 'lili', 'lala']`, so `"lala"` is the max word.

## Reference Implementations

<ExamplePair>

```js
function maxWord(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  const words = value
    .trim()
    .toLowerCase()
    .split(/\s+/)
    .filter(Boolean);

  if (words.length === 0) {
    return null;
  }

  const counts = new Map();
  let bestWord = words[0];
  let bestCount = 0;

  for (const word of words) {
    const nextCount = (counts.get(word) ?? 0) + 1;
    counts.set(word, nextCount);

    if (nextCount > bestCount) {
      bestWord = word;
      bestCount = nextCount;
    }
  }

  return bestWord;
}
```

## Follow-up Questions

- Return all top-frequency words instead of just one.
- Strip punctuation before tokenizing.
- Stream tokens from a large file instead of splitting the whole sentence at once.

## Common Pitfalls

- Splitting only on single spaces and counting empty tokens.
- Ignoring tie-breaking rules.
- Forgetting to normalize case when the prompt treats words case-insensitively.
