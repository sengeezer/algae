## Why It Surfaces

The brute-force version asks you to test every permutation of one array against the other, but the greedy insight is that optimal pairings never cross once both sides are ordered. Sorting both arrays and pairing equal-ranked elements collapses the search space from factorial to a single deterministic scan.

## Worked Example

For `a = [4, 1, 8, 7]` and `b = [2, 3, 6, 5]`, sort both arrays to get `[1, 4, 7, 8]` and `[2, 3, 5, 6]`. The minimum sum is `|1 - 2| + |4 - 3| + |7 - 5| + |8 - 6| = 6`.

<ExamplePair>

```js
function minimumSumOfAbsoluteDifferenceOfPairsOfTwoArrays(firstArray, secondArray) {
  if (firstArray.length !== secondArray.length) {
    return 0;
  }

  const sortedFirst = [...firstArray].sort((left, right) => left - right);
  const sortedSecond = [...secondArray].sort((left, right) => left - right);

  let totalDifference = 0;

  for (let index = 0; index < sortedFirst.length; index += 1) {
    totalDifference += Math.abs(sortedFirst[index] - sortedSecond[index]);
  }

  return totalDifference;
}
```

## Follow-up Questions

- Prove with an exchange argument why crossing pairings can always be uncrossed without increasing the cost.
- Return one explicit optimal pairing, not only the minimum sum.
- Explain what changes if the arrays can have different lengths and some elements may stay unmatched.

## Common Pitfalls

- Trying all permutations even though sorted pairing is sufficient.
- Sorting only one array and leaving the other in its original order.
- Forgetting to define behavior when the input arrays have different lengths.
