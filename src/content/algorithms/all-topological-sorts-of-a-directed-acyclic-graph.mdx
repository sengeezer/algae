## Why It Surfaces

Ordinary topological sort gives one valid ordering, but some problems need every possible one. That happens when unrelated tasks can swap positions freely and a later objective depends on testing each feasible dependency-respecting arrangement.

## Worked Example

If both vertices `0` and `2` currently have indegree zero, then both choices are valid next steps. Pick `0`, recurse, backtrack, then pick `2` and recurse again. Repeating that branching process produces every valid ordering.

```js
function allTopologicalSortsOfADirectedAcyclicGraph(nodeCount, edges) {
  const graph = Array.from({ length: nodeCount }, () => []);
  const indegree = new Array(nodeCount).fill(0);

  for (const [from, to] of edges) {
    graph[from].push(to);
    indegree[to] += 1;
  }

  const used = new Array(nodeCount).fill(false);
  const order = [];
  const allOrders = [];

  function backtrack() {
    let progressed = false;

    for (let node = 0; node < nodeCount; node += 1) {
      if (used[node] || indegree[node] !== 0) {
        continue;
      }

      progressed = true;
      used[node] = true;
      order.push(node);

      for (const neighbor of graph[node]) {
        indegree[neighbor] -= 1;
      }

      backtrack();

      for (const neighbor of graph[node]) {
        indegree[neighbor] += 1;
      }

      order.pop();
      used[node] = false;
    }

    if (!progressed && order.length === nodeCount) {
      allOrders.push([...order]);
    }
  }

  backtrack();
  return allOrders;
}
```

## Follow-up Questions

- How would you stop after finding only the lexicographically smallest valid order?
- How do you detect that the graph is not a DAG?
- When does enumerating all orders become computationally unrealistic?

## Common Pitfalls

- Returning after the first zero-indegree choice instead of exploring all of them.
- Forgetting to restore indegrees during backtracking.
- Using the single-order Kahn algorithm when the prompt explicitly wants every valid ordering.
