## Why It Surfaces

Each allowed modification changes one `a[i]` by `2`, so spending every modification on the same best index compounds the benefit linearly. The greedy task is therefore not to distribute updates, but to find the one paired product term whose value can be driven downward the most.

## Worked Example

For `a = [1, 2, -3]`, `b = [-2, 3, -5]`, and `k = 5`, the original sum is `19`. Applying all five updates to `a[2]` changes it from `-3` to `7`, so the final sum becomes `(1 * -2) + (2 * 3) + (7 * -5) = -31`, which is minimal.

```js
function minimumSumOfProductOfTwoArrays(firstArray, secondArray, operations) {
  if (firstArray.length !== secondArray.length) {
    return 0;
  }

  let originalSum = 0;
  let bestReduction = 0;

  for (let index = 0; index < firstArray.length; index += 1) {
    const currentProduct = firstArray[index] * secondArray[index];
    originalSum += currentProduct;

    let adjustedProduct = currentProduct;

    if (secondArray[index] > 0) {
      adjustedProduct = (firstArray[index] - 2 * operations) * secondArray[index];
    } else if (secondArray[index] < 0) {
      adjustedProduct = (firstArray[index] + 2 * operations) * secondArray[index];
    }

    bestReduction = Math.max(bestReduction, currentProduct - adjustedProduct);
  }

  return originalSum - bestReduction;
}
```

## Follow-up Questions

- Return the index chosen for the updates as well as the minimized sum.
- Prove why distributing the modifications across multiple indices cannot beat the best single-index reduction.
- Adapt the routine if both arrays may be modified instead of only the first one.

## Common Pitfalls

- Trying to spread the `k` updates across many indices without first checking the best one-index improvement.
- Forgetting that the direction of change depends only on the sign of the paired `b[i]` value.
- Reordering the arrays even though the source problem keeps the element pairing fixed.
