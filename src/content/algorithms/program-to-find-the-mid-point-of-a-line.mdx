## Why It Surfaces

This is one of the simplest coordinate-geometry formulas, but it still shows up because it is a building block for larger geometric routines. The key move is to recognize that the midpoint is just the average of the two x-values and the average of the two y-values.

## Worked Example

For endpoints `[-1, 2]` and `[3, -6]`, the x-coordinate is `(-1 + 3) / 2 = 1` and the y-coordinate is `(2 + -6) / 2 = -2`, so the midpoint is `[1, -2]`.

## Reference Implementations

<ExamplePair>

```js
function programToFindTheMidPointOfALine(start, end) {
  return [
    (start[0] + end[0]) / 2,
    (start[1] + end[1]) / 2,
  ];
}
```

```ts
export function programToFindTheMidPointOfALine(
  start: [number, number],
  end: [number, number],
): [number, number] {
  return [
    (start[0] + end[0]) / 2,
    (start[1] + end[1]) / 2,
  ];
}
```

## Common Pitfalls

- Averaging the coordinates across axes instead of axis by axis.
- Forgetting that the result can be fractional even when the inputs are integers.
- Mixing endpoint order into the calculation even though the midpoint is symmetric.
