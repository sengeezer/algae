## Why It Surfaces

This is the cleanest demonstration of XOR cancellation in practice. The solution is short, but it only makes sense if you understand the algebraic properties behind it.

## Worked Example

For `[4, 1, 2, 1, 2]`, XOR from left to right: `4 ^ 1 ^ 2 ^ 1 ^ 2`. The two `1` values cancel to `0`, the two `2` values cancel to `0`, and the result left behind is `4`.

## Reference Implementations

<ExamplePair>

```js
function uniqueElementInAnArray(values) {
  if (!Array.isArray(values) || values.length === 0) {
    throw new RangeError('Values must be a non-empty array');
  }

  let unique = 0;

  for (const value of values) {
    unique ^= value;
  }

  return unique;
}
```

```ts
export function uniqueElementInAnArray(values: number[]): number {
  if (values.length === 0) {
    throw new RangeError('Values must be a non-empty array');
  }

  let unique = 0;

  for (const value of values) {
    unique ^= value;
  }

  return unique;
}
```

## Follow-up Questions

- Adapt the solution when every other number appears three times instead of twice.
- Explain why order does not matter in the XOR reduction.
- Compare this constant-space solution with a frequency-map approach.

## Common Pitfalls

- Applying the XOR trick when the input does not satisfy the exact twice-except-one guarantee.
- Reaching for a hash map even when the prompt emphasizes constant extra space.
- Forgetting that JavaScript bitwise operators coerce values to 32-bit integers.
