## Why It Surfaces

Reversing an integer is a standard digit-manipulation exercise. It looks simple, but it tests whether you can be precise about negative signs, trailing zeros, and the difference between numeric and string approaches.

## Worked Example

`reverseNumber(-1200)` returns `-21`. Leading zeros in the reversed representation disappear because the result is still a number, not a formatted string.

## Reference Implementations

<ExamplePair>

```js
function reverseNumber(value) {
  if (!Number.isInteger(value)) {
    throw new RangeError('Value must be an integer');
  }

  const sign = value < 0 ? -1 : 1;
  let remaining = Math.abs(value);
  let reversed = 0;

  while (remaining > 0) {
    reversed = reversed * 10 + (remaining % 10);
    remaining = Math.floor(remaining / 10);
  }

  return sign * reversed;
}
```

## Follow-up Questions

- Detect overflow if the language has fixed-width integers.

- Compare this arithmetic approach with string reversal and discuss the tradeoffs.

## Common Pitfalls

- Ignoring negative inputs entirely.
- Expecting reversed trailing zeros to remain visible in numeric output.
- Accepting fractional inputs even though the routine is defined for integers.
