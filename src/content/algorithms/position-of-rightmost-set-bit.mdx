## Why It Surfaces

This is the natural follow-up to low-bit identities such as clearing the rightmost set bit. Interviewers use it to see whether you can move from “isolate the bit” to “turn that bit into an index.”

## Worked Example

`18` is `10010` in binary, so the first set bit from the right is at position `2`. For `19`, which is `10011`, the answer is `1`.

## Reference Implementations

<ExamplePair>

```js
function positionOfRightmostSetBit(value) {
  if (!Number.isInteger(value) || value < 0 || value > 0xffffffff) {
    throw new RangeError('Value must be an unsigned 32-bit integer');
  }

  const number = value >>> 0;

  if (number === 0) {
    return 0;
  }

  const isolated = (number & -number) >>> 0;
  return Math.log2(isolated) + 1;
}
```

## Follow-up Questions

- Return the isolated low-bit value instead of its position.
- Solve the same problem with a shift loop instead of `Math.log2`.
- Compare the one-based convention here with zero-based bit indexing prompts.

## Common Pitfalls

- Returning `-1` instead of `0` when the input has no set bits.
- Forgetting that the expected position is one-based from the right.
- Mixing up the rightmost set-bit position with the count of set bits.
