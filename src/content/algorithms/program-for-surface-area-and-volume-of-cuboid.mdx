## Why It Surfaces

Cuboid prompts are the rectangular-box version of the combined-measurement pattern. The main check is whether you decompose the area into the three unique face rectangles and remember that each occurs twice.

## Worked Example

For `length = 2`, `width = 3`, and `height = 4`, the total surface area is `52` and the volume is `24`.

## Reference Implementations

<ExamplePair>

```js
function programForSurfaceAreaAndVolumeOfCuboid(length, width, height) {
  const dimensions = [length, width, height];
  if (!dimensions.every(Number.isFinite) || dimensions.some((dimension) => dimension < 0)) {
    throw new RangeError("Cuboid dimensions must be non-negative numbers");
  }

  return {
    surfaceArea: 2 * (length * width + width * height + length * height),
    volume: length * width * height,
  };
}
```

```ts
export function programForSurfaceAreaAndVolumeOfCuboid(
  length: number,
  width: number,
  height: number,
): { surfaceArea: number; volume: number } {
  const dimensions = [length, width, height];
  if (!dimensions.every(Number.isFinite) || dimensions.some((dimension) => dimension < 0)) {
    throw new RangeError('Cuboid dimensions must be non-negative numbers');
  }

  return {
    surfaceArea: 2 * (length * width + width * height + length * height),
    volume: length * width * height,
  };
}
```

## Follow-up Questions

- Why are there only three distinct face-area products even though a cuboid has six faces?
- How does this simplify when all three dimensions are equal?
- If the prompt wanted lateral surface area only, which terms would remain?

## Common Pitfalls

- Forgetting the leading `2` in the total surface-area formula.
- Mixing up the three face products or dropping one of them.
- Returning positional results without documenting which value is area and which is volume.
