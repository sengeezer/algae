## Why It Surfaces

Minimum Cost Path is the weighted follow-up to simple grid-counting problems. Instead of asking how many ways to reach the destination, the interviewer asks for the cheapest route, so each cell stores the best cumulative cost seen so far.

## Worked Example

For `[[2, 1], [3, 1], [4, 2]]`, the cheapest route is `2 -> 1 -> 1 -> 2`, for a total of `6`. Each cell only needs the cheaper of the path from above or the path from the left.

<Callout title="State choice" tone="tip">
  Store cumulative cost, not the path itself. Once the cheapest cost table is correct, route reconstruction is a separate follow-up.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function minimumCostPath(matrix) {
  const rows = matrix.length;
  const cols = matrix[0].length;
  const dp = Array.from({ length: rows }, () => new Array(cols).fill(0));

  dp[0][0] = matrix[0][0];

  for (let col = 1; col < cols; col += 1) {
    dp[0][col] = dp[0][col - 1] + matrix[0][col];
  }

  for (let row = 1; row < rows; row += 1) {
    dp[row][0] = dp[row - 1][0] + matrix[row][0];
  }

  for (let row = 1; row < rows; row += 1) {
    for (let col = 1; col < cols; col += 1) {
      dp[row][col] = Math.min(dp[row - 1][col], dp[row][col - 1]) + matrix[row][col];
    }
  }

  return dp[rows - 1][cols - 1];
}
```

```ts
export function minimumCostPath(matrix: number[][]): number {
  const cols = matrix[0].length;
  const dp = new Array<number>(cols).fill(Number.POSITIVE_INFINITY);
  dp[0] = 0;

  for (const row of matrix) {
    dp[0] += row[0];

    for (let col = 1; col < cols; col += 1) {
      dp[col] = Math.min(dp[col], dp[col - 1]) + row[col];
    }
  }

  return dp[cols - 1];
}
```

## Follow-up Questions

- Reconstruct the actual cheapest path, not just its total cost.
- Explain when Dijkstra becomes necessary instead of grid DP.
- Add blocked cells or diagonal movement and update the recurrence.

## Common Pitfalls

- Treating this as an unweighted path-count problem.
- Forgetting to initialize the first row and first column separately.
- Using a greedy local choice instead of cumulative state.
