## Why It Surfaces

Count-letters problems are the simplest form of frequency mapping. They matter because the same pattern underlies anagram grouping, uniqueness checks, and many sliding-window string problems.

## Worked Example

For `"Hello"`, normalize to lowercase and count letters: `{ h: 1, e: 1, l: 2, o: 1 }`.

## Reference Implementations

<ExamplePair>

```js
function countLetters(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  if (!/^[a-z]+$/i.test(value)) {
    throw new TypeError('Value must contain letters only');
  }

  const counts = {};

  for (const char of value.toLowerCase()) {
    counts[char] = (counts[char] ?? 0) + 1;
  }

  return counts;
}
```

```ts
export function countLetters(value: string): Record<string, number> {
  if (!/^[a-z]+$/i.test(value)) {
    throw new TypeError('Value must contain letters only');
  }

  const counts: Record<string, number> = {};

  for (const char of value.toLowerCase()) {
    counts[char] = (counts[char] ?? 0) + 1;
  }

  return counts;
}
```

## Follow-up Questions

- Allow spaces and punctuation but ignore them instead of throwing.
- Return the most frequent letter and its count.
- Compare this direct map approach with an array-based count table for fixed alphabets.

## Common Pitfalls

- Forgetting case normalization.
- Mixing total length with per-letter counts.
- Not clarifying whether non-letter characters should be ignored or rejected.
