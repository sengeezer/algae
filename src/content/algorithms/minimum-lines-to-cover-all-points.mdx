## Why It Surfaces

At first glance this looks like a geometric covering problem, but the fixed pivot point simplifies it sharply. Every valid covering line is determined only by its slope through the pivot, so you just count how many distinct normalized directions the given points produce.

## Worked Example

For points `[[-1, 3], [4, 3], [2, 1], [-1, -2], [3, -3]]` and pivot `[1, 0]`, there are only two distinct slopes from the pivot, so the minimum number of lines is `2`.

```js
function minimumLinesToCoverAllPoints(points, origin) {
  function gcd(left, right) {
    let a = Math.abs(left);
    let b = Math.abs(right);

    while (b !== 0) {
      [a, b] = [b, a % b];
    }

    return a;
  }

  function normalizedSlope(point) {
    let dy = point[1] - origin[1];
    let dx = point[0] - origin[0];

    if (dx === 0) {
      return "1/0";
    }

    if (dy === 0) {
      return "0/1";
    }

    const divisor = gcd(dy, dx);
    dy /= divisor;
    dx /= divisor;

    if (dx < 0) {
      dy = -dy;
      dx = -dx;
    }

    return `${dy}/${dx}`;
  }

  const slopes = new Set();

  for (const point of points) {
    if (point[0] === origin[0] && point[1] === origin[1]) {
      continue;
    }

    slopes.add(normalizedSlope(point));
  }

  return slopes.size;
}
```

  points: Array<[number, number]>,
  origin: [number, number],
): number {
  const gcd = (left: number, right: number): number => {
    let a = Math.abs(left);
    let b = Math.abs(right);

    while (b !== 0) {
      [a, b] = [b, a % b];
    }

    return a;
  };

  const normalizedSlope = (point: [number, number]): string => {
    let dy = point[1] - origin[1];
    let dx = point[0] - origin[0];

    if (dx === 0) {
      return '1/0';
    }

    if (dy === 0) {
      return '0/1';
    }

    const divisor = gcd(dy, dx);
    dy /= divisor;
    dx /= divisor;

    if (dx < 0) {
      dy = -dy;
      dx = -dx;
    }

    return `${dy}/${dx}`;
  };

  const slopes = new Set<string>();

  for (const point of points) {
    if (point[0] === origin[0] && point[1] === origin[1]) {
      continue;
    }

    slopes.add(normalizedSlope(point));
  }

  return slopes.size;
}

## Follow-up Questions

- Why does fixing one required pivot point collapse the problem to counting slopes?
- How would the problem change if lines were not required to pass through one origin?
- Why is normalized direction safer than floating-point slope storage here too?

## Common Pitfalls

- Counting each point separately instead of collapsing equal slopes.
- Forgetting to skip the pivot point itself if it appears in the input.
- Mishandling vertical directions where the denominator is zero.
