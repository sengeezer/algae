## Why It Surfaces

This problem is valuable because it looks like an MST question at first, but the real move is completely different. Instead of greedily constructing one tree, you count every valid spanning tree by converting the graph into a Laplacian matrix and taking one determinant.

## Worked Example

If a connected graph has one triangle hanging off a tail, the cycle contributes multiple valid ways to remove one edge while still staying connected. Kirchhoff's theorem captures all of those possibilities at once by using the cofactor of the Laplacian matrix.

```js
function numberOfSpanningTreesInAGraph(graph) {
  const size = graph.length;
  if (size <= 1) {
    return 1;
  }

  const laplacian = Array.from({ length: size }, () => new Array(size).fill(0));

  for (let row = 0; row < size; row += 1) {
    let degree = 0;
    for (let column = 0; column < size; column += 1) {
      if (graph[row][column] === 0) {
        continue;
      }

      degree += graph[row][column];
      if (row !== column) {
        laplacian[row][column] = -graph[row][column];
      }
    }

    laplacian[row][row] = degree;
  }

  const minor = laplacian.slice(1).map((row) => row.slice(1));
  let determinant = 1;

  for (let pivot = 0; pivot < minor.length; pivot += 1) {
    let bestRow = pivot;
    for (let row = pivot + 1; row < minor.length; row += 1) {
      if (Math.abs(minor[row][pivot]) > Math.abs(minor[bestRow][pivot])) {
        bestRow = row;
      }
    }

    if (Math.abs(minor[bestRow][pivot]) < 1e-9) {
      return 0;
    }

    if (bestRow !== pivot) {
      [minor[pivot], minor[bestRow]] = [minor[bestRow], minor[pivot]];
      determinant *= -1;
    }

    determinant *= minor[pivot][pivot];
    for (let row = pivot + 1; row < minor.length; row += 1) {
      const factor = minor[row][pivot] / minor[pivot][pivot];
      for (let column = pivot; column < minor.length; column += 1) {
        minor[row][column] -= factor * minor[pivot][column];
      }
    }
  }

  return Math.round(determinant);
}
```

## Follow-up Questions

- Explain why removing any one row and any one column yields the same count.
- Extend the approach to weighted graphs with the weighted matrix-tree theorem.
- Compare this counting problem with constructing a minimum spanning tree.

## Common Pitfalls

- Confusing spanning-tree counting with MST construction.
- Building the adjacency matrix instead of the Laplacian before taking the determinant.
- Forgetting that the theorem uses a cofactor, not the full Laplacian determinant.
