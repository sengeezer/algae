## Why It Surfaces

The trick here is interpretive rather than algebraic. A cylinder is three-dimensional, so it does not have a standard planar perimeter, but the source prompt defines the measure using the side projection, which is just a rectangle with dimensions `diameter` and `height`.

## Worked Example

For `diameter = 5` and `height = 10`, the perimeter is `2 * (5 + 10) = 30`.

## Reference Implementations

<ExamplePair>

```js
function findThePerimeterOfACylinder(diameter, height) {
  if (!Number.isFinite(diameter) || !Number.isFinite(height) || diameter < 0 || height < 0) {
    throw new RangeError("Diameter and height must be non-negative numbers");
  }

  return 2 * (diameter + height);
}
```

```ts
export function findThePerimeterOfACylinder(diameter: number, height: number): number {
  if (!Number.isFinite(diameter) || !Number.isFinite(height) || diameter < 0 || height < 0) {
    throw new RangeError('Diameter and height must be non-negative numbers');
  }

  return 2 * (diameter + height);
}
```

## Common Pitfalls

- Confusing this side-view perimeter with the circumference of the base circle.
- Halving the diameter unnecessarily.
- Forgetting that the formula comes from a rectangle, not from a curved surface length.
