## Why It Surfaces

Decimal expansion is one of the cleanest demonstrations of cycle detection outside graph problems. Once a remainder repeats, the fractional digits must repeat too.

## Worked Example

For `1 / 6` in base `10`, the integer part is `0`, the fractional digits begin as `[1, 6]`, and the repeating cycle starts at index `1`, giving `0.1(6)`.

```js
function decimalExpansion(numerator, denominator, base = 10) {
  if (!Number.isInteger(numerator) || !Number.isInteger(denominator) || denominator === 0) {
    throw new RangeError('Numerator and denominator must be integers, and denominator cannot be zero');
  }

  if (!Number.isInteger(base) || base < 2 || base > 10) {
    throw new RangeError('Base must be an integer from 2 to 10');
  }

  const sign = numerator * denominator < 0 ? -1 : 1;
  const left = Math.abs(numerator);
  const right = Math.abs(denominator);
  const integerPart = Math.floor(left / right);
  const digits = [];
  const seenRemainders = new Map();
  let remainder = left % right;

  while (remainder !== 0 && !seenRemainders.has(remainder)) {
    seenRemainders.set(remainder, digits.length);
    remainder *= base;
    digits.push(Math.floor(remainder / right));
    remainder %= right;
  }

  return {
    sign,
    integerPart,
    digits,
    cycleStart: remainder === 0 ? undefined : seenRemainders.get(remainder),
  };
}
```

## Follow-up Questions

- Render the result as a human-readable string like `-3.14(2857)`.
- Support bases above `10` by mapping digits to alphabetic symbols.
- Explain why the cycle length is bounded by the number of possible nonzero remainders.

## Common Pitfalls

- Forgetting to guard against division by zero.
- Trying to detect repetition from floating-point values instead of remainders.
- Losing the sign when the integer part is `0`.
