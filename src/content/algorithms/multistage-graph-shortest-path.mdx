## Why It Surfaces

This problem matters because it looks like a shortest-path question, but the real win comes from the graph's stage structure. Once nodes are ordered by stage, the best cost from any node depends only on already-solved costs in later stages.

## Worked Example

If node `0` can go to `1`, `2`, or `3`, and each of those can only move to later stages, then the shortest distance from `0` is the minimum of its outgoing edge cost plus the already-computed distance from the chosen next node to the sink. That makes a backward pass from the sink enough.

## Reference Implementations

<ExamplePair>

```js
function multistageGraphShortestPath(cost, source = 0, target = cost.length - 1) {
  const size = cost.length;
  if (size === 0) {
    return { distance: 0, path: [] };
  }

  const distance = new Array(size).fill(Number.POSITIVE_INFINITY);
  const nextNode = new Array(size).fill(-1);
  distance[target] = 0;

  for (let node = target - 1; node >= source; node -= 1) {
    for (let neighbor = node + 1; neighbor <= target; neighbor += 1) {
      if (!Number.isFinite(cost[node][neighbor]) || distance[neighbor] === Number.POSITIVE_INFINITY) {
        continue;
      }

      const candidate = cost[node][neighbor] + distance[neighbor];
      if (candidate < distance[node]) {
        distance[node] = candidate;
        nextNode[node] = neighbor;
      }
    }
  }

  if (distance[source] === Number.POSITIVE_INFINITY) {
    return { distance: -1, path: [] };
  }

  const path = [];
  for (let node = source; node !== -1; node = nextNode[node]) {
    path.push(node);
    if (node === target) {
      break;
    }
  }

  return { distance: distance[source], path };
}
```

## Common Pitfalls

- Ignoring the stage structure and reaching for a heavier generic algorithm immediately.
- Assuming the locally cheapest outgoing edge must lie on the best global route.
- Forgetting that the backward DP depends on nodes being ordered consistently with stages.
