## Why It Surfaces

Date parsing looks simple until validation starts to matter. Interviewers use it to see whether you stop at regex matching or keep going until the parsed date is actually possible on a calendar.

## Worked Example

`29/02/2024` parses successfully to `{ day: 29, month: 2, year: 2024 }`, but `29/02/2023` must be rejected because February has only `28` days in that year.

```js
function isLeapYear(year) {
  return year % 400 === 0 || (year % 4 === 0 && year % 100 !== 0);
}

function getMonthDays(month, year) {
  if (!Number.isInteger(month) || month < 1 || month > 12) {
    throw new RangeError('Month must be an integer from 1 to 12');
  }

  if ([1, 3, 5, 7, 8, 10, 12].includes(month)) {
    return 31;
  }

  if ([4, 6, 9, 11].includes(month)) {
    return 30;
  }

  return isLeapYear(year) ? 29 : 28;
}

function parseDate(input) {
  if (typeof input !== 'string') {
    throw new TypeError('Expected a date string');
  }

  const match = /^(\d{1,2})\/(\d{1,2})\/(\d{4})$/.exec(input);
  if (!match) {
    throw new RangeError('Date must use dd/mm/yyyy format');
  }

  const day = Number(match[1]);
  const month = Number(match[2]);
  const year = Number(match[3]);
  const maxDay = getMonthDays(month, year);

  if (day < 1 || day > maxDay) {
    throw new RangeError('Invalid day value for the parsed month and year');
  }

  return { day, month, year };
}
```

```ts
export function parseDate(input: string): { day: number; month: number; year: number } {
  const match = /^(\d{1,2})\/(\d{1,2})\/(\d{4})$/.exec(input);
  if (!match) {
    throw new RangeError('Date must use dd/mm/yyyy format');
  }

  const day = Number(match[1]);
  const month = Number(match[2]);
  const year = Number(match[3]);
  const maxDay = getMonthDays(month, year);

  if (day < 1 || day > maxDay) {
    throw new RangeError('Invalid day value for the parsed month and year');
  }

  return { day, month, year };
```

## Follow-up Questions

- Accept ISO-style `yyyy-mm-dd` input as an alternate format.
- Explain why parsing and validation should stay separate in larger systems.
- Reuse this parser inside `dateToDay` or `dateDayDifference` implementations.

## Common Pitfalls

- Accepting month `0` or `13` because the regex matched two digits.
- Validating the string shape but not the day range for the parsed month.
- Forgetting leap-year rules for February.
