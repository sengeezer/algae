## Why It Surfaces

Title casing turns a simple ASCII offset trick into a stateful string problem. The interesting part is deciding when a new word begins and making sure repeated whitespace survives unchanged.

## Worked Example

`aLgAe   cATALOG` becomes `Algae   Catalog`. The first letter of each space-delimited word is uppercased, and later ASCII letters in the same word are lowercased.

## Reference Implementations

```js
function toUpperAscii(char) {
  const code = char.charCodeAt(0);
  return code >= 97 && code <= 122 ? String.fromCharCode(code - 32) : char;
}

function toLowerAscii(char) {
  const code = char.charCodeAt(0);
  return code >= 65 && code <= 90 ? String.fromCharCode(code + 32) : char;
}

function titleCaseConversion(input) {
  if (typeof input !== 'string') {
    throw new TypeError('Expected a string input');
  }

  let atWordStart = true;

  return input
    .split('')
    .map((char) => {
      if (/\s/.test(char)) {
        atWordStart = true;
        return char;
      }

      if (atWordStart) {
        atWordStart = false;
        return toUpperAscii(char);
      }

      return toLowerAscii(char);
    })
    .join('');
}
```

## Follow-up Questions

- Preserve small stop words like `a` and `of` in lowercase except at the start of the title.
- Treat punctuation such as `-` or `'` as additional word boundaries if a product spec requires it.
- Compare this manual approach with locale-aware casing APIs.

## Common Pitfalls

- Collapsing repeated spaces when splitting on whole words.
- Uppercasing the first letter of a word but forgetting to lowercase the rest.
- Assuming ASCII character offsets are correct for every language.
