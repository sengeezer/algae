## Why It Surfaces

Interpolation search is a lower-frequency follow-up for sorted arrays. Instead of probing the midpoint, it predicts where the target should sit based on how far its value lies between the current endpoints.

## Worked Example

If the range is `[10, 20, 30, 40, 50]` and the target is `40`, the next probe should land closer to the right side than the middle because `40` is much closer to `50` than to `10`.

<Callout title="When it helps" tone="note">
  This strategy only pays off when the data distribution is fairly even. On skewed data, the estimate can be poor and the worst case becomes linear.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function interpolationSearch(nums, target) {
  let left = 0;
  let right = nums.length - 1;

  while (
    left <= right
    && target >= nums[left]
    && target <= nums[right]
  ) {
    if (nums[left] === nums[right]) {
      return nums[left] === target ? left : -1;
    }

    const position = left + Math.floor(
      ((target - nums[left]) * (right - left)) / (nums[right] - nums[left]),
    );

    if (nums[position] === target) {
      return position;
    }

    if (nums[position] < target) {
      left = position + 1;
    } else {
      right = position - 1;
    }
  }

  return -1;
}
```

## Common Pitfalls

- Dividing by zero when both endpoints hold the same value.
- Using interpolation search on unsorted or badly skewed data.
- Letting the estimated probe fall outside the current range.
