## Why It Surfaces

Bell numbers are the natural follow-up when counting subsets is no longer enough and the interviewer wants complete set partitions instead. The useful shift is that every new element either creates a new singleton block or gets inserted into one of the existing blocks counted by a smaller subproblem.

## Worked Example

For `n = 3`, the five set partitions are `{{1}, {2}, {3}}`, `{{1, 2}, {3}}`, `{{1, 3}, {2}}`, `{{2, 3}, {1}}`, and `{{1, 2, 3}}`. That is why the third Bell number is `5`.

## Reference Implementations

<ExamplePair>

```js
function bellNumbersNumberOfWaysToPartitionASet(size) {
  if (size < 0) {
    throw new RangeError('size must be non-negative');
  }

  const dp = new Array(size + 1).fill(0);
  dp[0] = 1;

  for (let row = 1; row <= size; row += 1) {
    let previous = dp[0];
    dp[0] = dp[row - 1];

    for (let col = 1; col <= row; col += 1) {
      const current = dp[col];
      dp[col] = previous + dp[col - 1];
      previous = current;
    }
  }

  return dp[0];
}
```

## Follow-up Questions

- Compute the Stirling numbers of the second kind for a fixed number of blocks instead of summing all block counts.
- Return the first `n` Bell numbers as a sequence.
- Explain how the Bell triangle relates to the recurrence for Stirling numbers.

## Common Pitfalls

- Confusing set partitions with subset enumeration or combinations.
- Forgetting that empty blocks are not allowed in the counted partitions.
- Recomputing the full Bell triangle when only the previous row is needed.
