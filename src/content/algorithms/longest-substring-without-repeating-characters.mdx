## Why It Surfaces

This is one of the standard sliding-window interview questions because it forces you to maintain a valid contiguous segment incrementally. The key skill is repairing the window immediately after a repeated character appears.

## Worked Example

For `"abcabcbb"`, the longest valid window starts as `"abc"`. When the second `a` arrives, the left boundary jumps past the earlier `a`, keeping the window duplicate-free instead of restarting from scratch.

<Callout title="Substring, not subsequence" tone="warning">
  The answer must stay contiguous. If you mentally skip characters, you have switched to a different problem.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function lengthOfLongestSubstring(text) {
  const lastSeen = new Map();
  let left = 0;
  let best = 0;

  for (let right = 0; right < text.length; right += 1) {
    const char = text[right];

    if (lastSeen.has(char) && lastSeen.get(char) >= left) {
      left = lastSeen.get(char) + 1;
    }

    lastSeen.set(char, right);
    best = Math.max(best, right - left + 1);
  }

  return best;
}
```

```ts
export function lengthOfLongestSubstring(text: string): number {
  const lastSeen = new Map<string, number>();
  let left = 0;
  let best = 0;

  for (let right = 0; right < text.length; right += 1) {
    const char = text[right];
    const previous = lastSeen.get(char);

    if (previous !== undefined && previous >= left) {
      left = previous + 1;
    }

    lastSeen.set(char, right);
    best = Math.max(best, right - left + 1);
  }

  return best;
}
```

## Follow-up Questions

- Return the actual substring, not only its length.
- Adapt the pattern to at most `k` distinct characters.
- Explain why deleting one character at a time can still be linear.

## Common Pitfalls

- Resetting the window completely when only part of it is invalid.
- Confusing substring and subsequence.
- Letting the left pointer move backward after seeing an older duplicate.
