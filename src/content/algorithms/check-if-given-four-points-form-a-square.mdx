## Why It Surfaces

The prompt sounds shape-specific, but the real trick is to normalize away the input order. Once you compute all six pairwise squared distances, a valid square has exactly four equal non-zero side distances and two equal diagonal distances, with each diagonal equal to twice a side distance.

## Worked Example

For points `[20, 10]`, `[10, 20]`, `[20, 20]`, and `[10, 10]`, the four side distances are equal and the two diagonals match, so the points form a square.

```js
function checkIfGivenFourPointsFormASquare(p1, p2, p3, p4) {
  function distSq(a, b) {
    const dx = a[0] - b[0];
    const dy = a[1] - b[1];
    return dx * dx + dy * dy;
  }

  const distances = [
    distSq(p1, p2),
    distSq(p1, p3),
    distSq(p1, p4),
    distSq(p2, p3),
    distSq(p2, p4),
    distSq(p3, p4),
  ].sort((left, right) => left - right);

  return (
    distances[0] > 0
    && distances[0] === distances[1]
    && distances[1] === distances[2]
    && distances[2] === distances[3]
    && distances[4] === distances[5]
    && distances[4] === 2 * distances[0]
  );
}
```

## Common Pitfalls

- Checking only that four sides are equal and forgetting the diagonal condition.
- Assuming the points arrive in cyclic order.
- Using floating-point distances when integer squared distances are enough.
