## Why It Surfaces

This is a greedy subsequence problem where the critical choice is when to commit a high-valued character. If a character still appears at least `k` times in the remaining suffix, taking all of those copies dominates any answer that starts with a smaller letter.

## Worked Example

For `s = "banana"` and `k = 2`, the valid subsequences include `"aa"`, `"nn"`, and `"aa"`-prefixed variants from different positions. Since `n` is lexicographically larger than `a` and it appears twice in order, the answer is `"nn"`.

## Reference Implementations

<ExamplePair>

```js
function lexicographicallyLargestSubsequenceSuchThatEveryCharacterOccursAtLeastKTimes(
  text,
  k,
) {
  if (k <= 0) {
    return text;
  }

  let start = 0;
  let result = '';

  for (let code = 'z'.charCodeAt(0); code >= 'a'.charCodeAt(0); code -= 1) {
    const char = String.fromCharCode(code);
    let count = 0;
    let lastIndex = -1;

    for (let index = start; index < text.length; index += 1) {
      if (text[index] !== char) {
        continue;
      }

      count += 1;
      lastIndex = index;
    }

    if (count < k) {
      continue;
    }

    result += char.repeat(count);
    start = lastIndex + 1;
  }

  return result;
}
```

## Follow-up Questions

- Explain why taking only some of a chosen character block cannot improve the answer.
- Adapt the routine to uppercase letters or a larger custom alphabet.
- Return the indices of the chosen subsequence instead of only the string.

## Common Pitfalls

- Treating the problem like a substring instead of a subsequence.
- Committing a character that appears fewer than `k` times in the remaining suffix.
- Scanning from low characters upward, which ruins the lexicographic objective.
