## Why It Surfaces

This problem is the clean comparison point for 0/1 knapsack. The structure is almost the same, but the loop order changes because taking an item should still leave that same item available again.

## Worked Example

For weights `[2, 3, 4]`, values `[40, 50, 70]`, and capacity `6`, the best answer is `120` by taking the weight-`2` item three times.

## Reference Implementations

<ExamplePair>

```js
function unboundedKnapsackRepetitionOfItemsAllowed(weights, values, capacity) {
  const dp = new Array(capacity + 1).fill(0);

  for (let space = 0; space <= capacity; space += 1) {
    for (let item = 0; item < weights.length; item += 1) {
      if (weights[item] <= space) {
        dp[space] = Math.max(dp[space], dp[space - weights[item]] + values[item]);
      }
    }
  }

  return dp[capacity];
}
```

      }
    }
  }

  return dp[capacity];
}

## Common Pitfalls

- Iterating capacity backward and accidentally turning the problem back into 0/1 knapsack.
- Confusing unbounded value maximization with coin-change counting.
- Forgetting that a lower-weight item can dominate by being reusable more times.
