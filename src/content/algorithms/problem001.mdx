## Why It Surfaces

This is a classic warm-up for converting a brute-force sum into a constant-time formula. The important step is subtracting the multiples of `15` once because they were counted twice.

## Worked Example

Below `10`, the valid numbers are `3`, `5`, `6`, and `9`, which sum to `23`. The optimized version gets the same result by summing the multiples of `3` and `5` separately and subtracting the multiples of `15`.

## Reference Implementations

<ExamplePair>

```js
function arithmeticMultipleSum(step, limit) {
  const count = Math.floor((limit - 1) / step);
  return (step * count * (count + 1)) / 2;
}

function problem001(limit) {
  if (!Number.isInteger(limit) || limit < 1) {
    throw new RangeError('Limit must be a positive integer');
  }

  return (
    arithmeticMultipleSum(3, limit) +
    arithmeticMultipleSum(5, limit) -
    arithmeticMultipleSum(15, limit)
  );
}
```

## Follow-up Questions

- Compare the constant-time formula with a simple loop.
- Generalize the idea to any two divisors.
- Explain why the multiples of `15` need to be subtracted exactly once.

## Common Pitfalls

- Double-counting numbers divisible by both `3` and `5`.
- Summing up to and including the limit when the prompt says “below”.
- Using a loop when the closed-form progression formula is expected.
