## Why It Surfaces

Base conversion is a standard arithmetic interview helper. It checks whether you understand how remainders reveal the digits of a positional numeral system from least significant to most significant.

## Worked Example

`125` in base `2` becomes `1111101` because repeated division by `2` yields the binary digits in reverse remainder order.

## Reference Implementations

<ExamplePair>

```js
function intToBase(value, base) {
  if (![value, base].every((number) => Number.isInteger(number))) {
    throw new TypeError('Value and base must be integers');
  }

  if (base < 2 || base > 10) {
    throw new RangeError('Base must be between 2 and 10');
  }

  if (value === 0) {
    return '0';
  }

  let remaining = Math.abs(value);
  let converted = '';

  while (remaining > 0) {
    converted = String(remaining % base) + converted;
    remaining = Math.trunc(remaining / base);
  }

  return value < 0 ? `-${converted}` : converted;
}
```

## Follow-up Questions

- Extend the converter to bases above `10` with alphabetic digits.
- Convert from an arbitrary base back to decimal.
- Compare repeated-division conversion with Horner-style evaluation in the reverse direction.

## Common Pitfalls

- Forgetting that the remainders arrive from least significant to most significant.
- Accepting invalid base values like `1` or `16` when the implementation only supports digits `0` through `9`.
- Dropping the sign when the input is negative.
