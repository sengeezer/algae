## Why It Surfaces

This is the midpoint formula with uneven weights. Instead of splitting the segment equally, you scale each endpoint by the opposite ratio term, which gives the coordinates of the internal dividing point in one step.

## Worked Example

For endpoints `[2, 4]` and `[4, 6]` with ratio `2:3`, the section point is `[(3 * 2 + 2 * 4) / 5, (3 * 4 + 2 * 6) / 5] = [2.8, 4.8]`.

## Reference Implementations

<ExamplePair>

```js
function sectionFormulaPointThatDividesALineInGivenRatio(start, end, m, n) {
  const total = m + n;

  if (total === 0) {
    throw new RangeError("Ratio parts cannot sum to zero");
  }

  return [
    (n * start[0] + m * end[0]) / total,
    (n * start[1] + m * end[1]) / total,
  ];
}
```

```ts
export function sectionFormulaPointThatDividesALineInGivenRatio(
  start: [number, number],
  end: [number, number],
  m: number,
  n: number,
): [number, number] {
  const total = m + n;

  if (total === 0) {
    throw new RangeError('Ratio parts cannot sum to zero');
  }

  return [
    (n * start[0] + m * end[0]) / total,
    (n * start[1] + m * end[1]) / total,
  ];
}
```

## Follow-up Questions

- Why does midpoint fall out when `m = n = 1`?
- How would you adapt this for an external division point?
- Why do the weights use the opposite ratio parts?

## Common Pitfalls

- Using `m` with the first point and `n` with the second without checking the formula direction.
- Forgetting to reject the degenerate case where `m + n = 0`.
- Assuming the point must have integer coordinates.
