## Why It Surfaces

This is the shortest useful bit identity most interviewers expect you to know. It rewards understanding the binary shape of powers of two instead of brute-force arithmetic.

## Worked Example

`16` is `10000` in binary, so `16 & 15` becomes `10000 & 01111 = 0`. By contrast, `12` is `1100`, and `12 & 11` becomes `1100 & 1011 = 1000`, so it is not a power of two.

## Reference Implementations

<ExamplePair>

```js
function isPowerOfTwo(value) {
  return value > 0 && (value & (value - 1)) === 0;
}
```

```ts
export function isPowerOfTwo(value: number): boolean {
  return value > 0 && (value & (value - 1)) === 0;
}
```

</ExamplePair>

## Follow-up Questions

- Extend the same reasoning to powers of four.
- Explain why zero and negative values must be rejected first.
- Compare the bitwise test with repeated division by two.

## Common Pitfalls

- Forgetting that `1` is typically treated as `2^0` and therefore valid.
- Omitting the positivity check and accidentally accepting `0`.
- Replacing a constant-time bit test with a slower arithmetic loop.
