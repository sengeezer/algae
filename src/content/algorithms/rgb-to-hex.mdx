## Why It Surfaces

RGB-to-hex conversion is a practical example of base conversion that many developers actually use. Each channel is just one byte, so the algorithm converts three decimal values into three padded hexadecimal pairs.

## Worked Example

`(255, 99, 71)` becomes `(FF, 63, 47)`, so the final CSS color string is `#FF6347`. The important detail is padding single-digit hex outputs to two characters.

## Reference Implementations

<ExamplePair>

```js
function channelToHex(channel) {
  if (!Number.isInteger(channel) || channel < 0 || channel > 255) {
    throw new RangeError('RGB channels must be integers from 0 to 255');
  }

  return channel.toString(16).toUpperCase().padStart(2, '0');
}

function rgbToHex(red, green, blue) {
  return `#${channelToHex(red)}${channelToHex(green)}${channelToHex(blue)}`;
}
```

```ts
function channelToHex(channel: number): string {
  if (!Number.isInteger(channel) || channel < 0 || channel > 255) {
    throw new RangeError('RGB channels must be integers from 0 to 255');
  }

  return channel.toString(16).toUpperCase().padStart(2, '0');
}

export function rgbToHex(red: number, green: number, blue: number): string {
  return `#${channelToHex(red)}${channelToHex(green)}${channelToHex(blue)}`;
}
```

## Follow-up Questions

- Explain why each channel always maps to exactly two hex digits.
- Add support for optional alpha channels like `#RRGGBBAA`.
- Return lowercase output when integrating with a style guide that prefers it.

## Common Pitfalls

- Forgetting to pad one-digit hex outputs.
- Accepting channel values outside the `0` to `255` range.
- Mixing numeric conversion with percentage-based color formats without normalizing first.
