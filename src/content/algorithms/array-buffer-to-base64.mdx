## Why It Surfaces

Base64 encoding is a practical bridge between binary and text systems. It matters whenever raw bytes need to move through channels that only reliably support printable characters, such as JSON payloads, HTML attributes, or form fields.

## Worked Example

The byte sequence for `Hello` becomes `SGVsbG8=`. The encoder packs bytes into 24-bit chunks, splits them into four six-bit indexes, and pads the final chunk when the byte count is not divisible by three.

## Reference Implementations

<ExamplePair>

```js
const BASE64_ALPHABET = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/';

function arrayBufferToBase64(buffer) {
  if (!(buffer instanceof ArrayBuffer)) {
    throw new TypeError('Expected an ArrayBuffer input');
  }

  const bytes = new Uint8Array(buffer);
  let encoded = '';

  for (let index = 0; index < bytes.length; index += 3) {
    const remaining = bytes.length - index;
    const byte1 = bytes[index];
    const byte2 = remaining > 1 ? bytes[index + 1] : 0;
    const byte3 = remaining > 2 ? bytes[index + 2] : 0;
    const chunk = (byte1 << 16) | (byte2 << 8) | byte3;

    encoded += BASE64_ALPHABET[(chunk >> 18) & 63];
    encoded += BASE64_ALPHABET[(chunk >> 12) & 63];
    encoded += remaining > 1 ? BASE64_ALPHABET[(chunk >> 6) & 63] : '=';
    encoded += remaining > 2 ? BASE64_ALPHABET[chunk & 63] : '=';
  }

  return encoded;
}
```

## Common Pitfalls

- Mishandling the final one-byte or two-byte chunk.
- Forgetting that Base64 padding is part of the textual format.
- Treating an ArrayBuffer like a normal array without creating a typed view first.
