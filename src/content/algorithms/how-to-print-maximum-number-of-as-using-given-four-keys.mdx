## Why It Surfaces

This problem is a clean DP exercise in delayed payoff. Typing `A` directly is always safe, but the optimal answer comes from choosing the right moment to switch into `select all -> copy -> paste` so future keystrokes multiply earlier work.

## Worked Example

With `7` keystrokes, the best plan is `A, A, A, Ctrl-A, Ctrl-C, Ctrl-V, Ctrl-V`, which produces `9` `A` characters. Typing seven direct letters would only produce `7`.

## Reference Implementations

<ExamplePair>

```js
function howToPrintMaximumNumberOfAsUsingGivenFourKeys(keystrokes) {
  if (keystrokes <= 0) {
    return 0;
  }

  const dp = new Array(keystrokes + 1).fill(0);

  for (let current = 1; current <= keystrokes; current += 1) {
    dp[current] = current;

    for (let breakpoint = 1; breakpoint <= current - 3; breakpoint += 1) {
      const produced = dp[breakpoint] * (current - breakpoint - 1);
      dp[current] = Math.max(dp[current], produced);
    }
  }

  return dp[keystrokes];
}
```

## Common Pitfalls

- Forgetting to reserve two keystrokes for `select all` and `copy` before counting pastes.
- Assuming the best answer always comes from the latest possible breakpoint.
- Overlooking the simple baseline that typing `A` directly remains valid for every step count.
