## Why It Surfaces

Exponential search appears when a normal binary search cannot start because you do not yet know a safe right boundary. By doubling the bound, you quickly bracket the target and then reuse the simpler binary-search logic inside that window.

## Worked Example

If the target is at index `13`, the bounds evolve like `1`, `2`, `4`, `8`, `16`. Once `16` is large enough, the answer must lie in the window `[8, 16]`, which is small enough for ordinary binary search.

## Reference Implementations

<ExamplePair>

```js
function exponentialSearch(nums, target) {
  if (nums.length === 0) {
    return -1;
  }

  if (nums[0] === target) {
    return 0;
  }

  let bound = 1;

  while (bound < nums.length && nums[bound] < target) {
    bound *= 2;
  }

  let left = Math.floor(bound / 2);
  let right = Math.min(bound, nums.length - 1);

  while (left <= right) {
    const mid = left + Math.floor((right - left) / 2);

    if (nums[mid] === target) {
      return mid;
    }

    if (nums[mid] < target) {
      left = mid + 1;
    } else {
      right = mid - 1;
    }
  }

  return -1;
}
```

- Explain when exponential search beats a linear scan for early targets.

## Common Pitfalls

- Forgetting to clamp the right boundary to the array length.
- Letting the doubling loop overrun when the target is smaller than the current bound.
- Rewriting binary search incorrectly after finding the candidate window.
