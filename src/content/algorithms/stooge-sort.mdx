## Why It Surfaces

Stooge sort is useful precisely because it is such a bad idea. It shows that recursion alone does not make an algorithm elegant or efficient. The subproblems overlap so heavily that the runtime becomes dreadful.

## Worked Example

On `[4, 3, 2, 1]`, stooge sort compares the ends, swaps them if needed, then recursively sorts the first two-thirds, the last two-thirds, and the first two-thirds again. The overlap is what makes it explode in cost.

## Reference Implementations

<ExamplePair>

```js
function stoogeSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const items = [...values];

  function sortRange(left, right) {
    if (items[right - 1] < items[left]) {
      [items[left], items[right - 1]] = [items[right - 1], items[left]];
    }

    const length = right - left;

    if (length > 2) {
      const third = Math.floor(length / 3);
      sortRange(left, right - third);
      sortRange(left + third, right);
      sortRange(left, right - third);
    }
  }

  if (items.length > 1) {
    sortRange(0, items.length);
  }

  return items;
}
```

export function stoogeSort(values: number[]): number[] {
  const items = [...values];

  function sortRange(left: number, right: number): void {
    if (items[right - 1] < items[left]) {
      [items[left], items[right - 1]] = [items[right - 1], items[left]];
    }

    const length = right - left;

    if (length > 2) {
      const third = Math.floor(length / 3);
      sortRange(left, right - third);
      sortRange(left + third, right);
      sortRange(left, right - third);
    }
  }

  if (items.length > 1) {
    sortRange(0, items.length);
  }

  return items;
}

## Follow-up Questions

- Compare stooge sort's recursion tree with merge sort's.
- Explain why the overlapping thirds create so much repeated work.
- Use the recurrence to estimate the exponent in the runtime.

## Common Pitfalls

- Mistaking divide-and-conquer structure for efficiency.
- Forgetting that the range is half-open in many recursive implementations.
- Using it as if it were a serious alternative to standard recursive sorts.
