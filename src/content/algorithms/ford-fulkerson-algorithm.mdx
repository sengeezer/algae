## Why It Surfaces

Ford-Fulkerson is still the cleanest way to explain max flow from first principles. It makes the residual graph, bottleneck edge, and reverse-capacity updates explicit before you move on to faster variants such as Dinic or Push Relabel.

## Worked Example

Start with zero flow. Each BFS finds one source-to-sink path in the residual graph, computes the smallest residual capacity along it, pushes that amount, and then adds reverse residual capacity so later augmentations can reroute previously chosen flow.

```js
function fordFulkersonAlgorithm(capacity, source, sink) {
  const size = capacity.length;
  const residual = capacity.map((row) => [...row]);
  const parent = new Array(size).fill(-1);
  let maxFlow = 0;

  function bfs() {
    parent.fill(-1);
    parent[source] = source;
    const queue = [source];

    for (let head = 0; head < queue.length; head += 1) {
      const node = queue[head];
      for (let neighbor = 0; neighbor < size; neighbor += 1) {
        if (parent[neighbor] === -1 && residual[node][neighbor] > 0) {
          parent[neighbor] = node;
          queue.push(neighbor);
        }
      }
    }

    return parent[sink] !== -1;
  }

  while (bfs()) {
    let pushed = Number.POSITIVE_INFINITY;
    for (let node = sink; node !== source; node = parent[node]) {
      pushed = Math.min(pushed, residual[parent[node]][node]);
    }

    for (let node = sink; node !== source; node = parent[node]) {
      residual[parent[node]][node] -= pushed;
      residual[node][parent[node]] += pushed;
    }

    maxFlow += pushed;
  }

  return maxFlow;
}
```

## Common Pitfalls

- Forgetting to add reverse residual capacity after pushing flow.
- Confusing residual capacity with original capacity.
- Claiming the BFS implementation still has the generic Ford-Fulkerson worst-case bound.
