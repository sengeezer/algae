## Why It Surfaces

This is a pattern-recognition problem disguised as grid construction. The diagonals are determined entirely by the corners of each odd-sized square ring.

## Worked Example

For a `5 x 5` spiral, the diagonal sum is `101`. The outer `5 x 5` ring adds corners `25`, `21`, `17`, and `13`, and each larger odd ring follows the same spacing pattern.

## Reference Implementations

<ExamplePair>

```js
function problem028(dimension = 1001) {
  if (!Number.isInteger(dimension) || dimension < 1) {
    throw new RangeError('Dimension must be a positive integer');
  }

  if (dimension % 2 === 0) {
    throw new RangeError('Dimension must be odd');
  }

  let total = 1;

  for (let side = 3; side <= dimension; side += 2) {
    total += 4 * side * side - 6 * (side - 1);
  }

  return total;
}
```

## Follow-up Questions

- Derive the same result as a single closed-form polynomial in the number of layers.
- Return the four corner values for each ring along with the cumulative sum.
- Explain why even dimensions do not produce the same symmetric center-based spiral setup.

## Common Pitfalls

- Building the full spiral matrix when only the ring corners matter.
- Forgetting that the problem is defined for odd dimensions centered on `1`.
- Double-counting the center value when summing layers.
