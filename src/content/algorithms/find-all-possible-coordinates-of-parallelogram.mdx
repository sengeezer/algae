## Why It Surfaces

Three vertices are not enough to pin down a unique parallelogram. The useful observation is that depending on which pair is treated as adjacent sides or as a diagonal, there are exactly three vector-addition completions for the fourth point.

## Worked Example

For `A = (0, 0)`, `B = (1, 0)`, and `C = (0, 1)`, the three possible fourth vertices are `(1, -1)`, `(-1, 1)`, and `(1, 1)`.

```js
function findAllPossibleCoordinatesOfParallelogram(pointA, pointB, pointC) {
  const points = [pointA, pointB, pointC];
  if (
    !points.every(
      (point) => Array.isArray(point) && point.length === 2 && point.every(Number.isFinite),
    )
  ) {
    throw new RangeError("Each point must be a pair of finite numbers");
  }

  const twiceArea =
    pointA[0] * (pointB[1] - pointC[1])
    + pointB[0] * (pointC[1] - pointA[1])
    + pointC[0] * (pointA[1] - pointB[1]);

  if (twiceArea === 0) {
    throw new RangeError("Points must form a non-degenerate triangle");
  }

  return [
    [pointA[0] + pointB[0] - pointC[0], pointA[1] + pointB[1] - pointC[1]],
    [pointA[0] + pointC[0] - pointB[0], pointA[1] + pointC[1] - pointB[1]],
    [pointB[0] + pointC[0] - pointA[0], pointB[1] + pointC[1] - pointA[1]],
  ];
}
```

## Common Pitfalls

- Returning only one completion and forgetting the diagonal interpretations.
- Reproducing the source article's last y-coordinate typo instead of using `pointB[1] + pointC[1] - pointA[1]`.
- Ignoring the collinear case where the area collapses to zero.
