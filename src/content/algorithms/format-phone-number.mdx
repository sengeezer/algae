## Why It Surfaces

Phone-number formatting questions are really about safe, fixed-shape rendering. The interviewer wants to see whether you validate the raw input before inserting punctuation into a template.

## Worked Example

`"1234567890"` becomes `"(123) 456-7890"` by filling the template from left to right with the ten digits.

## Reference Implementations

<ExamplePair>

```js
function formatPhoneNumber(value) {
  if (typeof value !== 'string' || !/^\d{10}$/.test(value)) {
    throw new TypeError('Value must be exactly 10 digits');
  }

  let index = 0;

  return '(XXX) XXX-XXXX'.replace(/X/g, () => value[index++]);
}
```

```ts
export function formatPhoneNumber(value: string): string {
  if (!/^\d{10}$/.test(value)) {
    throw new TypeError('Value must be exactly 10 digits');
  }

  let index = 0;

  return '(XXX) XXX-XXXX'.replace(/X/g, () => value[index++]);
}
```

</ExamplePair>

## Follow-up Questions

- Accept input containing spaces or punctuation and normalize it first.
- Support optional country codes or extensions.
- Format the digits using `slice` calls instead of a substitution template.

## Common Pitfalls

- Using loose numeric checks that accept scientific notation or whitespace.
- Forgetting to reject inputs that are not exactly 10 digits long.
- Treating this as a full international phone-number parser when it is just a fixed local formatter.
