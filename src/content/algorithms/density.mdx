## Why It Surfaces

Density is the fastest way to summarize whether a graph is mostly empty or close to complete. It matters because many graph choices, from storage layouts to expected traversal behavior, depend on how many edges exist relative to the maximum possible.

## Worked Example

An undirected graph with `5` nodes can have at most `10` edges. If it currently has `4`, the density is `4 / 10 = 0.4`, which means it uses 40% of the possible connections.

## Reference Implementations

<ExamplePair>

```js
function density(numberOfNodes, numberOfEdges, isDirected = false) {
  if (numberOfNodes < 2) {
    return 0;
  }

  const multiplier = isDirected ? 1 : 2;
  return (multiplier * numberOfEdges) / (numberOfNodes * (numberOfNodes - 1));
}
```

```ts
export function density(numberOfNodes: number, numberOfEdges: number, isDirected = false): number {
  if (numberOfNodes < 2) {
    return 0;
  }

  const multiplier = isDirected ? 1 : 2;
  return (multiplier * numberOfEdges) / (numberOfNodes * (numberOfNodes - 1));
}
```

## Follow-up Questions

- Explain why undirected graphs need the factor of `2` in the numerator.
- Compare density for sparse road networks and dense social cliques.
- Use density to justify choosing an adjacency list or adjacency matrix.

## Common Pitfalls

- Dividing by zero when the graph has fewer than two nodes.
- Using the undirected denominator for a directed graph.
- Treating multi-edges or self-loops as if the simple-graph formula still applies unchanged.
