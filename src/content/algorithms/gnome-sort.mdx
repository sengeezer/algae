## Why It Surfaces

Gnome sort is a good teaching algorithm because it exposes local repair directly. Instead of finding an insertion point first, it keeps swapping a misplaced value leftward until the neighboring order is fixed.

## Worked Example

On `[4, 2, 3, 1]`, compare `4` and `2`, swap them, then step back to verify the repaired position. The algorithm keeps doing this until it can safely walk forward again.

## Reference Implementations

<ExamplePair>

```js
function gnomeSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const items = [...values];
  let index = 1;

  while (index < items.length) {
    if (index === 0 || items[index - 1] <= items[index]) {
      index += 1;
    } else {
      [items[index - 1], items[index]] = [items[index], items[index - 1]];
      index -= 1;
    }
  }

  return items;
}
```

## Common Pitfalls

- Forgetting the `index === 0` guard before checking the left neighbor.
- Assuming gnome sort shifts values instead of swapping them.
- Describing it as asymptotically better than insertion sort.
