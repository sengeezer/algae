## Why It Surfaces

Set Bit is one of the smallest bit-manipulation building blocks, but it becomes important once masks represent subsets, permissions, or visited states.

## Worked Example

`10` is `1010` in binary. Setting bit position `2` uses `10 | (1 << 2)`, which becomes `1010 | 0100 = 1110`, or `14`.

## Reference Implementations

<ExamplePair>

```js
function setBit(value, bitPosition) {
  if (!Number.isInteger(value) || !Number.isInteger(bitPosition)) {
    throw new TypeError('Inputs must be integers');
  }

  if (bitPosition < 0 || bitPosition > 30) {
    throw new RangeError('Bit position must be between 0 and 30');
  }

  return value | (1 << bitPosition);
}
```


## Follow-up Questions

- Write the matching clear-bit and toggle-bit operations.
- Adjust the function if the prompt uses one-based bit positions instead of zero-based indexing.
- Explain why OR sets a bit permanently while XOR toggles it.

## Common Pitfalls

- Mixing up zero-based and one-based bit positions.
- Using XOR and unintentionally toggling an already-set bit back to zero.
- Forgetting that JavaScript bitwise operators are limited to 32-bit signed integer behavior.
