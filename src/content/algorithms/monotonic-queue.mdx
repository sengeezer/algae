## Why It Surfaces

Monotonic queue is the upgrade path when plain sliding window is not enough. The interviewer usually wants the maximum or minimum for every window, and recomputing that value from scratch would make the solution too slow.

## JavaScript

```js
function maxSlidingWindow(values, windowSize) {
  const deque = [];
  const result = [];

  for (let index = 0; index < values.length; index += 1) {
    while (deque.length > 0 && deque[0] <= index - windowSize) {
      deque.shift();
    }

    while (deque.length > 0 && values[deque[deque.length - 1]] <= values[index]) {
      deque.pop();
    }

    deque.push(index);

    if (index >= windowSize - 1) {
      result.push(values[deque[0]]);
    }
  }

  return result;
}
```

## Interview Notes

- Store indices, not just values, so you can evict elements that leave the window.
- Be precise about the monotonic property: decreasing deque for window maximum, increasing deque for window minimum.
- If the interviewer suggests a heap, explain why lazy deletion still makes the deque cleaner for fixed windows.
