## Why It Surfaces

Isolating the fractional part is small, but it forces you to define the behavior for negative numbers instead of relying on whatever the language's remainder operator happens to do.

## Worked Example

`decimalIsolate(-12.375)` returns `0.375`. The sign is removed because the goal here is the magnitude of the fractional part, not a signed remainder.

## Reference Implementations

<ExamplePair>

```js
function decimalIsolate(value) {
  if (typeof value !== 'number' || !Number.isFinite(value)) {
    throw new TypeError('Expected a finite numeric input');
  }

  const magnitude = Math.abs(value);
  return magnitude - Math.trunc(magnitude);
}
```

```ts
export function decimalIsolate(value: number): number {
  if (!Number.isFinite(value)) {
    throw new TypeError('Expected a finite numeric input');
  }

  const magnitude = Math.abs(value);
  return magnitude - Math.trunc(magnitude);
}
```

## Common Pitfalls

- Letting negative numbers produce confusing signed fractions by accident.
- Using rounding instead of truncation before subtraction.
- Forgetting that floating-point representation can make some fractions approximate.
