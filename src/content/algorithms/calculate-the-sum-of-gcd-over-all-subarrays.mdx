## Why It Surfaces

The brute-force solution is obvious and unusable. The real trick is noticing that as you extend subarrays ending at position `i`, their GCDs can only stay the same or decrease, so many different subarrays collapse into the same GCD bucket.

## Worked Example

For `[1, 2, 3, 4, 5]`, the sum of GCDs over all subarrays is `25`. For `[2, 2, 2, 3, 5, 5]`, the same total becomes `41`.

function gcd(left, right) {
  let a = Math.abs(left);
  let b = Math.abs(right);

  while (b !== 0) {
    const remainder = a % b;
    a = b;
    b = remainder;
  }

  return a;
}

function calculateTheSumOfGcdOverAllSubarrays(values) {
  if (!Array.isArray(values) || values.length === 0) {
    return 0n;
  }

  let suffixGcdGroups = [];
  let total = 0n;

  for (const value of values) {
    if (!Number.isSafeInteger(value) || value <= 0) {
      throw new RangeError('Values must be positive safe integers');
    }

    const nextGroups = [[value, 1n]];

    for (const [currentGcd, count] of suffixGcdGroups) {
      const nextGcd = gcd(currentGcd, value);
      const lastGroup = nextGroups[nextGroups.length - 1];

      if (lastGroup[0] === nextGcd) {
        lastGroup[1] += count;
      } else {
        nextGroups.push([nextGcd, count]);
      }
    }

    suffixGcdGroups = nextGroups;

    for (const [currentGcd, count] of suffixGcdGroups) {
      total += BigInt(currentGcd) * count;
    }
  }

  return total;
}

## Common Pitfalls

- Enumerating every subarray and recomputing its GCD from scratch.
- Failing to merge adjacent groups that collapse to the same GCD after adding a new value.
- Using `number` for the final sum when the count of subarrays becomes large.
