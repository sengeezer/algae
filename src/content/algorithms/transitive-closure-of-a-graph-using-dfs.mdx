## Why It Surfaces

This problem is a good interview pivot from ordinary DFS because the task is no longer about one traversal order. Instead, you need a full matrix of reachability answers, which makes the repeated-source perspective the key idea.

## Worked Example

If vertex `0` can reach `1`, `1` can reach `2`, and `2` can reach `3`, then the closure row for source `0` should mark every vertex from `0` through `3` as reachable. Running DFS from each start vertex fills one row of the matrix at a time.

## Reference Implementations

<ExamplePair>

```js
function transitiveClosureOfAGraphUsingDFS(graph) {
  const size = graph.length;
  const closure = Array.from({ length: size }, () => new Array(size).fill(false));

  function dfs(source, node) {
    closure[source][node] = true;
    for (const neighbor of graph[node]) {
      if (!closure[source][neighbor]) {
        dfs(source, neighbor);
      }
    }
  }

  for (let source = 0; source < size; source += 1) {
    dfs(source, source);
  }

  return closure.map((row) => row.map((reachable) => (reachable ? 1 : 0)));
}
```

export function transitiveClosureOfAGraphUsingDFS(graph: number[][]): number[][] {
  const size = graph.length;
  const closure = Array.from({ length: size }, () => new Array<boolean>(size).fill(false));

  function dfs(source: number, node: number): void {
    closure[source][node] = true;
    for (const neighbor of graph[node]) {

      }

  }

  for (let source = 0; source < size; source += 1) {
    dfs(source, source);
  }

  return closure.map((row) => row.map((reachable) => (reachable ? 1 : 0)));
}

## Common Pitfalls

- Solving only one source-to-destination query instead of filling the whole matrix.
- Resetting visited state globally instead of per source row.
- Forgetting that dense graphs push the repeated DFS cost toward cubic time.
