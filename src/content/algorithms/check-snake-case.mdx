## Why It Surfaces

Snake-case checks show up in tooling and formatter-style prompts where identifier shape matters. The logic is straightforward, but sloppy patterns tend to accept invalid repeated separators.

## Worked Example

`"user_name"` passes because the segments are lowercase and underscore-separated. `"user__name"` fails because it contains an empty segment between two underscores.

## Reference Implementations

<ExamplePair>

```js
function checkSnakeCase(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  return /^[a-z]+(?:_[a-z]+)*$/.test(value);
}
```

```ts
export function checkSnakeCase(value: string): boolean {
  return /^[a-z]+(?:_[a-z]+)*$/.test(value);
}
```

</ExamplePair>

## Common Pitfalls

- Accepting repeated or trailing underscores.
- Allowing uppercase letters without meaning to.
- Using a loose substring regex instead of an anchored full-string check.
