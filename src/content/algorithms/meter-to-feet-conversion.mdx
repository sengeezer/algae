## Why It Surfaces

This is a compact example of reciprocal conversions. Once one factor is known, the reverse direction follows immediately, which makes it a good sanity-check exercise for measurement utilities.

## Worked Example

`3` meters becomes about `9.84252` feet, and `10` feet becomes `3.048` meters. The same factor `0.3048` connects both directions.

## Reference Implementations

<ExamplePair>

```js
function feetToMeter(feet) {
  if (!Number.isFinite(feet)) {
    throw new TypeError('Feet must be a finite number');
  }

  return feet * 0.3048;
}

function meterToFeet(meter) {
  if (!Number.isFinite(meter)) {
    throw new TypeError('Meters must be a finite number');
  }

  return meter / 0.3048;
}
```

## Common Pitfalls

- Mixing up the direction of the conversion factor.
- Embedding rounded constants that drift across repeated conversions.
- Returning strings too early when downstream code still needs numeric output.
