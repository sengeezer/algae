## Why It Surfaces

Tree diameter is one of the cleanest examples of exploiting tree structure. The farthest node from any starting point lies on a diameter, so a second traversal from that farthest node finds the answer.

## JavaScript

```js
function treeDiameter(adjacency) {
  const [start] = bfs(0, adjacency);
  const [, distance] = bfs(start, adjacency);
  return distance;
}

function bfs(source, adjacency) {
  const queue = [[source, 0]];
  const seen = new Set([source]);
  let farthestNode = source;
  let farthestDistance = 0;

  while (queue.length > 0) {
    const [node, distance] = queue.shift();

    if (distance > farthestDistance) {
      farthestNode = node;
      farthestDistance = distance;
    }

    for (const neighbor of adjacency[node] ?? []) {
      if (!seen.has(neighbor)) {
        seen.add(neighbor);
        queue.push([neighbor, distance + 1]);
      }
    }
  }

  return [farthestNode, farthestDistance];
}
```

## TypeScript

```ts
export function treeDiameter(adjacency: number[][]): number {
  const [start] = bfs(0, adjacency);
  const [, distance] = bfs(start, adjacency);
  return distance;
}

function bfs(source: number, adjacency: number[][]): [number, number] {
  const queue: Array<[number, number]> = [[source, 0]];
  const seen = new Set<number>([source]);
  let front = 0;
  let farthestNode = source;
  let farthestDistance = 0;

  while (front < queue.length) {
    const [node, distance] = queue[front];
    front += 1;

    if (distance > farthestDistance) {
      farthestNode = node;
      farthestDistance = distance;
    }

    for (const neighbor of adjacency[node] ?? []) {
      if (!seen.has(neighbor)) {
        seen.add(neighbor);
        queue.push([neighbor, distance + 1]);
      }
    }
  }

  return [farthestNode, farthestDistance];
}
```

## Interview Notes

- Call out that the two-pass method works because the graph is specifically a tree.
- If edge weights are non-unit, replace BFS with DFS accumulation or Dijkstra depending on the constraints.
- Be clear whether the answer is measured in edges or nodes; interviews use both conventions.
