## Why It Surfaces

Farey approximation is a nice interview-friendly example of iterative refinement. It checks whether you can maintain valid lower and upper fraction bounds while moving toward a target decimal.

## Worked Example

For a target near `0.333`, the successive mediants quickly settle near `1 / 3`, which is the simple rational you would expect.

## Reference Implementations

```js
function fareyApproximation(decimal, repeat = 20) {
  if (typeof decimal !== 'number' || !Number.isFinite(decimal)) {
    throw new TypeError('Decimal value must be a finite number');
  }

  if (!(decimal > 0 && decimal < 1)) {
    throw new RangeError('Decimal value must lie strictly between 0 and 1');
  }

  if (!Number.isInteger(repeat) || repeat <= 0) {
    throw new RangeError('Repeat count must be a positive integer');
  }

  let a = 0;
  let b = 1;
  let c = 1;
  let d = 1;
  let numerator = 0;
  let denominator = 1;

  for (let iteration = 0; iteration < repeat; iteration += 1) {
    numerator = a + c;
    denominator = b + d;

    if (decimal > numerator / denominator) {
      a = numerator;
      b = denominator;
    } else {
      c = numerator;
      d = denominator;
    }
  }

  return { numerator, denominator };
}
```

## Follow-up Questions

- Stop once the approximation error drops below a tolerance instead of using a fixed iteration budget.
- Compare Farey approximation with continued fractions.
- Reduce the returned fraction explicitly and explain why mediants often stay reduced.

## Common Pitfalls

- Allowing targets outside `(0, 1)` without adapting the method.
- Forgetting that the mediant of `a / b` and `c / d` is `(a + c) / (b + d)`.
- Returning the bounds instead of the last mediant.
