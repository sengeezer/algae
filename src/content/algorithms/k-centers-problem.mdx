## Why It Surfaces

K centers shows up when shortest paths and MSTs stop being the right objective. The goal is no longer total cost or pairwise optimality, but the worst-case distance any point must travel to its nearest chosen center.

## Worked Example

Start by choosing an arbitrary first center. Then repeatedly measure each point's distance to its nearest chosen center and add whichever point is currently farthest away; that greedy rule keeps attacking the maximum uncovered radius directly.

## Reference Implementations

<ExamplePair>

```js
function kCentersProblem(distances, centerCount) {
  const nodeCount = distances.length;
  if (nodeCount === 0 || centerCount <= 0) {
    return [];
  }

  const centers = [0];
  const chosen = new Array(nodeCount).fill(false);
  const bestDistance = distances[0].slice();
  chosen[0] = true;
  bestDistance[0] = 0;

  while (centers.length < Math.min(centerCount, nodeCount)) {
    let candidate = -1;
    let farthestDistance = -1;

    for (let node = 0; node < nodeCount; node += 1) {
      if (!chosen[node] && bestDistance[node] > farthestDistance) {
        farthestDistance = bestDistance[node];
        candidate = node;
      }
    }

    if (candidate === -1) {
      break;
    }

    chosen[candidate] = true;
    centers.push(candidate);

    for (let node = 0; node < nodeCount; node += 1) {
      bestDistance[node] = Math.min(bestDistance[node], distances[candidate][node]);
    }
  }

  return centers;
}
```

## Follow-up Questions

## Common Pitfalls

- Optimizing total distance instead of the maximum nearest-center distance.
- Using the greedy rule on non-metric data and still claiming the same guarantee.
- Forgetting to update each point's nearest-center distance after adding a new center.
