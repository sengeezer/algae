## Why It Surfaces

Most graph interview questions analyze a graph that is already given. Degree-sequence construction flips that around and asks whether the constraints themselves can be realized as a simple graph, which is why Havel-Hakimi style greedy reasoning is the natural fit.

## Worked Example

If the current largest remaining degree is `3`, that vertex must connect to three other vertices with positive remaining degree. After making those edges, all four affected degrees drop, the sequence is resorted, and any negative value or shortage of available partners proves the construction is impossible.

## Reference Implementations

<ExamplePair>

```js
function constructGraphFromDegreeSequence(degrees) {
  const remaining = degrees.map((degree, vertex) => ({ degree, vertex }));
  const edges = [];

  while (true) {
    remaining.sort((left, right) => right.degree - left.degree || left.vertex - right.vertex);

    if (remaining[0].degree === 0) {
      return edges;
    }

    const current = remaining.shift();
    if (current.degree < 0 || current.degree > remaining.length) {
      return [];
    }

    for (let index = 0; index < current.degree; index += 1) {
      if (remaining[index].degree === 0) {
        return [];
      }

      remaining[index].degree -= 1;
      if (remaining[index].degree < 0) {
        return [];
      }

      edges.push([current.vertex, remaining[index].vertex]);
    }
  }
}
```

export function constructGraphFromDegreeSequence(degrees: number[]): Array<[number, number]> {
  const remaining = degrees.map((degree, vertex) => ({ degree, vertex }));
  const edges: Array<[number, number]> = [];

  while (true) {
    remaining.sort((left, right) => right.degree - left.degree || left.vertex - right.vertex);

    if (remaining[0].degree === 0) {
      return edges;
    }

    const current = remaining.shift()!;
    if (current.degree < 0 || current.degree > remaining.length) {
      return [];
    }

    for (let index = 0; index < current.degree; index += 1) {
      if (remaining[index].degree === 0) {
        return [];
      }

      remaining[index].degree -= 1;
      if (remaining[index].degree < 0) {
        return [];
      }

      edges.push([current.vertex, remaining[index].vertex]);
    }
  }
}

## Follow-up Questions

- Return an explicit failure reason when the sequence is not graphical.
- Compare this construction with the purely decision version of Havel-Hakimi.
- Explain why self-loops and parallel edges are excluded in this formulation.

## Common Pitfalls

- Connecting a vertex to itself or reusing the same neighbor twice in one reduction step.
- Forgetting to resort after each degree update.
- Treating any degree sequence as realizable without checking for impossible remainders.
