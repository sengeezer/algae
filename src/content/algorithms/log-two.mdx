## Why It Surfaces

Log Two is less about logarithms and more about reading binary structure. Interviewers use it when they want the highest set bit without leaning on floating-point helpers.

## Worked Example

For `20`, the binary form is `10100`. Shift right repeatedly: `20 -> 10 -> 5 -> 2 -> 1`. That takes four shifts, so `floor(log2(20)) = 4`.

## Reference Implementations

<ExamplePair>

```js
function logTwo(value) {
  if (!Number.isInteger(value) || value < 1) {
    throw new RangeError('Value must be a positive integer');
  }

  let result = 0;
  let remaining = value;

  while (remaining >> 1) {
    remaining >>= 1;
    result += 1;
  }

  return result;
}
```

```ts

  return result;
```

## Follow-up Questions

- Replace the loop with `Math.clz32` and explain the tradeoff.
- Adapt the idea for `bigint` inputs where 32-bit bitwise operators are insufficient.
- Contrast `floor(log2(n))` with the exact floating-point `Math.log2(n)`.

## Common Pitfalls

- Passing `0` or negative values even though the integer binary logarithm is defined here only for positive inputs.
- Forgetting that JavaScript bitwise operators truncate to 32-bit integers.
- Mixing up the highest set-bit position with the number of set bits.
