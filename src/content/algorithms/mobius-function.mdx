## Why It Surfaces

The Mobius function is a compact number-theory routine that forces you to separate square-free detection from the parity of distinct prime factors.

## Worked Example

`30 = 2 * 3 * 5` is square-free with `3` distinct prime factors, so `mu(30) = -1`. `12 = 2 * 2 * 3` has a squared prime factor, so `mu(12) = 0`. `1` is the empty-factorization base case, so `mu(1) = 1`.

## Reference Implementations

<ExamplePair>

```js
function mobiusFunction(value) {
  if (!Number.isInteger(value) || value <= 0) {
    throw new RangeError('Value must be a positive integer');
  }

  if (value === 1) {
    return 1;
  }

  let remaining = value;
  let distinctPrimeCount = 0;

  for (let divisor = 2; divisor * divisor <= remaining; divisor += 1) {
    let occurrences = 0;

    while (remaining % divisor === 0) {
      remaining /= divisor;
      occurrences += 1;
    }

    if (occurrences > 1) {
      return 0;
    }

    if (occurrences === 1) {
      distinctPrimeCount += 1;
    }
  }

  if (remaining > 1) {
    distinctPrimeCount += 1;
  }

  return distinctPrimeCount % 2 === 0 ? 1 : -1;
}
```

## Follow-up Questions

- Compare `mu(n)` with the Liouville function on square-free and non-square-free inputs.
- Precompute values for all `n <= limit` with a sieve-based approach.
- Explain why `mu(n) = 0` as soon as any prime factor repeats.

## Common Pitfalls

- Counting repeated prime factors as if they were only distinct.
- Forgetting that `mu(1)` is `1`.
- Continuing unnecessary factor scans after you already found a squared prime factor.
