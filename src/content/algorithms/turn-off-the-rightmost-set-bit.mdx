## Why It Surfaces

This is one of the core bit-manipulation identities interviewers expect you to recognize on sight. The same update powers faster set-bit counting, subset iteration, and power-of-two checks, so it is more useful than the small problem statement suggests.

## Worked Example

`12` is `1100` in binary. Subtracting `1` gives `1011`, and `1100 & 1011 = 1000`, which is `8`, so the rightmost set bit has been cleared.

## Reference Implementations

<ExamplePair>

```js
function turnOffTheRightmostSetBit(value) {
  if (!Number.isInteger(value) || value < 0) {
    throw new TypeError('Value must be a non-negative integer');
  }

  return value & (value - 1);
}
```

```ts
export function turnOffTheRightmostSetBit(value: number): number {
  if (!Number.isInteger(value) || value < 0) {
    throw new TypeError('Value must be a non-negative integer');
  }

  return value & (value - 1);
}
```

## Follow-up Questions

- Explain why the same trick makes `value === 0 || (value & (value - 1)) === 0` a power-of-two test.
- Use repeated lowest-bit clearing to count set bits in `O(k)` time.
- Compare this constant-time trick with a loop that searches for the first `1` bit manually.

## Common Pitfalls

- Forgetting that `0` should stay `0` because there is no set bit to clear.
- Confusing this clear-bit trick with XOR-based toggling.
- Ignoring that JavaScript bitwise operators coerce numbers to 32-bit signed integers.
