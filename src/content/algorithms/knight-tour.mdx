## Why It Surfaces

Knight Tour is a pure search-discipline problem. Every move must satisfy the knight constraint, every square may be visited once, and every wrong branch must be rolled back cleanly before the next move is tried.

## Worked Example

Starting at the top-left square, mark it as step `0`, then recursively try each legal knight move as step `1`. If no sequence completes the full board, undo the last move and try the next candidate.

```js
function knightTour(size) {
  const board = Array.from({ length: size }, () => new Array(size).fill(-1));
  const moves = [
    [2, 1],
    [1, 2],
    [-1, 2],
    [-2, 1],
    [-2, -1],
    [-1, -2],
    [1, -2],
    [2, -1],
  ];

  function dfs(row, col, step) {
    board[row][col] = step;

    if (step === size * size - 1) {
      return true;
    }

    for (const [rowOffset, colOffset] of moves) {
      const nextRow = row + rowOffset;
      const nextCol = col + colOffset;

      if (
        nextRow < 0
        || nextRow >= size
        || nextCol < 0
        || nextCol >= size
        || board[nextRow][nextCol] !== -1
      ) {
        continue;
      }

      if (dfs(nextRow, nextCol, step + 1)) {
        return true;
      }
    }

    board[row][col] = -1;
    return false;
  }

  return dfs(0, 0, 0) ? board : [];
}
```

## Common Pitfalls

- Forgetting to unmark a cell when a branch fails.
- Confusing legal knight moves with adjacent grid moves.
- Expecting brute force to scale without heuristic help.
