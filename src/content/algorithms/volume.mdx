## Why It Surfaces

Volume helpers are direct, but they still test whether you match the right formula to the right solid and whether you reject impossible negative dimensions instead of silently computing nonsense.

## Worked Example

A cylinder with radius `3` and height `5` has volume `pi * 3^2 * 5 = 45pi`. A sphere with radius `2` has volume `4 / 3 * pi * 2^3 = 32pi / 3`.

function validatePositive(value, label) {
  if (typeof value !== 'number' || !Number.isFinite(value) || value < 0) {
    throw new RangeError(`${label} must be a non-negative number`);
  }
}

function volumeCuboid(width, length, height) {
  validatePositive(width, 'Width');
  validatePositive(length, 'Length');
  validatePositive(height, 'Height');
  return width * length * height;
}

function volumeCube(length) {
  validatePositive(length, 'Length');
  return length ** 3;
}

function volumeCone(radius, height) {

  validatePositive(height, 'Height');
  return (Math.PI * radius ** 2 * height) / 3;
}

  validatePositive(radius, 'Radius');
  validatePositive(height, 'Height');
  return Math.PI * radius ** 2 * height;
}

  return (4 * Math.PI * radius ** 3) / 3;
}

## Follow-up Questions

- Add pyramid, triangular prism, and hemisphere helpers to the same module.
- Compare how area formulas grow into volume formulas when a third dimension is introduced.
- Explain why negative radii and heights should be rejected instead of absolutized.

## Common Pitfalls

- Mixing surface area formulas with volume formulas.
- Forgetting the `1 / 3` factor for cones and pyramids.
- Allowing negative or non-finite dimensions to flow into geometry formulas.
