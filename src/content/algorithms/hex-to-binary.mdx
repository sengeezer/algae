## Why It Surfaces

Hex-to-binary is the inverse of binary-to-hex and uses the same alignment fact in reverse: one hex digit always maps to one nibble. That makes the algorithm a direct per-character transform instead of a whole-number conversion.

## Worked Example

`4D` maps to `0100` and `1101`, so the full binary output is `01001101`. There is no carry or division step because every symbol expands independently.

```js
const hexToBits = {
  '0': '0000',
  '1': '0001',
  '2': '0010',
  '3': '0011',
  '4': '0100',
  '5': '0101',
  '6': '0110',
  '7': '0111',
  '8': '1000',
  '9': '1001',
  A: '1010',
  B: '1011',
  C: '1100',
  D: '1101',
  E: '1110',
  F: '1111',
};

function hexToBinary(hexString) {
  if (!/^[0-9A-F]+$/i.test(hexString)) {
    throw new TypeError('Hex strings may only contain digits 0-9 and letters A-F');
  }

  return hexString
    .toUpperCase()
    .split('')
    .map((digit) => hexToBits[digit])
    .join('');
}
```

## Follow-up Questions

- Explain why hex digits and four-bit groups line up perfectly.
- Trim leading zeros only when the surrounding format allows it.
- Generalize the idea to octal-to-binary by regrouping in threes.

## Common Pitfalls

- Forgetting to normalize lowercase hex letters.
- Accepting invalid characters outside the hex alphabet.
- Stripping leading zeros when the caller expects fixed-width nibbles.
