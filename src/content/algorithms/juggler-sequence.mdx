## Why It Surfaces

Juggler sequence prompts are small recurrence exercises that still test whether you can track parity correctly and build the full sequence rather than only the next term.

## Worked Example

Starting from `3`, the sequence is `3, 5, 11, 36, 6, 2, 1`.

## Reference Implementations

<ExamplePair>

```js
function jugglerSequence(value) {
  if (!Number.isInteger(value) || value <= 0) {
    throw new RangeError('Value must be a positive integer');
  }

  const sequence = [value];
  let current = value;

  while (current !== 1) {
    current = current % 2 === 0 ? Math.floor(Math.sqrt(current)) : Math.floor(current ** 1.5);
    sequence.push(current);
  }

  return sequence;
}
```

```ts
export function jugglerSequence(value: number): number[] {
  if (!Number.isInteger(value) || value <= 0) {
    throw new RangeError('Value must be a positive integer');
  }

  const sequence: number[] = [value];
  let current = value;

  while (current !== 1) {
    current = current % 2 === 0 ? Math.floor(Math.sqrt(current)) : Math.floor(current ** 1.5);
    sequence.push(current);
  }

  return sequence;
}
```

## Follow-up Questions

- Track the peak value reached before the sequence hits `1`.
- Compare the juggler sequence with the Collatz sequence.
- Stream terms to a callback instead of storing the whole array.

## Common Pitfalls

- Forgetting to include the starting value in the returned sequence.
- Mixing up the even and odd update rules.
- Using fractional powers without flooring back to an integer term.
