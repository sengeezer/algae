## Why It Surfaces

Pancake sort is a constrained-operation problem masquerading as a sorting problem. The challenge is not discovering the maximum element, but expressing the sort entirely with prefix reversals.

## Worked Example

For `[3, 6, 1, 5, 2, 4]`, first flip the prefix ending at `6` to bring the maximum to the front, then flip the full active prefix to move `6` to the end. Repeat on the shortened prefix.

## Reference Implementations

<ExamplePair>

```js
function pancakeSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const items = [...values];

  function flip(end) {
    for (let left = 0, right = end; left < right; left += 1, right -= 1) {
      [items[left], items[right]] = [items[right], items[left]];
    }
  }

  for (let size = items.length; size > 1; size -= 1) {
    let maxIndex = 0;

    for (let index = 1; index < size; index += 1) {
      if (items[index] > items[maxIndex]) {
        maxIndex = index;
      }
    }

    if (maxIndex === size - 1) {
      continue;
    }

    if (maxIndex > 0) {
      flip(maxIndex);
    }

    flip(size - 1);
  }

  return items;
}
```

## Common Pitfalls

- Flipping even when the maximum is already at the end of the active prefix.
- Forgetting the two-flip pattern needed when the maximum is not already in front.
- Using full-array reversals when only a prefix reversal is allowed.
