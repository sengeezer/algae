## Why It Surfaces

Eulerian classification tells you whether a traversal is possible, but Hierholzer is what you use when the interviewer wants the traversal itself. The core move is to keep following unused edges until you get stuck, then backtrack and splice those local cycles into one global route.

## Worked Example

Suppose a directed graph has an Eulerian circuit starting at `0`. Hierholzer follows one outgoing edge after another until it reaches a vertex with no remaining unused edge, adds that vertex to the answer, and backtracks through the stack until every consumed edge has been placed into the final route.

## Reference Implementations

<ExamplePair>

```js
function hierholzersAlgorithmDirectedGraph(graph, start = 0) {
  const remaining = graph.map((neighbors) => [...neighbors].reverse());
  const stack = [start];
  const route = [];

  while (stack.length > 0) {
    const node = stack[stack.length - 1];

    if (remaining[node].length === 0) {
      route.push(stack.pop());
      continue;
    }

    stack.push(remaining[node].pop());
  }

  return route.reverse();
}
```

```ts
export function hierholzersAlgorithmDirectedGraph(
  graph: number[][],
  start = 0,
): number[] {
  const remaining = graph.map((neighbors) => [...neighbors].reverse());
  const stack: number[] = [start];
  const route: number[] = [];

  while (stack.length > 0) {
    const node = stack[stack.length - 1];

    if (remaining[node].length === 0) {
      route.push(stack.pop()!);
      continue;
    }

    stack.push(remaining[node].pop()!);
  }

  return route.reverse();
}
```

## Common Pitfalls

- Running the construction before checking that the directed graph is Eulerian.
- Forgetting that edges, not vertices, are consumed exactly once.
- Mutating the original adjacency lists when the caller still needs them later.
