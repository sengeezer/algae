## Why It Surfaces

This is the natural next step after the one-singleton XOR trick. XORing the whole array no longer returns a complete answer, but it does reveal one bit where the two unique values differ, which is enough to split the input into two independently canceling groups.

## Worked Example

For `[1, 2, 3, 2, 1, 4]`, the duplicated values cancel under XOR, leaving `3 ^ 4`. The rightmost set bit of that XOR separates `3` and `4` into different groups, and XOR within each group returns the two answers.

```js
function findTwoNonRepeatingElementsInAnArrayOfRepeatingElements(values) {
  if (!Array.isArray(values) || values.length < 2) {
    throw new RangeError('Values must be an array with at least two elements');
  }

  if (!values.every((value) => Number.isInteger(value) && value >= 0 && value <= 0xffffffff)) {
    throw new RangeError('Values must contain unsigned 32-bit integers');
  }

  let xorValue = 0;

  for (const value of values) {
    xorValue ^= value >>> 0;
  }

  const separatingBit = xorValue & -xorValue;
  let first = 0;
  let second = 0;

  for (const value of values) {
    if (((value >>> 0) & separatingBit) === 0) {
      first ^= value >>> 0;
    } else {
      second ^= value >>> 0;
    }
  }

  return first < second ? [first, second] : [second, first];
}
```

## Common Pitfalls

- Reusing the single-singleton XOR trick without the second partitioning pass.
- Forgetting to sort or normalize the final pair when the prompt requires a stable output order.
- Applying this method when repeated numbers do not appear exactly twice.
