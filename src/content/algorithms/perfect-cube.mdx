## Why It Surfaces

Perfect-cube checks usually appear as a follow-up once a perfect-square helper is already on the table. The real question is whether you can make the numerical root check precise enough for ordinary inputs.

## Worked Example

`27` is a perfect cube because `Math.cbrt(27)` rounds to `3`, and `3 ** 3 = 27`. `-8` is also a perfect cube because `(-2) ** 3 = -8`. `26` is not, because the nearest integer cube root squares back to `27`, not `26`.

## Reference Implementations

<ExamplePair>

```js
function perfectCube(value) {
  return Number.isFinite(value) && Math.round(Math.cbrt(value)) ** 3 === value;
}
```

```ts
export function perfectCube(value: number): boolean {
  return Number.isFinite(value) && Math.round(Math.cbrt(value)) ** 3 === value;
}
```

</ExamplePair>

## Follow-up Questions

- Compare this check with an integer binary search over possible roots.
- Explain why negative values can still be perfect cubes.
- Generalize the pattern to perfect fourth powers or arbitrary integer powers.

## Common Pitfalls

- Rejecting negative perfect cubes even though odd powers preserve sign.
- Using truncation instead of rounding and misclassifying values near floating-point boundaries.
- Assuming the same floating-point check stays exact for arbitrarily large integers.
