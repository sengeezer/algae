## Why It Surfaces

Armstrong numbers are a standard digit-manipulation check. The main interview question is whether you can combine digit counting and per-digit exponentiation without getting lost in off-by-one behavior.

## Worked Example

`370` is an Armstrong number because it has `3` digits and $3^3 + 7^3 + 0^3 = 370$.

## Reference Implementations

<ExamplePair>

```js
function armstrongNumber(value) {
  if (!Number.isInteger(value) || value < 0) {
    return false;
  }

  if (value === 0) {
    return true;
  }

  const digits = [];
  let remaining = value;

  while (remaining > 0) {
    digits.push(remaining % 10);
    remaining = Math.floor(remaining / 10);
  }

  const power = digits.length;
  let total = 0;

  for (const digit of digits) {
    total += digit ** power;
  }

  return total === value;
}
```

## Follow-up Questions

- Generalize the check to other bases besides decimal.
- Cache single-digit powers if you need to test many nearby numbers.
- Compare an arithmetic digit loop with a string-based implementation.

## Common Pitfalls

- Forgetting that `0` is an Armstrong number.
- Using the wrong exponent instead of the digit count.
- Accepting fractional inputs even though the predicate is defined here for integers.
