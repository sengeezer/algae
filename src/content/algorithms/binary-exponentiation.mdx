## Why It Surfaces

Binary exponentiation is a classic interview optimization because it replaces `n` multiplications with repeated squaring. Once the exponent is large enough, anything linear is obviously too slow.

## Worked Example

To compute `3^13`, inspect the exponent in binary. Multiply the answer by the current base when the current bit is `1`, square the base every step, and shift the exponent right until it reaches zero.

## Reference Implementations

<ExamplePair>

```js
function binaryExponentiation(base, exponent) {
  let value = base;
  let power = exponent;
  let answer = 1;

  while (power > 0) {
    if (power % 2 === 1) {
      answer *= value;
    }

    value *= value;
    power = Math.floor(power / 2);
  }

  return answer;
}
```

```ts
export function binaryExponentiation(base: number, exponent: number): number {
  let value = base;
  let power = exponent;
  let answer = 1;

  while (power > 0) {
    if (power % 2 === 1) {
      answer *= value;
    }

    value *= value;
    power = Math.floor(power / 2);
  }

  return answer;
}
```

## Follow-up Questions

- Add a modulus to avoid overflow and support cryptography-style prompts.
- Compare the iterative and recursive formulations.
- Use the same idea to exponentiate matrices.

## Common Pitfalls

- Falling back to `O(n)` repeated multiplication.
- Forgetting to multiply the answer when the current exponent bit is odd.
- Mishandling the `exponent = 0` base case.
