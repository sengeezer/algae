## Why It Surfaces

Signum is tiny, but it exposes whether you treat zero carefully and whether you separate classification from magnitude. That pattern comes up in larger algorithms all the time.

## Worked Example

`signum(-8)` returns `-1`, `signum(0)` returns `0`, and `signum(19)` returns `1`. The numeric size does not matter once the sign has been classified.

## Reference Implementations

<ExamplePair>

```js
function signum(value) {
  if (typeof value !== 'number') {
    throw new TypeError('Expected a numeric input');
  }

  if (Number.isNaN(value)) {
    return NaN;
  }

  if (value === 0) {
    return 0;
  }

  return value > 0 ? 1 : -1;
}
```

```ts
export function signum(value: number): -1 | 0 | 1 | number {
  if (Number.isNaN(value)) {
    return NaN;
  }

  if (value === 0) {
    return 0;
  }

  return value > 0 ? 1 : -1;
}
```

</ExamplePair>

## Follow-up Questions

- Compare this helper with `Math.sign`, especially around `NaN` and `-0`.
- Return a string label like `negative`, `zero`, or `positive` instead of numeric codes.
- Use signum inside a comparator or normalization routine.

## Common Pitfalls

- Forgetting that `0` needs its own branch.
- Treating `NaN` as positive or negative instead of propagating it.
- Returning the input value instead of the normalized sign class.
