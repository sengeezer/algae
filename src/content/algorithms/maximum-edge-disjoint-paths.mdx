## Why It Surfaces

Maximum edge-disjoint paths is one of the clearest examples of reduction to flow. The graph question sounds different from a capacity problem at first, but giving every edge capacity one makes each unit of flow correspond to one path that cannot reuse an already claimed edge.

## Worked Example

If two source-to-target routes must not share any edge, assign capacity `1` to each edge and compute max flow. Every augmenting path that succeeds consumes one unit of capacity along its edges, preventing later paths from using them again.

## Reference Implementations

<ExamplePair>

```js
function maximumEdgeDisjointPaths(capacity, source, sink) {
  const size = capacity.length;
  const residual = capacity.map((row) => [...row]);
  const parent = new Array(size).fill(-1);
  let paths = 0;

  function bfs() {
    parent.fill(-1);
    parent[source] = source;
    const queue = [source];

    for (let head = 0; head < queue.length; head += 1) {
      const node = queue[head];

      for (let neighbor = 0; neighbor < size; neighbor += 1) {
        if (parent[neighbor] === -1 && residual[node][neighbor] > 0) {
          parent[neighbor] = node;
          queue.push(neighbor);
        }
      }
    }

    return parent[sink] !== -1;
  }

  while (bfs()) {
    for (let node = sink; node !== source; node = parent[node]) {
      residual[parent[node]][node] -= 1;
      residual[node][parent[node]] += 1;
    }

    paths += 1;
  }

  return paths;
}
```

## Follow-up Questions

- Reconstruct the actual set of disjoint paths, not just the count.
- Compare edge-disjoint paths with vertex-disjoint paths.
- Explain why unit capacities are the correct reduction here.

## Common Pitfalls

- Using arbitrary capacities when the goal is specifically edge-disjoint path counting.
- Forgetting to add reverse residual capacity after each augmentation.
- Confusing edge-disjointness with simply choosing shortest distinct paths.
