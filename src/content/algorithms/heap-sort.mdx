## Why It Surfaces

Heap sort is the sorting algorithm that asks you to explain the heap property clearly. Once the array is heapified, the largest value sits at the root, so each extraction places one final element at the end of the array.

## JavaScript

```js
function heapSort(values) {
  const nums = [...values];

  function siftDown(size, index) {
    let candidate = index;

    while (true) {
      const left = candidate * 2 + 1;
      const right = candidate * 2 + 2;
      let largest = candidate;

      if (left < size && nums[left] > nums[largest]) {
        largest = left;
      }

      if (right < size && nums[right] > nums[largest]) {
        largest = right;
      }

      if (largest === candidate) {
        return;
      }

      [nums[candidate], nums[largest]] = [nums[largest], nums[candidate]];
      candidate = largest;
    }
  }

  for (let index = Math.floor(nums.length / 2) - 1; index >= 0; index -= 1) {
    siftDown(nums.length, index);
  }

  for (let end = nums.length - 1; end > 0; end -= 1) {
    [nums[0], nums[end]] = [nums[end], nums[0]];
    siftDown(end, 0);
  }

  return nums;
}
```

## Interview Notes

- Separate the explanation into heap construction and repeated extraction.
- Mention that heap sort is in-place but not stable.
- If the interviewer asks why build-heap is O(n), focus on how most nodes are near the bottom and sift down only short distances.
