## Why It Surfaces

Insertion sort matters because it is both easy to reason about and genuinely useful in hybrid algorithms. The main invariant is simple: everything to the left of the current index is already sorted.

## Worked Example

For `[5, 2, 4, 6, 1, 3]`, when the algorithm reaches `4`, it shifts `5` one position right and inserts `4` between `2` and `5`. That same pattern repeats for each new element.

## Reference Implementations

<ExamplePair>

```js
function insertionSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const items = [...values];

  for (let index = 1; index < items.length; index += 1) {
    const current = items[index];
    let position = index - 1;

    while (position >= 0 && items[position] > current) {
      items[position + 1] = items[position];
      position -= 1;
    }

    items[position + 1] = current;
  }

  return items;
}
```

## Common Pitfalls

- Swapping repeatedly instead of shifting and inserting once.
- Losing the current value before shifting larger elements.
- Missing the fact that nearly sorted inputs make the best case linear.
