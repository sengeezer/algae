## Why It Surfaces

Coordinate-distance helpers are small, but they are a clean way to test whether you can translate geometry formulas into code and keep different metrics straight.

## Worked Example

For points `(2, 2)` and `(14, 11)`, the Euclidean distance is `15` because the coordinate differences are `12` and `9`, and `12^2 + 9^2 = 225`.

## Reference Implementations

<ExamplePair>

```js
function euclideanCoordinateDistance(x1, y1, x2, y2) {
  if (![x1, y1, x2, y2].every((value) => typeof value === 'number' && Number.isFinite(value))) {
    throw new TypeError('Coordinates must be finite numbers');
  }

  const width = x2 - x1;
  const height = y2 - y1;
  return Math.sqrt(width * width + height * height);
}

function manhattanCoordinateDistance(x1, y1, x2, y2) {
  if (![x1, y1, x2, y2].every((value) => typeof value === 'number' && Number.isFinite(value))) {
    throw new TypeError('Coordinates must be finite numbers');
  }

  return Math.abs(x2 - x1) + Math.abs(y2 - y1);
}
```

## Follow-up Questions

- Generalize the helpers to vectors of any dimension.
- Explain where Manhattan distance is a better model than Euclidean distance.
- Add Chebyshev distance as a third metric.

## Common Pitfalls

- Mixing Euclidean and Manhattan formulas in the same helper.
- Forgetting to validate that all coordinates are numeric.
- Squaring after taking the square root instead of before it.
