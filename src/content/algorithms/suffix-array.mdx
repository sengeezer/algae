## Why It Surfaces

Suffix arrays appear when the same string will be queried repeatedly. The key idea is that once all suffixes are sorted, substring queries can work against that sorted order instead of scanning the full text from scratch each time.

## Worked Example

For `banana`, the sorted suffixes are `a`, `ana`, `anana`, `banana`, `na`, and `nana`, which correspond to starting indices `[5, 3, 1, 0, 4, 2]`. That ordered index list is the suffix array.

## Reference Implementations

<ExamplePair>

```js
function buildSuffixArray(text) {
  return Array.from({ length: text.length }, (_, index) => index)
    .sort((left, right) => text.slice(left).localeCompare(text.slice(right)));
}

function containsPattern(text, pattern) {
  const suffixArray = buildSuffixArray(text);
  let left = 0;
  let right = suffixArray.length - 1;

  while (left <= right) {
    const mid = Math.floor((left + right) / 2);
    const suffix = text.slice(suffixArray[mid]);

    if (suffix.startsWith(pattern)) {
      return true;
    }

    if (suffix < pattern) {
      left = mid + 1;
    } else {
      right = mid - 1;
    }
  }

  return false;
}
```

## Common Pitfalls

- Claiming the naive suffix-sort build is linear or `O(n log n)`.
- Storing full copied suffix strings when index-based comparisons would be enough.
- Forgetting that binary search compares against suffix order, not original positions.
