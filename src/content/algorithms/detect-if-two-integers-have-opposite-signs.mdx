## Why It Surfaces

This is a compact sign-bit question dressed up as a comparison problem. Interviewers use it to check whether you understand that XOR highlights exactly the bit positions where two integers differ, including the high sign bit.

## Worked Example

`-5` and `10` have opposite signs, so the function returns `true`. By contrast, `7` and `15` are both non-negative, so the function returns `false`.

## Reference Implementations

<ExamplePair>

```js
function detectIfTwoIntegersHaveOppositeSigns(left, right) {
  if (!Number.isInteger(left) || !Number.isInteger(right)) {
    throw new TypeError('Inputs must be integers');
  }

  return (left ^ right) < 0;
}
```

```ts
export function detectIfTwoIntegersHaveOppositeSigns(left: number, right: number): boolean {
  if (!Number.isInteger(left) || !Number.isInteger(right)) {
    throw new TypeError('Inputs must be integers');
  }

  return (left ^ right) < 0;
}
```

## Follow-up Questions

- Compare the XOR test with a direct relational check such as `left < 0 ? right >= 0 : right < 0`.
- Explain how zero is treated in this signed-versus-non-negative framing.
- Show how the same sign-bit idea appears in overflow detection and signed comparisons.

## Common Pitfalls

- Forgetting that JavaScript bitwise operators coerce values to signed 32-bit integers.
- Assuming the prompt always excludes zero even though many formulations treat negative versus non-negative as opposite.
- Replacing the XOR test with multiplication and risking overflow in fixed-width languages.
