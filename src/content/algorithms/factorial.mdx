## Why It Surfaces

Factorial is one of the first counting helpers interviewers reach for because it is simple on the surface but ties directly into permutations, combinations, and overflow awareness.

## Worked Example

For `5`, multiply `1 * 2 * 3 * 4 * 5` to get `120`. For `0`, the result is `1` by definition.

## Reference Implementations

<ExamplePair>

```js
function factorial(value) {
  if (!Number.isInteger(value) || value < 0) {
    throw new RangeError('Value must be a non-negative integer');
  }

  let result = 1;

  for (let factor = 2; factor <= value; factor += 1) {
    result *= factor;
  }

  return result;
}
```

```ts
export function factorial(value: number): number {
  if (!Number.isInteger(value) || value < 0) {
    throw new RangeError('Value must be a non-negative integer');
  }

  let result = 1;

  for (let factor = 2; factor <= value; factor += 1) {
    result *= factor;
  }

  return result;
}
```

## Follow-up Questions

- Rework the function with `bigint` once ordinary numbers overflow.
- Count trailing zeroes in `n!` without computing the full factorial.
- Compare iterative and recursive versions in terms of stack usage and clarity.

## Common Pitfalls

- Forgetting that `0! = 1`.
- Accepting negative inputs even though factorial is defined here only for non-negative integers.
- Ignoring how quickly the result outgrows safe numeric ranges.
