## Why It Surfaces

This is the geometry version of interval overlap. Instead of trying to compute the intersection region directly, you write down the two ways overlap can fail in an axis-aligned plane and negate them.

## Worked Example

With `l1 = [0, 10]`, `r1 = [10, 0]`, `l2 = [5, 5]`, and `r2 = [15, 0]`, the rectangles overlap because neither rectangle is strictly to the left of or above the other.

## Reference Implementations

<ExamplePair>

```js
function findIfTwoRectanglesOverlap(l1, r1, l2, r2) {
  if (l1[0] > r2[0] || l2[0] > r1[0]) {
    return false;
  }

  if (r1[1] > l2[1] || r2[1] > l1[1]) {
    return false;
  }

  return true;
}
```

```ts
type Point = [number, number];

export function findIfTwoRectanglesOverlap(
  l1: Point,
  r1: Point,
  l2: Point,
  r2: Point,
): boolean {
  if (l1[0] > r2[0] || l2[0] > r1[0]) {
    return false;
  }

  if (r1[1] > l2[1] || r2[1] > l1[1]) {
    return false;
  }

  return true;
}
```

## Common Pitfalls

- Forgetting that this shortcut assumes axis-aligned rectangles.
- Using `>=` instead of `>` when the problem treats touching edges as overlap.
- Swapping top-left and bottom-right conventions without noticing the sign change.
