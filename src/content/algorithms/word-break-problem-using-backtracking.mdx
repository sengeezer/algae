## Why It Surfaces

This is the natural follow-up once the standard Word Break feasibility question is understood. Instead of stopping as soon as one segmentation works, you keep exploring all valid prefixes and emit every full sentence that reaches the end of the string.

## Worked Example

For `s = "catsanddog"` and dictionary `{"cat", "cats", "and", "sand", "dog"}`, the valid answers are `"cat sand dog"` and `"cats and dog"`. Both survive because each recursive level keeps trying longer prefixes after a shorter valid one succeeds.

## Reference Implementations

<ExamplePair>

```js
function wordBreakProblemUsingBacktracking(text, words) {
  const dictionary = new Set(words);
  const sentence = [];
  const result = [];

  function search(start) {
    if (start === text.length) {
      result.push(sentence.join(" "));
      return;
    }

    for (let end = start + 1; end <= text.length; end += 1) {
      const word = text.slice(start, end);

      if (!dictionary.has(word)) {
        continue;
      }

      sentence.push(word);
      search(end);
      sentence.pop();
    }
  }

  search(0);
  return result;
}
```

## Common Pitfalls

- Reusing the boolean DP intuition and stopping after the first valid sentence.
- Forgetting to pop the last word after each recursive call.
- Using linear dictionary scans instead of constant-time membership checks.
