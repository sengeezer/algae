## Why It Surfaces

Oddness checks are the mirror image of evenness checks, but they still matter because many recurrences and number properties split precisely on parity.

## Worked Example

`17` is odd because `17 % 2` is `1`. `14` is not odd because its remainder modulo `2` is `0`.

## Reference Implementations

<ExamplePair>

```js
function isOdd(value) {
  return Number.isInteger(value) && Math.abs(value % 2) === 1;
}

function isOddBitwise(value) {
  return Number.isInteger(value) && Boolean(value & 1);
}
```

```ts
export function isOdd(value: number): boolean {
  return Number.isInteger(value) && Math.abs(value % 2) === 1;
}

export function isOddBitwise(value: number): boolean {
  return Number.isInteger(value) && Boolean(value & 1);
}
```

## Follow-up Questions

- Compare how modulo behaves for negative values in JavaScript.
- Explain why the bitwise form still works for signed integers.
- Use oddness to simplify a recurrence like Collatz or fast exponentiation.

## Common Pitfalls

- Assuming `% 2 === 1` works for negative odd values in JavaScript.
- Treating fractional inputs as odd or even.
- Forgetting that bitwise operators coerce to 32-bit integers.
