## Why It Surfaces

This problem is really a small linear-algebra exercise wrapped in geometry language. Once both lines are in standard form, the intersection is just the solution to two equations in two unknowns, with the determinant deciding whether the lines are parallel.

## Worked Example

For lines through `[1, 1]` and `[4, 4]`, and `[1, 8]` and `[2, 4]`, solving the two standard-form equations gives the intersection point `[2.4, 2.4]`.

## Reference Implementations

<ExamplePair>

```js
function programForPointOfIntersectionOfTwoLines(firstA, firstB, secondA, secondB) {
  function coefficients(left, right) {
    const a = right[1] - left[1];
    const b = left[0] - right[0];
    const c = a * left[0] + b * left[1];
    return [a, b, c];
  }

  const [a1, b1, c1] = coefficients(firstA, firstB);
  const [a2, b2, c2] = coefficients(secondA, secondB);
  const determinant = a1 * b2 - a2 * b1;

  if (determinant === 0) {
    return null;
  }

  return [
    (c1 * b2 - c2 * b1) / determinant,
    (a1 * c2 - a2 * c1) / determinant,
  ];
}
```

export function programForPointOfIntersectionOfTwoLines(
  firstA: [number, number],
  firstB: [number, number],
  secondA: [number, number],
  secondB: [number, number],
): [number, number] | null {
  const coefficients = (
    left: [number, number],
    right: [number, number],
  ): [number, number, number] => {
    const a = right[1] - left[1];
    const b = left[0] - right[0];
    const c = a * left[0] + b * left[1];
    return [a, b, c];
  };

  const [a1, b1, c1] = coefficients(firstA, firstB);
  const [a2, b2, c2] = coefficients(secondA, secondB);
  const determinant = a1 * b2 - a2 * b1;

  if (determinant === 0) {
    return null;
  }

  return [
    (c1 * b2 - c2 * b1) / determinant,
    (a1 * c2 - a2 * c1) / determinant,
  ];
}

## Common Pitfalls

- Forgetting that parallel lines have no unique intersection point.
- Solving for the line intersection but skipping the segment-bounds check in a segment variant.
- Mixing up the determinant formulas for `x` and `y`.
