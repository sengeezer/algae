## Why It Surfaces

This is connected-components reasoning with a lighter vocabulary. Once the interviewer says the graph is a forest, you no longer need to worry about cycles inside a component. Each DFS or BFS launched from an unvisited node identifies exactly one tree.

## Worked Example

If the edges are `(0, 1)`, `(0, 2)`, and `(3, 4)`, then the graph has two connected components: `{0, 1, 2}` and `{3, 4}`. Because the input is already a forest, those two components are the two trees in the answer.

## Reference Implementations

<ExamplePair>

```js
function countNumberOfTreesInAForest(nodeCount, edges) {
  const graph = Array.from({ length: nodeCount }, () => []);

  for (const [from, to] of edges) {
    graph[from].push(to);
    graph[to].push(from);
  }

  const visited = new Array(nodeCount).fill(false);
  let trees = 0;

  for (let node = 0; node < nodeCount; node += 1) {
    if (visited[node]) {
      continue;
    }

    trees += 1;
    const stack = [node];
    visited[node] = true;

    while (stack.length > 0) {
      const current = stack.pop();

      for (const neighbor of graph[current]) {
        if (visited[neighbor]) {
          continue;
        }

        visited[neighbor] = true;
        stack.push(neighbor);
      }
    }
  }

  return trees;
}
```

```ts
export function countNumberOfTreesInAForest(
  nodeCount: number,
  edges: Array<[number, number]>,
): number {
  const graph = Array.from({ length: nodeCount }, () => [] as number[]);

  for (const [from, to] of edges) {
    graph[from].push(to);
    graph[to].push(from);
  }

  const visited = new Array<boolean>(nodeCount).fill(false);
  let trees = 0;

  for (let node = 0; node < nodeCount; node += 1) {
    if (visited[node]) {
      continue;
    }

    trees += 1;
    const stack: number[] = [node];
    visited[node] = true;

    while (stack.length > 0) {
      const current = stack.pop()!;

      for (const neighbor of graph[current]) {
        if (visited[neighbor]) {
          continue;
        }

        visited[neighbor] = true;
        stack.push(neighbor);
      }
    }
  }

  return trees;
}
```

## Follow-up Questions

- Remove the forest guarantee and reject any component that contains a cycle.
- Count the size of each tree instead of only the number of trees.
- Solve the same problem with union-find when edges arrive incrementally.

## Common Pitfalls

- Forgetting that isolated vertices count as one-node trees.
- Spending extra work on cycle detection even though the input is already promised to be a forest.
- Treating the graph as directed when the forest definition here is undirected.
