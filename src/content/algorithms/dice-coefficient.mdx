## Why It Surfaces

Dice coefficient is the lightweight alternative when you need a quick similarity score instead of an exact transformation cost. It works well on short strings because shared bigrams capture local structure without building a full DP table.

## Worked Example

`"night"` has bigrams `ni`, `ig`, `gh`, `ht`, while `"nacht"` has `na`, `ac`, `ch`, `ht`. The overlap is one bigram, so the score is `2 * 1 / (4 + 4) = 0.25`.

```js
function mapBigrams(text) {
  const counts = new Map();

  for (let index = 0; index < text.length - 1; index += 1) {
    const bigram = text.slice(index, index + 2);
    counts.set(bigram, (counts.get(bigram) ?? 0) + 1);
  }

  return counts;
}

function diceCoefficient(left, right) {
  if (left === right) {
    return 1;
  }

  if (left.length < 2 || right.length < 2) {
    return 0;
  }

  const leftBigrams = mapBigrams(left);
  let overlap = 0;

  for (let index = 0; index < right.length - 1; index += 1) {
    const bigram = right.slice(index, index + 2);
    const count = leftBigrams.get(bigram) ?? 0;

    if (count > 0) {
      leftBigrams.set(bigram, count - 1);
      overlap += 1;
    }
  }

  return (2 * overlap) / (left.length + right.length - 2);
}
```

## Follow-up Questions

- Compare this score with edit distance on the same string pair.
- Normalize case or strip punctuation before building bigrams.
- Extend the approach to trigrams or token-level similarity.

## Common Pitfalls

- Forgetting the special handling for strings shorter than length `2`.
- Counting shared bigrams without respecting multiplicity.
- Expecting the coefficient to explain the exact edit operations between strings.
