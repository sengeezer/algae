## Why It Surfaces

This is a threshold problem over a recurrence. The useful trick is that you do not need to count digits on every step with `toString`; you can stop once the Fibonacci term is at least `10^(targetDigits - 1)`.

## Worked Example

`F12 = 144`, so index `12` is the first Fibonacci term with `3` digits. The same logic scales by advancing the sequence until the current term reaches the smallest number with the desired width.

## Reference Implementations

<ExamplePair>

```js
function problem025(targetDigits = 1000) {
  if (!Number.isInteger(targetDigits) || targetDigits < 1) {
    throw new RangeError('Target digits must be a positive integer');
  }

  const threshold = 10n ** BigInt(targetDigits - 1);
  let previous = 0n;
  let current = 1n;
  let index = 1;

  while (current < threshold) {
    [previous, current] = [current, previous + current];
    index += 1;
  }

  return index;
}
```

## Follow-up Questions

- Return the Fibonacci value itself in addition to the index.

- Explain why comparing against a power of ten is faster than converting every term to a string.

## Common Pitfalls

- Counting digits by string conversion on every iteration when a threshold comparison is enough.
- Using `Number` instead of `BigInt` and losing exactness for large terms.
- Misaligning Fibonacci indexing when the sequence starts at `F1 = 1`, `F2 = 1`.
