## Why It Surfaces

This is the version of BFS interviewers use when they want more than a reachability check. The search still relies on breadth-first layering, but the twist is that you also preserve parent links so the final route can be reconstructed backward from the destination.

## Worked Example

If BFS reaches node `7` from source `0` through parent chain `7 <- 3 <- 0`, then the shortest path is `0 -> 3 -> 7`. Because BFS discovered `7` at the earliest possible layer, that reconstructed route has the minimum number of edges.

```js
function shortestPathInAnUnweightedGraph(nodeCount, edges, source, destination) {
  const graph = Array.from({ length: nodeCount }, () => []);
  for (const [from, to] of edges) {
    graph[from].push(to);
    graph[to].push(from);
  }

  const distance = new Array(nodeCount).fill(Number.POSITIVE_INFINITY);
  const parent = new Array(nodeCount).fill(-1);
  const queue = [source];
  distance[source] = 0;

  for (let head = 0; head < queue.length; head += 1) {
    const node = queue[head];
    if (node === destination) {
      break;
    }

    for (const neighbor of graph[node]) {
      if (distance[neighbor] !== Number.POSITIVE_INFINITY) {
        continue;
      }

      distance[neighbor] = distance[node] + 1;
      parent[neighbor] = node;
      queue.push(neighbor);
    }
  }

  if (distance[destination] === Number.POSITIVE_INFINITY) {
    return [];
  }

  const path = [];
  for (let node = destination; node !== -1; node = parent[node]) {
    path.push(node);
  }

  path.reverse();
  return path;
}
```

## Follow-up Questions

- Return both the shortest distance and the path.
- How would the solution change for a directed graph?
- Why does BFS fail once edge weights are no longer uniform?

## Common Pitfalls

- Using DFS and hoping the first found route is shortest.
- Forgetting to store parents while traversing.
- Treating the problem like weighted shortest path when every edge cost is one.
