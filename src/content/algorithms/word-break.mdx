## Why It Surfaces

Word Break is the string counterpart to many reachability DPs. Each prefix is either segmentable or not, and a longer prefix becomes valid only if some earlier prefix was already valid and the substring between them is a dictionary word.

## Worked Example

For `text = "leetcode"` and dictionary `{"leet", "code"}`, the answer is true because the valid split after `"leet"` leaves `"code"`, which is also in the dictionary.

## Reference Implementations

<ExamplePair>

```js
function wordBreak(text, words) {
  const dictionary = new Set(words);
  const reachable = new Array(text.length + 1).fill(false);
  reachable[0] = true;

  for (let end = 1; end <= text.length; end += 1) {
    for (let start = 0; start < end; start += 1) {
      if (reachable[start] && dictionary.has(text.slice(start, end))) {
        reachable[end] = true;
        break;
      }
    }
  }

  return reachable[text.length];
}
```

## Follow-up Questions

- Return one valid segmentation instead of only a boolean.
- Count the number of valid segmentations.
- Limit the search with the maximum dictionary word length.

## Common Pitfalls

- Forgetting the empty-prefix base case at index `0`.
- Checking only the latest possible split instead of all earlier segmentable prefixes.
- Using repeated dictionary scans instead of constant-time membership checks.
