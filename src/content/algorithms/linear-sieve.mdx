## Why It Surfaces

Linear sieve is the more technical sibling of the classical sieve. It matters when the interviewer or problem constraints care about constant factors and about reusing the sieve for downstream number-theory work.

## Worked Example

As you sweep upward through the numbers, each prime discovered is pushed into a list. Every composite `i * p` is marked using the current `i` and a prime `p`, and the loop stops when `p` divides `i`, ensuring that composite is generated by its smallest prime factor only once.

<Callout title="Key invariant" tone="tip">
  The linear-time win comes from breaking the inner loop as soon as the current prime divides the current base value.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function linearSieve(limit) {
  const primes = [];
  const isComposite = new Array(limit + 1).fill(false);

  for (let value = 2; value <= limit; value += 1) {
    if (!isComposite[value]) {
      primes.push(value);
    }

    for (const prime of primes) {
      const composite = value * prime;
      if (composite > limit) {
        break;
      }

      isComposite[composite] = true;

      if (value % prime === 0) {
        break;
      }
    }
  }

  return primes;
}
```

## Common Pitfalls

- Forgetting the break when `prime` divides the current value.
- Treating the boolean array as primality instead of composite status and inverting the logic incorrectly.
- Reintroducing repeated composite markings and losing the linear-time property.
