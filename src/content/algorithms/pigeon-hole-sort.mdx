## Why It Surfaces

Pigeon hole sort is a good reminder that not every sort needs pairwise comparisons. If the domain is a tight integer range, you can count occurrences directly and rebuild the sorted output in one pass over that range.

## Worked Example

For `[8, 3, 2, 7, 4, 6, 8]`, the minimum is `2` and the maximum is `8`, so the holes cover `2` through `8`. Count each occurrence, then emit values back in numeric order to produce `[2, 3, 4, 6, 7, 8, 8]`.

## Reference Implementations

<ExamplePair>

```js
function pigeonHoleSort(values) {
  if (values.some((value) => !Number.isInteger(value))) {
    throw new RangeError('Values must be integers');
  }

  if (values.length < 2) {
    return [...values];
  }

  const minValue = Math.min(...values);
  const maxValue = Math.max(...values);
  const counts = Array.from({ length: maxValue - minValue + 1 }, () => 0);

  for (const value of values) {
    counts[value - minValue] += 1;
  }

  const sorted = [];

  for (let offset = 0; offset < counts.length; offset += 1) {
    for (let count = counts[offset]; count > 0; count -= 1) {
      sorted.push(offset + minValue);
    }
  }

  return sorted;
}
```

## Follow-up Questions

- Compare pigeon hole sort with counting sort and bucket sort.
- Explain how the minimum offset lets the algorithm support negative integers.
- Estimate when the range becomes too large for this approach to be practical.

## Common Pitfalls

- Forgetting to offset by the minimum value when negatives are present.
- Using the algorithm when the range is huge and the count array dominates memory.
- Treating it as a generic comparison sort for arbitrary objects.
