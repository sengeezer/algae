## Why It Surfaces

This problem looks like coin change, which usually pushes you toward dynamic programming. The twist is that Fibonacci denominations are special enough for a greedy choice to work: taking the largest usable Fibonacci number never blocks an optimal minimum-term decomposition.

## Worked Example

For `K = 19`, generate Fibonacci values up to `19`: `1, 1, 2, 3, 5, 8, 13`. Take `13` first, leaving `6`, then take `5`, leaving `1`, then take `1`. The answer is `3` terms: `13 + 5 + 1`.

## Reference Implementations

<ExamplePair>

```js
function minimumFibonacciTermsWithSumEqualToK(target) {
  if (target < 0) {
    throw new RangeError('target must be non-negative');
  }

  if (target === 0) {
    return 0;
  }

  const fibonacci = [1, 1];

  while (fibonacci[fibonacci.length - 1] < target) {
    fibonacci.push(
      fibonacci[fibonacci.length - 1] + fibonacci[fibonacci.length - 2],
    );
  }

  let remaining = target;
  let count = 0;

  for (let index = fibonacci.length - 1; index >= 0 && remaining > 0; index -= 1) {
    while (remaining >= fibonacci[index]) {
      remaining -= fibonacci[index];
      count += 1;
    }
  }

  return count;
}
```

## Common Pitfalls

- Reaching for generic coin-change DP before checking whether the denomination family has a stronger greedy property.
- Forgetting to generate Fibonacci numbers only up to the needed target.
- Treating the answer as distinct-term only when the prompt allows repeated Fibonacci numbers.
