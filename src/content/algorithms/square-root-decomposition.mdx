## Why It Surfaces

Square root decomposition is a useful interview midpoint. It is more powerful than raw prefix sums for repeated queries with updates, but simpler to explain and implement than a full segment tree.

## Worked Example

If an array has `16` values, use blocks of size `4`. A query over `[3, 13]` sums the partial prefix of the first block, then the full middle blocks in constant time each, then the partial tail of the last block.

```js
class SquareRootDecomposition {
  constructor(values) {
    this.values = [...values];
    this.blockSize = Math.ceil(Math.sqrt(Math.max(1, values.length)));
    this.blocks = new Array(Math.ceil(values.length / this.blockSize)).fill(0);

    for (let index = 0; index < values.length; index += 1) {
      this.blocks[Math.floor(index / this.blockSize)] += values[index];
    }
  }

  query(left, right) {
    let sum = 0;
    let index = left;

    while (index <= right && index % this.blockSize !== 0) {
      sum += this.values[index];
      index += 1;
    }

    while (index + this.blockSize - 1 <= right) {
      sum += this.blocks[Math.floor(index / this.blockSize)];
      index += this.blockSize;
    }

    while (index <= right) {
      sum += this.values[index];
      index += 1;
    }

    return sum;
  }

  update(index, value) {
    const block = Math.floor(index / this.blockSize);
    this.blocks[block] += value - this.values[index];
    this.values[index] = value;
  }
}
```

## Follow-up Questions

- Adapt the block aggregate from sum to minimum or maximum.
- Compare this design with a Fenwick tree and a segment tree.
- Extend it to two-dimensional queries.

## Common Pitfalls

- Choosing a block size inconsistently with the array length.
- Forgetting to update the block aggregate after a point change.
- Assuming it handles heavy range updates as naturally as lazy propagation.
