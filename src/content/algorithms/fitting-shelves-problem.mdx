## Why It Surfaces

This problem looks combinatorial, but the greedy core is local once the larger-shelf count is fixed: fill the remaining wall with as many smaller shelves as possible. Enumerating all feasible larger-shelf counts is cheap, and the tie-break naturally prefers the solution with more larger shelves when leftover space is equal.

## Worked Example

For `w = 29`, `m = 3`, and `n = 9`, trying larger-shelf counts from `0` to `3` shows that `3` large shelves use `27` units of wall, leaving `2` units empty. No other combination leaves less space, so the answer is `0` smaller shelves, `3` larger shelves, and `2` units of empty space.

```js
function fittingShelvesProblem(wallLength, firstShelfLength, secondShelfLength) {
  const smallerShelfLength = Math.min(firstShelfLength, secondShelfLength);
  const largerShelfLength = Math.max(firstShelfLength, secondShelfLength);

  let bestSmallerShelfCount = 0;
  let bestLargerShelfCount = 0;
  let minimumEmptySpace = wallLength;

  for (
    let largerShelfCount = 0;
    largerShelfCount * largerShelfLength <= wallLength;
    largerShelfCount += 1
  ) {
    const remainingLength = wallLength - largerShelfCount * largerShelfLength;
    const smallerShelfCount = Math.floor(remainingLength / smallerShelfLength);
    const emptySpace = remainingLength % smallerShelfLength;

    if (
      emptySpace < minimumEmptySpace ||
      (emptySpace === minimumEmptySpace &&
        largerShelfCount > bestLargerShelfCount)
    ) {
      minimumEmptySpace = emptySpace;
      bestSmallerShelfCount = smallerShelfCount;
      bestLargerShelfCount = largerShelfCount;
    }
  }

  return {
    smallerShelfCount: bestSmallerShelfCount,
    largerShelfCount: bestLargerShelfCount,
    emptySpace: minimumEmptySpace,
  };
}
```

## Follow-up Questions

- Return the counts mapped back to the caller's original shelf order when `m > n` is allowed.
- Explain why the tie-break can be applied only after minimizing empty space without changing correctness.
- Compare this bounded enumeration with a dynamic-programming knapsack approach.

## Common Pitfalls

- Preferring more larger shelves before confirming that leftover space is minimal.
- Assuming the smaller shelf length is always passed first and hard-coding that requirement.
- Exploring all pairs of shelf counts explicitly instead of fixing the larger-shelf count and greedily filling the remainder.
