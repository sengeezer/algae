## Why It Surfaces

Some interval problems are not about merging everything. They are about selecting the best compatible subset. In that case, the right move is usually to sort by end time and greedily take the next interval that still fits.

> The proof matters here. Interviewers often care more about why earliest finish is optimal than about the scan itself.

## Worked Example

For intervals `[[1, 3], [2, 4], [3, 5], [6, 8]]`, sort by end time. Take `[1, 3]`, skip `[2, 4]` because it overlaps, take `[3, 5]` if touching endpoints are allowed, then take `[6, 8]`.

## JavaScript

```js
function maxNonOverlapping(intervals) {
  const sorted = [...intervals].sort((a, b) => a[1] - b[1]);
  let count = 0;
  let currentEnd = -Infinity;

  for (const [start, end] of sorted) {
    if (start >= currentEnd) {
      count += 1;
      currentEnd = end;
    }
  }

  return count;
}
```

## Interview Notes

- Clarify whether touching endpoints count as overlap.
- Be ready to justify the greedy choice, not just state it.
- Distinguish this problem from interval merging or meeting-room counting.
