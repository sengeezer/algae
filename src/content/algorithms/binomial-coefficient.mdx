## Why It Surfaces

Binomial coefficient is the shortest route from combinatorics theory into interview code. It often appears as a counting subroutine, and a good solution avoids full factorials when the numbers get large.

## Worked Example

`C(5, 2)` counts how many 2-element groups can be chosen from 5 items. Using the multiplicative view, the answer becomes `(5 * 4) / (2 * 1) = 10`.

## Reference Implementations

<ExamplePair>

```js
function binomialCoefficient(n, k) {
  if (k < 0 || k > n) {
    return 0;
  }

  let choose = Math.min(k, n - k);
  let result = 1;

  for (let step = 1; step <= choose; step += 1) {
    result = (result * (n - choose + step)) / step;
  }

  return result;
}
```

```ts
export function binomialCoefficient(n: number, k: number): number {
  if (k < 0 || k > n) {
    return 0;
  }

  const choose = Math.min(k, n - k);
  let result = 1;

  for (let step = 1; step <= choose; step += 1) {
    result = (result * (n - choose + step)) / step;
  }

  return result;
}
```

## Common Pitfalls

- Computing full factorials and overflowing earlier than necessary.
- Forgetting the symmetry reduction with `min(k, n - k)`.
- Ignoring invalid cases where `k < 0` or `k > n`.
