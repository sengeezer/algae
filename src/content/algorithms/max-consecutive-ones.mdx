## Why It Surfaces

This problem is the stripped-down version of many longer window questions. Because there is no replacement budget and no character-frequency logic, the solution is just a running count with resets.

## Worked Example

For `[1, 1, 0, 1, 1, 1]`, the first zero resets the current run from `2` back to `0`, then the final streak grows to `3`, which becomes the answer.

## Reference Implementations

<ExamplePair>

```js
function maxConsecutiveOnes(nums) {
  let current = 0;
  let best = 0;

  for (const value of nums) {
    if (value === 1) {
      current += 1;
      best = Math.max(best, current);
    } else {
      current = 0;
    }
  }

  return best;
}
```

```ts
export function maxConsecutiveOnes(nums: number[]): number {
  let current = 0;
  let best = 0;

  for (const value of nums) {
    if (value === 1) {
      current += 1;
      best = Math.max(best, current);
    } else {
      current = 0;
    }
  }

  return best;
}
```

## Follow-up Questions

- Allow flipping at most one zero, then at most `k` zeros.
- Return the bounds of the best run, not just its length.
- Generalize the same idea to longest repeated-character streaks in strings.

## Common Pitfalls

- Forgetting to reset the current streak after a zero.
- Overengineering the solution with a full sliding window.
- Updating the best length only at the end instead of during the scan.
