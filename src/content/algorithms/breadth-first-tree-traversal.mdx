## Why It Surfaces

Breadth-first tree traversal is the standard answer when the question says level order, nearest ancestor layer, or one layer at a time. The queue makes the traversal order explicit and easy to reason about.

## Worked Example

For a tree rooted at `1` with children `2` and `3`, the traversal begins `[1]`, then `[2, 3]`, then the children of `2` and `3`. The queue preserves exactly that frontier order.

## Reference Implementations

<ExamplePair>

```js
function breadthFirstTreeTraversal(root) {
  if (!root) {
    return [];
  }

  const queue = [root];
  const values = [];

  for (let index = 0; index < queue.length; index += 1) {
    const node = queue[index];
    values.push(node.value);

    if (node.left) {
      queue.push(node.left);
    }

    if (node.right) {
      queue.push(node.right);
    }
  }

  return values;
}
```

type TreeNode = {
  value: number;

  if (!root) {
    return [];

    if (node.left) {
      queue.push(node.left);
    }

    if (node.right) {
      queue.push(node.right);
    }
  }

  return values;
}

## Follow-up Questions

- Return values grouped by level instead of one flat list.
- Compute the right-side view or level averages using the same queue.
- Compare this with recursive depth-first tree traversals.

## Common Pitfalls

- Using a stack and accidentally switching to depth-first order.
- Forgetting to guard the empty-tree case.
- Mutating the queue incorrectly while iterating a level-based solution.
