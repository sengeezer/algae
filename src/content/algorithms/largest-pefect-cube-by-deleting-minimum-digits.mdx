## Why It Surfaces

The brute-force instinct is to try every deleted-digit subsequence, which is exponential. A better move is to iterate through perfect cubes up to the input value and keep the largest cube whose digits can be matched as a subsequence of the original number.

## Worked Example

For `n = 4125`, the cube `125` is valid because its digits appear in order after deleting the leading `4`. No larger cube can be formed as a subsequence, so the answer is `125`.

```js
function largestPefectCubeByDeletingMinimumDigits(value) {
  const digits = String(value);
  let root = Math.floor(Math.cbrt(value));

  while ((root + 1) ** 3 <= value) {
    root += 1;
  }

  while (root ** 3 > value) {
    root -= 1;
  }

  function isSubsequence(candidate) {
    let candidateIndex = 0;

    for (const digit of digits) {
      if (digit === candidate[candidateIndex]) {
        candidateIndex += 1;

        if (candidateIndex === candidate.length) {
          return true;
        }
      }
    }

    return candidateIndex === candidate.length;
  }

  for (let cubeRoot = root; cubeRoot >= 0; cubeRoot -= 1) {
    const cube = cubeRoot ** 3;

    if (isSubsequence(String(cube))) {
      return cube;
    }
  }

  return 0;
}
```

## Follow-up Questions

- Return how many digits were deleted along with the resulting cube.
- Explain why iterating candidate cubes is better than generating every subsequence.
- Adapt the technique to perfect squares or powers of two.

## Common Pitfalls

- Generating all subsequences explicitly, which becomes exponential in the number of digits.
- Treating the task like a permutation problem and reordering digits.
- Forgetting that `0` is also a perfect cube candidate if no larger valid cube exists.
