## Why It Surfaces

Interviewers often accept the knapsack value table first, then immediately ask for the actual chosen items. The recovery step is where you prove that you understand what the DP states mean, not just how to fill them.

## Worked Example

For weights `[1, 3, 4, 5]`, values `[1, 4, 5, 7]`, and capacity `7`, an optimal answer is value `9` from taking the items with weights `3` and `4`.

## Reference Implementations

<ExamplePair>

```js
function printingItemsIn01Knapsack(weights, values, capacity) {
  const itemCount = weights.length;
  const dp = Array.from({ length: itemCount + 1 }, () => new Array(capacity + 1).fill(0));

  for (let item = 1; item <= itemCount; item += 1) {
    for (let space = 0; space <= capacity; space += 1) {
      dp[item][space] = dp[item - 1][space];
      if (weights[item - 1] <= space) {
        dp[item][space] = Math.max(
          dp[item][space],
          dp[item - 1][space - weights[item - 1]] + values[item - 1],
        );
      }
    }
  }

  const chosenIndices = [];
  let remainingCapacity = capacity;

  for (let item = itemCount; item > 0; item -= 1) {
    if (dp[item][remainingCapacity] !== dp[item - 1][remainingCapacity]) {
      chosenIndices.push(item - 1);
      remainingCapacity -= weights[item - 1];
    }
  }

  return {
    maxValue: dp[itemCount][capacity],
    chosenIndices: chosenIndices.reverse(),
  };
}
```

## Follow-up Questions

- Return the chosen weights and values, not only the indices.
- Explain why the one-row optimized knapsack table is not enough for straightforward reconstruction.
- Handle ties by preferring fewer items or lexicographically earlier indices.

## Common Pitfalls

- Reconstructing from a one-dimensional DP array that no longer preserves item history.
- Forgetting to subtract the chosen item's weight while walking backward.
- Assuming equal values in adjacent table cells always mean the item was skipped for a unique reason.
