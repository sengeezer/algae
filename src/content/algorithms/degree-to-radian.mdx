## Why It Surfaces

Angle conversion is one of the simplest possible numeric transforms, which is exactly why interviewers sometimes use it as a calibration problem. It checks whether you know the formula and still validate the input sensibly.

## Worked Example

`90` degrees becomes `pi / 2`, or approximately `1.5707963268` radians. The mapping is linear because degrees and radians are just different units for the same angle.

## Reference Implementations

<ExamplePair>

```js
function degreeToRadian(degrees) {
  if (typeof degrees !== 'number' || Number.isNaN(degrees)) {
    throw new TypeError('Expected a numeric degree value');
  }

  return degrees * (Math.PI / 180);
}
```

```ts
export function degreeToRadian(degrees: number): number {
  if (Number.isNaN(degrees)) {
    throw new TypeError('Expected a numeric degree value');
  }

  return degrees * (Math.PI / 180);
}
```

## Common Pitfalls

- Mixing degrees and radians in the same code path without labeling units.
- Using `180 / pi` instead of `pi / 180` for this direction.
- Forgetting that floating-point output is usually approximate, not symbolic.
