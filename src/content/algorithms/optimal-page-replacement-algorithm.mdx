## Why It Surfaces

Optimal page replacement is the textbook offline greedy benchmark: if you can see the future, the safest page to evict is the one needed farthest from now, or never again. Real operating systems cannot do that perfectly, but this algorithm gives the target that practical heuristics try to approximate.

## Worked Example

With `frameCount = 3` and reference string `[7, 0, 1, 2, 0, 3, 0, 4, 2, 3]`, the algorithm records `6` page faults and `4` page hits. One valid final frame state is `[2, 4, 3]`.

```js
function optimalPageReplacementAlgorithm(referenceString, frameCount) {
  const frames = [];
  let pageHits = 0;
  let pageFaults = 0;

  for (let index = 0; index < referenceString.length; index += 1) {
    const page = referenceString[index];

    if (frames.includes(page)) {
      pageHits += 1;
      continue;
    }

    pageFaults += 1;

    if (frames.length < frameCount) {
      frames.push(page);
      continue;
    }

    let replacementIndex = 0;
    let farthestUse = -1;

    for (let frameIndex = 0; frameIndex < frames.length; frameIndex += 1) {
      let nextUse = Number.POSITIVE_INFINITY;

      for (
        let futureIndex = index + 1;
        futureIndex < referenceString.length;
        futureIndex += 1
      ) {
        if (referenceString[futureIndex] === frames[frameIndex]) {
          nextUse = futureIndex;
          break;
        }
      }

      if (nextUse === Number.POSITIVE_INFINITY) {
        replacementIndex = frameIndex;
        break;
      }

      if (nextUse > farthestUse) {
        farthestUse = nextUse;
        replacementIndex = frameIndex;
      }
    }

    frames[replacementIndex] = page;
  }

  return {
    pageHits,
    pageFaults,
    finalFrames: frames,
  };
}
```

## Follow-up Questions

- Compare the miss count against FIFO or LRU on the same reference string.
- Return the full frame state after each reference, not only the final result.
- Explain why the algorithm is optimal even though it is unusable online.

## Common Pitfalls

- Replacing the page used soonest instead of the page used farthest in the future.
- Forgetting that a page with no future reference is always the best eviction choice.
- Describing the algorithm as deployable in a real OS even though it requires perfect future knowledge.
