## Why It Surfaces

Painting Fence looks simple until the adjacency rule forces you to separate states. The clean DP insight is to count arrangements that end with the last two posts matching separately from those that end with different colors.

## Worked Example

For `posts = 3` and `colors = 2`, the valid paintings are `AAB`, `ABA`, `ABB`, `BAA`, `BAB`, and `BBA`, so the answer is `6`. `AAA` and `BBB` are excluded because they create a run of three equal posts.

## Reference Implementations

<ExamplePair>

```js
function paintingFenceAlgorithm(posts, colors) {
  if (posts === 0 || colors === 0) {
    return 0;
  }

  if (posts === 1) {
    return colors;
  }

  let same = colors;
  let different = colors * (colors - 1);

  for (let post = 3; post <= posts; post += 1) {
    const nextSame = different;
    const nextDifferent = (same + different) * (colors - 1);
    same = nextSame;
    different = nextDifferent;
  }

  return same + different;
}
```

## Follow-up Questions

- Change the rule so no two adjacent posts may match.
- Return the answer modulo a large prime.
- Explain why the `same` state can only come from a previously `different` state.

## Common Pitfalls

- Treating every new post as if it always had `colors` choices.
- Forgetting that the first two posts need explicit initialization.
- Collapsing the same-ending and different-ending states too early.
