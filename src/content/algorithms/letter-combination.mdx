## Why It Surfaces

This problem looks simple, but it is a clean test of recursive state construction. Each digit expands into a small set of letters, and the search must choose one letter per digit until a full word is formed.

## Worked Example

For `digits = "23"`, the first digit contributes `a`, `b`, or `c`, and the second contributes `d`, `e`, or `f`. The final answer is the Cartesian product of those two sets, such as `ad`, `ae`, `af`, and so on.

```js
function letterCombination(digits) {
  if (digits.length === 0) {
    return [];
  }

  const mapping = {
    2: "abc",
    3: "def",
    4: "ghi",
    5: "jkl",
    6: "mno",
    7: "pqrs",
    8: "tuv",
    9: "wxyz",
  };
  const results = [];

  function backtrack(index, path) {
    if (index === digits.length) {
      results.push(path);
      return;
    }

    for (const letter of mapping[digits[index]]) {
      backtrack(index + 1, path + letter);
    }
  }

  backtrack(0, "");
  return results;
}
```

      return;

    }
  }

  backtrack(0, "");
  return results;
}

## Follow-up Questions

- Handle invalid digits like `0` and `1` explicitly.
- Generate the combinations iteratively with a queue.
- Compare this search space with permutations and subsets.

## Common Pitfalls

- Treating the problem like permutations instead of position-by-position expansion.
- Forgetting the empty-input base case.
- Reusing shared mutable state without undoing it on backtrack.
