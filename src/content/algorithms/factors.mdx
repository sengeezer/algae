## Why It Surfaces

Factors is a basic number-theory helper that turns up in divisibility proofs, optimization shortcuts, and brute-force search pruning. The interview-friendly version uses divisor pairs instead of a full scan.

## Worked Example

For `36`, the divisor pairs are `(1, 36)`, `(2, 18)`, `(3, 12)`, `(4, 9)`, and `(6, 6)`. The full sorted factor list is `[1, 2, 3, 4, 6, 9, 12, 18, 36]`.

## Reference Implementations

<ExamplePair>

```js
function factors(value) {
  if (!Number.isInteger(value) || value < 1) {
    throw new RangeError('Value must be a positive integer');
  }

  const lowFactors = [];
  const highFactors = [];

  for (let divisor = 1; divisor * divisor <= value; divisor += 1) {
    if (value % divisor === 0) {
      lowFactors.push(divisor);

      const pairedDivisor = value / divisor;
      if (pairedDivisor !== divisor) {
        highFactors.push(pairedDivisor);
      }
    }
  }

  return lowFactors.concat(highFactors.reverse());
}
```

## Follow-up Questions

- Return only proper divisors by excluding the number itself.
- Use prime factorization to count divisors without listing them.
- Compare the `O(sqrt(n))` factor-pair method with the naive full scan.

## Common Pitfalls

- Double-counting the square root when the input is a perfect square.
- Scanning all the way to `n` instead of stopping at the square root.
- Confusing full factors with prime factors.
