## Why It Surfaces

This problem is really about converting one representation into another. Once the string becomes a sequence of adjacent character gaps, the task reduces to checking whether that sequence is nondecreasing.

## Worked Example

For `"delete"`, normalize to uppercase `DELETE` and compute adjacent gaps: `[1, 7, 7, 15, 15]`. Because each gap is at least as large as the previous one, the word qualifies as exceeding.

## Reference Implementations

<ExamplePair>

```js
function checkExceeding(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  const letters = value.toUpperCase().replace(/[^A-Z]/g, '');
  let previousGap = -1;

  for (let index = 0; index < letters.length - 1; index += 1) {
    const gap = Math.abs(
      letters.charCodeAt(index) - letters.charCodeAt(index + 1),
    );

    if (gap === 0) {
      continue;
    }

    if (gap < previousGap) {
      return false;
    }

    previousGap = gap;
  }

  return true;
}
```

## Follow-up Questions

- Require the gaps to be strictly increasing instead of merely nondecreasing.
- Return the computed gap sequence along with the boolean result.
- Decide whether repeated letters should count as a zero gap or be skipped.

## Common Pitfalls

- Forgetting to normalize case before comparing ASCII or Unicode code points.
- Comparing signed deltas instead of absolute distances.
- Not clarifying how to handle punctuation or repeated letters.
