## Why It Surfaces

Selection sort is the simplest example of repeatedly picking the next final element. The interview value is usually in clearly separating the sorted prefix from the unsorted suffix and explaining why the minimum selection is correct.

## Worked Example

For `[64, 25, 12, 22, 11]`, the first pass finds `11` and swaps it into position `0`. The second pass scans the remaining suffix for `12`, and the process repeats until the prefix spans the full array.

## Reference Implementations

<ExamplePair>

```js
function selectionSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const items = [...values];

  for (let start = 0; start < items.length - 1; start += 1) {
    let minIndex = start;

    for (let index = start + 1; index < items.length; index += 1) {
      if (items[index] < items[minIndex]) {
        minIndex = index;
      }
    }

    if (minIndex !== start) {
      [items[start], items[minIndex]] = [items[minIndex], items[start]];
    }
  }

  return items;
}
```

## Follow-up Questions

- Count how many writes selection sort performs compared with bubble sort.
- Sort in descending order by selecting the maximum each pass.
- Compare repeated minimum extraction with heap sort.

## Common Pitfalls

- Re-swapping even when the minimum is already in place.
- Confusing the minimum value with its index during the scan.
- Claiming the best case improves asymptotically when it does not.
