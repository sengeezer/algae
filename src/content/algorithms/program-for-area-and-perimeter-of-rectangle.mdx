## Why It Surfaces

This looks trivial, but it is still distinct from a generic area helper because the prompt asks for two outputs with one shape-specific input surface. The answer is just the pair of canonical rectangle formulas with minimal validation.

## Worked Example

For `length = 5` and `width = 6`, the area is `30` and the perimeter is `22`.

## Reference Implementations

<ExamplePair>

```js
function programForAreaAndPerimeterOfRectangle(length, width) {
  if (
    typeof length !== "number"
    || typeof width !== "number"
    || !Number.isFinite(length)
    || !Number.isFinite(width)
    || length <= 0
    || width <= 0
  ) {
    throw new RangeError("Rectangle side lengths must be positive numbers");
  }

  return {
    area: length * width,
    perimeter: 2 * (length + width),
  };
}
```

```ts
export function programForAreaAndPerimeterOfRectangle(
  length: number,
  width: number,
): { area: number; perimeter: number } {
  if (
    !Number.isFinite(length)
    || !Number.isFinite(width)
    || length <= 0
    || width <= 0
  ) {
    throw new RangeError('Rectangle side lengths must be positive numbers');
  }

  return {
    area: length * width,
    perimeter: 2 * (length + width),
  };
}
```
