## Why It Surfaces

Decimal-to-octal conversion is the same repeated-division story as binary or hexadecimal, but with base `8`. It is useful because it makes the general radix pattern feel reusable rather than tied to one familiar base.

## Worked Example

For `65`, repeated division by `8` gives remainders `1`, `0`, `1` as the quotient sequence goes `65 -> 8 -> 1 -> 0`. Reversing those remainders produces `101` in octal.

## Reference Implementations

<ExamplePair>

```js
function decimalToOctal(value) {
  if (!Number.isInteger(value) || value < 0) {
    throw new RangeError('The input must be a non-negative integer');
  }

  if (value === 0) {
    return '0';
  }

  const digits = [];
  let current = value;

  while (current > 0) {
    digits.push(String(current % 8));
    current = Math.floor(current / 8);
  }

  return digits.reverse().join('');
}
```

```ts
export function decimalToOctal(value: number): string {
  if (!Number.isInteger(value) || value < 0) {
    throw new RangeError('The input must be a non-negative integer');
  }

  if (value === 0) {
    return '0';
  }

  const digits: string[] = [];
  let current = value;

  while (current > 0) {
    digits.push(String(current % 8));
    current = Math.floor(current / 8);
  }

  return digits.reverse().join('');
}
```

## Follow-up Questions

- Explain why the remainder sequence must be reversed.
- Generalize the routine to any base between `2` and `36`.
- Compare octal grouping with binary regrouping in threes.

## Common Pitfalls

- Returning a numeric value and losing leading-zero formatting concerns.
- Forgetting the special case for `0`.
- Using floating-point or negative inputs without defining behavior.
