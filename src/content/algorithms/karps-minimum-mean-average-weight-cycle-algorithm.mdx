## Why It Surfaces

This problem is useful because it changes the optimization target. Standard shortest-path thinking minimizes total weight, but here the right answer is the cycle with the smallest average weight per edge, which requires a different dynamic-programming view.

## Worked Example

If a cycle has total weight `5` over `3` edges, its mean weight is `5 / 3`. Karp's algorithm compares exact-edge-count path costs and uses a min-max expression to infer which vertex lies on the best average-weight cycle.

```js
function karpsMinimumMeanAverageWeightCycleAlgorithm(nodeCount, edges) {
  const inf = Number.POSITIVE_INFINITY;
  const dp = Array.from({ length: nodeCount + 1 }, () => new Array(nodeCount).fill(inf));
  dp[0][0] = 0;

  for (let edgeCount = 1; edgeCount <= nodeCount; edgeCount += 1) {
    for (const [from, to, weight] of edges) {
      if (dp[edgeCount - 1][from] === inf) {
        continue;
      }

      dp[edgeCount][to] = Math.min(dp[edgeCount][to], dp[edgeCount - 1][from] + weight);
    }
  }

  let answer = inf;

  for (let node = 0; node < nodeCount; node += 1) {
    if (dp[nodeCount][node] === inf) {
      continue;
    }

    let candidate = Number.NEGATIVE_INFINITY;
    for (let edgeCount = 0; edgeCount < nodeCount; edgeCount += 1) {
      if (dp[edgeCount][node] === inf) {
        continue;
      }

      candidate = Math.max(
        candidate,
        (dp[nodeCount][node] - dp[edgeCount][node]) / (nodeCount - edgeCount),
      );
    }

    answer = Math.min(answer, candidate);
  }

  return Number.isFinite(answer) ? answer : null;
}
```

## Common Pitfalls

- Optimizing for total cycle weight when the prompt asks for average weight.
- Forgetting that the algorithm reasons over exact numbers of edges.
- Losing track of the outer `min` and inner `max` roles in Karp's formula.
