## Why It Surfaces

This problem tests whether you can reason about structure and identity at the same time. The graph can have cycles and shared neighbors, so the core requirement is not just traversing it, but guaranteeing that each original node corresponds to one and only one clone.

## Worked Example

If node `0` points to nodes `1` and `2`, and both `1` and `2` point back to `0`, the clone must preserve that shared structure without creating multiple copies of `0`. A memo table from original nodes to cloned nodes makes that identity preservation explicit.

## Reference Implementations

<ExamplePair>

```js
class GraphNode {
  constructor(val) {
    this.val = val;
    this.neighbors = [];
  }
}

function cloneAnUndirectedGraph(node) {
  if (node === null) {
    return null;
  }

  const clones = new Map();
  const queue = [node];
  clones.set(node, new GraphNode(node.val));

  for (let head = 0; head < queue.length; head += 1) {
    const current = queue[head];
    const currentClone = clones.get(current);

    for (const neighbor of current.neighbors) {
      if (!clones.has(neighbor)) {
        clones.set(neighbor, new GraphNode(neighbor.val));
        queue.push(neighbor);
      }

      currentClone.neighbors.push(clones.get(neighbor));
    }
  }

  return clones.get(node);
}
```

```ts
class GraphNode {
  val: number;
  neighbors: GraphNode[];

  constructor(val: number) {
    this.val = val;
    this.neighbors = [];
  }
}

export function cloneAnUndirectedGraph(node: GraphNode | null): GraphNode | null {
  if (node === null) {
    return null;
  }

  const clones = new Map<GraphNode, GraphNode>();
  const queue: GraphNode[] = [node];
  clones.set(node, new GraphNode(node.val));

  for (let head = 0; head < queue.length; head += 1) {
    const current = queue[head];
    const currentClone = clones.get(current)!;

    for (const neighbor of current.neighbors) {
      if (!clones.has(neighbor)) {
        clones.set(neighbor, new GraphNode(neighbor.val));
        queue.push(neighbor);
      }

      currentClone.neighbors.push(clones.get(neighbor)!);
    }
  }

  return clones.get(node)!;
}
```

## Common Pitfalls

- Recreating the same neighbor multiple times because there is no original-to-clone map.
- Returning a shallow copy that reuses old neighbor references.
- Forgetting that cycles can cause infinite recursion without memoization.
