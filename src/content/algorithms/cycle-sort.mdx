## Why It Surfaces

Cycle sort is interesting because it optimizes a different cost than most interview sorts. Instead of minimizing comparisons or maximizing stability, it tries to write each element as few times as possible by sending it directly into its final cycle position.

## Worked Example

Suppose the first value belongs three positions to the right in the fully sorted array. Move it there, pick up the displaced value, and continue rotating until the cycle closes back at the starting index.

```js
function cycleSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const items = [...values];

  for (let cycleStart = 0; cycleStart < items.length - 1; cycleStart += 1) {
    let item = items[cycleStart];
    let position = cycleStart;

    for (let index = cycleStart + 1; index < items.length; index += 1) {
      if (items[index] < item) {
        position += 1;
      }
    }

    if (position === cycleStart) {
      continue;
    }

    while (item === items[position]) {
      position += 1;
    }

    [items[position], item] = [item, items[position]];

    while (position !== cycleStart) {
      position = cycleStart;

      for (let index = cycleStart + 1; index < items.length; index += 1) {
        if (items[index] < item) {
          position += 1;
        }
      }

      while (item === items[position]) {
        position += 1;
      }

      [items[position], item] = [item, items[position]];
    }
  }

  return items;
}
```

## Follow-up Questions

- Count the number of writes and compare it with selection sort.
- Explain how duplicate values complicate cycle placement.

## Common Pitfalls

- Forgetting to skip over duplicates already occupying the target position.
- Confusing the displaced carried value with the array value at the current index.
- Using cycle sort when readability matters more than write minimization.
