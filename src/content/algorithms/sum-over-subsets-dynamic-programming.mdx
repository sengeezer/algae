## Why It Surfaces

SOS DP is what you use when every bitmask answer needs information from all of its submasks. Enumerating those submasks repeatedly works for small inputs, but the real pattern is to push contributions across one bit position at a time until every mask has accumulated the values it should inherit.

## Worked Example

For `values = [7, 12, 14, 16]`, the masks are `00`, `01`, `10`, and `11`. The subset sums become `[7, 19, 21, 49]` because mask `11` must include all four original entries while mask `01` includes only the submasks `00` and `01`.

```js
function sumOverSubsetsDynamicProgramming(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('values must be an array');
  }

  if (values.length === 0) {
    return [];
  }

  for (const value of values) {
    if (!Number.isFinite(value)) {
      throw new TypeError('values must contain only finite numbers');
    }
  }

  const bitCount = Math.log2(values.length);
  if (!Number.isInteger(bitCount)) {
    throw new RangeError('values length must be a power of two');
  }

  const sums = [...values];

  for (let bit = 0; bit < bitCount; bit += 1) {
    for (let mask = 0; mask < sums.length; mask += 1) {
      if (mask & (1 << bit)) {
        sums[mask] += sums[mask ^ (1 << bit)];
      }
    }
  }

  return sums;
}
```

## Interview Notes

- State clearly what each mask represents before talking about transitions.
- Mention the progression from brute force to submask iteration to the `O(n * 2^n)` transform.
- Connect the pattern to other bitmask DP problems that aggregate over subsets or supersets.

## Common Pitfalls

- Treating the input length as arbitrary when SOS DP expects `2^n` masks.
- Recomputing all submasks for every mask instead of applying the bit transform.
- Confusing subset aggregation with subset-sum problems over arbitrary values.
