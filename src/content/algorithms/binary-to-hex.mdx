## Why It Surfaces

Binary-to-hex conversion is the most practical base-conversion shortcut in systems work because the bases align perfectly. Instead of decoding the whole number, you can translate each nibble directly.

## Worked Example

`1001101` becomes `0100 1101` after left padding. Those nibbles map to `4` and `D`, so the final answer is `4D`.

## Reference Implementations

<ExamplePair>

```js
const HEX_DIGITS = '0123456789ABCDEF';

function binaryToHex(binaryString) {
  if (!/^[01]+$/.test(binaryString)) {
    throw new TypeError('Binary strings may only contain 0 and 1');
  }

  const paddedLength = Math.ceil(binaryString.length / 4) * 4;
  const padded = binaryString.padStart(paddedLength, '0');
  const output = [];

  for (let index = 0; index < padded.length; index += 4) {
    const chunk = padded.slice(index, index + 4);
    output.push(HEX_DIGITS[Number.parseInt(chunk, 2)]);
  }

  return output.join('').replace(/^0+(?=[0-9A-F])/, '') || '0';
}
```

## Follow-up Questions

- Explain why hex aligns so well with binary compared with decimal.
- Convert the routine into binary-to-octal by regrouping in threes.
- Return lowercase hex digits when a downstream format requires them.

## Common Pitfalls

- Padding on the wrong side instead of the most significant side.
- Forgetting to validate that the input only contains binary digits.
- Returning an empty string when the input is all zeros.
