## Why It Surfaces

LCA is the classic tree recursion question because the return value from each subtree carries real semantic meaning. It is a good interview signal for whether you can write recursive tree code that combines results from the left and right sides.

## JavaScript

```js
function lowestCommonAncestor(root, p, q) {
  if (root === null || root === p || root === q) {
    return root;
  }

  const left = lowestCommonAncestor(root.left, p, q);
  const right = lowestCommonAncestor(root.right, p, q);

  if (left !== null && right !== null) {
    return root;
  }

  return left ?? right;
}
```

## TypeScript

```ts
type TreeNode = {
  value: number;
  left: TreeNode | null;
  right: TreeNode | null;
};

export function lowestCommonAncestor(
  root: TreeNode | null,
  first: TreeNode,
  second: TreeNode,
): TreeNode | null {
  if (root === null || root === first || root === second) {
    return root;
  }

  const left = lowestCommonAncestor(root.left, first, second);
  const right = lowestCommonAncestor(root.right, first, second);

  if (left !== null && right !== null) {
    return root;
  }

  return left ?? right;
}
```

## Interview Notes

- Explain what each recursive call returns: either a target, an ancestor, or null.
- Mention that binary lifting or parent pointers matter only for many repeated queries.
- Test one case where one target is itself the ancestor of the other.
