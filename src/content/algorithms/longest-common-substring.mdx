## Why It Surfaces

Longest Common Substring is the standard trap after someone has learned LCS. The recurrence looks similar, but contiguity changes everything: when the characters stop matching, the current shared suffix length has to reset to zero.

## Worked Example

For `left = "abcdxyz"` and `right = "xyzabcd"`, the answer is `4` because `"abcd"` is the longest contiguous block appearing in both strings.

## Reference Implementations

<ExamplePair>

```js
function longestCommonSubstring(left, right) {
  const dp = new Array(right.length + 1).fill(0);
  let best = 0;

  for (let row = 1; row <= left.length; row += 1) {
    let diagonal = 0;

    for (let col = 1; col <= right.length; col += 1) {
      const previous = dp[col];
      if (left[row - 1] === right[col - 1]) {
        dp[col] = diagonal + 1;
        best = Math.max(best, dp[col]);
      } else {
        dp[col] = 0;
      }
      diagonal = previous;
    }
  }

  return best;
}
```

## Common Pitfalls

- Solving longest common subsequence by mistake and allowing gaps.
- Forgetting to reset the current state to zero on mismatch.
- Overwriting the diagonal dependency too early in the space-optimized version.
