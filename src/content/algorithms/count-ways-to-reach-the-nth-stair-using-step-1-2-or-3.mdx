## Why It Surfaces

This is the smallest possible upgrade from classic climbing stairs that still forces you to rethink the recurrence. Instead of summing the previous two states, you now need the previous three, which makes the sequence tribonacci-shaped rather than Fibonacci-shaped.

## Worked Example

For `n = 4`, the valid endings come from stair `3`, stair `2`, or stair `1`, so `ways[4] = ways[3] + ways[2] + ways[1] = 4 + 2 + 1 = 7`.

## Reference Implementations

<ExamplePair>

```js
function countWaysToReachTheNthStairUsingStep123(steps) {
  if (steps < 0) {
    return 0;
  }

  if (steps === 0) {
    return 1;
  }

  if (steps === 1) {
    return 1;
  }

  if (steps === 2) {
    return 2;
  }

  let waysMinusThree = 1;
  let waysMinusTwo = 1;
  let waysMinusOne = 2;

  for (let current = 3; current <= steps; current += 1) {
    const waysHere = waysMinusOne + waysMinusTwo + waysMinusThree;
    waysMinusThree = waysMinusTwo;
    waysMinusTwo = waysMinusOne;
    waysMinusOne = waysHere;
  }

  return waysMinusOne;
}
```

## Follow-up Questions

- Allow a blocked stair that cannot be stepped on.
- Return the count modulo a large prime.
- Generalize the transition to an arbitrary allowed move set.

## Common Pitfalls

- Reusing the two-step Fibonacci recurrence without adding the third predecessor.
- Forgetting that `n = 0` has one empty-path base case.
- Mixing counting questions with minimum-jumps or minimum-cost variants.
