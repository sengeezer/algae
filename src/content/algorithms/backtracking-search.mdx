## Why It Surfaces

Backtracking shows up when an interviewer wants a complete search, but not a careless brute-force search. You are expected to build a path incrementally, prune impossible states, and undo each choice cleanly before exploring the next branch.

<Callout title="Path hygiene" tone="warning">
  The pattern is usually summarized as choose, recurse, unchoose. If that last step is missing, the solution leaks state across branches.
</Callout>

## Worked Example

For `nums = [1, 2, 3]`, subset generation starts with the empty path. At each index, you either include the current value and continue, or skip it and move on. The answer collects every path that reaches the end.

## Reference Implementations

<ExamplePair>

```js
function subsets(nums) {
  const results = [];
  const path = [];

  function dfs(index) {
    if (index === nums.length) {
      results.push([...path]);
      return;
    }

    path.push(nums[index]);
    dfs(index + 1);
    path.pop();

    dfs(index + 1);
  }

  dfs(0);
  return results;
}
```


## Interview Notes

- State exactly what the recursion parameters mean.
- Mention pruning as soon as a branch violates the constraint.
- Restore shared state immediately after the recursive call.
