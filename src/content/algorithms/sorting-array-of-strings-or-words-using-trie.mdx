## Why It Surfaces

This problem is really about exploiting shared prefixes. Once all words are inserted, a lexicographic DFS over child edges produces dictionary order without comparing whole strings against each other the way a standard comparison sort would.

## Worked Example

For `['geeks', 'for', 'geeks', 'a']`, the trie stores each distinct word once. A lexicographic traversal visits `a`, then `for`, then `geeks`, so the sorted unique output is `['a', 'for', 'geeks']`.

```js
class TrieSortNode {
  constructor() {
    this.children = new Map();
    this.isWord = false;
  }
}

function sortingArrayOfStringsOrWordsUsingTrie(words) {
  if (!Array.isArray(words)) {
    throw new TypeError('Words must be an array of strings');
  }

  const root = new TrieSortNode();

  for (const word of words) {
    if (typeof word !== 'string' || word.length === 0) {
      throw new RangeError('Words must be non-empty strings');
    }

    let node = root;
    for (const char of word) {
      if (!node.children.has(char)) {
        node.children.set(char, new TrieSortNode());
      }
      node = node.children.get(char);
    }
    node.isWord = true;
  }

  const orderedWords = [];

  const dfs = (node, prefix) => {
    if (node.isWord) {
      orderedWords.push(prefix);
    }

    const entries = [...node.children.entries()].sort((left, right) =>
      left[0].localeCompare(right[0]),
    );

    for (const [char, child] of entries) {
      dfs(child, prefix + char);
    }
  };

  dfs(root, '');
  return orderedWords;
}
```

## Follow-up Questions

- How would this compare to `words.sort()` when the alphabet is fixed but prefixes overlap heavily?
- What changes if duplicates should be preserved instead of collapsed?

## Common Pitfalls

- Forgetting that a trie naturally removes duplicates unless you track multiplicity.
- Traversing child nodes in insertion order instead of lexicographic order.
- Reconstructing entire strings at each step with unnecessary repeated work.
