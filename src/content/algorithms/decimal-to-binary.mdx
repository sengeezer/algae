## Why It Surfaces

Decimal-to-binary conversion is the mirror image of binary parsing. Instead of accumulating place values, you repeatedly peel off the least significant bit with `% 2` and build the answer in reverse.

## Worked Example

For `13`, the remainders are `1`, `0`, `1`, `1` as the quotient sequence goes `13 -> 6 -> 3 -> 1 -> 0`. Reversing those remainders gives `1101`.

## Reference Implementations

<ExamplePair>

```js
function decimalToBinary(value) {
  if (!Number.isInteger(value) || value < 0) {
    throw new RangeError('The input must be a non-negative integer');
  }

  if (value === 0) {
    return '0';
  }

  const bits = [];
  let current = value;

  while (current > 0) {
    bits.push(String(current % 2));
    current = Math.floor(current / 2);
  }

  return bits.reverse().join('');
}
```

```ts
export function decimalToBinary(value: number): string {
  if (!Number.isInteger(value) || value < 0) {
    throw new RangeError('The input must be a non-negative integer');
  }

  if (value === 0) {
    return '0';
  }

  const bits: string[] = [];
  let current = value;

  while (current > 0) {
    bits.push(String(current % 2));
    current = Math.floor(current / 2);
  }

  return bits.reverse().join('');
}
```

## Follow-up Questions

- Explain why the remainder sequence arrives in reverse order.
- Support negative integers with two's-complement formatting rules.
- Generalize the routine to any base greater than one.

## Common Pitfalls

- Returning an empty string for the input `0`.
- Using bitwise operators on values that may exceed 32-bit integer limits.
- Forgetting to reject non-integer or negative input.
