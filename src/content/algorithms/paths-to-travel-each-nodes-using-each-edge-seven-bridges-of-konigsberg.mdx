## Why It Surfaces

This is the construction counterpart to Eulerian classification. Knowing that zero or two odd-degree vertices are allowed is not enough here; you also need to emit the actual route, and the adjacency-matrix input forces you to think about consuming undirected edge counts directly.

## Worked Example

If exactly two vertices have odd degree, the trail must start at one of them. As the stack walks unused edges, vertices are appended to the answer only after all remaining incident edges are exhausted, so the final path appears in reverse and must be flipped at the end.

```js
function pathsToTravelEachNodesUsingEachEdgeSevenBridgesOfKonigsberg(adj) {
  const nodeCount = adj.length;
  if (nodeCount === 0) {
    return [];
  }

  const degree = new Array(nodeCount).fill(0);
  const odd = [];
  let totalEdges = 0;
  let start = -1;

  for (let row = 0; row < nodeCount; row += 1) {
    for (let col = 0; col < nodeCount; col += 1) {
      degree[row] += adj[row][col];
      if (col > row) {
        totalEdges += adj[row][col];
      }
    }

    if (degree[row] > 0 && start === -1) {
      start = row;
    }

    if (degree[row] % 2 === 1) {
      odd.push(row);
    }
  }

  if (start === -1) {
    return [1];
  }

  if (odd.length !== 0 && odd.length !== 2) {
    return [-1];
  }

  if (odd.length === 2) {
    start = odd[0];
  }

  const remaining = adj.map((row) => [...row]);
  const stack = [start];
  const path = [];

  while (stack.length > 0) {
    const node = stack[stack.length - 1];
    let next = -1;

    for (let neighbor = 0; neighbor < nodeCount; neighbor += 1) {
      if (remaining[node][neighbor] > 0) {
        next = neighbor;
        break;
      }
    }

    if (next === -1) {
      path.push(stack.pop() + 1);
      continue;
    }

    remaining[node][next] -= 1;
    remaining[next][node] -= 1;
    stack.push(next);
  }

  if (path.length !== totalEdges + 1) {
    return [-1];
  }

  path.reverse();
  return path;
}
```

## Follow-up Questions

- Convert the same idea to adjacency lists for better sparse-graph performance.

## Common Pitfalls

- Forgetting that an undirected Euler trail allows either zero or two odd-degree vertices.
- Building the route in stack-pop order and forgetting to reverse it before returning.
- Assuming degree checks alone are enough without verifying that every edge was actually consumed.
