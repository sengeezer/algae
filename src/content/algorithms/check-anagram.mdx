## Why It Surfaces

Anagram questions are usually really normalization questions. The interviewer wants to see whether you recognize that order is irrelevant once character counts are fixed.

## Worked Example

`"listen"` and `"silent"` are anagrams because each letter appears the same number of times in both strings. After counting the letters, every frequency matches exactly.

## Reference Implementations

<ExamplePair>

```js
function normalize(input) {
  return input.toLowerCase().replace(/[^a-z0-9]/g, '');
}

function checkAnagram(left, right) {
  if (typeof left !== 'string' || typeof right !== 'string') {
    throw new TypeError('Both arguments must be strings');
  }

  const normalizedLeft = normalize(left);
  const normalizedRight = normalize(right);

  if (normalizedLeft.length !== normalizedRight.length) {
    return false;
  }

  const counts = new Map();

  for (const char of normalizedLeft) {
    counts.set(char, (counts.get(char) ?? 0) + 1);
  }

  for (const char of normalizedRight) {
    const nextCount = (counts.get(char) ?? 0) - 1;
    if (nextCount < 0) {
      return false;
    }
    counts.set(char, nextCount);
  }

  return true;
}
```

## Follow-up Questions

- Group an array of words by their anagram signature.
- Compare the frequency-map approach with sorting both strings first.
- Clarify how punctuation, spaces, and case should be handled.

## Common Pitfalls

- Comparing strings directly without removing ordering from the problem.
- Forgetting to normalize punctuation or case when the prompt treats them as irrelevant.
- Sorting the strings by default when a linear-time frequency count is enough.
