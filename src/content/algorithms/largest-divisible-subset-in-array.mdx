## Why It Surfaces

Largest divisible subset is a useful DP follow-up to LIS because the structure looks familiar but the ordering rule changes. Once the array is sorted, each value tries to extend the longest earlier chain ending at a divisor of that value.

## Worked Example

For `[1, 2, 3, 8, 4, 16]`, sorting gives `[1, 2, 3, 4, 8, 16]`. The best divisible chain is `[1, 2, 4, 8, 16]`, which is longer than any chain that goes through `3`.

```js
function largestDivisibleSubsetInArray(values) {
  if (values.length === 0) {
    return [];
  }

  const sorted = [...values].sort((left, right) => left - right);
  const lengths = new Array(sorted.length).fill(1);
  const previous = new Array(sorted.length).fill(-1);
  let bestIndex = 0;

  for (let current = 0; current < sorted.length; current += 1) {
    for (let candidate = 0; candidate < current; candidate += 1) {
      if (
        sorted[current] % sorted[candidate] === 0 &&
        lengths[candidate] + 1 > lengths[current]
      ) {
        lengths[current] = lengths[candidate] + 1;
        previous[current] = candidate;
      }
    }

    if (lengths[current] > lengths[bestIndex]) {
      bestIndex = current;
    }
  }

  const subset = [];
  for (let index = bestIndex; index !== -1; index = previous[index]) {
    subset.push(sorted[index]);
  }

  return subset.reverse();
}
```

## Follow-up Questions

- Return only the maximum size instead of reconstructing the subset.
- Explain why sorting is enough to make the predecessor relation one-directional.
- Discuss how ties between multiple optimal subsets can be broken.

## Common Pitfalls

- Skipping the sort and losing the simple predecessor ordering.
- Checking only adjacent numbers instead of all earlier divisibility candidates.
- Forgetting to reconstruct the chain after computing the best lengths.
