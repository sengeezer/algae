## Why It Surfaces

This is a fixed-window scan over digits. The useful insight is that only contiguous runs matter, and zeros can cheaply eliminate whole windows from consideration.

## Worked Example

If the digit string is `1234512345` and the window size is `3`, the products are `1*2*3`, `2*3*4`, `3*4*5`, and so on. The best window is the one with the largest adjacent product, not just the globally largest digits in any order.

```js
function problem008(series, span) {
  if (typeof series !== 'string' || series.trim() === '') {
    throw new TypeError('Series must be a non-empty digit string');
  }

  if (!Number.isInteger(span) || span < 1 || span > series.length) {
    throw new RangeError('Span must be a positive integer within the series length');
  }

  const digits = series.replace(/\s+/g, '').split('').map((character) => {
    const digit = Number(character);
    if (!Number.isInteger(digit) || digit < 0 || digit > 9) {
      throw new TypeError('Series must contain only digits and whitespace');
    }
    return digit;
  });

  let best = 0;
  for (let start = 0; start <= digits.length - span; start += 1) {
    let product = 1;
    for (let offset = 0; offset < span; offset += 1) {
      product *= digits[start + offset];
    }
    if (product > best) {
      best = product;
    }
  }

  return best;
}
```

## Follow-up Questions

- Add a zero-aware rolling window to skip unnecessary multiplications.
- Return the winning digit slice along with its product.
- Explain why unordered digit selection would solve a different problem entirely.

## Common Pitfalls

- Forgetting to strip newlines or spaces from the input digit block.
- Treating the problem as a combination search instead of a contiguous window scan.
- Allowing non-digit characters through the parser.
