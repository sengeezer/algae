## Why It Surfaces

Pangram checks are simple set problems disguised as string questions. The trick is filtering down to alphabet letters and then stopping once full coverage is achieved.

## Worked Example

`"The quick brown fox jumps over the lazy dog"` is a pangram because, after ignoring spaces and case, the set of letters contains all 26 uppercase letters.

## Reference Implementations

<ExamplePair>

```js
function checkPangram(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  const letters = new Set();

  for (const char of value.toUpperCase()) {
    if (char >= 'A' && char <= 'Z') {
      letters.add(char);
    }
  }

  return letters.size === 26;
}
```

```ts
export function checkPangram(value: string): boolean {
  const letters = new Set<string>();

  for (const char of value.toUpperCase()) {
    if (char >= 'A' && char <= 'Z') {
      letters.add(char);
    }
  }

  return letters.size === 26;
}
```

## Follow-up Questions

- Return the missing letters instead of only a boolean result.
- Replace the set with a 26-bit mask.
- Adapt the routine for other alphabets or Unicode scripts.

## Common Pitfalls

- Counting punctuation or spaces as letters.
- Forgetting case normalization.
- Using a frequency table when only presence or absence matters.
