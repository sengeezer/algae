## Why It Surfaces

This is a good hybrid problem when interviewers want both preprocessing and fast online queries. Once divisor counts are computed for every number up to a limit, the rest is just a maximum-range query over a static array.

## Worked Example

In `[1, 10]`, the largest divisor count is `4` because the numbers `6`, `8`, and `10` each have four divisors. In `[10, 48]`, the maximum becomes `10`, and a segment tree can return that without refactoring every number in the range.

```js
class MaximumDivisorCountQuery {
  constructor(limit) {
    if (!Number.isInteger(limit) || limit < 1) {
      throw new RangeError('Limit must be a positive integer');
    }

    this.limit = limit;
    const smallestPrimeFactor = this.buildSmallestPrimeFactor(limit);
    this.divisorCounts = this.buildDivisorCounts(limit, smallestPrimeFactor);
    this.size = limit;
    this.tree = new Array(this.size * 2).fill(0);

    for (let value = 1; value <= limit; value += 1) {
      this.tree[this.size + value - 1] = this.divisorCounts[value];
    }

    for (let index = this.size - 1; index > 0; index -= 1) {
      this.tree[index] = Math.max(this.tree[index * 2], this.tree[index * 2 + 1]);
    }
  }

  buildSmallestPrimeFactor(limit) {
    const smallestPrimeFactor = new Array(limit + 1).fill(0);

    for (let value = 2; value <= limit; value += 1) {
      if (smallestPrimeFactor[value] !== 0) {
        continue;
      }

      smallestPrimeFactor[value] = value;
      if (value <= Math.floor(limit / value)) {
        for (let multiple = value * value; multiple <= limit; multiple += value) {
          if (smallestPrimeFactor[multiple] === 0) {
            smallestPrimeFactor[multiple] = value;
          }
        }
      }
    }

    return smallestPrimeFactor;
  }

  buildDivisorCounts(limit, smallestPrimeFactor) {
    const divisorCounts = new Array(limit + 1).fill(0);
    divisorCounts[1] = 1;

    for (let value = 2; value <= limit; value += 1) {
      let remaining = value;
      let total = 1;

      while (remaining > 1) {
        const prime = smallestPrimeFactor[remaining];
        let exponent = 0;

        while (remaining % prime === 0) {
          remaining /= prime;
          exponent += 1;
        }

        total *= exponent + 1;
      }

      divisorCounts[value] = total;
    }

    return divisorCounts;
  }

  query(queryLeft, queryRight) {
    this.validateRange(queryLeft, queryRight);

    let left = queryLeft - 1 + this.size;
    let right = queryRight - 1 + this.size;
    let answer = 0;

    while (left <= right) {
      if (left % 2 === 1) {
        answer = Math.max(answer, this.tree[left]);
        left += 1;
      }

      if (right % 2 === 0) {
        answer = Math.max(answer, this.tree[right]);
        right -= 1;
      }

      left = Math.floor(left / 2);
      right = Math.floor(right / 2);
    }

    return answer;
  }

  validateRange(queryLeft, queryRight) {
    if (
      !Number.isInteger(queryLeft)
      || !Number.isInteger(queryRight)
      || queryLeft < 1
      || queryRight < queryLeft
      || queryRight > this.limit
    ) {
      throw new RangeError('Query range is invalid');
    }
  }
}
```

## Follow-up Questions

- Return one number that achieves the maximum divisor count, not just the count itself.
- Replace the segment tree with a sparse table when the limit is fixed and there are no updates.
- Explain why the smallest-prime-factor sieve speeds up divisor counting compared with trial division.

## Common Pitfalls

- Recomputing prime factorizations from scratch for every query.
- Confusing the divisor count with the number that attains the count.
- Forgetting that the numeric range here is 1-based while the tree storage is 0-based.
