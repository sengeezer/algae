## Why It Surfaces

Bridges are the edge-level counterpart to articulation points. The interview value is in understanding the low-link invariant well enough to say whether a child subtree has any back edge that bypasses the parent edge.

## Worked Example

If a DFS tree edge from `2` to `5` leads into a subtree whose `low[5]` remains greater than `disc[2]`, then nothing below `5` reconnects to `2` or above without using the edge `(2, 5)`. That edge is therefore a bridge.

```js
function bridgesInGraph(graph) {
  const visited = new Array(graph.length).fill(false);
  const disc = new Array(graph.length).fill(0);
  const low = new Array(graph.length).fill(0);
  const bridges = [];
  let time = 0;

  function dfs(node, parent) {
    visited[node] = true;
    disc[node] = time;
    low[node] = time;
    time += 1;

    for (const neighbor of graph[node]) {
      if (neighbor === parent) {
        continue;
      }

      if (!visited[neighbor]) {
        dfs(neighbor, node);
        low[node] = Math.min(low[node], low[neighbor]);

        if (low[neighbor] > disc[node]) {
          bridges.push([node, neighbor]);
        }
      } else {
        low[node] = Math.min(low[node], disc[neighbor]);
      }
    }
  }

  for (let node = 0; node < graph.length; node += 1) {
    if (!visited[node]) {
      dfs(node, -1);
    }
  }

  return bridges;
}
```

## Follow-up Questions

- Output articulation points and bridges in one DFS pass.
- Explain the difference between the bridge condition `>` and the articulation condition `>=`.
- Extend the reasoning to 2-edge-connected components.

## Common Pitfalls

- Forgetting that the algorithm is for undirected graphs.
- Updating low-link values with the parent edge.
- Counting an undirected edge twice without a parent check.
