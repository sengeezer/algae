## Why It Surfaces

This is the side-recovery counterpart to the angle-recovery problem from the prior batch. When two sides and the included angle are known, the law of cosines gives the missing side immediately without converting the triangle into coordinates.

## Worked Example

For `a = 5`, `b = 8`, and included angle `49` degrees, the third side is about `6.04279`.

## Reference Implementations

<ExamplePair>

```js
function thirdSideOfTriangleUsingLawOfCosines(a, b, angleDegrees) {
  if (
    !Number.isFinite(a)
    || !Number.isFinite(b)
    || !Number.isFinite(angleDegrees)
    || a <= 0
    || b <= 0
    || angleDegrees <= 0
    || angleDegrees >= 180
  ) {
    throw new RangeError("Sides must be positive and the included angle must be between 0 and 180 degrees");
  }

  const angleRadians = angleDegrees * Math.PI / 180;
  const squared = a * a + b * b - 2 * a * b * Math.cos(angleRadians);

  return Math.sqrt(Math.max(0, squared));
}
```

## Common Pitfalls

- Feeding degrees into `Math.cos` without converting to radians.
- Using a non-included angle and expecting the formula to stay valid.
- Accepting angles of `0` or `180` degrees, which collapse the triangle.
