## Why It Surfaces

This is a standard recognition problem: the brute-force scan over all sizes works, but the cleaner solution is to stop thinking about rectangles as filled regions and instead think about choosing their boundary lines. A rectangle is determined by any two distinct horizontal lines and any two distinct vertical lines.

## Worked Example

For `N = 2` and `M = 2`, the total is `9`: four `1 x 1` rectangles, two `1 x 2`, two `2 x 1`, and one `2 x 2` rectangle.

## Reference Implementations

<ExamplePair>

```js
function numberOfRectanglesInNMGrid(rows, columns) {
  if (!Number.isInteger(rows) || !Number.isInteger(columns) || rows < 0 || columns < 0) {
    throw new RangeError("Grid dimensions must be non-negative integers");
  }

  return (rows * (rows + 1) * columns * (columns + 1)) / 4;
}
```

## Common Pitfalls

- Mixing up rows and columns with the number of grid lines.
- Re-deriving the answer with nested loops when the closed form is already available.
- Forgetting that zero-sized dimensions should be rejected or treated consistently.
