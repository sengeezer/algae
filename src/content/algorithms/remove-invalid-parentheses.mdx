## Why It Surfaces

The tricky part is not checking validity, it is guaranteeing minimum removals while returning every answer at that minimum depth. BFS is a natural fit because the first balanced level corresponds exactly to the fewest deleted parentheses.

## Worked Example

For `()())()`, deleting one parenthesis can produce `()()()` or `(())()`. BFS finds both at depth `1` and stops before exploring deeper states that would remove too many characters.

```js
function removeInvalidParentheses(text) {
  if (typeof text !== 'string') {
    throw new TypeError('Input must be a string');
  }

  const results = [];
  const queue = [text];
  const visited = new Set([text]);
  let foundLevel = false;

  while (queue.length > 0) {
    const current = queue.shift();

    if (isBalanced(current)) {
      results.push(current);
      foundLevel = true;
    }

    if (foundLevel) {
      continue;
    }

    for (let index = 0; index < current.length; index += 1) {
      if (current[index] !== '(' && current[index] !== ')') {
        continue;
      }

      const next = current.slice(0, index) + current.slice(index + 1);
      if (!visited.has(next)) {
        visited.add(next);
        queue.push(next);
      }
    }
  }

  return [...new Set(results)];
}

function isBalanced(text) {
  let balance = 0;

  for (const char of text) {
    if (char === '(') {
      balance += 1;
    } else if (char === ')') {
      balance -= 1;
      if (balance < 0) {
        return false;
      }
    }
  }

  return balance === 0;
}
```

## Common Pitfalls

- Continuing to deeper BFS levels after valid answers have already been found.
- Forgetting to deduplicate intermediate strings.
- Returning only one balanced string when the prompt asks for all of them.
