## Why It Surfaces

Two Clique is a good interview problem because the direct statement sounds unfamiliar, but the solution is mostly graph translation. Once you realize that two cliques in the original graph correspond to two independent sets in the complement, the problem becomes an ordinary bipartite check.

## Worked Example

If two vertices are missing an edge in the original graph, they cannot belong to the same clique. In the complement graph that missing edge becomes a real edge, so a successful two-coloring there tells you exactly how to split the original vertices into two cliques.

```js
function twoCliqueProblem(graph) {
  const size = graph.length;
  const complement = Array.from({ length: size }, () => []);

  for (let row = 0; row < size; row += 1) {
    for (let column = 0; column < size; column += 1) {
      if (row !== column && graph[row][column] === 0) {
        complement[row].push(column);
      }
    }
  }

  const color = new Array(size).fill(-1);

  for (let start = 0; start < size; start += 1) {
    if (color[start] !== -1) {
      continue;
    }

    const queue = [start];
    color[start] = 0;

    for (let head = 0; head < queue.length; head += 1) {
      const node = queue[head];

      for (const neighbor of complement[node]) {
        if (color[neighbor] === -1) {
          color[neighbor] = 1 - color[node];
          queue.push(neighbor);
        } else if (color[neighbor] === color[node]) {
          return false;
        }
      }
    }
  }

  return true;
}
```

## Follow-up Questions

- Return the actual two cliques instead of only answering yes or no.
- Explain why the complement graph is the right transformation instead of the original graph itself.
- Compare this with ordinary bipartite graph validation.

## Common Pitfalls

- Forgetting to exclude self-loops when building the complement graph.
- Checking bipartiteness on the original graph rather than the complement.
- Treating a missing edge inside a clique as harmless when it actually breaks the definition.
