## Why It Surfaces

This is a frequency-count problem disguised as a palindrome problem. The core observation is that a palindrome can have at most one character with an odd count.

## Worked Example

`"carrace"` can be rearranged into `"racecar"`. Its counts are `c: 2`, `a: 2`, `r: 2`, `e: 1`, so only one character has an odd frequency.

## Reference Implementations

<ExamplePair>

```js
function checkRearrangePalindrome(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  const oddChars = new Set();

  for (const char of value.toLowerCase().replace(/[^a-z0-9]/g, '')) {
    if (oddChars.has(char)) {
      oddChars.delete(char);
    } else {
      oddChars.add(char);
    }
  }

  return oddChars.size <= 1;
}
```

```ts
export function checkRearrangePalindrome(value: string): boolean {
  const oddChars = new Set<string>();

  for (const char of value.toLowerCase().replace(/[^a-z0-9]/g, '')) {
    if (oddChars.has(char)) {
      oddChars.delete(char);
    } else {
      oddChars.add(char);
    }
  }

  return oddChars.size <= 1;
}
```

## Follow-up Questions

- Construct one valid palindrome instead of returning only a boolean.
- Preserve spaces and punctuation instead of normalizing them away.
- Solve the same problem with a bitmask when the alphabet is fixed.

## Common Pitfalls

- Checking whether the original string is already a palindrome instead of whether a permutation can be one.
- Forgetting that at most one odd frequency is allowed.
- Leaving normalization rules undefined when the input may contain spaces or punctuation.
