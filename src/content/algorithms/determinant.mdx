## Why It Surfaces

Determinant problems are a good test of whether you can decompose a matrix operation into recursive subproblems and keep the shape validation straight.

## Worked Example

For the matrix `[[1, 2], [3, 4]]`, the determinant is `1 * 4 - 2 * 3 = -2`.

```js
function validateSquareMatrix(matrix) {
  if (!Array.isArray(matrix) || matrix.length === 0 || !matrix.every((row) => Array.isArray(row) && row.length === matrix.length)) {
    throw new RangeError('Input must be a non-empty square matrix');
  }
}

function minorMatrix(matrix, rowToSkip, colToSkip) {
  return matrix
    .filter((_, row) => row !== rowToSkip)
    .map((row) => row.filter((_, col) => col !== colToSkip));
}

function determinant(matrix) {
  validateSquareMatrix(matrix);

  if (matrix.length === 1) {
    return matrix[0][0];
  }

  if (matrix.length === 2) {
    return matrix[0][0] * matrix[1][1] - matrix[0][1] * matrix[1][0];
  }

  let total = 0;

  for (let col = 0; col < matrix.length; col += 1) {
    const sign = col % 2 === 0 ? 1 : -1;
    total += sign * matrix[0][col] * determinant(minorMatrix(matrix, 0, col));
  }

  return total;
}
```

## Follow-up Questions

- Compare Laplace expansion with Gaussian elimination for larger matrices.
- Explain how the determinant changes under row swaps and row scaling.
- Use the determinant to test whether a matrix is singular.

## Common Pitfalls

- Forgetting to validate that the matrix is square.
- Dropping the alternating cofactor sign during expansion.
- Using Laplace expansion on large matrices without acknowledging the factorial growth.
