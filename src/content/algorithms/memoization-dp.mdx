## Why It Surfaces

Memoization is what turns a workable recursive idea into an efficient solution. Instead of recomputing the same suffix, index, or remaining capacity over and over, you solve that state once and store the result.

<Callout title="State test" tone="tip">
  If you can name the recursive state clearly, memoization is often only one hash map away.
</Callout>

## Worked Example

The climbing-stairs recurrence repeats the same smaller stair counts many times. Caching `n` after it is solved once reduces the recursion from exponential growth to linear work.

## Reference Implementations

<ExamplePair>

```js
function climbStairs(n) {
  const memo = new Map();

  function solve(stepsRemaining) {
    if (stepsRemaining <= 2) {
      return stepsRemaining;
    }

    if (memo.has(stepsRemaining)) {
      return memo.get(stepsRemaining);
    }

    const ways = solve(stepsRemaining - 1) + solve(stepsRemaining - 2);
    memo.set(stepsRemaining, ways);
    return ways;
  }

  return solve(n);
}
```

```ts
export function climbStairs(n: number): number {
  const memo = new Map<number, number>();

  function solve(stepsRemaining: number): number {
    if (stepsRemaining <= 2) {
      return stepsRemaining;
    }

    const cached = memo.get(stepsRemaining);
    if (cached !== undefined) {
      return cached;
    }

    const ways = solve(stepsRemaining - 1) + solve(stepsRemaining - 2);
    memo.set(stepsRemaining, ways);
    return ways;
  }

  return solve(n);
}
```

</ExamplePair>

## Interview Notes

- Define the state before writing the recursion.
- Use a cache key that uniquely identifies equivalent subproblems.
- Mention when a bottom-up table might be more space-efficient.
