## Why It Surfaces

This is the point where brute-force divisibility checking becomes obviously wasteful. The useful insight is that the answer is just the lcm of all integers from `1` to `n`.

## Worked Example

For `10`, the smallest number divisible by every integer from `1` through `10` is `2520`. You can reach it by folding `lcm` from left to right across the range.

## Reference Implementations

<ExamplePair>

```js
function gcd(left, right) {
  let a = left;
  let b = right;

  while (b !== 0) {
    [a, b] = [b, a % b];
  }

  return a;
}

function lcm(left, right) {
  return (left / gcd(left, right)) * right;
}

function problem005(limit) {
  if (!Number.isInteger(limit) || limit < 1) {
    throw new RangeError('Limit must be a positive integer');
  }

  let result = 1;
  for (let value = 2; value <= limit; value += 1) {
    result = lcm(result, value);
  }

  return result;
}
```

## Follow-up Questions

- Compare the lcm fold with a brute-force incrementing search.
- Explain why dividing before multiplying is safer in the lcm formula.
- Extend the same idea to a custom array of divisors.

## Common Pitfalls

- Testing candidate numbers one by one instead of folding lcm.
- Forgetting that every earlier divisor is already covered once the lcm is updated.
- Multiplying before dividing and inflating intermediate values unnecessarily.
