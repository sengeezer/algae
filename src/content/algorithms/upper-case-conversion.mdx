## Why It Surfaces

Uppercasing is the mirror image of lowercasing, so it is a clean way to test whether someone actually understands ASCII offsets instead of memorizing one direction. It is also a useful building block for normalization pipelines.

## Worked Example

`AlGae 123` becomes `ALGAE 123`. Only ASCII lowercase letters change; digits, spaces, and punctuation are copied through untouched.

## Reference Implementations

<ExamplePair>

```js
function upperCaseConversion(input) {
  if (typeof input !== 'string') {
    throw new TypeError('Expected a string input');
  }

  return input
    .split('')
    .map((char) => {
      const code = char.charCodeAt(0);
      if (code >= 97 && code <= 122) {
        return String.fromCharCode(code - 32);
      }
      return char;
    })
    .join('');
}
```

```ts

}
```

## Common Pitfalls

- Applying the ASCII offset to characters outside the lowercase range.
- Forgetting that locale-aware casing has edge cases not captured here.
- Returning early after the first lowercase match instead of transforming the full string.
