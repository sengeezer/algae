## Why It Surfaces

Deque questions show up when a normal queue is almost enough but the problem also needs efficient eviction or insertion at the opposite end. The structure is especially useful when candidates must stay ordered while old values fall out of a window.

## Worked Example

In a capacity-5 deque, if the front index is `1` and the size is `3`, then the live elements occupy logical positions `1`, `2`, and `3`. Inserting at the front moves the front index to `(1 - 1 + 5) % 5 = 0` without shifting anything.

<ExamplePair>

```js
class Deque {
  constructor(capacity) {
    this.values = new Array(capacity);
    this.front = 0;
    this.count = 0;
  }

  insertFront(value) {
    if (this.count === this.values.length) {
      return false;
    }

    this.front = (this.front - 1 + this.values.length) % this.values.length;
    this.values[this.front] = value;
    this.count += 1;
    return true;
  }

  insertRear(value) {
    if (this.count === this.values.length) {
      return false;
    }

    const rear = (this.front + this.count) % this.values.length;
    this.values[rear] = value;
    this.count += 1;
    return true;
  }

  deleteFront() {
    if (this.count === 0) {
      return undefined;
    }

    const value = this.values[this.front];
    this.values[this.front] = undefined;
    this.front = (this.front + 1) % this.values.length;
    this.count -= 1;
    return value;
  }

  deleteRear() {
    if (this.count === 0) {
      return undefined;
    }

    const rear = (this.front + this.count - 1) % this.values.length;
    const value = this.values[rear];
    this.values[rear] = undefined;
    this.count -= 1;
    return value;
  }
}
```

## Common Pitfalls

- Shifting the whole array instead of wrapping indices.
- Losing track of full versus empty state.
- Forgetting that rear indexing must also wrap with modulo arithmetic.
