## Why It Surfaces

Manacher's algorithm is what you reach for when center expansion is conceptually right but still too slow. Its real trick is not “expand faster,” but “reuse what a mirrored center already proved” while the current palindrome boundary still covers you.

## Worked Example

For `abba`, transform the text into `^#a#b#b#a#$`. The separators make even and odd palindromes look the same, so the algorithm can store one radius per center and mirror known values across the current best palindrome.

```js
function manachersAlgorithm(text) {
  if (typeof text !== 'string') {
    throw new TypeError('Expected a string');
  }

  if (text.length === 0) {
    return '';
  }

  const transformed = ['^'];
  for (const char of text) {
    transformed.push('#', char);
  }
  transformed.push('#', '$');

  const radius = new Array(transformed.length).fill(0);
  let center = 0;
  let right = 0;

  for (let index = 1; index < transformed.length - 1; index += 1) {
    const mirror = 2 * center - index;

    if (index < right) {
      radius[index] = Math.min(right - index, radius[mirror]);
    }

    while (transformed[index + radius[index] + 1] === transformed[index - radius[index] - 1]) {
      radius[index] += 1;
    }

    if (index + radius[index] > right) {
      center = index;
      right = index + radius[index];
    }
  }

  let bestCenter = 0;
  for (let index = 1; index < radius.length - 1; index += 1) {
    if (radius[index] > radius[bestCenter]) {
      bestCenter = index;
    }
  }

  const start = Math.floor((bestCenter - radius[bestCenter]) / 2);
  return text.slice(start, start + radius[bestCenter]);
}
```

## Follow-up Questions

- Explain why the transformed string removes the odd-versus-even case split.

## Common Pitfalls

- Forgetting the sentinels that prevent bounds checks from becoming messy.
- Mixing transformed-string indices with original-string indices.
- Re-expanding work that the mirror case already proved unnecessary.
