## Why It Surfaces

Kishnamurthy-number checks are a good digit-manipulation exercise because they reward a clean digit loop and a small factorial cache instead of repeated expensive recomputation.

## Worked Example

`145` is a Kishnamurthy number because `1! + 4! + 5! = 1 + 24 + 120 = 145`.

## Reference Implementations

<ExamplePair>

```js
const DIGIT_FACTORIALS = [1, 1, 2, 6, 24, 120, 720, 5040, 40320, 362880];

function checkKishnamurthyNumber(value) {
  if (!Number.isInteger(value) || value <= 0) {
    return false;
  }

  let remaining = value;
  let total = 0;

  while (remaining > 0) {
    total += DIGIT_FACTORIALS[remaining % 10];
    remaining = Math.floor(remaining / 10);
  }

  return total === value;
}
```

## Follow-up Questions

- Compare Kishnamurthy numbers with Armstrong numbers and explain the changed digit transform.
- Cache the factorial table once instead of recomputing it per digit.
- Generalize the routine to other bases with a base-specific digit set.

## Common Pitfalls

- Recomputing factorials from scratch inside the digit loop.
- Treating `0` or negative inputs as valid under this definition.
- Forgetting to stop once the remaining number has no digits left.
