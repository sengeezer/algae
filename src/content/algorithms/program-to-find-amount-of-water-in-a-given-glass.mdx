## Why It Surfaces

This problem looks like a probability or physics puzzle, but the implementation is pure state propagation. Once a glass exceeds one unit, its overflow is split evenly to the two glasses directly below, so the whole triangle can be built row by row.

## Worked Example

If `2` units are poured into the top glass, row `0` keeps `1`, and the extra `1` is split evenly into row `1`, producing `0.5` in both glasses below.

## Reference Implementations

<ExamplePair>

```js
function programToFindAmountOfWaterInAGivenGlass(poured, targetRow, targetGlass) {
  const glasses = Array.from({ length: targetRow + 2 }, (_, row) => new Array(row + 1).fill(0));
  glasses[0][0] = poured;

  for (let row = 0; row <= targetRow; row += 1) {
    for (let glass = 0; glass <= row; glass += 1) {
      const overflow = Math.max(0, glasses[row][glass] - 1);
      if (overflow > 0) {
        glasses[row][glass] = 1;
        glasses[row + 1][glass] += overflow / 2;
        glasses[row + 1][glass + 1] += overflow / 2;
      }
    }
  }

  return Math.min(1, glasses[targetRow][targetGlass] ?? 0);
}
```

## Follow-up Questions

- Return the whole triangle of fill levels up to the queried row.
- Use a rolling row representation instead of storing the entire triangle.
- Explain why no glass ever needs to store more than `1` unit after processing overflow.

## Common Pitfalls

- Forgetting to cap the current glass at `1` before distributing the overflow.
- Propagating negative overflow when a glass is not full.
- Allocating too few rows and losing spill data for the queried depth.
