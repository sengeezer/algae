## Why It Surfaces

This problem is less about clever data structures than about picking the right level of work. You do not need to sort the array to know the second-largest distinct value. You only need to remember the largest and runner-up seen so far.

## Worked Example

For `[9, 4, 9, 7, 2]`, the best value is `9`. The second-best distinct value becomes `7`, not the repeated `9`, because duplicates do not count as a new rank.

## Reference Implementations

<ExamplePair>

```js
function findSecondLargestElement(values) {
  if (!Array.isArray(values) || values.length < 2) {
    throw new RangeError('Values must contain at least two numbers');
  }

  let largest = -Infinity;
  let secondLargest = -Infinity;

  for (const value of values) {
    if (value > largest) {
      secondLargest = largest;
      largest = value;
    } else if (value > secondLargest && value < largest) {
      secondLargest = value;
    }
  }

  if (secondLargest === -Infinity) {
    throw new RangeError('Values must contain at least two distinct numbers');
  }

  return secondLargest;
}
```

## Follow-up Questions

- Return the third-largest distinct value as well.
- Adapt the scan for streaming input where values arrive one at a time.
- Compare this with using a heap or full sort for the same task.

## Common Pitfalls

- Sorting first when the prompt only asks for one rank statistic.
- Letting duplicates of the maximum overwrite the second-largest value.
- Returning a sentinel silently when the input lacks two distinct numbers.
