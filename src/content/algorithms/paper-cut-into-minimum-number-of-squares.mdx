## Why It Surfaces

This problem is useful precisely because the first greedy instinct is wrong. Cutting the largest possible square at every step fails on `6 x 7`, so the real interview move is to define a reusable state for the remaining paper and memoize the search.

## Worked Example

For `a = 6` and `b = 7`, greedy would cut a `6 x 6` square first and then leave six `1 x 1` squares, for a total of `7`. The optimal answer is `5`, which is why you need a stateful search over possible square sizes rather than a locally largest choice.

```js
function paperCutIntoMinimumNumberOfSquares(height, width) {
  if (height <= 0 || width <= 0) {
    return 0;
  }

  const rows = Math.min(height, width);
  const columns = Math.max(height, width);
  const memo = new Map();

  function solve(remainingHeights) {
    const stateKey = remainingHeights.join(',');
    const cached = memo.get(stateKey);

    if (cached !== undefined) {
      return cached;
    }

    let tallest = 0;
    let start = -1;

    for (let column = 0; column < columns; column += 1) {
      if (remainingHeights[column] > tallest) {
        tallest = remainingHeights[column];
        start = column;
      }
    }

    if (tallest === 0) {
      return 0;
    }

    let end = start;

    while (end < columns && remainingHeights[end] === tallest) {
      end += 1;
    }

    const maxSquareSize = Math.min(tallest, end - start);
    let best = Number.POSITIVE_INFINITY;

    for (let size = maxSquareSize; size >= 1; size -= 1) {
      const next = [...remainingHeights];

      for (let offset = 0; offset < size; offset += 1) {
        next[start + offset] -= size;
      }

      best = Math.min(best, 1 + solve(next));
    }

    memo.set(stateKey, best);
    return best;
  }

  return solve(new Array(columns).fill(rows));
}
```

## Follow-up Questions

- Explain exactly why the largest-square greedy rule fails on `6 x 7`.
- Return one optimal sequence of square cuts instead of only the minimum count.
- Discuss how normalizing the rectangle so `height <= width` shrinks the state space.

## Common Pitfalls

- Assuming the problem is greedy because the source index lists it on a Greedy page.
- Storing the whole remaining grid instead of a compressed skyline state.
- Forgetting that equivalent skyline states should reuse the same memoized answer.
