## Why It Surfaces

This question tests whether you can do clean pointer work under sorted-order constraints. Each step has only one real choice: splice in the smaller front node and advance that list.

## Worked Example

If the lists start as `1 -> 3 -> 7` and `1 -> 2 -> 5`, the merged result begins `1 -> 1 -> 2 -> 3 -> 5 -> 7` without creating new data nodes.

<Callout title="Implementation cue" tone="tip">
  Use a dummy head and a `tail` pointer. That removes almost all first-node edge cases from the code.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function mergeLists(first, second) {
  const dummy = { next: null };
  let tail = dummy;
  let left = first;
  let right = second;

  while (left !== null && right !== null) {
    if (left.value <= right.value) {
      tail.next = left;
      left = left.next;
    } else {
      tail.next = right;
      right = right.next;
    }

    tail = tail.next;
  }

  tail.next = left ?? right;
  return dummy.next;
}
```

```ts
type Node = {
  value: number;
  next: Node | null;
};

export function mergeLists(first: Node | null, second: Node | null): Node | null {
  const dummy: Node = { value: 0, next: null };
  let tail = dummy;
  let left = first;
  let right = second;

  while (left !== null && right !== null) {
    if (left.value <= right.value) {
      tail.next = left;
      left = left.next;
    } else {
      tail.next = right;
      right = right.next;
    }

    tail = tail.next;
  }

  tail.next = left ?? right;
  return dummy.next;
}
```

## Common Pitfalls

- Recreating nodes unnecessarily instead of relinking the existing ones.
- Forgetting to attach the remainder once one list ends.
- Mishandling the head of the merged list without a dummy node.
