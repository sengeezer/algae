## Why It Surfaces

This is a classic backtracking problem where the search tree is easy to visualize. It is useful because it makes the factorial cost of exhaustive generation explicit.

## Worked Example

For `"abc"`, pick each character as the first position and recurse on the remaining two characters. The result set is `['abc', 'acb', 'bac', 'bca', 'cab', 'cba']`.

## Reference Implementations

<ExamplePair>

```js
function createPermutations(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  if (value.length <= 1) {
    return [value];
  }

  const permutations = [];

  for (let index = 0; index < value.length; index += 1) {
    const picked = value[index];
    const rest = value.slice(0, index) + value.slice(index + 1);

    for (const suffix of createPermutations(rest)) {
      permutations.push(picked + suffix);
    }
  }

  return permutations;
}
```

## Follow-up Questions

- Deduplicate the output when the string contains repeated characters.
- Generate permutations lazily instead of building them all at once.
- Stop early once a permutation satisfies a predicate.

## Common Pitfalls

- Underestimating the factorial output size.
- Forgetting the base case for empty or one-character strings.
- Accidentally mutating shared state across recursive branches.
