## Why It Surfaces

This is a counting problem disguised as pathfinding. Since every valid route contains the same number of right and down steps, the answer is just the number of ways to choose where one step type appears.

## Worked Example

For a `2 x 2` grid, every valid route contains two rights and two downs. The count is therefore `C(4, 2) = 6`.

## Reference Implementations

<ExamplePair>

```js
function problem015(gridSize = 20) {
  if (!Number.isInteger(gridSize) || gridSize < 0) {
    throw new RangeError('Grid size must be a non-negative integer');
  }

  let paths = 1;

  for (let step = 1; step <= gridSize; step += 1) {
    paths = (paths * (gridSize + step)) / step;
  }

  return paths;
}
```

## Common Pitfalls

- Enumerating every path instead of counting combinations directly.
- Forgetting that the route has exactly `2n` moves in an `n x n` grid.
- Ignoring safe-integer limits if the function is generalized to larger grids.
