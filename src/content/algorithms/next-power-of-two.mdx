## Why It Surfaces

This question is usually about sizing rather than pure bit theory. The key distinction is whether the prompt wants the next larger power or the next power that is at least the input.

## Worked Example

For `19`, keep doubling until the running power exceeds the highest set bit of `19`, producing `32`. For `16`, the answer stays `16` because it is already a power of two.

## Reference Implementations

<ExamplePair>

```js
function nextPowerOfTwo(value) {
  if (value <= 1) {
    return 1;
  }

  if ((value & (value - 1)) === 0) {
    return value;
  }

  let result = 1;
  let remaining = value;

  while (remaining > 0) {
    result <<= 1;
    remaining >>= 1;
  }

  return result;
}
```

## Follow-up Questions

- Implement the same routine with bit spreading instead of a loop.
- Explain how this behaves for `0`, `1`, and negative inputs.
- Pair the routine with a power-of-two validator and simplify the branch logic.

## Common Pitfalls

- Returning the next larger power even when the input is already a power of two.
- Ignoring small base cases around `0` and `1`.
- Forgetting that JavaScript bit shifts operate on 32-bit integers.
