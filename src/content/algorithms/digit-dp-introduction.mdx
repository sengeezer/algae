## Why It Surfaces

Digit DP is the technique interviewers reach for when the bound is huge but the property lives entirely in the decimal representation. The trick is to stop thinking about every integer one by one and instead count all valid completions of a digit prefix in a single DP state.

## Worked Example

The article's introductory problem asks for the sum of all digits appearing in integers from `5` through `11`. Writing the answer as `G(11) - G(4)` yields `38`, because the digits are `5 + 6 + 7 + 8 + 9 + 1 + 0 + 1 + 1`.

```js
function digitDPIntroduction(lower, upper) {
  if (typeof lower !== 'bigint' || typeof upper !== 'bigint') {
    throw new TypeError('lower and upper must be bigint values');
  }

  if (lower < 0n || upper < lower) {
    throw new RangeError('expected 0n <= lower <= upper');
  }

  function sumOfDigitsUpTo(limit) {
    if (limit < 0n) {
      return 0n;
    }

    const digits = Array.from(limit.toString(), (digit) => Number(digit));
    const memo = new Map();

    function solve(index, tight, sum) {
      if (index === digits.length) {
        return BigInt(sum);
      }

      const key = `${index}|${tight ? 1 : 0}|${sum}`;
      const cached = memo.get(key);
      if (cached !== undefined) {
        return cached;
      }

      let total = 0n;
      const maxDigit = tight ? digits[index] : 9;

      for (let digit = 0; digit <= maxDigit; digit += 1) {
        total += solve(index + 1, tight && digit === maxDigit, sum + digit);
      }

      memo.set(key, total);
      return total;
    }

    return solve(0, true, 0);
  }

  return sumOfDigitsUpTo(upper) - sumOfDigitsUpTo(lower - 1n);
}
```

## Interview Notes

- Say `answer = G(b) - G(a - 1)` early; it shows you know the pattern.
- Choose a state that captures only what future digits still need to know.
- Mention `tight` explicitly, because it is what separates unrestricted digits from bound-constrained digits.

## Common Pitfalls

- Iterating the whole range even though the limits are too large.
- Carrying unnecessary state that blows up the memo table.
- Forgetting to handle leading zeros consistently when defining the digit property.
