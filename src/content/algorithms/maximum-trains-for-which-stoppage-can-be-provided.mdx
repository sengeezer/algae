## Why It Surfaces

This looks like one large railway scheduling problem, but the fixed-platform constraint simplifies it. Once each train is locked to a platform, trains on different platforms never conflict, so the problem decomposes into one interval-scheduling scan per platform.

## Worked Example

If several trains target platform `1`, sort only those by departure time and keep the earliest-finishing compatible set. Do the same for platform `2`, platform `3`, and so on, then sum the accepted counts. The key is that a conflict on one platform says nothing about another platform.

## Reference Implementations

<ExamplePair>

```js
function maximumTrainsForWhichStoppageCanBeProvided(platformCount, trains) {
  const grouped = Array.from({ length: platformCount + 1 }, () => []);

  for (const [arrival, departure, platform] of trains) {
    grouped[platform].push([arrival, departure]);
  }

  let accepted = 0;

  for (const platformTrains of grouped) {
    if (platformTrains.length === 0) {
      continue;
    }

    platformTrains.sort((left, right) => left[1] - right[1] || left[0] - right[0]);
    let currentDeparture = -Infinity;

    for (const [arrival, departure] of platformTrains) {
      if (arrival < currentDeparture) {
        continue;
      }

      accepted += 1;
      currentDeparture = departure;
    }
  }

  return accepted;
}
```

```ts
export function maximumTrainsForWhichStoppageCanBeProvided(
  platformCount: number,
  trains: Array<[number, number, number]>,
): number {
  const grouped: Array<Array<[number, number]>> = Array.from(
    { length: platformCount + 1 },
    () => [],
  );

  for (const [arrival, departure, platform] of trains) {
    grouped[platform].push([arrival, departure]);
  }

  let accepted = 0;

  for (const platformTrains of grouped) {
    if (platformTrains.length === 0) {
      continue;
    }

    platformTrains.sort((left, right) => left[1] - right[1] || left[0] - right[0]);
    let currentDeparture = Number.NEGATIVE_INFINITY;

    for (const [arrival, departure] of platformTrains) {
      if (arrival < currentDeparture) {
        continue;
      }

      accepted += 1;
      currentDeparture = departure;
    }
  }

  return accepted;
}
```

## Follow-up Questions

- Explain why equal departure and arrival times allow platform reuse in this version.
- Return the accepted train IDs for each platform rather than only the count.
- Contrast this problem with `Minimum Platforms Required`, where platforms are interchangeable instead of fixed per train.

## Common Pitfalls

- Treating the problem as one global interval set instead of decomposing by platform.
- Applying the overlap rule from `Minimum Platforms Required` even though same-time reuse is allowed here.
- Forgetting that the optimal rule within each platform is still earliest departure first.
