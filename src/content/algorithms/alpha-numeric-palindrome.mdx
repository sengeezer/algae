## Why It Surfaces

This is a palindrome question with a normalization layer on top. The interviewer usually wants to see whether you separate the input-cleaning rule from the actual palindrome check.

## Worked Example

`"A man, a plan, a canal: Panama!"` becomes `"amanaplanacanalpanama"` after removing non-alphanumeric characters and lowercasing, so it reads the same forward and backward.

## Reference Implementations

<ExamplePair>

```js
function alphaNumericPalindrome(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  const normalized = value.replace(/[^a-z0-9]+/gi, '').toLowerCase();

  let left = 0;
  let right = normalized.length - 1;

  while (left < right) {
    if (normalized[left] !== normalized[right]) {
      return false;
    }

    left += 1;
    right -= 1;
  }

  return true;
}
```

## Follow-up Questions

- Solve it with two pointers directly on the original string without building a normalized copy.
- Support full Unicode letters and digits instead of ASCII-only normalization.
- Return the longest palindromic normalized substring instead of a boolean.

## Common Pitfalls

- Forgetting to ignore punctuation or spaces when the prompt requires it.
- Comparing case-sensitively after normalization is expected.
- Using reversal alone without clarifying the memory cost.
