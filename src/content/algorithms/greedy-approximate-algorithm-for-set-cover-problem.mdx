## Why It Surfaces

Set cover is one of the classic NP-hard interview prompts where the exact solution is usually too expensive, so the discussion shifts to a principled approximation. The greedy rule is to keep choosing the set with the best cost-effectiveness ratio, meaning the lowest `cost / newlyCoveredElements` among the remaining candidates.

## Worked Example

Let the universe be `[1, 2, 3, 4, 5]` and the weighted sets be `S1 = {4, 1, 3}` with cost `5`, `S2 = {2, 5}` with cost `10`, and `S3 = {1, 4, 3, 2}` with cost `3`. The greedy heuristic picks `S3` first because `3 / 4` is the best ratio, then adds `S2` to cover `5`, producing a total cost of `13`.

```js
function greedyApproximateAlgorithmForSetCoverProblem(universe, sets) {
  const uncovered = new Set(universe);
  const selectedSetIds = [];
  let totalCost = 0;

  while (uncovered.size > 0) {
    let bestSet = null;
    let bestNewElements = [];
    let bestRatio = Number.POSITIVE_INFINITY;

    for (const candidate of sets) {
      const newElements = candidate.elements.filter((element) => uncovered.has(element));

      if (newElements.length === 0) {
        continue;
      }

      const ratio = candidate.cost / newElements.length;

      if (
        ratio < bestRatio ||
        (ratio === bestRatio && newElements.length > bestNewElements.length) ||
        (ratio === bestRatio &&
          newElements.length === bestNewElements.length &&
          candidate.cost < bestSet.cost) ||
        (ratio === bestRatio &&
          newElements.length === bestNewElements.length &&
          candidate.cost === bestSet.cost &&
          candidate.id < bestSet.id)
      ) {
        bestSet = candidate;
        bestNewElements = newElements;
        bestRatio = ratio;
      }
    }

    if (bestSet === null) {
      break;
    }

    selectedSetIds.push(bestSet.id);
    totalCost += bestSet.cost;

    for (const element of bestNewElements) {
      uncovered.delete(element);
    }
  }

  return {
    selectedSetIds,
    totalCost,
    coveredElements: universe.filter((element) => !uncovered.has(element)),
    uncoveredElements: Array.from(uncovered),
  };
}
```

## Follow-up Questions

- Return the chosen sets themselves, not only their identifiers and total cost.
- Add support for arbitrary element types instead of assuming numeric universe members.
- Compare this approximation with an exact bitmask dynamic-programming solution for small universes.

## Common Pitfalls

- Choosing the cheapest set overall instead of the cheapest set per newly covered element.
- Forgetting to ignore elements that are already covered when recomputing each ratio.
- Assuming the greedy result is guaranteed to be globally optimal.
