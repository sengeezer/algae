## Why It Surfaces

Bin packing is a classic approximation problem: the exact optimum is hard, but practical systems still need a fast packing policy. The source page surveys several heuristics, and this reference focuses on First Fit Decreasing because it is a strong interview-friendly baseline that usually packs more tightly than the simpler online variants.

## Worked Example

For weights `[2, 5, 4, 7, 1, 3, 8]` and `capacity = 10`, First Fit Decreasing sorts the items to `[8, 7, 5, 4, 3, 2, 1]` and produces bins `[[8, 2], [7, 3], [5, 4, 1]]`. The algorithm uses `3` bins.

## Reference Implementations

<ExamplePair>

```js
function binPackingProblemMinimizeNumberOfUsedBins(weights, capacity) {
  const sortedWeights = [...weights].sort((left, right) => right - left);
  const bins = [];
  const remainingCapacities = [];

  for (const weight of sortedWeights) {
    let placed = false;

    for (let binIndex = 0; binIndex < bins.length; binIndex += 1) {
      if (remainingCapacities[binIndex] >= weight) {
        bins[binIndex].push(weight);
        remainingCapacities[binIndex] -= weight;
        placed = true;
        break;
      }
    }

    if (!placed) {
      bins.push([weight]);
      remainingCapacities.push(capacity - weight);
    }
  }

  return {
    binCount: bins.length,
    bins,
    remainingCapacities,
  };
}
```

## Common Pitfalls

- Calling the result optimal even though First Fit Decreasing is still an approximation.
- Forgetting that the algorithm is offline because it sorts items first.
- Recomputing used capacity from scratch for every placement instead of tracking remaining room per bin.
