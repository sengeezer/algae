## Why It Surfaces

Hex-to-RGB is the inverse of RGB-to-hex and shows how compact hex color codes are really just three byte values written in base 16. It is a very common format bridge in UI and graphics work.

## Worked Example

`#FF6347` splits into `FF`, `63`, and `47`, which decode to `{ r: 255, g: 99, b: 71 }`. The work is just validation plus three base-16 parses.

## Reference Implementations

<ExamplePair>

```js
function hexToRgb(hexString) {
  if (typeof hexString !== 'string') {
    throw new TypeError('Expected a string input');
  }

  const normalized = hexString.startsWith('#') ? hexString.slice(1) : hexString;
  if (!/^[0-9A-Fa-f]{6}$/.test(normalized)) {
    throw new RangeError('Expected a 6-digit hexadecimal color');
  }

  return {
    r: Number.parseInt(normalized.slice(0, 2), 16),
    g: Number.parseInt(normalized.slice(2, 4), 16),
    b: Number.parseInt(normalized.slice(4, 6), 16),
  };
}
```

## Follow-up Questions

- Add support for shorthand forms like `#FFF`.
- Explain why each color channel is one byte.
- Normalize the output for downstream APIs that want percentages or floats from `0` to `1`.

## Common Pitfalls

- Forgetting to handle an optional leading `#`.
- Accepting malformed lengths like `#FFFF` or non-hex characters.
- Mixing up the channel order when slicing the string.
