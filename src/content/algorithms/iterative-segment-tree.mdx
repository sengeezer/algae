## Why It Surfaces

Interviewers sometimes want the segment tree idea without recursive bookkeeping. The iterative layout stores leaves at indices `[n, 2n)` and lets queries walk inward from both ends while updates climb back to the root.

## Worked Example

For `[2, 6, 7, 5, 18, 86, 54, 2]`, querying `[2, 7]` returns `86`. If index `3` is updated from `5` to `90`, a second query over `[2, 6]` returns `90` after only the affected parent chain is recomputed.

```js
class IterativeSegmentTree {
  constructor(values) {
    if (!Array.isArray(values) || values.length === 0) {
      throw new RangeError('Values must be a non-empty array');
    }

    this.size = values.length;
    this.tree = new Array(this.size * 2).fill(-Infinity);

    for (let index = 0; index < this.size; index += 1) {
      this.tree[this.size + index] = values[index];
    }

    for (let index = this.size - 1; index > 0; index -= 1) {
      this.tree[index] = Math.max(this.tree[index * 2], this.tree[index * 2 + 1]);
    }
  }

  update(index, value) {
    this.validateIndex(index);

    let position = index + this.size;
    this.tree[position] = value;

    while (position > 1) {
      position = Math.floor(position / 2);
      this.tree[position] = Math.max(this.tree[position * 2], this.tree[position * 2 + 1]);
    }
  }

  query(queryLeft, queryRight) {
    this.validateRange(queryLeft, queryRight);

    let left = queryLeft + this.size;
    let right = queryRight + this.size;
    let answer = -Infinity;

    while (left <= right) {
      if (left % 2 === 1) {
        answer = Math.max(answer, this.tree[left]);
        left += 1;
      }

      if (right % 2 === 0) {
        answer = Math.max(answer, this.tree[right]);
        right -= 1;
      }

      left = Math.floor(left / 2);
      right = Math.floor(right / 2);
    }

    return answer;
  }

  validateIndex(index) {
    if (!Number.isInteger(index) || index < 0 || index >= this.size) {
      throw new RangeError('Index is out of bounds');
    }
  }

  validateRange(queryLeft, queryRight) {
    if (
      !Number.isInteger(queryLeft)
      || !Number.isInteger(queryRight)
      || queryLeft < 0
      || queryRight < queryLeft
      || queryRight >= this.size
    ) {
      throw new RangeError('Query range is invalid');
    }
  }
}
```

## Follow-up Questions

- Swap `Math.max` for sums, minimums, or bitwise OR and explain what changes.
- Compare the bottom-up layout with the classic `4n` recursive array layout.
- Extend the structure to answer half-open ranges `[l, r)` if that is the team's house style.

## Common Pitfalls

- Mixing inclusive ranges with the half-open style used in some iterative templates.
- Forgetting to rebuild every parent after a point update.
- Choosing a neutral element that breaks the combine operation.
