## Why It Surfaces

This is a small Boolean-algebra check disguised as a programming problem. The interviewer wants to know whether you understand what XOR means at the bit level, not just whether you remember the symbol for it.

## Worked Example

For `1` and `2`, the binary forms are `01` and `10`. Their XOR is `11`, so the answer is `3`.

## Reference Implementations

<ExamplePair>

```js
function findXorOfTwoNumberWithoutUsingXorOperator(left, right) {
  if (!Number.isInteger(left) || !Number.isInteger(right)) {
    throw new TypeError('Inputs must be integers');
  }

  return (left | right) & (~left | ~right);
}
```

```ts
export function findXorOfTwoNumberWithoutUsingXorOperator(left: number, right: number): number {
  if (!Number.isInteger(left) || !Number.isInteger(right)) {
    throw new TypeError('Inputs must be integers');
  }

  return (left | right) & (~left | ~right);
}
```

## Follow-up Questions

- Derive the same result using arithmetic and `a & b` instead.
- Explain why `x ^ x = 0` still follows from the derived formula.
- Compare the constant-time Boolean identity with the slower per-bit scan.

## Common Pitfalls

- Reintroducing the `^` operator in a helper and defeating the point of the prompt.
- Forgetting that JavaScript bitwise operators work on 32-bit signed integers.
- Mixing up XOR with OR and leaving shared `1` bits in the answer.
