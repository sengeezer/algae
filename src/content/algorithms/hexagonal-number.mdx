## Why It Surfaces

This is a simple closed-form sequence problem. It checks whether you recognize when a formula completely removes the need for iteration.

## Worked Example

The fourth hexagonal number is `4 * (2 * 4 - 1) = 28`.

## Reference Implementations

<ExamplePair>

```js
function hexagonalNumber(value) {
  if (!Number.isInteger(value) || value <= 0) {
    throw new RangeError('Value must be a positive integer');
  }

  return value * (2 * value - 1);
}
```

```ts
export function hexagonalNumber(value: number): number {
  if (!Number.isInteger(value) || value <= 0) {
    throw new RangeError('Value must be a positive integer');
  }

  return value * (2 * value - 1);
}
```

## Follow-up Questions

- Derive the formula from the differences between consecutive terms.
- Compare hexagonal numbers with triangular and square numbers.
- Check whether a given number is hexagonal instead of generating one by index.

## Common Pitfalls

- Accepting zero or negative indices without defining a sequence convention.
- Iterating through all previous terms when a closed form already exists.
- Confusing hexagonal numbers with centered hexagonal numbers, which use a different formula.
