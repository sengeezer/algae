## Why It Surfaces

This is the binary-string version of column addition. Interviewers use it to check carry handling, pointer movement from right to left, and whether you can avoid unnecessary numeric conversion.

## Worked Example

Adding `"1101"` and `"111"` gives `"10100"`. The digit-wise sum is `13 + 7 = 20`, and `20` in binary is `10100`.

## Reference Implementations

<ExamplePair>

```js
function addTwoBitStrings(left, right) {
  if (typeof left !== 'string' || typeof right !== 'string' || /[^01]/.test(left) || /[^01]/.test(right)) {
    throw new TypeError('Inputs must be binary strings');
  }

  const normalizedLeft = left.replace(/^0+(?!$)/, '');
  const normalizedRight = right.replace(/^0+(?!$)/, '');
  let leftIndex = normalizedLeft.length - 1;
  let rightIndex = normalizedRight.length - 1;
  let carry = 0;
  const digits = [];

  while (leftIndex >= 0 || rightIndex >= 0 || carry > 0) {
    const leftDigit = leftIndex >= 0 ? normalizedLeft.charCodeAt(leftIndex) - 48 : 0;
    const rightDigit = rightIndex >= 0 ? normalizedRight.charCodeAt(rightIndex) - 48 : 0;
    const sum = leftDigit + rightDigit + carry;

    digits.push(String(sum & 1));
    carry = sum >> 1;
    leftIndex -= 1;
    rightIndex -= 1;
  }

  return digits.reverse().join('').replace(/^0+(?!$)/, '');
}
```

## Follow-up Questions

## Common Pitfalls

- Forgetting to keep a final carry digit after both strings are exhausted.
- Returning leading zeros even though the prompt asks to trim them.
- Converting long binary strings into `Number` values and losing precision.
