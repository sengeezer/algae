## Why It Surfaces

This problem combines greedy thinking with controlled backtracking. The greedy part says each position should hold the largest digit still reachable. The backtracking part is needed because repeated max digits can produce different downstream opportunities, so one local choice is not always enough.

## Worked Example

For `s = "7599"` and `k = 2`, the optimal answer is `"9975"`. One valid sequence is swapping the `5` with the rightmost `9` to get `7995`, then swapping the `7` with the remaining `9` to get `9975`.

```js
function maximumNumberPossibleByDoingAtMostKSwaps(text, maxSwaps) {
  const digits = text.split("");
  let best = text;

  function backtrack(index, remainingSwaps) {
    const candidate = digits.join("");

    if (candidate > best) {
      best = candidate;
    }

    if (remainingSwaps === 0 || index === digits.length) {
      return;
    }

    let maxDigit = digits[index];

    for (let cursor = index + 1; cursor < digits.length; cursor += 1) {
      if (digits[cursor] > maxDigit) {
        maxDigit = digits[cursor];
      }
    }

    if (maxDigit === digits[index]) {
      backtrack(index + 1, remainingSwaps);
      return;
    }

    for (let cursor = digits.length - 1; cursor > index; cursor -= 1) {
      if (digits[cursor] !== maxDigit) {
        continue;
      }

      [digits[index], digits[cursor]] = [digits[cursor], digits[index]];
      backtrack(index + 1, remainingSwaps - 1);
      [digits[index], digits[cursor]] = [digits[cursor], digits[index]];
    }
  }

  backtrack(0, maxSwaps);
  return best;
}
```

## Follow-up Questions

- How would you modify the recursion to return one concrete swap sequence as well as the best number?
- Why is it safe to skip swaps that do not improve the current digit?
- How would you support inputs that may contain leading zeroes after swaps?

## Common Pitfalls

- Exploring all pair swaps even when the current digit is already optimal.
- Forgetting that repeated maximum digits may lead to different later states.
- Treating the result as a numeric type and losing leading-zero behavior that belongs to the string problem.
