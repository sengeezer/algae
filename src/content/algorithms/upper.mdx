## Why It Surfaces

This is a simple transformation problem that exposes how characters are encoded. The interview value is usually in explaining the gap between a hand-written ASCII solution and a robust Unicode-aware one.

## Worked Example

`"He_llo"` becomes `"HE_LLO"`: lowercase letters are mapped to uppercase, while underscores and existing uppercase letters stay unchanged.

## Reference Implementations

<ExamplePair>

```js
function upper(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  return value.replace(/[a-z]/g, (char) =>
    String.fromCharCode(char.charCodeAt(0) - 32),
  );
}
```

```ts
export function upper(value: string): string {
  return value.replace(/[a-z]/g, (char) =>
    String.fromCharCode(char.charCodeAt(0) - 32),
  );
}
```

## Follow-up Questions

- Implement the lowercase transformation symmetrically.
- Compare the manual ASCII approach with `value.toUpperCase()`.
- Handle locale-sensitive case mapping requirements.

## Common Pitfalls

- Assuming ASCII conversion is enough for all languages.
- Mutating character codes incorrectly by using the wrong offset.
- Forgetting that non-letter characters should remain unchanged.
