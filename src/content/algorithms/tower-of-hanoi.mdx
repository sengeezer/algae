## Why It Surfaces

Tower of Hanoi is a pure recursion exercise. There is no greedy shortcut; the only way forward is to define the smaller subproblem correctly and trust the recursive structure.

## Worked Example

To move `3` disks from peg `A` to peg `C`, first move `2` disks from `A` to `B`, then move the largest disk from `A` to `C`, then move the `2` disks from `B` to `C`.

<Callout title="Forced structure" tone="note">
  The middle move is never optional. Once the top n-1 disks are out of the way, the largest disk has exactly one legal move.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function towerOfHanoi(disks, fromPeg, toPeg, auxPeg, moves = []) {
  if (disks === 1) {
    moves.push(`Move disk 1 from rod ${fromPeg} to rod ${toPeg}`);
    return moves;
  }

  towerOfHanoi(disks - 1, fromPeg, auxPeg, toPeg, moves);
  moves.push(`Move disk ${disks} from rod ${fromPeg} to rod ${toPeg}`);
  towerOfHanoi(disks - 1, auxPeg, toPeg, fromPeg, moves);
  return moves;
}
```

```ts
  moves.push(`Move disk ${disks} from rod ${fromPeg} to rod ${toPeg}`);
  towerOfHanoi(disks - 1, auxPeg, toPeg, fromPeg, moves);
  return moves;
}
```

## Follow-up Questions

- Return only the number of moves instead of the full move list.
- Explain why the minimum move count is `2^n - 1`.
- Compare this deterministic recursion with branching backtracking problems.

## Common Pitfalls

- Swapping the auxiliary and destination pegs in the recursive calls.
- Forgetting the single-disk base case.
- Treating the recursion as optional rather than structurally forced by the rules.
