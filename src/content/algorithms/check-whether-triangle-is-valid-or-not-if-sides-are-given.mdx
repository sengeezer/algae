## Why It Surfaces

This is the smallest standalone triangle predicate. The entire problem reduces to whether each pair of sides is strictly longer than the remaining side. If any one inequality fails, the shape collapses into a segment or becomes impossible.

## Worked Example

For sides `7`, `10`, and `5`, all three inequalities hold: `7 + 10 > 5`, `7 + 5 > 10`, and `10 + 5 > 7`, so the triangle is valid.

## Reference Implementations

<ExamplePair>

```js
function checkWhetherTriangleIsValidOrNotIfSidesAreGiven(a, b, c) {
  if (![a, b, c].every((side) => typeof side === "number" && Number.isFinite(side) && side > 0)) {
    return false;
  }

  return a + b > c && a + c > b && b + c > a;
}
```

```ts
export function checkWhetherTriangleIsValidOrNotIfSidesAreGiven(
  a: number,
  b: number,
  c: number,
): boolean {
  if (![a, b, c].every((side) => Number.isFinite(side) && side > 0)) {
    return false;
  }

  return a + b > c && a + c > b && b + c > a;
}
```

## Common Pitfalls

- Accepting degenerate triples where two sides sum exactly to the third.
- Ignoring zero or negative side lengths.
- Sorting the sides first when the direct inequalities already suffice.
