## Why It Surfaces

The key greedy observation is that the same cut cost becomes more expensive later because it must cross more segments. That means high-cost cuts should be taken first, which turns the problem into merging two descending cost lists.

## Worked Example

With `x = [2, 1, 3, 1, 4]` and `y = [4, 1, 2]`, start from one horizontal strip and one vertical strip. Taking `4` from `x` first costs `4`, then `4` from `y` costs `8`, and continuing with the largest remaining cut each time yields the optimal total of `42`.

```js
function minimumCostToCutABoardIntoSquares(verticalCutCosts, horizontalCutCosts) {
  const vertical = [...verticalCutCosts].sort((left, right) => right - left);
  const horizontal = [...horizontalCutCosts].sort((left, right) => right - left);

  let verticalIndex = 0;
  let horizontalIndex = 0;
  let verticalSegments = 1;
  let horizontalSegments = 1;
  let totalCost = 0;

  while (verticalIndex < vertical.length && horizontalIndex < horizontal.length) {
    if (vertical[verticalIndex] >= horizontal[horizontalIndex]) {
      totalCost += vertical[verticalIndex] * horizontalSegments;
      verticalSegments += 1;
      verticalIndex += 1;
      continue;
    }

    totalCost += horizontal[horizontalIndex] * verticalSegments;
    horizontalSegments += 1;
    horizontalIndex += 1;
  }

  while (verticalIndex < vertical.length) {
    totalCost += vertical[verticalIndex] * horizontalSegments;
    verticalSegments += 1;
    verticalIndex += 1;
  }

  while (horizontalIndex < horizontal.length) {
    totalCost += horizontal[horizontalIndex] * verticalSegments;
    horizontalSegments += 1;
    horizontalIndex += 1;
  }

  return totalCost;
}
```

## Follow-up Questions

- Explain why taking a cheaper cut before a more expensive cut can only increase or preserve the final cost.
- Adapt the routine if a third dimension adds depth-wise cuts.
- Return the actual cut order as well as the minimum total cost.

## Common Pitfalls

- Multiplying a vertical cut by the wrong segment count.
- Sorting ascending and then scanning forward without compensating for the order.
- Forgetting to finish the remaining cuts once one list is exhausted.
