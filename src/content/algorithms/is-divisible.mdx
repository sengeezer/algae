## Why It Surfaces

This is one of the simplest arithmetic predicates, but it still checks whether you define the divisor-zero edge case instead of letting the runtime decide for you implicitly.

## Worked Example

`10` is divisible by `5` because `10 % 5` is `0`, while `99` is not divisible by `5` because the remainder is `4`.

## Reference Implementations

<ExamplePair>

```js
function isDivisible(value, divisor) {
  if (![value, divisor].every((number) => Number.isFinite(number))) {
    throw new TypeError('Expected finite numbers');
  }

  if (divisor === 0) {
    return false;
  }

  return value % divisor === 0;
}
```

```ts
export function isDivisible(value: number, divisor: number): boolean {
  if (![value, divisor].every((number) => Number.isFinite(number))) {
    throw new TypeError('Expected finite numbers');
  }

  if (divisor === 0) {
    return false;
  }

  return value % divisor === 0;
}
```

## Follow-up Questions

- Restrict the helper to integers only and explain the tradeoff.
- Use divisibility rules for `2`, `3`, `5`, or `9` as fast prechecks.
- Compose this predicate into a filter over an array or range.

## Common Pitfalls

- Forgetting to define the `divisor === 0` case.
- Treating all non-integer inputs as invalid even when the implementation is intentionally written for finite numbers.
- Confusing divisibility of one pair with counting divisible values across a range.
