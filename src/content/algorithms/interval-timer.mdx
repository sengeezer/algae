## Why It Surfaces

Timer utilities are small, but they test whether you can manage mutable state without confusing interval handles, runtime, and elapsed checkpoints. The interesting part is not `setInterval` itself, but the bookkeeping around it.

## Worked Example

If a timer runs a callback every `50` milliseconds, `getRunTime()` should report total time since `startTimer()`, while `getElapsedTime()` should report the delta since the previous checkpoint.

```js
class IntervalTimer {
  constructor(intervalMs = 10, callback = () => {}) {
    if (!Number.isInteger(intervalMs) || intervalMs <= 0) {
      throw new RangeError('Interval must be a positive integer number of milliseconds');
    }

    if (typeof callback !== 'function') {
      throw new TypeError('Callback must be a function');
    }

    this.intervalMs = intervalMs;
    this.callback = callback;
    this.timerId = null;
    this.startedAt = null;
    this.lastCheckpointAt = null;
  }

  startTimer() {
    if (this.timerId !== null) {
      return;
    }

    const now = Date.now();
    this.startedAt = now;
    this.lastCheckpointAt = now;
    this.timerId = setInterval(this.callback, this.intervalMs);
  }

  stopTimer() {
    if (this.timerId !== null) {
      clearInterval(this.timerId);
      this.timerId = null;
    }

    return this.getRunTime();
  }

  getElapsedTime(offset = 0) {
    if (this.lastCheckpointAt === null) {
      return 0;
    }

    const now = Date.now();
    const elapsed = Math.max(0, now - this.lastCheckpointAt - offset);
    this.lastCheckpointAt = now;
    return elapsed;
  }

  getRunTime() {
    return this.startedAt === null ? 0 : Date.now() - this.startedAt;
  }

  resetTimer() {
    const elapsed = this.getRunTime();
    if (this.timerId !== null) {
      clearInterval(this.timerId);
    }

    this.timerId = null;
    this.startedAt = null;
    this.lastCheckpointAt = null;
    return elapsed;
  }
}
```

## Follow-up Questions

- Swap `Date.now()` for a monotonic source like `performance.now()` in environments where that matters.
- Add pause and resume without losing the accumulated runtime.
- Explain why timer callbacks can drift relative to the requested interval.

## Common Pitfalls

- Treating an interval handle as if it were elapsed time.
- Restarting an already running timer and leaking multiple intervals.
- Forgetting to clear the interval when resetting the timer.
