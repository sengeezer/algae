## Why It Surfaces

The prompt looks geometric, but the core is just a distance comparison. Once you know the perpendicular distance from a point to a line, the whole problem reduces to comparing that distance with the circle radius to decide whether the line stays outside, touches once, or cuts through the circle.

## Worked Example

For center `(0, 0)`, radius `5`, and line `3x + 4y + 25 = 0`, the distance from the center to the line is `25 / sqrt(3^2 + 4^2) = 5`, so the line touches the circle.

<ExamplePair>

```js
function checkIfALineTouchesOrIntersectsACircle(a, b, c, centerX, centerY, radius) {
  if (
    !Number.isFinite(a)
    || !Number.isFinite(b)
    || !Number.isFinite(c)
    || !Number.isFinite(centerX)
    || !Number.isFinite(centerY)
    || !Number.isFinite(radius)
    || radius < 0
    || (a === 0 && b === 0)
  ) {
    throw new RangeError("Invalid line or circle parameters");
  }

  const numerator = a * centerX + b * centerY + c;
  const numeratorSq = numerator * numerator;
  const denominator = a * a + b * b;
  const radiusSqScaled = radius * radius * denominator;

  if (numeratorSq === radiusSqScaled) {
    return "touch";
  }

  return numeratorSq < radiusSqScaled ? "intersect" : "outside";
}
```

## Follow-up Questions

- Why is comparing squared quantities cleaner than calling `sqrt`?

## Common Pitfalls

- Forgetting that `a = 0` and `b = 0` do not define a valid line.
- Computing the distance with integer truncation.
- Treating a tangent line as an intersection without distinguishing the exact-equality case.
