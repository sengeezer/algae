## Why It Surfaces

Most interview explanations of topological sort focus on indegrees or an implicit DFS stack. This variant makes the DFS reasoning explicit: once a vertex has finished exploring all outgoing edges, its departure time tells you where it belongs relative to every descendant.

## Worked Example

If DFS finishes vertex `3` before vertex `1`, then `1` gets a later departure time. Reading vertices in decreasing departure time puts `1` before `3`, which is exactly what a topological order needs when `1 -> 3` is an edge.

## Reference Implementations

<ExamplePair>

```js
function topologicalSortOfAGraphUsingDepartureTimeOfVertex(graph) {
  const nodeCount = graph.length;
  const visited = new Array(nodeCount).fill(false);
  const departure = new Array(nodeCount);
  let time = 0;

  function dfs(node) {
    visited[node] = true;

    for (const neighbor of graph[node]) {
      if (!visited[neighbor]) {
        dfs(neighbor);
      }
    }

    departure[time] = node;
    time += 1;
  }

  for (let node = 0; node < nodeCount; node += 1) {
    if (!visited[node]) {
      dfs(node);
    }
  }

  departure.reverse();
  return departure;
}
```

  const nodeCount = graph.length;
  const visited = new Array<boolean>(nodeCount).fill(false);
  const departure = new Array<number>(nodeCount);
  let time = 0;

    }

## Common Pitfalls

- Mixing up arrival times with departure times.
- Reversing the order before DFS is complete.
- Forgetting that the method depends on the graph being acyclic.
