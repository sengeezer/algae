## Why It Surfaces

This is the standard example of how one small bit trick can change the effective loop cost. Instead of checking every bit position, you let the binary representation tell you how many iterations are necessary.

## Worked Example

For `13`, whose binary form is `1101`, the first pass clears the last `1` to get `1100`, the next clears to `1000`, and the last clears to `0000`. The count is `3`.

## Reference Implementations

<ExamplePair>

```js
function binaryCountSetBits(value) {
  if (!Number.isInteger(value) || value < 0) {
    throw new TypeError('Value must be a non-negative integer');
  }

  let remaining = value;
  let count = 0;

  while (remaining !== 0) {
    remaining &= remaining - 1;
    count += 1;
  }

  return count;
}
```


## Follow-up Questions

## Common Pitfalls

- Forgetting that JavaScript bitwise operators work on 32-bit integers.
- Applying the routine to non-integer input.
- Using a bit-position scan when the set-bit count is much smaller than the word size.
