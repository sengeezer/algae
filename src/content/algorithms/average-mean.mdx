## Why It Surfaces

Mean is simple, but it is a common way to check whether you can combine iteration, validation, and a formula without overcomplicating the solution. Empty arrays and non-numeric elements are the usual edge cases.

## Worked Example

For `[10, 40, 100, 20]`, the total is `170`, and `170 / 4 = 42.5`, so the arithmetic mean is `42.5`.

## Reference Implementations

<ExamplePair>

```js
function averageMean(values) {
  if (!Array.isArray(values) || values.length === 0) {
    throw new RangeError('Expected a non-empty array of numbers');
  }

  let total = 0;

  for (const value of values) {
    if (typeof value !== 'number' || Number.isNaN(value)) {
      throw new TypeError('Array values must be numbers');
    }

    total += value;
  }

  return total / values.length;
}
```

```ts
export function averageMean(values: number[]): number {
  if (values.length === 0) {
    throw new RangeError('Expected a non-empty array of numbers');
  }

  let total = 0;

  for (const value of values) {
    if (Number.isNaN(value)) {
      throw new TypeError('Array values must be numbers');
    }

    total += value;
  }

  return total / values.length;
}
```

## Follow-up Questions

- Add a weighted-mean version that accepts values and weights.
- Compare mean with median on outlier-heavy data.
- Stream values one at a time instead of receiving a full array.

## Common Pitfalls

- Dividing by zero when the array is empty.
- Forgetting to reject `NaN` values.
- Accidentally using integer-style truncation in languages where division behavior differs.
