## Why It Surfaces

This is an operator-ban arithmetic prompt, not a general fast-power problem. The interviewer usually wants to see whether you can treat multiplication as repeated shifted additions once one factor is written in binary.

## Worked Example

For `12`, the square is `144`. One factor of `12` is `1100` in binary, so only the shifted copies for the `8` and `4` places contribute: `12 << 3` plus `12 << 2`, which sums to `96 + 48 = 144`.

## Reference Implementations

<ExamplePair>

```js
function calculateSquareOfANumberWithoutUsingMultiplicationDivisionOrPow(value) {
  if (!Number.isSafeInteger(value)) {
    throw new TypeError('Value must be a safe integer');
  }

  let multiplier = BigInt(Math.abs(value));
  let multiplicand = multiplier;
  let result = 0n;

  while (multiplier > 0n) {
    if ((multiplier & 1n) === 1n) {
      result += multiplicand;
    }

    multiplicand <<= 1n;
    multiplier >>= 1n;
  }

  const square = Number(result);
  if (!Number.isSafeInteger(square)) {
    throw new RangeError('Square exceeds JavaScript safe integer range');
  }

  return square;
}
```

## Follow-up Questions

- Compare this square-specific helper with full binary exponentiation.
- Adapt the same shift-and-add idea to multiply two different numbers.
- Return a `BigInt` instead of throwing when the square exceeds the safe-integer range.

## Common Pitfalls

- Treating this as ordinary fast power and reintroducing multiplication.
- Forgetting that negative inputs square to the same result as their absolute value.
- Using JavaScript `<<` on large `number` values and silently truncating to 32 bits.
