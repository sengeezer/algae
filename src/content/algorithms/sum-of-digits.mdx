## Why It Surfaces

Sum-of-digits questions are small, but they test whether you can reason about decimal representation without leaning on string conversion by default.

## Worked Example

For `-255`, take the absolute value first. Then peel digits with `% 10`: `5 + 5 + 2 = 12`, so the digit sum is `12`.

## Reference Implementations

<ExamplePair>

```js
function sumOfDigits(value) {
  if (!Number.isInteger(value)) {
    throw new RangeError('Value must be an integer');
  }

  let remaining = Math.abs(value);
  let total = 0;

  while (remaining > 0) {
    total += remaining % 10;
    remaining = Math.floor(remaining / 10);
  }

  return total;
}
```

```ts
export function sumOfDigits(value: number): number {
  if (!Number.isInteger(value)) {
    throw new RangeError('Value must be an integer');

  let remaining = Math.abs(value);
  let total = 0;

  while (remaining > 0) {
    total += remaining % 10;
    remaining = Math.floor(remaining / 10);
  }

  return total;
}
```

## Follow-up Questions

- Implement the same routine recursively.
- Generalize the logic to other bases besides decimal.
- Reduce the repeated digit sum to a digital root.

## Common Pitfalls

- Forgetting to take the absolute value for negative inputs.
- Mishandling `0`, whose digit sum should still be `0`.
- Accepting fractional inputs even though decimal digits are defined here only for integers.
