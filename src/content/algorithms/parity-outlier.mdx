## Why It Surfaces

This is a small classification problem, but it still rewards careful handling of ambiguous inputs and non-integer values instead of assuming the happy path.

## Worked Example

In `[1, 3, 5, 8, 9]`, four values are odd and one is even, so the parity outlier is `8`. In `[2, 4, 7, 8, 10]`, the outlier is `7`.

## Reference Implementations

<ExamplePair>

```js
function parityOutlier(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Expected an array of integers');
  }

  let oddCount = 0;
  let evenCount = 0;
  let oddValue = null;
  let evenValue = null;

  for (const value of values) {
    if (!Number.isInteger(value)) {
      return null;
    }

    if (value % 2 === 0) {
      evenCount += 1;
      evenValue = value;
    } else {
      oddCount += 1;
      oddValue = value;
    }
  }

  if (oddCount === 1 && evenCount > 1) {
    return oddValue;
  }

  if (evenCount === 1 && oddCount > 1) {
    return evenValue;
  }

  return null;
}
```

## Follow-up Questions

- Return the index of the outlier instead of the value.
- Short-circuit once the majority parity becomes obvious from the first few values.
- Explain how negative odd numbers behave under the same parity test.

## Common Pitfalls

- Returning the first odd or even value before confirming it is actually the unique outlier.
- Ignoring arrays that contain both multiple odds and multiple evens.
- Treating non-integer inputs as valid parity candidates.
