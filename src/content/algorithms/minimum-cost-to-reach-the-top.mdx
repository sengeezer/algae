## Why It Surfaces

This problem looks like ordinary climbing stairs until the objective changes. Once every visited step has a cost, the transition stops adding counts and starts taking the cheaper of the last two ways forward.

## Worked Example

For costs `[10, 15, 20]`, you can start on index `0` or `1`. Taking step `1` for cost `15` and then moving directly to the top beats stepping on both `10` and `20`, so the minimum total is `15`.

## Reference Implementations

<ExamplePair>

```js
function minimumCostToReachTheTop(costs) {
  let previousTwo = 0;
  let previousOne = 0;

  for (const cost of costs) {
    const current = cost + Math.min(previousOne, previousTwo);
    previousTwo = previousOne;
    previousOne = current;
  }

  return Math.min(previousOne, previousTwo);
}
```

```ts
export function minimumCostToReachTheTop(costs: number[]): number {
  let previousTwo = 0;
  let previousOne = 0;

  for (const cost of costs) {
    const current = cost + Math.min(previousOne, previousTwo);
    previousTwo = previousOne;
    previousOne = current;
  }

  return Math.min(previousOne, previousTwo);
}
```

## Follow-up Questions

- Return one cheapest sequence of step indices, not just the total cost.
- Allow jumps of one, two, or three steps and update the recurrence.
- Compare the top-down memoized version with the rolling-state tabulation.

## Common Pitfalls

- Counting ways instead of minimizing accumulated cost.
- Forgetting that the top sits one move beyond the last index.
- Paying the top itself as if it had an extra step cost.
