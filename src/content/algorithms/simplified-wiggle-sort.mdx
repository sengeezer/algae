## Why It Surfaces

Simplified wiggle sort is a good reminder that not every ordering problem wants a globally sorted answer. The goal is only local structure: odd indexes should become peaks and even indexes should become valleys.

## Worked Example

For `[1, 5, 1, 1, 6, 4]`, one valid simplified wiggle arrangement is `[1, 6, 1, 5, 1, 4]`, which satisfies `1 <= 6 >= 1 <= 5 >= 1 <= 4`.

## Reference Implementations

<ExamplePair>

```js
function simplifiedWiggleSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const sorted = [...values].sort((left, right) => left - right);
  const result = new Array(sorted.length);
  let lower = Math.floor((sorted.length - 1) / 2);
  let upper = sorted.length - 1;

  for (let index = 0; index < result.length; index += 1) {
    if (index % 2 === 0) {
      result[index] = sorted[lower];
      lower -= 1;
    } else {
      result[index] = sorted[upper];
      upper -= 1;
    }
  }

  return result;
}
```

## Follow-up Questions

- Tighten the condition to require strict `<` and `>` where possible.
- Explain why placing larger values in odd indexes creates the peaks.
- Replace the full sort with a quickselect-based median strategy.

## Common Pitfalls

- Solving the wrong problem by fully sorting the array and stopping there.
- Forgetting that the simplified version allows equal neighboring values.
- Filling the output left-to-right from the sorted array and losing the wiggle pattern.
