## Why It Surfaces

The matrices multiply to the same final result regardless of grouping, but the intermediate dimensions can make one order dramatically cheaper than another. The interview test is whether you can model that choice as an interval DP instead of brute-forcing every parenthesization.

## Worked Example

For `dimensions = [1, 2, 3, 4, 3]`, the best answer is `30`. The cheapest order is `(((A B) C) D)`, which beats the alternatives because it keeps the intermediate matrices small.

## Reference Implementations

<ExamplePair>

```js
function matrixChainMultiplication(dimensions) {
  const size = dimensions.length;
  if (size <= 2) {
    return 0;
  }

  const dp = Array.from({ length: size }, () => new Array(size).fill(0));

  for (let length = 2; length < size; length += 1) {
    for (let left = 0; left + length < size; left += 1) {
      const right = left + length;
      dp[left][right] = Number.POSITIVE_INFINITY;

      for (let mid = left + 1; mid < right; mid += 1) {
        const cost =
          dp[left][mid]
          + dp[mid][right]
          + dimensions[left] * dimensions[mid] * dimensions[right];

        if (cost < dp[left][right]) {
          dp[left][right] = cost;
        }
      }
    }
  }

  return dp[0][size - 1];
}
```

## Follow-up Questions

- Return the optimal parenthesization, not only the minimum cost.
- Explain why the DP table is filled diagonally instead of row by row.
- Compare the bottom-up table with a memoized recursive solution.

## Common Pitfalls

- Treating the problem like ordinary matrix multiplication instead of order optimization.
- Using matrix indices directly instead of the dimension array boundaries.
- Forgetting that a single matrix has zero multiplication cost.
