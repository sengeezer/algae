## Why It Surfaces

This problem starts from standard LCS, then asks for reconstruction at full combinatorial scale. The DP table still tells you the best achievable length from every suffix pair, but you need a second pass to enumerate every string that actually achieves that optimum.

## Worked Example

For `"abc"` and `"bac"`, the LCS length is `2`. The valid longest common subsequences are `"ac"` and `"bc"`, and the required output order is `['ac', 'bc']`.

```js
function printAllLongestCommonSubSequencesInLexicographicalOrder(first, second) {
  const rowCount = first.length;
  const columnCount = second.length;
  const dp = Array.from({ length: rowCount + 1 }, () => Array(columnCount + 1).fill(0));

  for (let row = rowCount - 1; row >= 0; row -= 1) {
    for (let column = columnCount - 1; column >= 0; column -= 1) {
      if (first[row] === second[column]) {
        dp[row][column] = dp[row + 1][column + 1] + 1;
      } else {
        dp[row][column] = Math.max(dp[row + 1][column], dp[row][column + 1]);
      }
    }
  }

  if (dp[0][0] === 0) {
    return [];
  }

  const memo = new Map();

  function build(row, column) {
    const key = `${row}:${column}`;

    if (memo.has(key)) {
      return memo.get(key);
    }

    const result = new Set();

    if (row === rowCount || column === columnCount) {
      result.add("");
    } else if (first[row] === second[column]) {
      for (const suffix of build(row + 1, column + 1)) {
        result.add(first[row] + suffix);
      }
    } else {
      if (dp[row + 1][column] === dp[row][column]) {
        for (const sequence of build(row + 1, column)) {
          result.add(sequence);
        }
      }

      if (dp[row][column + 1] === dp[row][column]) {
        for (const sequence of build(row, column + 1)) {
          result.add(sequence);
        }
      }
    }

    memo.set(key, result);
    return result;
  }

  return [...build(0, 0)].filter((sequence) => sequence.length === dp[0][0]).sort();
}
```

## Follow-up Questions

- How would you reduce memory if you needed only the LCS length and not the sequences?
- Why does a `Set` matter during reconstruction even though the DP table already stores optimal lengths?
- How would you stream the answers in sorted order without materializing the full list first?

## Common Pitfalls

- Reconstructing from both DP branches without deduplicating equal strings.
- Printing every common subsequence instead of only those with maximal length.
- Forgetting to sort the final answers lexicographically.
