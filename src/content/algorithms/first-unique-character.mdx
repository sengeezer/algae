## Why It Surfaces

This problem checks whether you can separate counting from selection. The count map solves frequency, but the second pass is what preserves the first occurrence requirement.

## Worked Example

For `"leetcode"`, the counts show `l`, `t`, `c`, `o`, `d`, and `e` frequencies, and the first character with count `1` is `l` at index `0`.

## Reference Implementations

<ExamplePair>

```js
function firstUniqueCharacter(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  const counts = new Map();

  for (const char of value) {
    counts.set(char, (counts.get(char) ?? 0) + 1);
  }

  for (let index = 0; index < value.length; index += 1) {
    if (counts.get(value[index]) === 1) {
      return index;
    }
  }

  return -1;
}
```

```ts
export function firstUniqueCharacter(value: string): number {
  const counts = new Map<string, number>();

  for (const char of value) {
    counts.set(char, (counts.get(char) ?? 0) + 1);
  }

  for (let index = 0; index < value.length; index += 1) {
    if (counts.get(value[index]) === 1) {
      return index;
    }
  }

  return -1;
}
```

## Follow-up Questions

- Return the character instead of its index.
- Optimize for a fixed lowercase alphabet with an array count table.
- Extend the idea to a stream where characters arrive over time.

## Common Pitfalls

- Returning the smallest-count character instead of the first unique one.
- Trying to decide uniqueness in one pass without keeping enough state.
- Forgetting to return `-1` when no unique character exists.
