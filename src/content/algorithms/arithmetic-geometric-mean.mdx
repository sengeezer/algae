## Why It Surfaces

AGM is a compact numerical-methods exercise. It checks whether you can reason about convergence, validate the domain of the square root, and stop with a tolerance instead of relying on exact floating-point equality.

## Worked Example

Starting from `a = 24` and `g = 6`, the next pair is $(15, 12)$, then roughly $(13.5, 13.4164)$, and the values quickly converge to the same limit.

## Reference Implementations

<ExamplePair>

```js
function arithmeticGeometricMean(a, g, tolerance = 1e-12, maxIterations = 1000) {
  if (![a, g, tolerance].every((value) => typeof value === 'number' && Number.isFinite(value))) {
    throw new TypeError('Inputs and tolerance must be finite numbers');
  }

  if (a < 0 || g < 0) {
    throw new RangeError('AGM is defined here only for non-negative inputs');
  }

  let arithmetic = a;
  let geometric = g;

  for (let iteration = 0; iteration < maxIterations; iteration += 1) {
    if (Math.abs(arithmetic - geometric) <= tolerance) {
      return arithmetic;
    }

    const nextArithmetic = (arithmetic + geometric) / 2;
    const nextGeometric = Math.sqrt(arithmetic * geometric);
    arithmetic = nextArithmetic;
    geometric = nextGeometric;
  }

  return arithmetic;
}
```

## Follow-up Questions

- Explain why the arithmetic mean stays above the geometric mean during the iteration.
- Compare a tolerance-based stop condition with exact equality.
- Use AGM as a building block in a more advanced numerical approximation.

## Common Pitfalls

- Accepting negative inputs and then taking the square root of a negative product.
- Using exact equality as the only convergence test.
- Forgetting that floating-point convergence speed depends on the chosen tolerance.
