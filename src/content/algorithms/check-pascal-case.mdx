## Why It Surfaces

PascalCase checks are usually paired with other naming-style classifiers. The interesting part is encoding just enough structure to reject separators and lowercase-leading identifiers.

## Worked Example

`"UserName"` passes because it starts uppercase and contains only letters. `"userName"` fails because that is camelCase instead.

## Reference Implementations

<ExamplePair>

```js
function checkPascalCase(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  return /^[A-Z][A-Za-z]*$/.test(value);
}
```

```ts
export function checkPascalCase(value: string): boolean {
  return /^[A-Z][A-Za-z]*$/.test(value);
}
```

</ExamplePair>

## Follow-up Questions

- Convert PascalCase to spaced title case.

## Common Pitfalls

- Treating any uppercase letter anywhere as enough to qualify.
- Accepting separators like hyphens or underscores.
- Mixing PascalCase rules with camelCase rules.
