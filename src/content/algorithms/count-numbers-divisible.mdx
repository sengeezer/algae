## Why It Surfaces

This problem is a good check on whether you can replace an obvious loop with arithmetic. The main insight is that counting multiples up to a bound is a floor-division problem.

## Worked Example

In the range `[4, 19]`, the numbers divisible by `3` are `6`, `9`, `12`, `15`, and `18`, so the count is `5`.

## Reference Implementations

<ExamplePair>

```js
function countNumbersDivisible(left, right, divisor) {
  if (![left, right, divisor].every((value) => Number.isInteger(value))) {
    throw new RangeError('Bounds and divisor must be integers');
  }

  if (divisor <= 0) {
    throw new RangeError('Divisor must be a positive integer');
  }

  if (left > right) {
    throw new RangeError('Left bound must be less than or equal to right bound');
  }

  const multiplesUpToRight = Math.floor(right / divisor);
  const multiplesBeforeLeft = Math.floor((left - 1) / divisor);
  return multiplesUpToRight - multiplesBeforeLeft;
}
```

## Follow-up Questions

- Extend the logic to exclude both endpoints explicitly.
- Count values divisible by one number but not another.
- Explain why arithmetic beats iteration for large intervals.

## Common Pitfalls

- Forgetting that the interval is inclusive.
- Allowing a zero or negative divisor without defining the behavior.
- Iterating across the full range instead of using floor arithmetic.
