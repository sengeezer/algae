## Why It Surfaces

The trap here is that a single left-to-right pass with two independent comparisons per element is not comparison-optimal. Once you compare values in pairs first, only the smaller member of the pair can challenge the global minimum and only the larger member can challenge the global maximum.

## Worked Example

For `[3, 5, 4, 1, 9]`, initialize both answers from the first odd element `3`, then process `(5, 4)` and `(1, 9)` as pairs. The result is `[1, 9]`.

```js
function maximumAndMinimumOfAnArrayUsingMinimumNumberOfComparisons(values) {
  if (!Array.isArray(values) || values.length === 0) {
    throw new RangeError("Values must be a non-empty array");
  }

  let index = 0;
  let minimum;
  let maximum;

  if (values.length % 2 === 1) {
    minimum = values[0];
    maximum = values[0];
    index = 1;
  } else {
    if (values[0] < values[1]) {
      minimum = values[0];
      maximum = values[1];
    } else {
      minimum = values[1];
      maximum = values[0];
    }

    index = 2;
  }

  while (index < values.length - 1) {
    let localMinimum = values[index];
    let localMaximum = values[index + 1];

    if (localMinimum > localMaximum) {
      [localMinimum, localMaximum] = [localMaximum, localMinimum];
    }

    minimum = Math.min(minimum, localMinimum);
    maximum = Math.max(maximum, localMaximum);
    index += 2;
  }

  return [minimum, maximum];
}
```

## Common Pitfalls

- Initializing even-length arrays incorrectly and double-counting the first two values.
- Comparing each element to both global extremes without the pairwise shortcut.
- Forgetting that odd-length arrays need a different starting state than even-length arrays.
