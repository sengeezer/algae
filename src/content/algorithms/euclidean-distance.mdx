## Why It Surfaces

This is the default distance metric most people know, but it still tests whether you can translate the math formula into a careful loop over coordinates.

## Worked Example

For points `[1, 2]` and `[4, 6]`, subtract coordinate-wise to get differences `3` and `4`, square them to get `9` and `16`, sum to `25`, and take the square root to get `5`.

## Reference Implementations

<ExamplePair>

```js
function euclideanDistance(left, right) {
  if (left.length !== right.length) {
    throw new Error('Vectors must have the same length');
  }

  let sumOfSquares = 0;

  for (let index = 0; index < left.length; index += 1) {
    const difference = left[index] - right[index];
    sumOfSquares += difference * difference;
  }

  return Math.sqrt(sumOfSquares);
}
```

## Follow-up Questions

- Return squared distance when the square root is unnecessary for comparisons.
- Generalize the routine to Manhattan or Chebyshev distance.
- Explain why vectors of different lengths should be rejected.

## Common Pitfalls

- Forgetting to validate equal dimensions.
- Summing absolute differences instead of squared differences.
- Taking the square root before accumulating all coordinate contributions.
