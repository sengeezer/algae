## Why It Surfaces

Counting digits is a small arithmetic problem that exposes whether you think carefully about edge cases. `0` is the usual trap because it has one digit even though a loop that only runs while the number is positive would report none.

## Worked Example

`-42050` has `5` digits. The sign does not count toward the total, so the routine works on the absolute value before peeling digits away.

## Reference Implementations

<ExamplePair>

```js
function numberOfDigits(value) {
  if (!Number.isInteger(value)) {
    throw new RangeError('Value must be an integer');
  }

  let remaining = Math.abs(value);
  if (remaining === 0) {
    return 1;
  }

  let digits = 0;
  while (remaining > 0) {
    digits += 1;
    remaining = Math.floor(remaining / 10);
  }

  return digits;
}
```

## Follow-up Questions

- Implement the logarithm-based alternative and compare its edge cases.
- Generalize the routine to bases other than decimal.
- Reuse the logic in a palindrome or reverse-number check.

## Common Pitfalls

- Returning `0` for the input `0`.
- Counting the negative sign as if it were a digit.
- Accepting fractional inputs when the prompt is about integers.
