## Why It Surfaces

Cocktail shaker sort is the cleanest answer when someone asks how to improve bubble sort without changing its basic adjacent-swap model. The point is to move large items rightward and small items leftward in the same round.

## Worked Example

On `[5, 1, 4, 2, 8, 0]`, the forward pass pushes `8` toward the end, and the backward pass then pulls `0` toward the front. Each round tightens both boundaries instead of only the right boundary.

## Reference Implementations

<ExamplePair>

```js
function cocktailShakerSort(values) {
  if (!Array.isArray(values)) {
    throw new TypeError('Values must be an array of numbers');
  }

  const items = [...values];
  let start = 0;
  let end = items.length - 1;
  let swapped = true;

  while (swapped && start < end) {
    swapped = false;

    for (let index = start; index < end; index += 1) {
      if (items[index] > items[index + 1]) {
        [items[index], items[index + 1]] = [items[index + 1], items[index]];
        swapped = true;
      }
    }

    end -= 1;

    for (let index = end; index > start; index -= 1) {
      if (items[index - 1] > items[index]) {
        [items[index - 1], items[index]] = [items[index], items[index - 1]];
        swapped = true;
      }
    }

    start += 1;
  }

  return items;
}
```

## Follow-up Questions

- Compare cocktail shaker sort with plain bubble sort on nearly sorted input.
- Add explicit early-stop checks for both passes.
- Explain why the asymptotic runtime does not improve even though constants can.

## Common Pitfalls

- Forgetting to shrink both the left and right boundaries.
- Carrying a stale swapped flag between passes.
- Claiming the bidirectional sweep changes the worst-case complexity.
