## Why It Surfaces

This is the streamlined version of bit counting. Instead of caring how many set bits exist, you only care whether that count is odd or even, which means a single toggled state is enough.

## Worked Example

`13` is `1101` in binary, which contains three `1` bits. Three is odd, so the function returns `'odd'`.

## Reference Implementations

<ExamplePair>

```js
function programToFindParity(value) {
  if (!Number.isInteger(value) || value < 0 || value > 0xffffffff) {
    throw new RangeError('Value must be an unsigned 32-bit integer');
  }

  let number = value >>> 0;
  let oddParity = false;

  while (number !== 0) {
    oddParity = !oddParity;
    number &= number - 1;
  }

  return oddParity ? 'odd' : 'even';
}
```

```ts
export function programToFindParity(value: number): 'odd' | 'even' {
  if (!Number.isInteger(value) || value < 0 || value > 0xffffffff) {
    throw new RangeError('Value must be an unsigned 32-bit integer');
  }

  let number = value >>> 0;
  let oddParity = false;

  while (number !== 0) {
    oddParity = !oddParity;
    number &= number - 1;
  }

  return oddParity ? 'odd' : 'even';
}
```

## Follow-up Questions

- Return `1` and `0` instead of string labels for odd and even parity.
- Compare the toggling approach with computing the full popcount and taking `% 2`.
- Use nibble lookup tables to reduce the number of arithmetic steps further.

## Common Pitfalls

- Confusing parity of the value with parity of its count of set bits.
- Counting every bit position even though only set bits need to drive the loop.
- Returning a boolean without clarifying how it maps to odd versus even parity.
