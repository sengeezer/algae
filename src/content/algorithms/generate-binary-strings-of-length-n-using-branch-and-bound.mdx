## Why It Surfaces

This problem is mostly pedagogical. The source labels it branch and bound, but the real mechanic is breadth-first generation over a binary decision tree where each level fixes one more bit. That makes it a clean example of expanding partial solutions systematically until they become complete outputs.

## Worked Example

For `N = 3`, start from the empty prefix, then expand to `0` and `1`, then to `00`, `01`, `10`, `11`, and finally to `000` through `111`. The final output is `['000', '001', '010', '011', '100', '101', '110', '111']`.

## Reference Implementations

<ExamplePair>

```js
function generateBinaryStringsOfLengthNUsingBranchAndBound(length) {
  if (!Number.isInteger(length) || length < 0) {
    throw new RangeError("Length must be a non-negative integer");
  }

  const queue = [""];
  const result = [];

  while (queue.length > 0) {
    const prefix = queue.shift();

    if (prefix.length === length) {
      result.push(prefix);
      continue;
    }

    queue.push(`${prefix}0`);
    queue.push(`${prefix}1`);
  }

  return result;
}
```

## Follow-up Questions

- Why is this closer to plain state-space enumeration than to a pruning-heavy branch-and-bound search?
- How would you switch from queue-based generation to recursive backtracking?
- What changes if the strings must avoid consecutive `1` bits?

## Common Pitfalls

- Claiming the algorithm prunes the search when it actually enumerates every leaf.
- Forgetting that the empty string is the only correct output when `N = 0`.
- Building all integers first and converting them later when direct prefix expansion is clearer.
