## Why It Surfaces

Vigenere cipher is the natural follow-up once a fixed Caesar shift feels too simple. The algorithm is still lightweight, but it introduces the important bookkeeping problem of coordinating the message index and the key index correctly.

## Worked Example

Encrypting `"ATTACK"` with key `"LEMON"` uses shifts `L, E, M, O, N, L`, producing `"LXFOPV"`. The keyword wraps when it runs out.

## Reference Implementations

<ExamplePair>

```js
function vigenereCipher(message, key, mode = 'encrypt') {
  if (typeof message !== 'string' || typeof key !== 'string') {
    throw new TypeError('Message and key must be strings');
  }

  const normalizedKey = key.toLowerCase().replace(/[^a-z]/g, '');
  if (normalizedKey.length === 0) {
    throw new Error('Key must contain at least one letter');
  }

  let keyIndex = 0;

  return message.replace(/[a-z]/gi, (char) => {
    const base = /[A-Z]/.test(char) ? 65 : 97;
    const shift = normalizedKey.charCodeAt(keyIndex % normalizedKey.length) - 97;
    const direction = mode === 'encrypt' ? shift : -shift;
    const index = char.charCodeAt(0) - base;
    keyIndex += 1;
    return String.fromCharCode(base + (((index + direction) % 26 + 26) % 26));
  });
}
```

export function vigenereCipher(
  message: string,
  key: string,
  mode: 'encrypt' | 'decrypt' = 'encrypt',
): string {
  const normalizedKey = key.toLowerCase().replace(/[^a-z]/g, '');

  if (normalizedKey.length === 0) {
    throw new Error('Key must contain at least one letter');
  }

  let keyIndex = 0;

  return message.replace(/[a-z]/gi, (char) => {
    const base = /[A-Z]/.test(char) ? 65 : 97;
    const shift = normalizedKey.charCodeAt(keyIndex % normalizedKey.length) - 97;
    const direction = mode === 'encrypt' ? shift : -shift;
    const index = char.charCodeAt(0) - base;
    keyIndex += 1;
    return String.fromCharCode(base + (((index + direction) % 26 + 26) % 26));
  });
}

## Follow-up Questions

- Preserve punctuation without consuming a key character.
- Recover the plaintext when the key is known and the message contains mixed case.
- Compare Vigenere with one-time pad and explain why repeated keys are weaker.

## Common Pitfalls

- Advancing the key index on spaces or punctuation.
- Forgetting to normalize negative wraparound during decryption.
- Accepting an empty or nonalphabetic key.
