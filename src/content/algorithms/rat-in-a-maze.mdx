## Why It Surfaces

Rat in a Maze is a compact grid backtracking problem. It tests whether you can search for a path while keeping the current route separate from cells that belong to failed branches.

## Worked Example

Starting from the top-left cell, move only into open cells. If a route reaches a dead end, undo the last move, unmark the cell from the current path, and try the next direction.

<Callout title="State hygiene" tone="warning">
  Mark visited cells before recursing and unmark them when backtracking. Otherwise the search can either loop forever or incorrectly block valid routes.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function ratInAMaze(maze) {
  const size = maze.length;
  const path = Array.from({ length: size }, () => new Array(size).fill(0));

  function dfs(row, col) {
    if (
      row < 0 || row >= size || col < 0 || col >= size || maze[row][col] === 0 || path[row][col] === 1
    ) {
      return false;
    }

    path[row][col] = 1;

    if (row === size - 1 && col === size - 1) {
      return true;
    }

    if (dfs(row + 1, col) || dfs(row, col + 1) || dfs(row - 1, col) || dfs(row, col - 1)) {
      return true;
    }

    path[row][col] = 0;
    return false;
  }

  return dfs(0, 0) ? path : [];
}
```

## Follow-up Questions

- Return every valid path instead of the first one found.
- Restrict movement to only right and down.
- Compare this with flood fill and shortest-path grid problems.

## Common Pitfalls

- Forgetting to unmark cells when a branch fails.
- Walking into walls or revisiting cells in the current path.
- Confusing any valid path with the shortest path.
