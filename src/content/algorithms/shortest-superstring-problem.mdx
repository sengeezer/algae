## Why It Surfaces

Shortest superstring is a good interview checkpoint for recognizing when exact optimization is too expensive. The greedy version is easy to explain: at every step, merge the pair of strings with the biggest suffix-prefix overlap and keep going until one string remains.

## Worked Example

For `['catgc', 'ctaagt', 'gcta', 'ttca', 'atgcatc']`, the greedy merge starts by combining `catgc` with `atgcatc` into `catgcatc`, then combines `gcta` with `ctaagt` into `gctaagt`, and continues until one merged answer remains. One valid greedy result is `gctaagttcatgcatc`.

```js
function shortestSuperstringProblem(strings) {
  const pool = strings.filter(Boolean);
  if (pool.length <= 1) {
    return pool[0] ?? '';
  }

  function mergeWithMaximumOverlap(left, right) {
    let bestOverlap = -1;
    let merged = left + right;
    const limit = Math.min(left.length, right.length);

    for (let size = 1; size <= limit; size += 1) {
      if (left.slice(left.length - size) === right.slice(0, size) && size > bestOverlap) {
        bestOverlap = size;
        merged = left + right.slice(size);
      }

      if (right.slice(right.length - size) === left.slice(0, size) && size > bestOverlap) {
        bestOverlap = size;
        merged = right + left.slice(size);
      }
    }

    return { bestOverlap, merged };
  }

  while (pool.length > 1) {
    let bestLeft = 0;
    let bestRight = 1;
    let bestMerged = pool[0] + pool[1];
    let bestOverlap = -1;

    for (let leftIndex = 0; leftIndex < pool.length; leftIndex += 1) {
      for (let rightIndex = leftIndex + 1; rightIndex < pool.length; rightIndex += 1) {
        const candidate = mergeWithMaximumOverlap(pool[leftIndex], pool[rightIndex]);

        if (candidate.bestOverlap > bestOverlap) {
          bestOverlap = candidate.bestOverlap;
          bestMerged = candidate.merged;
          bestLeft = leftIndex;
          bestRight = rightIndex;
        }
      }
    }

    const nextPool = [];
    for (let index = 0; index < pool.length; index += 1) {
      if (index !== bestLeft && index !== bestRight) {
        nextPool.push(pool[index]);
      }
    }

    nextPool.push(bestMerged);
    pool.splice(0, pool.length, ...nextPool);
  }

  return pool[0];
}
```

## Follow-up Questions

- Remove strings that are already substrings of other strings before merging.
- Compare the greedy merge with an exact bitmask-DP solution.
- Return the merge order, not just the final superstring.

## Common Pitfalls

- Assuming the maximum-overlap greedy merge is always optimal.
- Checking overlap in only one direction instead of both suffix-prefix combinations.
- Forgetting that substring duplicates can distort the merge order.
