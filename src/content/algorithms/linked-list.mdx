## Why It Surfaces

Linked lists are less about the container itself and more about whether you can update pointers carefully. Once that mental model is solid, most linked-list interview problems are just constrained versions of the same rewiring work.

## Worked Example

Starting from `1 -> 2 -> 3`, inserting `0` at the head gives `0 -> 1 -> 2 -> 3`. Reversing the list walks through the nodes and flips each `next` pointer so the result becomes `3 -> 2 -> 1 -> 0`.

```js
class ListNode {
  constructor(value) {
    this.value = value;
    this.next = null;
  }
}

class LinkedList {
  constructor() {
    this.head = null;
  }

  insertAtHead(value) {
    const node = new ListNode(value);
    node.next = this.head;
    this.head = node;
  }

  search(value) {
    let current = this.head;

    while (current !== null) {
      if (current.value === value) {
        return true;
      }

      current = current.next;
    }

    return false;
  }

  delete(value) {
    if (this.head === null) {
      return;
    }

    if (this.head.value === value) {
      this.head = this.head.next;
      return;
    }

    let current = this.head;
    while (current.next !== null) {
      if (current.next.value === value) {
        current.next = current.next.next;
        return;
      }

      current = current.next;
    }
  }

  reverse() {
    let previous = null;
    let current = this.head;

    while (current !== null) {
      const next = current.next;
      current.next = previous;
      previous = current;
      current = next;
    }

    this.head = previous;
  }
}
```

## Follow-up Questions

- Add tail insertion and discuss the extra state it needs.
- Detect or remove duplicates from the list.
- Compare the tradeoffs with an array-backed list.

## Common Pitfalls

- Losing the rest of the list by overwriting `next` too early.
- Forgetting to special-case deletion at the head.
- Assuming linked lists support random access efficiently.
