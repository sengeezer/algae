## Why It Surfaces

This is a classic escalation from subset sum. Instead of filling one target subset once, you have to place every element into one of `k` buckets and maintain a global invariant that each bucket must eventually reach the same target.

## Worked Example

For `arr = [2, 1, 4, 5, 6]` and `k = 3`, the total sum is `18`, so every subset must sum to `6`. One valid grouping is `[6]`, `[1, 5]`, and `[2, 4]`, so the answer is true.

<ExamplePair>

```js
function partitionIntoKEqualSumSubsets(values, bucketCount) {
  const total = values.reduce((sum, value) => sum + value, 0);

  if (bucketCount === 1) {
    return true;
  }

  if (values.length < bucketCount || total % bucketCount !== 0) {
    return false;
  }

  const target = total / bucketCount;
  const sorted = [...values].sort((left, right) => right - left);

  if (sorted[0] > target) {
    return false;
  }

  const subsetSums = new Array(bucketCount).fill(0);

  function search(index) {
    if (index === sorted.length) {
      return subsetSums.every((sum) => sum === target);
    }

    const value = sorted[index];

    for (let bucket = 0; bucket < bucketCount; bucket += 1) {
      if (subsetSums[bucket] + value > target) {
        continue;
      }

      subsetSums[bucket] += value;

      if (search(index + 1)) {
        return true;
      }

      subsetSums[bucket] -= value;

      if (subsetSums[bucket] === 0) {
        break;
      }
    }

    return false;
  }

  return search(0);
}
```

## Common Pitfalls

- Skipping the total-sum divisibility check before recursing.
- Forgetting that buckets with the same current sum are symmetric states.
- Allowing a single element larger than the target bucket sum.
