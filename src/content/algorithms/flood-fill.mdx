## Why It Surfaces

Flood fill is a compact way to test grid traversal fundamentals. The region only grows through neighbors that match the original color, so the problem is really just a connected-component traversal with one extra rewrite step.

## Worked Example

Starting from `(1, 1)` in a region of `1`s, every adjacent `1` reachable through up, down, left, or right should change to the new color, while diagonally touching or different-valued cells should remain untouched.

<Callout title="Guard clause" tone="tip">
  If the new color already matches the starting color, return immediately. Otherwise the traversal can revisit cells forever or do pointless work.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function floodFill(image, startRow, startCol, nextColor) {
  const rows = image.length;
  const cols = image[0].length;
  const original = image[startRow][startCol];

  if (original === nextColor) {
    return image;
  }

  function dfs(row, col) {
    if (
      row < 0
      || row >= rows
      || col < 0
      || col >= cols
      || image[row][col] !== original
    ) {
      return;
    }

    image[row][col] = nextColor;
    dfs(row + 1, col);
    dfs(row - 1, col);
    dfs(row, col + 1);
    dfs(row, col - 1);
  }

  dfs(startRow, startCol);
  return image;
}
```

```ts
export function floodFill(
  image: number[][],
  startRow: number,
  startCol: number,
  nextColor: number,
): number[][] {
  const original = image[startRow][startCol];

  if (original === nextColor) {
    return image;
  }

  const rows = image.length;
  const cols = image[0].length;
  const queue: Array<[number, number]> = [[startRow, startCol]];
  image[startRow][startCol] = nextColor;

  while (queue.length > 0) {
    const [row, col] = queue.shift()!;

    for (const [rowOffset, colOffset] of [
      [1, 0],
      [-1, 0],
      [0, 1],
      [0, -1],
    ] as const) {
      const nextRow = row + rowOffset;
      const nextCol = col + colOffset;

      if (
        nextRow < 0
        || nextRow >= rows
        || nextCol < 0
        || nextCol >= cols
        || image[nextRow][nextCol] !== original
      ) {
        continue;
      }

      image[nextRow][nextCol] = nextColor;
      queue.push([nextRow, nextCol]);
    }
  }

  return image;
}
```

## Follow-up Questions

- Allow diagonal connectivity and explain what changes.
- Return the size of the filled region.
- Avoid mutating the original grid.

## Common Pitfalls

- Comparing neighbors against the new color instead of the original one.
- Forgetting the early return when the replacement color is unchanged.
- Marking cells too late and enqueueing them multiple times.
