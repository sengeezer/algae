## Why It Surfaces

Dynamic connectivity is what union-find looks like when the interviewer stops asking about the data structure in isolation and instead asks for query handling. The graph changes over time, but only by adding edges, which makes DSU the right tool.

## Worked Example

Start with all vertices disconnected. Each add-edge query unions two components, and each check query answers immediately by comparing the representatives of the queried vertices.

## Reference Implementations

<ExamplePair>

```js
function dynamicConnectivity(nodeCount, queries) {
  const parent = Array.from({ length: nodeCount }, (_, index) => index);
  const rank = new Array(nodeCount).fill(0);
  const answers = [];

  function find(node) {
    if (parent[node] !== node) {
      parent[node] = find(parent[node]);
    }
    return parent[node];
  }

  function union(left, right) {
    const rootLeft = find(left);
    const rootRight = find(right);
    if (rootLeft === rootRight) {
      return;
    }

    if (rank[rootLeft] < rank[rootRight]) {
      parent[rootLeft] = rootRight;
    } else if (rank[rootLeft] > rank[rootRight]) {
      parent[rootRight] = rootLeft;
    } else {
      parent[rootRight] = rootLeft;
      rank[rootLeft] += 1;
    }
  }

  for (const [type, left, right] of queries) {
    if (type === 1) {
      union(left, right);
      continue;
    }

    answers.push(find(left) === find(right));
  }

  return answers;
}
```

## Follow-up Questions

- Extend the interface to support decremental or fully dynamic connectivity.
- Return component sizes alongside connectivity answers.
- Explain why DSU works only because edges are added and never removed.

## Common Pitfalls

- Re-running graph traversal on every query instead of maintaining components.
- Forgetting path compression or union by rank when discussing complexity.
- Assuming the same approach handles deletions without additional machinery.
