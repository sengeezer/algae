## Why It Surfaces

This is the baseline string-search algorithm every optimized method improves on. The interviewer often wants to see the simple nested-loop version first before discussing better asymptotics.

## Worked Example

Searching for `"cada"` in `"abracadabra"` checks each possible start index until index `4`, where the next four characters match exactly.

## Reference Implementations

<ExamplePair>

```js
function patternMatching(text, pattern) {
  if (typeof text !== 'string' || typeof pattern !== 'string') {
    throw new TypeError('Both arguments must be strings');
  }

  const normalizedText = text.toLowerCase();
  const normalizedPattern = pattern.toLowerCase();

  if (normalizedPattern.length === 0) {
    return 0;
  }

  for (
    let start = 0;
    start <= normalizedText.length - normalizedPattern.length;
    start += 1
  ) {
    let matched = true;

    for (let offset = 0; offset < normalizedPattern.length; offset += 1) {
      if (normalizedText[start + offset] !== normalizedPattern[offset]) {
        matched = false;
        break;
      }
    }

    if (matched) {
      return start;
    }
  }

  return -1;
}
```

## Follow-up Questions

- Return every match index instead of only the first one.
- Make the search case-sensitive.
- Replace the naive search with KMP or Rabin-Karp for large repeated queries.

## Common Pitfalls

- Iterating the inner loop against the text length instead of the pattern length.
- Not defining the result for an empty pattern.
- Returning a formatted message string instead of a reusable index value.
