## Why It Surfaces

Find HCF is the most reusable arithmetic helper in the catalog. It often appears as the real subroutine hiding beneath lcm, coprimality, fraction reduction, and modular inverse questions.

## Worked Example

For `84` and `30`, compute `84 % 30 = 24`, then `30 % 24 = 6`, then `24 % 6 = 0`. The last non-zero divisor is `6`, so the HCF is `6`.

## Reference Implementations

<ExamplePair>

```js
function findHcf(left, right) {
  if (!Number.isInteger(left) || !Number.isInteger(right)) {
    throw new TypeError('Inputs must be whole numbers');
  }

  if (left < 1 || right < 1) {
    throw new RangeError('Inputs must be positive');
  }

  let a = left;
  let b = right;

  while (b !== 0) {
    [a, b] = [b, a % b];
  }

  return a;
}
```

## Common Pitfalls

- Scanning all divisors up to the smaller input instead of using remainders.
- Forgetting to validate positive integer inputs.
- Confusing HCF with LCM in follow-up formulas.
