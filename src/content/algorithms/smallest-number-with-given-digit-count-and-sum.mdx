## Why It Surfaces

To make the number as small as possible, large digits should be pushed toward the least significant positions. The only catch is that the leftmost digit cannot be zero, so you reserve one unit for it first and then distribute the rest from right to left.

## Worked Example

For `sum = 20` and `digitCount = 3`, reserve `1` for the first digit and distribute the remaining `19` from the right. That yields digits `2`, `9`, and `9`, producing `"299"`, which is the smallest three-digit number with digit sum `20`.

## Reference Implementations

<ExamplePair>

```js
function smallestNumberWithGivenDigitCountAndSum(sum, digitCount) {
  if (digitCount <= 0 || sum < 0) {
    return '-1';
  }

  if (sum === 0) {
    return digitCount === 1 ? '0' : '-1';
  }

  if (sum > 9 * digitCount) {
    return '-1';
  }

  const digits = new Array(digitCount).fill('0');
  let remaining = sum - 1;

  for (let index = digitCount - 1; index > 0; index -= 1) {
    const digit = Math.min(9, remaining);
    digits[index] = String(digit);
    remaining -= digit;
  }

  digits[0] = String(remaining + 1);
  return digits.join('');
}
```

## Follow-up Questions

- Explain why filling from left to right with the largest possible digits gives the wrong objective.
- Return `-1` immediately under which exact impossible conditions.
- Adapt the construction to return the largest number with the same digit count and sum.

## Common Pitfalls

- Letting the leading digit become zero for multi-digit answers.
- Forgetting the simple impossibility check `sum > 9 * digitCount`.
- Filling the number from the left first, which makes the result larger than necessary.
