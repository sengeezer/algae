## Why It Surfaces

Pronic numbers are a short closed-form classification problem. The main trick is recognizing that the relevant consecutive integers sit around the square root.

## Worked Example

`12` is pronic because `12 = 3 * 4`. `16` is not pronic because no pair of consecutive integers multiplies to `16`.

## Reference Implementations

<ExamplePair>

```js
function isPronic(value) {
  if (!Number.isInteger(value) || value < 0) {
    return false;
  }

  if (value === 0) {
    return true;
  }

  const root = Math.floor(Math.sqrt(value));
  return root * (root + 1) === value;
}
```

```ts
export function isPronic(value: number): boolean {
  if (!Number.isInteger(value) || value < 0) {
    return false;
  }

  if (value === 0) {
    return true;
  }

  const root = Math.floor(Math.sqrt(value));
  return root * (root + 1) === value;
}
```

## Follow-up Questions

- Derive why checking around the square root is enough.
- Compare pronic numbers with triangular and hexagonal numbers.
- Generate the first `k` pronic numbers instead of testing one value.

## Common Pitfalls

- Forgetting that `0 = 0 * 1` is pronic.
- Treating negative values as valid under this definition.
- Scanning all possible factors when the square root already narrows the candidate pair.
