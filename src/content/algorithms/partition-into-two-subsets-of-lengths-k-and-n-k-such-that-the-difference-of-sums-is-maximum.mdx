## Why It Surfaces

This looks like a constrained partition problem, but with non-negative values the only thing that matters is which subset gets to contain more of the biggest elements. The optimal strategy is to let the larger-cardinality subset take the largest available values, because every large value moved into that side increases the final difference twice: once by increasing that subset and once by decreasing the other.

## Worked Example

For `arr = [8, 4, 5, 2, 10]` and `k = 2`, the larger subset size is `3`. Sorting descending gives `[10, 8, 5, 4, 2]`, so the best larger subset sum is `10 + 8 + 5 = 23`, the remaining subset sum is `6`, and the maximum difference is `17`.

## Reference Implementations

<ExamplePair>

```js
function partitionIntoTwoSubsetsOfLengthsKAndNKSuchThatTheDifferenceOfSumsIsMaximum(
  values,
  subsetLength,
) {
  const sortedValues = [...values].sort((left, right) => right - left);
  const largerSubsetLength = Math.max(subsetLength, sortedValues.length - subsetLength);
  const totalSum = sortedValues.reduce((sum, value) => sum + value, 0);

  let largerSubsetSum = 0;

  for (let index = 0; index < largerSubsetLength; index += 1) {
    largerSubsetSum += sortedValues[index];
  }

  return largerSubsetSum - (totalSum - largerSubsetSum);
}
```

## Follow-up Questions

- Explain why the proof depends on the source constraint that all numbers are non-negative.
- Return one optimal pair of subsets, not only the maximum difference.
- Adapt the solution if the objective is to maximize the ratio instead of the difference.

## Common Pitfalls

- Treating the problem like contiguous subarrays even though the verified article is about subsets.
- Assigning the largest values to the smaller-cardinality subset without checking which side creates the larger difference.
- Forgetting that the answer is `largerSubsetSum - remainingSum`, not just the sum of the chosen prefix.
