## Why It Surfaces

This is a deliberately small helper, but it still tests whether you reach for the simplest correct approach instead of sorting or overcomplicating the problem.

## Worked Example

For the values `9`, `4`, `11`, and `2`, the running minimum moves from `9` to `4`, then stays there until it updates to `2`.

## Reference Implementations

<ExamplePair>

```js
function findMin(...values) {
  if (values.length === 0) {
    throw new TypeError('At least one value is required');
  }

  if (!values.every((value) => typeof value === 'number' && Number.isFinite(value))) {
    throw new TypeError('Values must be finite numbers');
  }

  let min = values[0];

  for (let index = 1; index < values.length; index += 1) {
    if (values[index] < min) {
      min = values[index];
    }
  }

  return min;
}
```

## Follow-up Questions

- Return both the minimum value and its index.
- Compare this one-pass scan with sorting the whole array.
- Extend the helper to support a selector over objects.

## Common Pitfalls

- Sorting the full input when only the minimum is needed.
- Forgetting to reject an empty argument list.
- Ignoring non-finite numeric values if the caller expects strict validation.
