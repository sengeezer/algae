## Why It Surfaces

This pattern is the string version of subset generation. Each character offers a binary decision, but unlike permutations, every accepted path must preserve the original order of the string.

## Worked Example

For `"abc"`, the non-empty subsequences are `a`, `ab`, `abc`, `ac`, `b`, `bc`, and `c`. The empty string is usually excluded unless the prompt explicitly asks for it.

## Reference Implementations

<ExamplePair>

```js
function subsequenceRecursive(text) {
  const results = new Set();

  function dfs(index, path) {
    if (index === text.length) {
      if (path.length > 0) {
        results.add(path);
      }
      return;
    }

    dfs(index + 1, path + text[index]);
    dfs(index + 1, path);
  }

  dfs(0, '');
  return [...results].sort();
}
```

```ts
export function subsequenceRecursive(text: string): string[] {
  const results = new Set<string>();

  function dfs(index: number, path: string): void {
    if (index === text.length) {
      if (path.length > 0) {
        results.add(path);
      }
      return;
    }

    dfs(index + 1, path + text[index]);
    dfs(index + 1, path);
  }

  dfs(0, '');
  return [...results].sort();
}
```

## Follow-up Questions

- Return the count only instead of materializing every subsequence.
- Avoid the final sort while still generating lexicographic order.
- Compare this branching pattern with longest common subsequence DP.

## Common Pitfalls

- Reordering characters and accidentally generating permutations instead.
- Including the empty string when the prompt says non-empty subsequences only.
- Emitting duplicates for repeated characters without deduplication.
