## Why It Surfaces

This is a balancing-by-pairing question. Once the array is sorted, pairing extremes is the move that pulls large values downward and small values upward, compressing the final spread of pair sums.

## Worked Example

For `arr = [11, 4, 3, 5, 7, 1]`, sort to `[1, 3, 4, 5, 7, 11]`. Pair `(1, 11)`, `(3, 7)`, and `(4, 5)`, which gives sums `12`, `10`, and `9`. The answer is `12 - 9 = 3`.

```js
function minimumDifferenceBetweenGroupsOfSizeTwo(values) {
  if (values.length % 2 !== 0) {
    throw new RangeError('values length must be even');
  }

  if (values.length === 0) {
    return 0;
  }

  const sorted = [...values].sort((left, right) => left - right);
  let lowestPairSum = Number.POSITIVE_INFINITY;
  let highestPairSum = Number.NEGATIVE_INFINITY;

  for (let index = 0; index < sorted.length / 2; index += 1) {
    const pairSum = sorted[index] + sorted[sorted.length - 1 - index];
    lowestPairSum = Math.min(lowestPairSum, pairSum);
    highestPairSum = Math.max(highestPairSum, pairSum);
  }

  return highestPairSum - lowestPairSum;
}
```

## Common Pitfalls

- Pairing nearby values instead of pairing from opposite ends after sorting.
- Forgetting the input must have even length.
- Building every possible pairing combination instead of using the sorted greedy invariant.
