## Why It Surfaces

Flatcase checks are a minimal version of naming-style validation. The problem is useful because it forces you to define the identifier contract precisely instead of informally saying it "looks lowercase enough."

## Worked Example

`"thisvariable"` passes because it is all lowercase letters with no separators. `"thisVariable"` fails because an uppercase letter appears in the middle.

## Reference Implementations

<ExamplePair>

```js
function checkFlatCase(value) {
  if (typeof value !== 'string') {
    throw new TypeError('Value must be a string');
  }

  return /^[a-z]*$/.test(value);
}
```

```ts
export function checkFlatCase(value: string): boolean {
  return /^[a-z]*$/.test(value);
}
```

</ExamplePair>

## Follow-up Questions

- Require at least one character instead of accepting the empty string.
- Convert flatcase into title case using a dictionary of known word boundaries.
- Classify a string across flatcase, camelCase, kebab-case, and snake_case.

## Common Pitfalls

- Accidentally accepting digits, spaces, or punctuation.
- Forgetting to anchor the regex to the full string.
- Leaving the empty-string rule undefined.
