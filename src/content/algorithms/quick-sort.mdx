## Why It Surfaces

Quick sort is less about raw sorting in interviews and more about partitioning. Once you explain the partition step clearly, you are close to quickselect and many selection-style questions.

## JavaScript

```js
function quickSort(nums, left = 0, right = nums.length - 1) {
  if (left >= right) {
    return nums;
  }

  const pivot = nums[right];
  let boundary = left;

  for (let index = left; index < right; index += 1) {
    if (nums[index] <= pivot) {
      [nums[index], nums[boundary]] = [nums[boundary], nums[index]];
      boundary += 1;
    }
  }

  [nums[boundary], nums[right]] = [nums[right], nums[boundary]];
  quickSort(nums, left, boundary - 1);
  quickSort(nums, boundary + 1, right);
  return nums;
}
```

## TypeScript

```ts
export function partition(values: number[], left: number, right: number): number {
  const pivot = values[right];
  let boundary = left;

  for (let index = left; index < right; index += 1) {
    if (values[index] <= pivot) {
      [values[index], values[boundary]] = [values[boundary], values[index]];
      boundary += 1;
    }
  }

  [values[boundary], values[right]] = [values[right], values[boundary]];
  return boundary;
}
```

## Interview Notes

- State the partition invariant before you swap.
- Mention randomized pivots if worst-case input is a concern.
- Connect the partition step directly to quickselect when asked for the kth largest element.
