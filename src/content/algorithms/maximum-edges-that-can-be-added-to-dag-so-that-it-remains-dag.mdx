## Why It Surfaces

Ordinary topological sort only orders the vertices. This problem asks what that order permits: once you know which vertices must come earlier than others, you can safely add every missing forward edge and stop exactly when the DAG becomes maximal.

## Worked Example

If one valid topological order is `0, 1, 2, 3`, then edges may only point from left to right in that order. If `0 -> 2` and `1 -> 3` are missing, both can be added safely, but `2 -> 0` would immediately violate the order and create a cycle.

function maximumEdgesThatCanBeAddedToDagSoThatItRemainsDag(graph) {
  const nodeCount = graph.length;
  const indegree = new Array(nodeCount).fill(0);
  const edgeSet = new Set();

  for (let from = 0; from < nodeCount; from += 1) {
    for (const to of graph[from]) {
      indegree[to] += 1;
      edgeSet.add(`${from}:${to}`);
    }
  }

  const queue = [];
  for (let node = 0; node < nodeCount; node += 1) {
    if (indegree[node] === 0) {
      queue.push(node);
    }
  }

  const order = [];
  for (let head = 0; head < queue.length; head += 1) {
    const node = queue[head];
    order.push(node);

    for (const neighbor of graph[node]) {
      indegree[neighbor] -= 1;
      if (indegree[neighbor] === 0) {
        queue.push(neighbor);
      }
    }
  }

  if (order.length !== nodeCount) {
    return [];
  }

  const additions = [];

  for (let left = 0; left < order.length; left += 1) {
    const from = order[left];

    for (let right = left + 1; right < order.length; right += 1) {
      const to = order[right];
      const key = `${from}:${to}`;

      if (edgeSet.has(key)) {
        continue;
      }

      additions.push([from, to]);
      edgeSet.add(key);
    }
  }

  return additions;
}

## Follow-up Questions

- If you only need the count, how does the closed-form `V * (V - 1) / 2 - E` relate to this construction?
- Why must every safe additional edge point forward in some topological order?
- How would you modify the solution if the input might contain a cycle?

## Common Pitfalls

- Adding edges without first fixing a topological order.
- Forgetting to skip edges that already exist in the graph.
- Using a construction that returns only the count when the interviewer asks for the actual added edges.
