## Why It Surfaces

This is the constant-time answer to weekday-from-date prompts. Instead of marching forward from an epoch, you use a formula that folds the day, month, year, century, and leap-year corrections into one arithmetic expression.

## Worked Example

For `2026-04-30`, apply the date directly because April is month `4`. The formula maps the date to weekday index `5`, which corresponds to `Thursday`.

```js
function zellersCongruenceAlgorithm(day, month, year) {
  if (!Number.isInteger(day) || !Number.isInteger(month) || !Number.isInteger(year)) {
    throw new TypeError('Arguments must be integers');
  }

  if (month < 1 || month > 12 || day < 1 || day > 31) {
    throw new RangeError('Date components are out of range');
  }

  let adjustedMonth = month;
  let adjustedYear = year;

  if (adjustedMonth < 3) {
    adjustedMonth += 12;
    adjustedYear -= 1;
  }

  const yearOfCentury = adjustedYear % 100;
  const zeroBasedCentury = Math.floor(adjustedYear / 100);
  const weekdayIndex = (
    day +
    Math.floor((13 * (adjustedMonth + 1)) / 5) +
    yearOfCentury +
    Math.floor(yearOfCentury / 4) +
    Math.floor(zeroBasedCentury / 4) +
    5 * zeroBasedCentury
  ) % 7;

  const weekdays = [
    'Saturday',
    'Sunday',
    'Monday',
    'Tuesday',
    'Wednesday',
    'Thursday',
    'Friday',
  ];

  return weekdays[weekdayIndex];
}
```

## Follow-up Questions

- Compare the formula with epoch-based day counting.
- Extend the validation so day bounds match the selected month and leap-year rules.
- Use the weekday result inside a full month-calendar renderer.

## Common Pitfalls

- Forgetting to treat January and February as months of the previous year.
- Mixing up Zeller's weekday indexing order.
- Trusting only day `1..31` validation when month-specific bounds still matter.
