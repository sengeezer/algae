## Why It Surfaces

This problem is popular because several good solutions exist, but only one or two match the follow-up constraints. The strongest interview move is to explain why the smaller boundary determines the water level at each step.

## Worked Example

For `height = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]`, the answer is `6`. Each trapped unit sits below the shorter of the best left wall and best right wall seen for that position.

<Callout title="Invariant" tone="note">
  If `leftMax <= rightMax`, the left pointer can be resolved immediately because no future bar on the right can lower the left side's limiting wall.
</Callout>

## Reference Implementations

<ExamplePair>

```js
function trap(height) {
  let left = 0;
  let right = height.length - 1;
  let leftMax = 0;
  let rightMax = 0;
  let water = 0;

  while (left < right) {
    if (height[left] <= height[right]) {
      leftMax = Math.max(leftMax, height[left]);
      water += leftMax - height[left];
      left += 1;
    } else {
      rightMax = Math.max(rightMax, height[right]);
      water += rightMax - height[right];
      right -= 1;
    }
  }

  return water;
}
```

```ts
export function trap(height: number[]): number {
  const stack: number[] = [];
  let water = 0;

  for (let index = 0; index < height.length; index += 1) {
    while (stack.length > 0 && height[index] > height[stack[stack.length - 1]]) {
      const bottom = stack.pop()!;

      if (stack.length === 0) {
        break;
      }

      const left = stack[stack.length - 1];
      const width = index - left - 1;
      const boundedHeight = Math.min(height[left], height[index]) - height[bottom];
      water += width * boundedHeight;
    }

    stack.push(index);
  }

  return water;
}
```

</ExamplePair>

## Follow-up Questions

- Compare the two-pointer and monotonic-stack solutions.
- Explain how you would solve the 2D version of the problem.
- Derive the prefix-maxima solution before optimizing it to constant space.

## Common Pitfalls

- Moving the wrong pointer and breaking the invariant.
- Forgetting that water depends on the smaller boundary, not the taller one.
- Off-by-one errors when computing the width in the stack-based solution.
