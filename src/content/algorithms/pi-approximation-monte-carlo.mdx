## Why It Surfaces

This is a classic randomized algorithm question. The main interview signal is whether you can connect the geometry to the probability ratio instead of treating the random simulation as magic.

## Worked Example

Generate many points `(x, y)` uniformly inside the unit square `[0, 1] x [0, 1]`. A point lands inside the quarter circle when `x^2 + y^2 <= 1`. If that happens for about `78.5%` of the samples, then multiplying by `4` gives an estimate near `3.14`.

## Reference Implementations

<ExamplePair>

```js
function piApproximationMonteCarlo(iterations = 100_000) {
  if (!Number.isInteger(iterations) || iterations <= 0) {
    throw new RangeError('Iteration count must be a positive integer');
  }

  let insideCircle = 0;

  for (let index = 0; index < iterations; index += 1) {
    const x = Math.random();
    const y = Math.random();

    if (x * x + y * y <= 1) {
      insideCircle += 1;
    }
  }

  return (insideCircle / iterations) * 4;
}
```

## Follow-up Questions

- Seed the randomness in a test harness so repeated runs become reproducible.
- Measure how the estimate changes as the iteration count increases.
- Compare this stochastic method with midpoint or Simpson-style deterministic approximations.

## Common Pitfalls

- Using too few samples and over-trusting a noisy estimate.
- Forgetting why the quarter-circle hit ratio needs to be multiplied by `4`.
- Treating the random estimate as exact instead of probabilistic.
