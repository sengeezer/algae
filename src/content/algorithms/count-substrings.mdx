## Why It Surfaces

This is often a disguised spec-reading problem. The algorithm is simple, but the interviewer is usually testing whether you ask about overlapping matches and empty substrings before writing the loop.

## Worked Example

In `"banana"`, the substring `"ana"` appears twice if overlaps count: once starting at index `1` and again at index `3`.

## Reference Implementations

<ExamplePair>

```js
function countSubstrings(value, target) {
  if (typeof value !== 'string' || typeof target !== 'string') {
    throw new TypeError('Both arguments must be strings');
  }

  if (target.length === 0) {
    return value.length + 1;
  }

  let count = 0;
  let position = value.indexOf(target);

  while (position !== -1) {
    count += 1;
    position = value.indexOf(target, position + 1);
  }

  return count;
}
```

```ts
export function countSubstrings(value: string, target: string): number {
  if (target.length === 0) {
    return value.length + 1;
  }

  let count = 0;
  let position = value.indexOf(target);

  while (position !== -1) {
    count += 1;
    position = value.indexOf(target, position + 1);
  }

  return count;
}
```

## Follow-up Questions

- Count only non-overlapping matches.
- Replace the repeated `indexOf` calls with KMP or Rabin-Karp for large-scale search.
- Return the starting indices of each match instead of only the total count.

## Common Pitfalls

- Forgetting that overlapping matches may still count.
- Not defining what should happen when the target is empty.
- Advancing by the target length when the prompt expects overlaps to be counted.
