## Why It Surfaces

Directed Eulerian checks are a common follow-up once you already know the undirected rule. The trap is that odd-degree counting no longer works by itself, so you need both balance and strong connectivity among the vertices that actually participate in edges.

## Worked Example

If every active vertex has matching in-degree and out-degree but one active region cannot reach another, a directed Euler circuit is impossible. If both connectivity and degree balance hold, then a directed circuit exists and a construction method such as Hierholzer can recover it.

function eulerCircuitInADirectedGraph(graph) {
  const size = graph.length;
  const transpose = Array.from({ length: size }, () => []);
  const inDegree = new Array(size).fill(0);
  const outDegree = new Array(size).fill(0);

  for (let from = 0; from < size; from += 1) {
    outDegree[from] = graph[from].length;
    for (const to of graph[from]) {
      transpose[to].push(from);
      inDegree[to] += 1;
    }
  }

  for (let node = 0; node < size; node += 1) {
    if (inDegree[node] !== outDegree[node]) {
      return false;
    }
  }

  const start = graph.findIndex((neighbors, node) => neighbors.length > 0 || inDegree[node] > 0);
  if (start === -1) {
    return true;
  }

  function dfs(adjacency, source, seen) {
    seen[source] = true;
    for (const neighbor of adjacency[source]) {
      if (!seen[neighbor]) {
        dfs(adjacency, neighbor, seen);
      }
    }
  }

  const seenForward = new Array(size).fill(false);
  dfs(graph, start, seenForward);
  const seenBackward = new Array(size).fill(false);
  dfs(transpose, start, seenBackward);

  for (let node = 0; node < size; node += 1) {
    const active = inDegree[node] > 0 || outDegree[node] > 0;
    if (active && (!seenForward[node] || !seenBackward[node])) {
      return false;
    }
  }

  return true;
}

## Common Pitfalls

- Applying the undirected odd-degree rule to a directed graph.
- Checking only weak connectivity instead of strong connectivity on active vertices.
- Forgetting to compare in-degree and out-degree for every active node.
