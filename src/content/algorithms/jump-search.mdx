## Why It Surfaces

Jump search is mostly a comparison tool for sorted arrays. It is easy to reason about: skip ahead in coarse blocks until the target could be inside one, then finish with a local linear scan.

## Worked Example

If `n = 25`, a block size around `5` is natural. Probe indices `4`, `9`, `14`, and so on until the array value crosses the target, then scan backward or within that block to find the exact match.

## Reference Implementations

<ExamplePair>

```js
function jumpSearch(nums, target) {
  const step = Math.floor(Math.sqrt(nums.length));
  let left = 0;
  let right = step;

  while (left < nums.length && nums[Math.min(right, nums.length) - 1] < target) {
    left = right;
    right += step;
  }

  for (let index = left; index < Math.min(right, nums.length); index += 1) {
    if (nums[index] === target) {
      return index;
    }
  }

  return -1;
}
```

## Follow-up Questions

- Explain why the best jump size is about `sqrt(n)`.
- Compare this strategy with binary search.
- Return the insertion region when the target is absent.

## Common Pitfalls

- Forgetting to clamp the right edge on the last block.
- Jumping past the correct block and never scanning it.
- Claiming logarithmic time instead of `O(sqrt(n))`.
