## Why It Surfaces

This problem is a clean way to test whether someone can separate sequence generation from a property check. The triangle numbers are cheap to produce; the interesting work is counting divisors efficiently enough.

## Worked Example

The triangle numbers start `1, 3, 6, 10, 15, 21, 28, ...`. Since `28` has six divisors, it is the first triangle number with more than five divisors.

function countDivisors(value) {
  let total = 0;
  const root = Math.floor(Math.sqrt(value));

  for (let divisor = 1; divisor <= root; divisor += 1) {
    if (value % divisor !== 0) {
      continue;
    }

    total += divisor * divisor === value ? 1 : 2;
  }

  return total;
}

function problem012(minDivisors = 500) {
  if (!Number.isInteger(minDivisors) || minDivisors < 1) {
    throw new RangeError('Divisor target must be a positive integer');
  }

  let triangle = 0;

  for (let n = 1; ; n += 1) {
    triangle += n;
    if (countDivisors(triangle) > minDivisors) {
      return triangle;
    }
  }
}

## Follow-up Questions

- Replace square-root scanning with a prime-factorization-based divisor count.
- Return both the triangle number and its divisor count.
- Explain why updating the running triangle value is cheaper than recomputing `n(n + 1) / 2` from scratch each time.

## Common Pitfalls

- Starting the divisor loop at `0`, which makes modulo logic invalid.
- Forgetting that a perfect square contributes only one divisor at its square root.
- Using `>=` when the prompt says “more than” a divisor threshold.
